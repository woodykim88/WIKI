| 리뷰 종류 | [DB] review_type | [DB] creater_type | [DB] platform | 주문 데이터 | createdAt |
| --- | --- | --- | --- | --- | --- |
| 텍스트 리뷰  | 1 (일반리뷰) | 1 (회원)
2 (비회원)
3 (NPAY 비회원) | 알파리뷰 | 있음 | 최초 작성 시점 (별점 클릭) |
| 빠른 리뷰 | 2 (빠른리뷰) | 1 (회원)
2 (비회원)

| SNS 리뷰 | 3 (SNS리뷰) | 1 (회원)
2 (비회원)
3 (NPAY 비회원) | 알파리뷰 | 있음 | 해시태그가 인식되어서 admin 에 유입된 시점이 createdAt 임. 게시 여부와는 별개임 |
- [회원 / 비회원 / NPAY 비회원] 구분
    - **회원** : 자사몰 회원
    - **비회원** : 자사몰 회원이 아니면서, 자사몰 내부 시스템으로 결제한 경우
    - **NPAY 비회원** : 자사몰 회원이 아니면서 npay 등의 간편결제로 결제한 경우 (이름만 NAPAY 이지, npay 결제 케이스만 의미하는 것이 아님)
- **텍스트 리뷰** vs **빠른 리뷰**
    - 리뷰 작성 유도 알림톡에서 버튼 클릭 시, 자동으로 “빠른 리뷰” 로 저장이 됨.
    - 이 “빠른 리뷰” 에서 텍스트를 추가로 입력/수정 시 “일반 리뷰” 로 review_type 이 변경됨. 텍스트 수정 없이 이미지만 추가할 경우에는 계속 “빠른 리뷰” 로 type 유지

### [2] 엔드 유저가 알파리뷰에서 직접 작성 (관리자가 엔드 유저에게 요청)

| 리뷰 종류 | [DB] review_type | [DB] creater_type | [DB] platform | 주문 데이터 | createdAt |
| --- | --- | --- | --- | --- | --- |
| 미검증 리뷰 | 1 (일반리뷰) | 11 (미검증 리뷰) | 알파리뷰 | 강제 생성됨 | 최초 작성시점 |
| 리뷰 요청 > 텍스트 리뷰 | 1 (일반리뷰) | 입력한 고객정보 (이름, 전화번호) 와 일치하는 고객 데이터가

1) 존재하면 ⇒ 해당 고객의 customer_type 값을 그대로 가져옴

2) 존재하지 않으면 ⇒ 4 (주문생성 비회원) | 알파리뷰 | 강제 생성됨 | (경로 : 알림톡)
최초 작성 시점 |
| 리뷰 요청 > 빠른 리뷰 | 2 (빠른리뷰) | 입력한 고객정보 (이름, 전화번호) 와 일치하는 고객 데이터가

1) 존재하면 ⇒ 해당 고객의 customer_type 값을 그대로 가져옴

2) 존재하지 않으면 ⇒ 4 (주문생성 비회원) | 알파리뷰 | 강제 생성됨 | (경로 : 알림톡)
최초 작성 시점 |
- 미검증 리뷰는 “빠른 리뷰” 를 쓸 수 없음
- **리뷰 요청 관련 특이사항 → 추후에 개선 필요해보임**
    - shop 에서 “비회원은 리뷰를 작성할 수 없다” 라고 설정해놨더라도, 리뷰 요청 생성 시 비회원도 리뷰를 작성하는 것은 가능함
        - 적립금 지급 > 해당 업체의 "비회원 관련 설정(지급 수준)"을 따름
        - 포인트 지급 설정 화면 정책 > 비회원 작성불가로 설정 시 포인트 설정 영역에서 비회원 미표시
            - 화면에서는 노출되지 않더라도 default 값이 있기 때문에, 이 default 값 기반으로, 리뷰 요청을 통해 리뷰를 작성한 비회원에게 적립금 지급됨 → 초큼 요상한 flow 긴 하네.. 비회원 포인트가 적용은 되는데, 비회원에 대해서 포인트를 설정하는 UI 는 사용자에게 표시되지 않음
    - 작성 제외 상품/카테고리에 속한 상품에 대해서 리뷰 요청시, 알림톡은 발송되나 리뷰 작성은 불가함 (리뷰 작성 페이지로 진입시, 작성 가능한 리뷰가 없다라고 뜸)
        - 해당 부분 개선 시, 리뷰 요청의 속도가 빨라질 것으로 기대됨 (카테고리 정보는 코어쪽에 있기 때문에 속도 이슈 있음)

### 기타

- 체험단 리뷰는 100% 외부 업체가 관리중. 체험단을 통해 수집된 리뷰인지 여부는 저장하지 않음.
- 리뷰 수정 시, 원본과 변경된 버전 2개가 creater_type 으로 구분되어서 저장됨
    - 기존 리뷰 → creater_type 이 원본(100) 으로 변경됨
    - 수정된 버전의 리뷰  → 원본 리뷰의 creater_type 을 따라감
- 리뷰 복사 시, 원본과 복사된 버전 2개가 creater_type 으로 구분되어서 저장됨
    - 원본 리뷰 → creater_type 이 그대로 유지
    - 복사된 리뷰 → creater_type 이 복사(29) 로 설정됨
- 사진이 포함된 리뷰 / 동영상이 포함된 리뷰 / 설문 리뷰 / 키워드 리뷰 → 얘네들은 리뷰의 종류가 아니라, 모든 리뷰들이 공통적으로 가지고 있는 속성임

            - 별로 (1,2점)

            - 만족 (4,5점)
            - 보통 (3점)
            - 아쉬움 (1,2점)
            
            ⇒ 기본으로 생성되어 있고 관리자가 생성/수정/삭제할 수 없음
            
        - 별점/만족도 별 템플릿이 여러개인 경우에는 그중에서 랜덤으로 선택됨

        - 인스타그램 계정 연동

        - 각 기준별로 (모든 상품 대상, 특정 상품 대상, 특정 카테고리 대상) 최대 3개까지만 생성 가능
        - 작성 페이지 내에선 모든 상품 - 상품 - 카테고리별 순으로 최대 5개까지만 해시태그가 표시됨? ([출처](https://guide.alph.kr/2adf9ae6-597f-4eab-9e41-7e37e2068d56#2adf9ae6-597f-4eab-9e41-7e37e2068d56))
            - 앞에서부터 그냥 5개 자름
    - 대상 리뷰
        - 일반 리뷰
        - “리뷰 요청” 으로 생성된 일반 리뷰
    - 인스타그램 게시글 연동 방식
        - 엔드 유저에게 정해진 해시태그, 특정코드 를 복붙할 수 있도록 제공
        - 게시물 최초 업로드 시에 지정된 해시태그, 특정코드가 포함되어 있어야지만 연동됨
        - 해시태그, 특정코드 중 하나라도 누락되면 연동 안됨
        - 이미 생성된 인스타그램 게시글에서 수정하는 방식으로 해시태그, 특정코드를 추가해도 연동 안됨
        - 특정코드는 어디에 따로 저장되어있는지? → order_item  에 shortcode
    - SNS 유지 기한
        - 역할이 뭔지?
        - 이것과 관련된 데이터 처리 로직 있는지 → 없음…
    - 엔드 유저가 SNS 리뷰를 작성할 수 있는 조건 ([출처](https://www.notion.so/SNS-1d246f3e549c80bfa8e9ef1a07c2f82e?pvs=21))
        1. 해당 주문상품의 해시태그가 있는지 확인
        2. 해당 주문상품의 리뷰작성이 가능한지 확인?
        3. 해당 주문상품의 SNS리뷰작성이 가능한지 확인?
        4. 해당 주문상품에 SNS리뷰가 쓰였던적이 있는지 확인?
        5. 대시보드에서 SNS리뷰 토글을 활성화하였는지 확인
    - 그외
        - 연결된 sns 계정은 어디에 저장? → 코어
        - 카테고리 목록은 core 에서 관리? ⇒ ㅇㅇ
        - 사진, 동영상 개수 제한은 없는지? ⇒ 없음
        - SNS 리뷰 위젯 말고 다른 위젯에서도 볼 수 있는지? 특정 상품과 관련있는 해시코드가 없다면 상품과의 연결이 어떻게 되는지?
            - 주문이 이미 존재하는 상태니까 주문, 상품 맵핑 관련

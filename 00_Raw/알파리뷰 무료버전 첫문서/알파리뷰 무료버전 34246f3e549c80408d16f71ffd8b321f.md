# 1. 배경 및 목적

### 배경

1. 알파앱스 통합의 주춧돌 또는 재료 중 하나
2. 긴 호흡의 온보딩으로서의 가치
3. 시장 장악력 확대로 경쟁자 차단

### 목적

1. 리뷰/업셀/푸시의 무료버전을 통해서 시장 장악력을 높이는 것 (경쟁사 견제 등)
2. 가능하다면 진정한 의미의 Freemium모델로 모든 기능들을 체험 또는 가능성을 살펴볼 수 있고, 평생 무료로 이용할 수도 있는 유용한 기능을 제공할 수 있음
3. 나아가서 알파앱스 통합 - 에이전틱 알파앱스에 영향을 줄 것이므로 고려하여 설계
4. 더 나아가 프라이싱 전략 수정까지 영향을 미치게 될 것이므로 고려하여 설계

### 무료의 목표는 유료다

- 현재의 목표 회고
    - 현재의 목표는 너무나도 나이브하다. “예상치 못한 바이럴”, “시장, 군중이라는 거대한 흐름”에 기대하여 무료버전이 생기면 더 많은 잠재고객을 끌어들일 수 있을 것을 가정하고 있다. 이러한 방식은  “B2C, 또는 더 넓은 타겟의 고객사를 대상으로”에 해당하는 것이고 우리에게는 어울리지 않을지도 모름.
    - **결국 “마케팅 비용 > 무료버전 운영 비용”이어야 함**
- **무료 플랜의 '자격'을 정의하라:** "누구나"가 아닌, "우리의 미래 유료 고객이 될 가능성이 있는" 초기 쇼핑몰 운영자를 타겟으로 무료 플랜을 설계해야 합니다. 예를 들어, 특정 월 매출 이하의 쇼핑몰에게만 혜택을 주는 방식을 고려할 수 있습니다.
    - 선별 기준
        - 쇼핑몰을 진지하게 운영할 마음이 있는 사람들(준비과정, 매출이나 상품이 없어도 사용해 보는 것은 가능함)이 운영에는 결국 알파앱스가 필요하겠구나를 인지 시키는 것이 필요)
        - 리뷰가 정말 필요하다고 느낄 수 있도록
            - 리뷰 관리에 시간이 드는 대상
            - 리뷰가 없었는데 리뷰가 작성되어 매출이 일어날 것 같은 대상
            - 리뷰로 마케팅을 하고 싶은 대상(SNS리뷰)
        - 핵심 기능은 유사 체험판으로 한정, 절대적인 한도로 제공
        - 월 주문건 100건 이상 넘어가면 기능이 오히려 다운그레이드 되도록 설정(한계치 제공)
- **'진짜 지표'에 집중하라:** 전체 가입자 수가 아닌, '유료 플랜 가격 페이지 조회 수', '유료 기능 클릭 수', '활성 사용자 중 유료 전환율' 등 비즈니스 건강과 직결된 진짜 지표를 추적하고 의사결정의 기준으로 삼아야 합니다.

> **⇒ 무료를 제공해서 유료화에 확실하게 도움이 될 수 있는 루프까지 만들어야 함**
> 

### ‘알파앱스’관점에선 굳이 리뷰를 무료로 제공하지 않아도 된다.

- 임팩트는 약하겠지만 지금처럼 무료를 “푸시” + “업셀”정도로 제한해도 된다. 알파앱스는 “마케팅 - 매출 자동화에 도움을 주는 툴로서의 역할”로 본다면 반드시 리뷰가 있어야 하는 것은 아니다.
- 사실 진짜 자동화의 역할은 캠페인을 자동화하고 운영을 자동화하는, 에이전트 크레딧을 제공하거나, 룰베이스의 자동화 툴을 제공하는 정도.
- 그러면 무료버전을 제공하는 이유가 무엇이냐? 다시 돌아가서 “미끼상품”, “부담없이 써보다가 자연스럽게 제품 주도 성장을 만들어내는 것”
- 매출을 자동화하는 관점에서 기능이 많아지고 범용성이 높아질 수록, 쓰길 원하는 고객이 많아지고 [“예상치 못한 바이럴”](https://www.notion.so/34246f3e549c80408d16f71ffd8b321f?pvs=21) 의 가치는 커질 것임

# 2. 무료버전 명세

### 제한 및 레버의 방식

- 대표적인 레버의 종류
    1. 기능 제한
    2. 사용 한도 Limit (월간 / 평생)
    3. 시간조건
    4. 비금전 대가
- 가능성 높은 레버들
    - 화이트리스트 방식
        - 핵심 기능을 화이트리스트 방식으로 목록에 나열된 것만 제공하고 나머지는 모두 제공 안함
    - 절대적 사용량 제한의 방식
        - 전체 리뷰수 N건까지 무료버전 사용 가능
        - 라이브 후 주문건 N건까지 무료버전 사용 가능
    - 절대적 기한의 방식
        - PG등록일로 부터 쇼핑몰 12개월까지만 무료 가능

        - 손실 현실화 — 놓치는 것을 수치로 보여줌

            **한계치 도달 전 단계별 알림강력 추천**

            **잠긴 기능에 "예상 효과" 표시강력 추천**
            
            단순히 자물쇠 아이콘이 아니라, "AI 리뷰 답글 사용 시 재구매율 평균 +12% 향상" 같은 수치를 함께 표시. "이걸 안 쓰면 이만큼 손해"라는 프레임.
            
            기회비용 수치화잠긴 기능 preview
            
            **이번 달 놓친 리뷰 요청 수 표시추천**
            
            한도 때문에 발송 못 한 리뷰 요청이 N건이라는 것을 보여줌. "발송했다면 예상 리뷰 +X건, 예상 매출 기여 +Y원" 형태로 기회손실을 구체화. 추정값이어도 체감 강함.
            
            **경쟁 쇼핑몰과 리뷰 수 비교조건부 추천**
            
            "같은 업종 쇼핑몰이 이번 달 평균 리뷰 47건 수집 중 — 내 쇼핑몰은 12건." 뒤처지는 것에 대한 손실감. 단, 초기엔 비교 데이터가 부족하므로 누적 후 적용.
            
            **손실 프레임의 언어 설계**
            
            "업그레이드하면 더 많이 얻을 수 있어요" (이득 프레임) vs "지금 이 기능 없이 매달 리뷰 N건을 놓치고 있어요" (손실 프레임). 후자가 전환율 2~3배 높다. 모든 넛지 카피를 손실 언어로 작성할 것.
            
            **셋업 체크리스트 + 진행률 표시강력 추천**
            
            가입 직후 "리뷰 앱 100% 준비까지 4단계 남음" 형태의 체크리스트. ① 위젯 설치 ② 첫 발송 설정 ③ 카카오채널 연동 ④ 카드 등록. 각 단계 완료 시 즉각적인 보상(기능 해제, 배지). 80%쯤 왔을 때 완성 충동이 가장 강함.
            
            4단계 체크리스트즉각 보상80% 효과
            
        - 완성 충동 — 미완성 상태가 불편하다
            
            Zeigarnik effect: 완료된 일보다 미완성된 일이 기억에 오래 남고 해결하고 싶은 충동을 만든다. 온보딩을 "완성"으로 설계해야 한다.
            
            **셋업 체크리스트 + 진행률 표시강력 추천**
            
            가입 직후 "리뷰 앱 100% 준비까지 4단계 남음" 형태의 체크리스트. ① 위젯 설치 ② 첫 발송 설정 ③ 카카오채널 연동 ④ 카드 등록. 각 단계 완료 시 즉각적인 보상(기능 해제, 배지). 80%쯤 왔을 때 완성 충동이 가장 강함.
            
            4단계 체크리스트즉각 보상80% 효과
            
            **"쇼핑몰 준비도" 점수 시각화추천**
            
            현재 상태를 0~100점으로 시각화. 카드 등록 +20점, 카카오채널 +20점, 첫 리뷰 수집 +30점 등. 점수가 낮으면 무언가 부족하다는 느낌이 들도록. "리뷰 준비도 40점 — 평균 쇼핑몰은 72점" 비교 가능.
            
            준비도 점수평균 비교행동 유도
            
            **잠긴 기능 개수 표시추천**
            
            대시보드 어딘가에 "아직 잠긴 기능 7개"가 보이면 완성되지 않은 상태로 느껴짐. 클릭하면 잠긴 기능 목록과 각각의 예상 효과가 보임. 완성 충동 + 손실 현시화 복합 효과.
            
            미완성 가시화클릭 유도기능 목록 노출
            
            **체크리스트 설계 시 주의점**
            
            카드 등록을 체크리스트 마지막 단계에 배치할 것. 앞 단계에서 이미 가치를 체감한 상태여야 카드 등록 저항이 낮아진다. 첫 번째 단계에 카드 등록을 두면 이탈률이 급등한다
            
        - 습관 형성 — 제품이 일상이 되게
            
            매일/매주 체크하는 루틴이 생기면, 없어졌을 때의 공백이 크게 느껴진다. 제품을 운영 루틴의 일부로 만드는 것이 목표.
            
            **주간 리뷰 성과 리포트강력 추천**
            
            매주 월요일 "이번 주 리뷰 7건 수집, 전환율 3.2%" 형태의 리포트를 카카오 또는 이메일로 발송. 받아보는 습관이 생기면 중단 시 공백감이 커짐. 리포트 안에 유료 기능 preview 삽입 가능.
            
            **실시간 리뷰 알림추천**
            
            새 리뷰가 달릴 때마다 푸시 알림. 긍정 리뷰는 기쁨, 부정 리뷰는 즉각 대응 욕구를 만든다. 알림을 끊으면 운영에 구멍이 생긴다는 인식 형성.
            
            **성과 대시보드 루틴추천**
            
            로그인할 때마다 "오늘의 리뷰 현황"이 첫 화면. 매일 열어보게 되면 이 화면이 없는 운영은 상상이 안 되는 상태가 됨. 주요 수치를 매일 확인하는 습관이 핵심.
            
            **리뷰 답글 알림조건부 추천**
            
            고객이 리뷰를 남기면 "답글 달 기회" 알림. 답글 달기는 유료 기능 → 알림은 무료로 오지만 실행은 유료. 욕구 형성 후 기능 잠금의 조합.
            
            **습관 형성의 핵심 지표**
            
            "D7 retention" — 가입 후 7일째에도 접속하는 비율. 이 수치가 40% 이상이면 습관이 형성되고 있는 것. 주간 리포트는 비접속 사용자도 다시 끌어들이는 재활성화 효과가 있다.
            
        - 사회 비교 — 남들과 비교해서 보여주기
            
            초기엔 데이터가 부족해 효과가 약하지만, 사용자 풀이 쌓이면 가장 강력한 넛지 중 하나가 됨. 데이터 수집과 함께 설계해야 할 미래 레버.
            
            **업종별 벤치마크중기 이후 적용**
            
            "패션 쇼핑몰 평균 리뷰 전환율 4.8% — 내 쇼핑몰 2.1%". 뒤처지고 싶지 않은 심리. 구체적인 업종 분류(패션/뷰티/식품)로 쪼개야 비교가 와닿음. 사용자 1,000개 이상 쌓인 후 신뢰도 확보.
            
            업종 평균전환율 비교갭 시각화
            
            **상위 N% 표시중기 이후 적용**
            
            "이번 달 리뷰 수집 기준 상위 23%". 상위권이면 유지 동기, 하위권이면 따라잡으려는 동기가 생김. 유료 기능 사용자 기준 상위 N% 표시는 업그레이드 넛지로 특히 효과적.
            
            퍼센타일 표시유료 사용자 대비
            
            **"이 기능 사용 중인 쇼핑몰 수" 표시지금 적용 가능**
            
            잠긴 기능 옆에 "지금 1,247개 쇼핑몰이 이 기능으로 리뷰 수집 중". 직접 비교 없이도 사회적 증거로 작동. 사용자 수 데이터는 초기부터 집계 가능.
            
            사용 쇼핑몰 수사회적 증거초기 적용 가능
            
            **사회 비교 넛지 사용 시 주의**
            
            비교 수치가 불리하게 나오면 의욕 저하 → 이탈로 이어질 수 있다. "상위 70%"보다 "평균보다 2배 빠른 성장 중"처럼 긍정적 프레임을 우선 사용. 심각하게 낮은 수치는 개선 방향과 함께 제시할 것.

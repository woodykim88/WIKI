## **매일의 운영에서 손을 떼고
매달의 성장을 에이전트에게.**

이 문서는 알파앱스 통합과 자동화라는 전략적 방향성을 위한 구조적 기반을 정리한다.
앞으로의 모든 제품·전략 논의는 이 문서의 세 축 논리(**세팅 → 운영 ↔ 매출**)를 디폴트 프레임으로 삼는다.

Why?

1. **통합 알파앱스**
    1. **필연적인 인지 범위 초과 문제**
        1. 프로덕트 다변화 : 리뷰-업셀-푸시-챗-캔버스-애즈 이렇게 많은 솔루션들이 주렁주렁 달려있다면 고객들은 인지하지도 못하고 가치 전달도 모호해질 것
        2. 기능 다변화 : 기능이 많아진다면
        3. 전체 일관성 악화 : 프로덕트 마다 체계가 다름(화이트리스트/블랙리스트의 UX)
    2. **도구에서 결과로**
        1. Tools로 접근했던 것 → 아무리 편하게 만들어도 어려워함 → 완전히 새로운 UX가 필요
    3. **도메인을 재설계 최적화**
        1. 여러 기능들을 "고객들이 이해할만한 형태로" 도메인별로 적절히 나눠서 가치 전달 필요가 있음
        2. 마케팅 & 세일즈의 목적 : 개념을 이해시키기 쉬워져야 함
        3. 실제 성과 목적 : 하나만 쓰는 것보다 여러개를 더 잘 쓸 때 더 큰 성과가 가능해야 함
        4. 편의성 증대 목적 : 지금보다 더 편리해져야 함
2. **통합 알파앱스의 목표**
    1. 알파앱스의 모든 기능은 매출향상 그자체 또는 매출향상을 위한 고객의 어려움을 해결해 주는 것
        1. 더 쉬운 이해 알파앱스가 제공하는 모든 기능을 유의미하게 쓰일 수 있도록 만들어 줌
        2. 더 많은 기능을 더 잘 사용하면 솔루션을 통한 더 큰 성과 확보 가능
3. **매출 자동화**
    1. **이커머스 회사들은 매출을 내기 위해 어떤 활동을 하는가?**
        1. **상품화**(상세페이지 생성, 썸네일 생성, 광고소재 제작)
            1. 자사몰 생성
            2. 각종 쇼핑몰 업로드(당장 없으면 내가 올려줌)
            3. 디자인 수정
        2. **쇼핑몰 운영**
            1. 가격 변동(자사몰 + 외부채널 + 아마존)
            2. 여러 세팅들 한방에 처리(채널톡, 리뷰, 업셀의 일부, 푸시의 일부 기능들, 웹사이트 개선)
            3. 배너 수정
        3. **판매 부스팅(자사몰 중심)**
            1. 씨딩
            2. 캠페인 : 애즈, 업셀, 푸시
        4. **채널링**
            1. 외부채널
            2. 글로벌 + 3PL + CS
    2. **이 중 반드시 해야 하고 뾰족한 일이 무엇인가?**
        1. 고반복 : 상품 진열(추천), 자동화 메시지, 팝업, 리뷰 관리, 상담
        2. 고시간 : 배너, 상담
        3. 고난도 : CRM 전략(세그먼트 분리, 실험, 이터레이션 반복), 광고 집행
        4. 고의존 : 프로모션/가격, 상담, CRM
4. **알파앱스의 3가지 축**
    1. 세팅
        - 현재: 각 제품의 개별 온보딩이 파편화되어 있음.
        - 목표: 회원가입 + 카페24 연동 → 10분 내 알파앱스 전 제품 기본 세팅 완료.
        - 핵심 작업: 공통 온보딩 레이어 설계, 외부 플랫폼 인증 통합, 브랜드 기반 기본값 자동 생성.
    2. 운영
        - 현재: Canvas·Review·Chat이 각자 운영 작업을 줄이지만 통합되지 않음.
        - 목표: "매일 로그인할 필요 없는 쇼핑몰". 사람이 필요한 순간에만 알림.
        - 핵심 작업: 운영 대시보드 통합, 운영자 개입 필요 순간 정의, 좁은 맥락 AI 판단 품질 고도화.
    3. 성장
        - 현재: 알파업셀·알파푸시가 프리셋 기반으로 동작. 알파애즈 개발 예정.
        - 목표: 3C 기반 캠페인 기획·소재 생성·집행·A/B 보고의 자율 사이클.
        - 핵심 작업: 매출 3제품 완성, 에이전트 판단 아키텍처 구축, 3C 데이터 파이프라인 구축.
    4. 운영 + 성장 + 세팅
        1. 더 큰 레이어, 그러나 크게 3개의 모든 기능이 연관된 유즈케이스는 아직 떠오르지 않음.
        2. 그러나, "목적 없는 기능 필요" : 상품 정보 수정, 재고 정보 수정, 주문 정보 수정… 마치 모든 API에 접근할 수 있는 권한만 획득한 상태 → 마스터 에이전트가 의사결정 실행

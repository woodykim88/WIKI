## #3 쿠폰

#### 공통 정책

쿠폰의 종류는 할인, 체험, 크레딧 쿠폰 존재

고객사는 발급받은 쿠폰코드를 대시보드에 입력하여 적용하는 형태로 쿠폰을 활성화 시킴

전체 쿠폰 사용 가능 시점

사이트 상태 중 “앱설치, 사이트 생성, 온보딩중, 필수세팅 완료, 라이브”에만 사용 가능하며, 결제실패중, 계약구독 중 등에 사용 불가(참고: [알파솔루션 기능 중단표 231023](https://docs.google.com/spreadsheets/d/1IHiCuDJTUvpeYIzUkJC0VRvjx3IMQ_5BAz0ITQyMbDk/edit#gid=0))

또한 아래 개별 쿠폰의 사용 가능 시점과 AND조건으로 작동

쿠폰 중단

슈퍼관리자는 이미 발행된 쿠폰을 비활성화 시켜서 새로운 고객사가 해당 쿠폰을 사용할 수 없도록 방지 가능

이미 고객사에 적용된 쿠폰은 정상적으로 사용이 가능합니다.

#### 적용 가능 조건

적용 범위에 포함되는 프로덕트/패키지가 최소 1개 이상 체험구독 상태가 “사용중” 또는 정기구독 중 하나라도 “예약중, 대기중, 사용중”일 때 쿠폰 사용 또는 예약 가능

또는 아직 구독이 없는 상태에서 쿠폰 사용 예약 가능

현재 아무런 과금중인 구독이 없고, 체험구독만 있다면 다음 예약중인 구독이 시작될 때 최초로 사용중인 구독에 적용됨

쿠폰 사용 후 계약구독 또는 슈퍼관리자 구독예약(구독 수정) 시에도 쿠폰 적용 불가

#### 적용 시점과 적용 기간

![](https://www.notion.so/image/https%3A%2F%2Fprod-files-secure.s3.us-west-2.amazonaws.com%2F0cf20186-0724-4a80-b423-3d24fc7470e1%2Fcb62331a-d2d1-46b7-bedd-8c7f14617cbc%2FUntitled.png?table=block&id=5af6a550-2de9-46b6-88e9-46b0bfbb3c9a&spaceId=0cf20186-0724-4a80-b423-3d24fc7470e1&width=1020&userId=2a4d872b-594c-813f-b33b-000213cafe76&cache=v2)

과금이 시작되기 전(즉, 정기구독이 예약중인 경우) 쿠폰이 먼저 적용되어 할인된 금액으로 결제 가능

결제 전, 대기중인 상태에서 예치금등이 이미 결제된 경우, 또는 과금이 이미 완료된 경우 쿠폰은 다음날 자정부터 적용됨(구독 변경이 일어나는 경우, 구독 날짜를 공유하기 때문에 쿠폰 적용으로 불이익을 받지 않도록 이같은 정책이 존재하는 것)

쿠폰이 적용된 기간은 구독 기간과 독립적으로 “사이트”단위에 적용됨

구독내역에는 해당 구독 기간 동안 적용되는 쿠폰의 할인액만 계산해서 보여줌(실제 적용될 수 있는 쿠폰의 기간이 더 길어도 현재 청구서 내에서의 할인 액을 보여준다는 뜻)

#### 최대 할인율과 여러 쿠폰 적용 시 기간 분리

여러 쿠폰이 적용되는 경우, 남은 구독 기간 동안 자동으로 최대 할인율이 적용되어 계산됨

여러 쿠폰이 적용 되고 쿠폰 기간이 서로 다른 경우 자동으로 계산 되어야 함

#### 할인 쿠폰 적용 후 구독 변경 조건

할인 쿠폰 적용 시 구독 변경이 일어나는 경우

과금구독 진행 중 쿠폰 적용 시 구독 변경이 일어남

할인 쿠폰 적용 시 구독변경이 일어나지 않는 경우

쿠폰 적용이 가격에 영향을 주지 못하는 경우: 구독을 새롭게 만들지 않고 현재구독의 쿠폰 적용 내역만 변경

현재 구독 내에서 더 높은 할인율의 쿠폰이 적용 되어 할인액의 변동이 일어나지 않는 경우

해당 구독 프로덕트와 패키지 종류에 할인 종류가 일치하지 않는 경우 ⇒ 쿠폰이 사용되지만 할인 금액이 적용되지는 않음

쿠폰이 적용되어 청구서 자체는 Update 되나, Revision되지 않음

참고: [📜구독 시스템 - 구독 변경 시 Update(청구서 수정이지, 구독변경이 아님)와 Revision(청구서 개정)](https://www.notion.so/Update-Revision-cdff839defda43c8acaaf87a0bcacd92?pvs=24#720c2f9044ba45209a9a99a6d33ee95d)

### 체험 쿠폰

| 설정 명칭 | 설명 | 기본값 | 범위 |
| --- | --- | --- | --- |
| 체험기간 연장 | 추가 연장될 기간을 지정 (ex, 기본 14일 체험이 제공되므로, 추가로 16일이 제공된다면 총 30일이제공됨) | 16 | 0초과 ~ 365 이하, 정수 |
| 적용 범위 | 모든 패키지와 프로덕트를 지정해서 할인이 적용될 범위 선택 가능, 최소 1개 선택 필수 | 모두 선택 | 최소 1개 필수 |

#### 적용 가능 조건

적용 범위에 포함되는 프로덕트/패키지가 최소 1개 이상 [재설치 상태](https://www.notion.so/cdff839defda43c8acaaf87a0bcacd92?pvs=25#e70848b5814f46e9a064988e5a801671) 가 아닌 경우, 그리고 해당 프로덕트/패키지에 쿠폰을 한 번도 적용한 적이 없어야 사용 가능.(즉, 프로덕트를 현재 이용하지 않아도 적용 가능하며 선착순 1번만 사용 가능하다는 뜻)

또는 아직 구독이 없는 상태에서 쿠폰 사용 예약 가능

현재 체험구독을 이용하고 있다면 즉시 체험기간이 연장 되며, 쿠폰을 사용하고 30일 이내까지 체험기간을 생성할 수 있다면 해당 프로덕트/패키지도 연장 가능

상술한 설명과 같은 의미지만, 쿠폰 적용 후 기간 내 다운그레이드 이후 재업그레이드 시([재설치 상태](https://www.notion.so/cdff839defda43c8acaaf87a0bcacd92?pvs=25#e70848b5814f46e9a064988e5a801671) 인 경우) 쿠폰 재적용 불가. 물론, 체험구독은 최초 1번만 생성되니 체험구독 자체가 없을 것.

계약구독 이용 시, 슈퍼관리자 구독예약 시 쿠폰 사용 불가

쿠폰 사용 후 계약구독 또는 슈퍼관리자 구독예약(구독 수정) 시에도 쿠폰 적용 불가

### 크레딧 쿠폰

| 설정 명칭 | 설명 | 기본값 | 범위 |
| --- | --- | --- | --- |
| 크레딧 금액 |  | 없음 | 0초과 ~ 100,000,000 이하 가능 |

쿠폰별 적용가능 조건은 따로 없음

프로모션1. 체험기간 연장: 업셀링 프로모션과 쿠폰

체험기간 연장 프로모션: 최초 서비스 개시 후 14일 이전까지 업그레이드(프로덕트 추가 시) +16일 기존 체험구독이 30일로 연장됨

최초 체험기간 연장 쿠폰: 최초 체험기간 14일 이내에만 사용할 수 있는 쿠폰 제공

체험기간 추가 쿠폰: 중복으로 적용 가능한 쿠폰 제공 프로덕트/패키지별 쿠폰 제공

프로모션2. 체험기간 내 카드등록 시 첫구독 50%할인 쿠폰 적용

구독이 없는 최초 온보딩 시점에 카드를 등록하고 온보딩이 완료되거나, 체험구독 기간 내에 카드를 등록한 경우 50%할인 쿠폰 적용 ( 연간 구독은 어떻게할까..? 적용 안됨? )

## #5 결제 수단

#### 결제 수단의 결제 상태

어떤 청구서 하나라도 결제 실패 시 결제수단상태는 “결제 실패중”으로 변동

어떤 청구서 하나라도 결제 성공 시 결제수단상태는 “결제 정상”으로 업데이트

“결제실패중”에서 → “결제 정상”으로 변화 시 모든 결제 실패중인 청구서 결제 진행 (즉, 타청구서 결제 성공 시 함께 결제)

#### 카드 등록/수정/삭제

최소한 결제 가능한 카드가 1개는 확보해야 함 즉, 카드 삭제 후 카드를 추가하는 것은 불가능하고 카드 추가 후 삭제만 가능

카드는 총 2개 등록 가능하며, 주/부결제 카드를 지정할 수 있음.

주카드 삭제 시 자동으로 부카드가 주카드로 변환

카드를 모두 삭제하고 싶다면 “해지완료” 상태에서만 카드 삭제 가능

무료버전구독이라도, 카드를 등록하지 않으면 서비스를 사용할 수 없음

#### 카드 미등록 상태와 체험 구독/무료버전구독

포인트 잔액만큼 메시지 발송 가능 (무료 버전구독, 체험구독)

잔액 이상 발송 원할 경우 포인트 충전, 카드 등록 필요

후불비용 보조청구서 생성

#### 카드 미등록 상태와 계약구독의 계좌이체 방식

계약구독의 계좌이체 방식 선택 시 결제 수단 상태 - “카드 미등록”상태에서 계약구독 청구서 발행 가능

계약구독의 계좌이체 방식 선택 시 “계좌이체 이용중” 종속 상태 적용, 계약구독-계좌이체 방식을 더이상 사용하지 않는다면 해당 종속 상태 제거 필요

카드등록면제 “카드미등록” 상태 시 “메시지포인트 자동결제 불가”상태로 메시지포인트를 자동 결제하지 않음, 슈퍼관리자에서 “구독 시작” 가능

#### 결제 수단 상태

메인 상태: 카드등록, 카드미등록

종속상태: 카드결제 정상, 카드결제 실패중, 계좌이체 이용중

\-: 아직 결제 시도가 되지 않은 초기 상태

“카드 결제 정상”: 모든 종류의 청구서 카드결제 시도 후 성공 시

#### 크레딧

오로지 쿠폰과 임의 청구서 의해 제공될 수 있는 금원으로, 프로모션 성격으로 지급되어 환급성이 없는 금원

#### 메시지 포인트

⚠️메시지 포인트에 관하여 후대에 알립니다..

메시지포인트의 음수 가능성

메시지 발송 요청 시점에 메시지포인트 잔액을 확인 후, 부족 시 코어시스템에 충전 요청 및 충전 완료 후 메시지발송 진행

메시지 포인트가 처음 예상보다 적어지거나 더 많아질 수 있음

알림톡 등 발송 실패 시 (예상보다 덜 씀)

알림톡 등 발송 후 대체문자 발송 시 (예상보다 더 많이 씀)

ex) 잔액 5,000원, 15,000원 발송 요청 시 → 10,000원 충전 필요, 예상으로는 발송 후 잔액은 0원이지만, 실제로는 메시지 발송에 완전히 실패하는 경우와 알림톡 발송 후 대체문자 발송이 가능하여, 메시지 비용이 예상보다 적거나 많이 소진될 수 있음 따라서 잔액이 음수 또는 그대로 가능

# P-Reinforce Wiki (The Autonomous Gardener)

P-Reinforce는 사용자가 입력한 파편화된 지식을 스스로 분류하고, 구조화하며, 연결하는 **"강화학습 기반 영속적 위키 결합 시스템"**입니다. 이 저장소는 AI 에이전트가 단독으로 1차 지식 큐레이션을 담당하며, 사용자의 피드백(칭찬, 수정, 방치)에 따라 스스로 지식의 폴더링 정책을 최적화해 나갑니다.

## 🎯 핵심 목표 (Core Mission)
1. **자동화된 지식 수집:** 사용자가 `00_Raw` 폴더에 던져둔 파편화된 원시 데이터를 실시간 리딩하여 구조화된 정보로 변환합니다.
2. **동적인 자율 폴더 구조:** 고정된 정적 분류 프레임 워크에 얽매이지 않고 지식의 맥락에 따라 스스로 '폴더 트리'를 설계하고 확장합니다.
3. **거대 외부 뇌 구축:** 단일 지식을 최소 2가지 이상의 기존 개념과 엮어 거대한 '양방향 그래프'로 만듭니다.
4. **타임라인 보존:** 구조가 변화하고 지식이 진화하는 모든 과정을 GitHub 커밋으로 남겨 언제든 과거 버전을 추적하게 합니다.

## 📂 디렉토리 구조 및 역할 (The Structure)
에이전트가 인식하고 자율적으로 관리하는 표준 폴더 계층도입니다.

```text
root/
├── 00_Raw/                 # [필수/입력단] 가공되지 않은 모든 원시 데이터를 날짜별로 보관
│   └── YYYY-MM-DD/         # 사용자는 이곳에만 텍스트 조각을 모아두면 됩니다.
│
├── 10_Wiki/                # [출력단] 에이전트가 가상 뇌구조를 구성하는 메인 무대
│   ├── 🛠️ Projects/        # (목표 중심) 현재 진행 중인 목표 별 묶음
│   ├── 💡 Topics/          # (개념 중심) 자율적으로 생성하고 관리하는 태그 및 챕터들
│   ├── ⚖️ Decisions/       # (의사 결정) 특정 문제에 어떻게 판별을 내렸는지 남기는 공간
│   └── 🚀 Skills/          # (실행 중심) 사용자가 직접 만든 에이전트 프롬프트나 패턴 보관
│
└── 20_Meta/                # [시스템 전용] 에이전트 최적화 데이터
    ├── Graph.json          # 지식 간 연결 그래프 좌표 (시각화 목적용)
    ├── Policy.md           # 사용자 피드백이 담긴 RL(강화학습) 분류 정책
    └── Index.md            # 전체 지식의 목차 (Table of Contents)
```

## 🧠 판단 및 학습 원리 (The RL Logic)

지침 설계에 따라 P-Reinforce 에이전트는 아래의 보상 함수($R$)를 1.0(Max)에 가깝게 유지하려 시도합니다.
`R = w1(범주 분류 정확도) + w2(서로 연결된 그래프의 조밀성) + w3(사용자 만족도)`

- **스스로 확장:** 기존 폴더와 유사도가 85% 이하면 새로운 신규 개념을 탐색하여 스스로 새 폴더를 팝니다.
- **스스로 리팩토링:** 단일 폴더에 문서가 12개를 초과하면 스스로 하위 목차를 구성하여 구조를 리팩토링할 것을 제안합니다.
- **문서 규격화:** 모든 문서는 `[UUID, 예측 신뢰도, 핵심 1줄 통찰(Karpathy Summary), 모순 발견 내역]` 등을 정형 마크다운 포맷에 맞게 재작성됩니다.

## 💡 에이전트 훈련 가이드 (사용자 피드백 체계)
이 저장소는 여러분의 액션에 따라 지식망의 방향이 달라집니다.

- **👍 칭찬(보상):** "이 분류 좋아, 완벽해." 라고 할 경우 유사 주제에 대한 분류 가중치가 극대화됩니다.
- **✏️ 수정(벌점):** "이 지식은 '코딩'이 아니라 '비즈니스' 폴더로 옮겨야 해." 라고 교정시, 카테고리의 경계선을 변경합니다.(Boundary Shift)
- **😶 방치(암묵적 동의):** 에이전트가 구성한 구조를 사용자가 그대로 쓴다면 에이전트는 이것이 최적의 상태임을 확신하고 정책을 고정시킵니다.

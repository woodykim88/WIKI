# [[리뷰 무료버전 ver 1.0]]

## 📌 한 줄 통찰 (The Karpathy Summary)
> 알파리뷰 무료버전은 당장의 라이트 요금제 수익을 일부 포기하더라도, 초저가 경쟁사로 이탈하는 셀러들을 알파앱스 생태계 내에 무기한 락인(Lock-in) 시켜 추후 업셀/푸시 등으로 교차 판매(Cross-sell)하기 위한 전략적 방어선이다.

## 📐 도메인 모델 개요 (Domain Model Overview)

| # | 도메인 | 핵심 개념 |
|---|--------|-----------|
| 1 | Problem (해결 과제) | 월 7.9만 원의 라이트 버전조차 부담으로 느끼는 셀러의 초저가/타사 무료 스킨 리뷰로의 이탈 방어. |
| 2 | Target (대상 고객) | 비용 부담으로 이탈하려는 기존 라이트 셀러 및 리뷰 에이드 등 초저가 경쟁 서비스 탐색 셀러. |
| 3 | Core Value (핵심 가치) | 필수 리뷰 기능(요청, 빠른 리뷰, 기본 게시판 위젯, 제한된 스태프 리뷰) 제공 극대화 + 락인. |
| 4 | Risk (카니발라이제이션) | 기존 유료 셀러의 무료 다운그레이드 위험. 이를 상쇄할 스펙 제한(메시지 비용 할증, 갯수 제한) 필요. |
| 5 | 기능 오픈/제한 정책 | 목록/관리/통계는 오픈. 워터마크 노출("알파리뷰 제공")로 제거 심리 유도. 설정 탭은 템플릿 기본값에 고정. |

## 📖 핵심 도메인별 상세 (Domain Deep-Dive)

### 📖-1. Problem & Target (왜 무료버전인가?)

- **문제 상황:** 알파리뷰의 최하위 과금 체계인 라이트(Light) 버전의 79,000원/월 비용조차 부담스러워하는 초기 또는 영세 셀러들이 존재.
- **이탈 경로:** 이들은 기능의 품질보다 '리뷰를 쌓을 수 있다'는 본질적 목표만 충족하면 되므로, 리뷰에이드 등 초저가 서비스나 타사(다자인) 스킨 내장 기본 리뷰로 이탈함.
- **목표 타겟:** 비용 부담으로 이탈하는 기존 라이트 셀러 및 초저가 경쟁사를 찾는 신규 셀러.
- **도입 대전제 (The Grand Strategy):** 이들을 놓치는 대신, 알파앱스 생태계 내에 확실한 락인(Lock-in) 효과를 확보함. 이들이 성장했을 때, 푸시(Push), 업셀(Upsell) 등 연계 서비스에 자연스럽게 노출시키는 그물망 구축.

---

### 📖-2. Core Value & Specification (제공 가치 및 제한)

유료 과금 유도를 위한 **기능 잠금(Lock)**과 **넛지(Nudge)**가 밸런스의 핵심.

**[제공 기능 제한 스펙]**
- **리뷰 요청 (알림톡):** 기본 제공이나, 무료 갯수 초과 시 건당 메시지 비용을 30원 등 **더 비싸게 할증** 부과 → 라이트 버전 전환 넛지 포인트.
- **빠른 리뷰:** 기본 기능 제공.
- **위젯:** 가장 기본적인 '게시판 리뷰' 위젯(형태) 1종류만 제공. 포토리뷰 제공 여부는 검토 중.
- **스태프 리뷰:** 라이트 버전의 30% 수준으로 생성 갯수 강력 제한. 최대값을 채우면 유료로 레버리지를 일으키도록 유도.
- **워터마크 추가 (아이디어):** 위젯에 UI적으로 *"알파리뷰가 제공했습니다"* 워터마크 강제 노출. 이를 제거하고 싶은 업체의 심리를 이용한 유료 결제 유도. (비용 지불 의사 확보)

---

### 📖-3. 기능 탭별 UI 제어 정책 (메뉴 권한)

무료 버전만의 가치 전달과 유료 결제 유도(설명 & 넛지 목적)를 위해 메뉴별로 권한을 나눔. 언급되지 않은 기능은 **Lock** 상태 유지.

| 메뉴 | 제공 정책 | 비고 |
|---|---|---|
| 통계 및 인사이트 | **제공** | 데이터 기반 동기 부여 |
| 리뷰 목록 / 관리 | **제공** | 본질적 리뷰 운영 기능 |
| 스태프 리뷰 | **제한적 제공** | 갯수 상한 존재 |
| 디자인 설정 | **Lock (기본값 제공)** | 커스텀 불가, 템플릿(기본값) 고정 |
| 리뷰 요청 설정 | **Lock (기본값 제공)** | 텍스트/이미지 변경 불가, 템플릿 고정 |
| 포인트 지급 설정 | **Lock (기본값 제공)** | 템플릿 고정 |
| 알림톡/메시지 설정 | **Lock (기본값 제공)** | 템플릿 고정 |
| 공지사항 설정 | **Lock (기본값 제공)** | 템플릿 고정 |
| 빠른 리뷰 설정 | **Lock (기본값 제공)** | 템플릿 고정 |

### 📖-4. 수익화 전략 및 이탈 위험 (Risk Management)

- **가장 큰 두려움(리스크):** 기존 라이트/베이직 등 유료로 결제하던 셀러들이, 기능에 만족하여 대거 무료버전으로 다운그레이드(하향 전환)하는 카니발라이제이션 위협.
- **방어 기제 전략:** 
    - 메시지 무료 발송 건수를 극도로 줄이고 초과분을 비싸게 과금.
    - 스태프 리뷰 작성 갯수 제한 (상한 도달 시 결제 유도).
    - 초기에는 적절한 밸런스를 찾을 때까지 제한 범위를 A/B 테스트.
- **Pricing 후속 전략:** 무료 버전 만족 고객이 유료로 전환 안 한다고 해서, 기존 라이트/베이직 요금을 올리는 것은 불가 (오히려 더 안 올라감). 차라리 푸시, 업셀 무료 버전까지 묶어 **'알파앱스 통합 권역'**으로 무과금 유저 풀(Pool)을 형성한 후, 통합 솔루션 관점에서 유료 전환을 거시적으로 시도하는 것이 효과적임.

## ⚠️ 모순 및 업데이트 (Contradictions & RL Update)

- **[포토리뷰 정책 미정]** 본문 아이디어 섹션에 "포토리뷰는 제공해야 하지 않을까?"라는 미해결 의문이 존재. 위젯 제공 시 사진 포함 여부는 전환율에 큰 영향을 미치므로 최종 검토 후 FSD 단계에서 확정 필요.
- **[이중 목표 모순 주의]** "타사 이탈을 막기 위해 쓸만한 무료 기능을 준다"와 "유료 결제를 유도하기 위해 기능을 불편하게(기본값 고정, 워터마크) 만든다" 사이의 텐션이 존재. 온보딩 경험 최적화를 통해 এই 모순을 시공간적으로 해소해야 함 (가입 초반은 쓸만하게, 사용할수록 제약을 느끼게).

# [[코어 시스템 Type/Status 정의]]

## 📌 한 줄 통찰 (The Karpathy Summary)
> 코어 시스템의 모든 비즈니스 도메인(앱, 플랜, 구독, 결제, 청구서, 웹훅, 알림)을 IntegerChoices/TextChoices Enum으로 정의한 단일 진실의 원천(SSOT)이다.

## 📐 도메인 모델 개요 (Domain Model Overview)

| # | 도메인 | 핵심 개념 |
|---|--------|-----------|
| 1 | 앱/서비스/플랜 | App__Type(5종), Service__Type(20종), Plan__Type(13단계) |
| 2 | 구독 라이프사이클 | Subscription__Type(4종), 기간(7종), 전환(3종), 변경(3종), 실행(12종), 생성사유(13종) |
| 3 | 결제/청구 | Payment__Method(2종), Bill__Type(3종), CustomBill__Type(16종), AssistantBill__Type(6종) |
| 4 | 정산 | AdjustmentSign(2종), Column(3종), Action(12종), DailyRecord 3종류 |
| 5 | 할인/쿠폰 | Coupon__Type(3종), Discount__Method(2종), Discount__Type(7종) |

### 📖-10. 상태(Status) 머신

**샵 메인상태:**
```

→ 구독종료=600 → 해지완료=700 / 리드이탈=800 / 앱삭제=900
```

**구독 상태:**
```

분기: 결제실패=400, 취소=500, 취소_구독변경=501, 취소_실패확정=502,

```

**결제 상태:**
```
대기=100 → 성공=200 / 실패=400 / 미수=402 / 실패확정=403 / 환불=600
```

**ShopService 상태:**

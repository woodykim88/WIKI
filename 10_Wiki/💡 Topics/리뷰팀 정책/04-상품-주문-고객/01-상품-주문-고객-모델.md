# 상품, 주문, 고객

## 1. 도메인 개요

### 1.1 이 도메인이 해결하는 비즈니스 문제

알파리뷰는 쇼핑몰의 리뷰 수집 및 관리 서비스이다. 리뷰가 작성되려면 "어떤 상품을", "누가 주문했는지"를 알아야 한다. **상품(Product)**, **주문(Order/OrderItem)**, **고객(CustomerOption)** 세 도메인은 리뷰 시스템의 기반 데이터를 제공하며, 다음과 같은 비즈니스 문제를 해결한다.

- **리뷰 작성 자격 판단**: 주문 상품(OrderItem)별로 리뷰 작성 허용 여부를 세분화하여 관리한다.
- **혜택(포인트) 지급 제어**: 상품 가격, 품절 상태, 정책 기반 제한 조건 등을 통해 혜택 지급 가능 여부를 결정한다.
- **리뷰 요청(알림톡) 발송 추적**: 주문 상태 변경에 따라 메시지 발송 이력을 비트맵으로 관리한다.
- **상품 간 리뷰 연결**: 관련 상품들의 리뷰를 묶어서 노출할 수 있도록 연결 그룹을 구성한다.
- **리뷰 통계 집계**: 상품별, 고객별 리뷰 수/평점/매출 등의 집계 데이터를 캐싱한다.

### 1.2 핵심 개념 및 용어 정의

| 용어 | 설명 |
|------|------|
| **Product** | 쇼핑몰 플랫폼(Cafe24 등)에서 동기화된 상품 정보. 리뷰 통계 캐시 데이터를 함께 보유한다. |
| **Order** | 하나의 주문 건. 고객 1명의 1회 주문에 대응한다. |
| **OrderItem** | 주문 내 개별 상품 단위. 리뷰 작성/혜택 지급의 실질적인 기준 단위이다. |
| **CustomerOption** | 쇼핑몰 내 고객 정보. 회원/비회원/네이버페이 구매자 등 고객 유형을 구분한다. |
| **ProductConnectionGroup** | 상품 간 리뷰 연결 그룹. 단방향 또는 그룹 연결로 리뷰를 공유한다. |
| **ProductOptionSet** | 상품 옵션명을 동일하게 처리하기 위한 매핑 설정이다. |
| **short_code** | Order, OrderItem, CustomerOption 각각에 생성되는 7자리 UUID 기반 고유코드이다. |
| **message_check** | 10자리 비트맵 문자열로, 각 자리가 특정 메시지 발송 상태를 나타낸다. |
| **permit_*  플래그** | OrderItem에 설정되는 리뷰 유형별 작성 허용 여부 불리언 필드들이다. |

### 2.1 상품 (Product)

#### 2.1.2 ProductOption

상품에 종속된 옵션 항목 (예: 색상, 사이즈).

| 필드 | 타입 | 설명 |
|------|------|------|
| `shop_id` | IntegerField | 쇼핑몰 고유 ID |
| `product` | ForeignKey(Product) | 상품 FK (`related_name="option_set"`) |
| `option_name` | CharField (null) | 옵션 이름 (예: "색상", "사이즈") |

#### 2.1.3 ProductOptionValue

옵션 항목의 구체적인 값 (예: "블랙", "XL").

| 필드 | 타입 | 설명 |
|------|------|------|
| `product_option` | ForeignKey(ProductOption) | 옵션 FK (`related_name="option_value_set"`) |
| `option_value` | TextField | 옵션값 이름 |

#### 2.1.4 ProductConnectionGroup

상품 간 리뷰를 연결하는 그룹 단위.

| 필드 | 타입 | 설명 |
|------|------|------|
| `shop_id` | IntegerField | 쇼핑몰 고유 ID |
| `connect_name` | CharField (null) | 리뷰 연결명 |
| `connect_type` | IntegerField | 리뷰 연결 유형 (choices: `ProductConnectionGroup__ConnectType`) |
| `connect_direction` | IntegerField (null) | 리뷰 연결 방향 (단방향 연결 시에만 사용. 그룹 연결인 경우 null) |

**`ProductConnectionGroup__ConnectType` 열거값:**

| 값 | 한글명 | 설명 |
|----|--------|------|
| 100 | 단방향연결 | A 상품의 리뷰를 B 상품에만 노출 (B -> A 방향 불가) |
| 200 | 그룹연결 | 그룹 내 모든 상품이 리뷰를 공유 |

**`ProductConnectionGroup__ConnectDirection` 열거값:**

| 값 | 한글명 | 설명 |
|----|--------|------|
| 100 | 오른쪽 | 화살표 방향 -> (연결상품에서 피연결상품으로) |
| 200 | 왼쪽 | 화살표 방향 <- (피연결상품에서 연결상품으로) |

#### 2.1.5 ProductConnectionTarget

리뷰 연결 그룹에 포함되는 개별 상품.

| 필드 | 타입 | 설명 |
|------|------|------|
| `product` | ForeignKey(Product) | 상품 FK (`related_name="connection_target_set"`) |
| `product_group_connect` | ForeignKey(ProductConnectionGroup) | 연결 그룹 FK (`related_name="connection_target_set"`) |
| `product_type` | IntegerField | 상품 연결 유형 (choices: `ProductConnectionTarget__ProductType`) |

**`ProductConnectionTarget__ProductType` 열거값:**

| 값 | 한글명 | 설명 |
|----|--------|------|
| 100 | 연결상품 | 리뷰를 보내는(send) 주체 상품 |
| 200 | 피연결상품 | 리뷰를 받는(receive) 대상 상품 |

> 그룹 연결인 경우 모든 상품이 "연결상품"으로 정의된다.

##### message_check 비트맵 상세

`message_check`는 10자리 문자열로 각 자릿수의 의미는 다음과 같다:

| 인덱스 | 의미 | 관련 메서드 |
|--------|------|-------------|
| 0 | 주문 직후 발송 여부 | `apply_after_order()` |
| 1 | 송장입력 직후 발송 여부 | `apply_after_invoice()` |
| 2 | 배송 완료 알림톡 발송 여부 | `apply_after_delivered_alimtalk()` |
| 3 | 비회원 알림톡 발송 여부 | `apply_after_nonmember_alimtalk()` |
| 4 | 비회원 링크 | - |
| 5 | SNS 리뷰 혜택 제공 여부 | `apply_sns_review_rewarded()` |
| 6 | 일반 리뷰 혜택 제공 여부 | - |
| 7 | (예약) | - |
| 8 | 리마인더 발송 여부 | `apply_reminder()` |
| 9 | (예약) | - |

- `"0"`: 미발송 (발송 가능)
- `"1"`: 발송 완료 (또는 발송 불가)

## 3. 비즈니스 규칙

### 3.1 상품-리뷰 연동 규칙

#### 3.1.1 리뷰 통계 갱신 메커니즘

상품의 리뷰 통계는 실시간이 아닌 **배치 갱신 방식**으로 관리된다.

1. 리뷰가 생성/수정/삭제되면 해당 상품의 `is_review_changed` 플래그가 `True`로 설정된다.
2. `Product__Feature.update_review_number()` 메서드가 실행되면:
   - `is_review_changed = True`인 모든 상품을 대상으로 리뷰 통계를 재계산한다.
   - 게시됨(`posting_status = 20`) 상태이고 노출 가능(`is_visible = True`)이며, 원본 리뷰(`creater_type != 100`)가 아닌 리뷰만 카운트한다.
   - `review_number_count`, `review_ratings`, `review_photo_count`, `review_video_count`, `review_survey_count`, `review_number_lately_count`를 갱신한다.
   - 연결된 상품(`connect_info`)이 있으면 연결 리뷰 통계(`review_number_connect_count`, `review_normal_connect_count`, `review_sns_connect_count`, `review_ratings_connect`)도 갱신한다.

3. 주문이 발생하면 `is_sales_changed = True`로 설정되며, 같은 배치에서 최근 14일 판매 개수(`sales_number_lately`)가 갱신된다.

#### 3.1.2 상품 간 리뷰 연결 규칙

리뷰 연결은 두 가지 유형을 지원한다:

**단방향 연결 (`connect_type = 100`):**
- 연결상품(send, `product_type = 100`)의 리뷰가 피연결상품(receive, `product_type = 200`)에 노출된다.
- 피연결상품의 `connect_info`에 연결상품의 `product_code`가 추가된다.
- 연결상품의 `connect_info`는 변경되지 않는다.

**그룹 연결 (`connect_type = 200`):**
- 그룹 내 모든 상품의 리뷰가 서로 공유된다.
- 모든 상품의 `connect_info`에 그룹 내 다른 상품들의 `product_code`가 추가된다.

연결 생성/삭제 시 관련 상품의 `is_review_changed`가 `True`로 설정되어, 다음 배치에서 연결 리뷰 통계가 갱신된다.

#### 3.1.3 카테고리 기반 리뷰 제한

`category_list` 필드는 `|` 구분자로 된 카테고리 플랫폼 ID 목록이다 (예: `"|111|222|333|"`).

리뷰 작성 또는 혜택 지급 시, `ShopPermitRelation`에 설정된 제외 카테고리와 상품의 `category_list`를 대조하여 허용 여부를 판단한다. 하위 카테고리까지 포함하여 검사한다.

#### 3.1.4 해시태그 구성 규칙

상품의 `hashtag_all`은 세 가지 원천의 해시태그를 합산하여 구성된다:
1. 전역 해시태그 (ShopOption의 hashtag)
2. 상품별 해시태그 (ReviewHashtagProduct)
3. 카테고리별 해시태그 (ReviewHashtagCategory)

최대 5개까지 공백으로 연결하여 저장한다. 카테고리 변경 시 해시태그가 자동으로 재계산된다.

### 3.2 주문-리뷰 요청 연동 규칙

#### 3.2.1 리뷰 작성 허용(permit) 결정 로직

OrderItem 생성 시, 각 리뷰 유형별 작성 허용 여부가 결정된다. 결정 로직은 `OrderItemMigration__Util.get_postcreated_data()` 및 `_get_permit()` 메서드에 구현되어 있다.

**판단 순서:**

1. **비회원/결제경로 체크** (선행 조건):
   - 비회원이면서 `use_posting_nonmember = False`이면 모든 permit이 `False`
   - 네이버페이 주문이면서 `use_posting_npay = False`이면 모든 permit이 `False`
   - 지그재그 주문이면서 `use_posting_zpay = False`이면 모든 permit이 `False`
   - 카카오페이 주문이면서 `use_posting_kakaopay = False`이면 모든 permit이 `False`
   - 스마트스토어 주문이면서 `use_posting_marketplus = False`이면 모든 permit이 `False`

2. **일반 리뷰(`permit_normal`) 판단**:
   - `ShopPermit`의 `permit_normal` 확인
   - 제외 상품 리스트 확인 (`ShopPermitProduct`)
   - 제외 카테고리 리스트 확인 (`ShopPermitCategory`, 하위 카테고리 포함)

3. **SNS 리뷰(`permit_sns`) 판단**: 일반 리뷰와 동일한 로직으로 독립 판단

4. **하위 리뷰 유형 판단** (일반 리뷰 허용 시에만):
   - `permit_quick`: 빠른 리뷰
   - `permit_survey`: 설문 리뷰
   - `permit_video`: 동영상 리뷰
   - `permit_keyword`: 키워드 리뷰
   - 각각 독립적으로 제외 상품/카테고리 체크

5. **전체 리뷰 허용(`permit_review`)**: `permit_normal` 또는 `permit_sns` 중 하나라도 `True`이면 `True`

#### 3.2.2 혜택 지급 제한 규칙

OrderItem 생성 시 혜택 지급 가능 여부(`is_permit_reward`)가 결정된다.

**혜택 지급 기한 (`reward_disabled_at`) 계산:**
- 일반 리뷰: `배송 시작 일시 + 3일 + 수령 후 지급기한(shop_policy.reward_condition_delivery)`
- 네이버 블로그 리뷰: `현재 시각 + 30일 + 수령 후 지급기한`

**최대 포인트 (`max_point`) 계산:**
- 정액제: `shop_policy.max_total_point_fixed` 값 사용
- 정률제: `상품가격 * shop_policy.max_total_point_percentage / 100`

**혜택 제한 사유 (`permit_reward_reason`) 발생 조건:**
- 상품 가격이 최소 금액(`reward_condition_price`)보다 낮은 경우: `"상품 최소 가격 미달"`
- 상품 또는 카테고리가 제외 목록에 포함된 경우: `"지급제한상품"`
- 미검증 리뷰이면서 미검증 혜택 불가 설정인 경우: `"미검증 리뷰 혜택 지급 불가"`

**품절/미진열/미판매 상품 혜택 지급:**
- `use_reward_sold_out`: 품절 상품도 혜택 지급할지 여부
- `use_reward_display_out`: 미진열 상품도 혜택 지급할지 여부
- `use_reward_selling_out`: 미판매 상품도 혜택 지급할지 여부

#### 3.2.3 자체 주문 생성 규칙

알파리뷰는 두 가지 상황에서 자체적으로 Order/OrderItem을 생성한다:

**1. 리뷰 요청 (`Order__Feature.create_order_and_order_item`):**
- `order_id = 0`, `order_item_id = 0`으로 생성
- `order_place_id = "created"`, `status_code = "created"`
- `order_status = "N40"` (배송완료) - 알림톡 발송을 위해
- `shipped_date = 현재 - 5일` - 배송 시작 시간을 소급 설정
- `message_check = "1100000000"` - 주문직후/송장입력직후 발송 완료로 처리
- `is_custom = True`
- 고객 전화번호(`cellphone`)와 이름(`name`)으로 CustomerOption을 get_or_create
- 상품가격 미입력 시 Product의 price를 사용

**2. 소셜(블로그) 리뷰용 임의 주문 (`Order__Feature.create_temporary_order`):**
- 혜택 지급을 위해 생성
- `message_check = "1111111111"` - 모든 메시지 발송 불가로 설정
- `shipped_date = 현재 시각`

**3. 미검증 리뷰용 주문 (`OrderItem__Feature.get_unverified_data`):**
- `order_status = "unverified"`, `status_code = "unverified"`
- `permit_quick = False`, `permit_sns = False` (미검증 리뷰는 빠른리뷰/SNS리뷰 비허용)
- `is_custom = True`

#### 3.2.4 주문 상태 변경에 따른 환불 처리

`OrderContext__Util.REFUND_ORDER_STATUS_PERMIT_MAP`에 정의된 환불 대상 주문 상태:

| 주문 상태 | 환불 사유 |
|-----------|-----------|
| `C40` (취소완료) | "취소" |
| `R40` (반품완료/환불완료) | "반품" |
| `E40` (교환완료) | "교환" |

이 상태로 변경 시:
1. `is_permit_reward = False`, `permit_reward_reason`에 사유 기록
2. 지급 전 리뷰의 `reward_status`를 "자동지급취소"로 변경, `total_point = 0`으로 초기화
3. 지급 완료 리뷰의 `reward_status`를 "회수예정"으로 변경

#### 3.2.5 알림톡 소급 적용

서비스 설치 14일 전까지의 배송완료 주문에 대해 알림톡 발송을 소급 활성화할 수 있다.
- `OrderItem__Feature.set_alimtalk_14days()`: `message_check`의 3번째 자리(인덱스 2)를 `"0"`으로 변경하여 배송완료 알림톡 발송을 허용한다.
- 대상: `order_status`가 `"N40"` 또는 `"N50"`이고, `message_check = "1111111111"`인 주문

#### 3.2.6 쇼핑몰 상태별 메시지 처리

- **자동수집(autocrawling)** 또는 **온보딩 중**(`ShopService__MainStatus.프로덕트온보딩중`) 상태에서 배송완료/구매확정 주문이 들어오면, `message_check`를 `"1111111111"`(전체 발송 불가)로 설정한다.

### 3.3 고객 옵션 관리

#### 3.3.1 고객 유형 결정 규칙

`CustomerOption__Feature.get_customer_type()` 메서드에 구현된 로직:

```
비회원이 아닌 경우 -> "member" (회원)
비회원인 경우:
  주문경로 = 네이버페이  -> "npay"
  주문경로 = 지그재그    -> "zpay"
  주문경로 = 카카오페이  -> "kakaopay"
  주문경로 = 스마트스토어 -> "marketplus"
  그 외                 -> "none" (비회원)
```

#### 3.3.2 고객 정보 동기화

- 코어에서 SQS를 통해 주문 데이터가 유입될 때, `CustomerOption__Feature.migrate_sqs_data()`에서 고객 정보를 조회/생성한다.
- 이미 존재하는 고객이면 `member_id`, `name`, `cellphone` 필드만 변경분을 감지하여 업데이트한다.
- 새로 생성 시 시스템 수집 포토 리뷰 수(`review_photo_total`)를 함께 계산한다.

#### 3.3.3 중복 구매 감지

`OrderItem__Manager.get_duplicate_repurchase_count()`: 동일 상품 + 동일 고객 조합의 OrderItem 수를 반환한다. 2건 이상이면 중복 구매로 판단한다.

#### 3.3.4 메시지 발송 횟수 관리

- `send_count`: 일반 리뷰 요청 발송 횟수
- `reminder_send_count`: 리마인더 발송 횟수
- `nonmeber_send_count`: 비회원 발송 횟수
- `CustomerOption__Manager.reset_send_limit()`: 특정 고객의 발송 횟수를 모두 0으로 초기화

## 4. 상태 전이

### 4.1 주문 상태(order_status) 전이 흐름

```
입금전(N00) --> 상품준비중(N10) --> 배송준비중(N20) --> 배송중(N30) --> 배송완료(N40) --> 구매확정(N50)
    |                                   |
    |                                   +--> 배송대기(N21)
    |                                   +--> 배송보류(N22)
    |
    +--> 입금전취소(C47/C48/C49)

[정상 주문에서 분기]
    +--> 취소신청(C00) --> 취소접수(C10) --> 취소완료(C40)
    +--> 반품신청(R00) --> 반품접수(R10) --> 반품수거완료(R20) --> 반품완료(R40)
    +--> 교환신청(E00) --> 교환접수(E10) --> 교환준비(E20) --> 교환완료(E40)
```

### 4.2 OrderItem의 리뷰 작성 상태 전이

```
주문 생성 (permit_review=True, 조건 충족 시)
    |
    +--> 빠른리뷰 작성 --> quick_written_rating > 0
    |
    +--> 일반리뷰 작성 --> is_review_normal_written = True
    |
    +--> SNS리뷰 작성 --> is_review_sns_written = True
    |
    +--> 취소/반품/교환 --> is_permit_reward = False
                           (이미 지급된 혜택 -> 회수예정)
                           (미지급 혜택 -> 자동지급취소)
```

### 4.3 message_check 상태 전이 예시

```
"0000000000"  (초기 상태: 전체 발송 가능)
      |
      v  주문직후 발송
"1000000000"
      |
      v  송장입력 직후 발송
"1100000000"
      |
      v  배송완료 알림톡 발송
"1110000000"
      |
      v  리마인더 발송
"1110000010"
```

## 5. Usecase 흐름

### 5.1 Cafe24 웹훅을 통한 주문 동기화

```
1. Cafe24에서 주문 생성/변경 웹훅 수신
2. Order__Feature.migrate_sqs_data() 호출
   2-1. CustomerOption 조회 또는 생성 (CustomerOption__Feature.migrate_sqs_data)
        - customer_type 결정 (회원/비회원/네이버페이 등)
   2-2. OrderContext__Util 생성 (ShopPolicy, ShopPermit, 제한 상품/카테고리 캐싱)
   2-3. Order get_or_create
        - 첫 생성 시 short_code 생성
   2-4. 각 order_item에 대해 OrderItem__Feature.migrate_sqs_data() 호출
        - Product 조회 -> is_sales_changed = True 설정
        - OrderItem get_or_create
        - 변경 데이터 처리 (가격, 배송일자, 주문상태 등)
        - 알파리뷰용 데이터 처리 (order_status_sub, ordered_date_sub)
        - 후처리 데이터 처리 (reward_disabled_at, max_point, 품절/미진열 혜택 설정)
        - 쇼핑몰 상태별 message_check 처리 (온보딩 중이면 전체 발송불가)
        - 첫 생성 시 permit_* 플래그 계산
        - 환불 상태 감지 시 자동 지급취소/회수 처리
```

### 5.2 리뷰 요청(수동 주문 생성)

```
1. 관리자가 리뷰 요청 API 호출 (고객정보 + 상품정보 + 주문번호)
2. Order__Feature.create_order_and_order_item() 호출
   2-1. CustomerOption get_or_create (전화번호 + 이름 + customer_type="created")
   2-2. product_price 결정 (입력값 없으면 Product.price 사용)
   2-3. Order get_or_create (order_place_id="created")
   2-4. OrderItem 생성
        - order_status = "N40" (배송완료)
        - shipped_date = 현재 - 5일
        - message_check = "1100000000"
        - 혜택 제한/permit 계산 수행
   2-5. Product.is_sales_changed = True 설정
3. 알림톡 발송 스케줄러에 의해 리뷰 요청 알림톡 발송
```

### 5.3 SNS 리뷰를 위한 주문 상품 조회

```
1. OrderItem__Manager.get_first_sns_reviewable_order_item() 호출
2. 조건:
   - permit_sns = True
   - 해당 OrderItem에 빠른 또는 일반 리뷰가 이미 작성되어 있어야 함
     (Review.review_type IN [일반(1), 빠른(2)], is_deleted = False)
3. 배송 완료일(shipped_date) 최신순으로 1건 반환
4. CustomerOption 정보를 select_related로 함께 로드
```

### 5.4 팝업/플로팅 모듈용 주문 상품 조회

```
1. OrderItem__Manager.get_list_for_popup_floating() 호출
2. 조건:
   - order_status_sub IN ["N40", "N50"] (배송완료 또는 구매확정)
   - tracking_no IS NOT NULL (송장번호 존재)
   - is_review_normal_written = False (일반 리뷰 미작성)
   - quick_written_rating = 0 (빠른 리뷰 미작성)
   - is_permit_reward = True
   - permit_review = True
   - permit_normal = True
3. 리뷰 작성 기한(review_write_limit) 설정 시: shipped_date >= (현재 - 기한일수)
4. replica DB에서 조회, 최신순 정렬
```

### 5.5 리뷰 수 최신화 (배치)

```
1. Product__Feature.update_review_number() 호출
2. ShopAutocrawling 상태를 "설치중"으로 변경
3. is_review_changed = True인 상품 순회
   3-1. 일반 리뷰 통계 갱신 (게시됨 + 노출가능 + 원본제외)
   3-2. 연결된 상품(connect_info)이 있으면 연결 리뷰 통계도 갱신
4. is_sales_changed = True인 상품 순회
   4-1. 최근 14일 판매 개수 갱신
   4-2. 최근 판매일시 갱신
5. ShopAutocrawling 상태를 "완료" 또는 "실패"로 변경
```

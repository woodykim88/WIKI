# 위젯 구성 및 렌더링

## 1. 도메인 개요

위젯 시스템은 알파리뷰 서비스의 핵심 프론트엔드 렌더링 파이프라인으로, 쇼핑몰 상세 페이지/메인 페이지/게시판 등에 리뷰 데이터를 다양한 형태로 노출하는 역할을 담당한다. 쇼핑몰 운영자는 대시보드에서 **위젯 세트(WidgetSet)** 단위로 위젯을 구성하고, 각 세트 안에 하나 이상의 **위젯(Widget)** 을 배치하며, 위젯마다 개별적인 **위젯 옵션(WidgetOption)** 을 설정하여 노출 방식을 제어한다.

### 주요 도메인 흐름

1. **구성 단계 (대시보드)**: 운영자가 위젯 세트를 생성하고, 위젯 타입을 선택하여 배치하고, 옵션을 설정한다.
2. **렌더링 단계 (스크립트/CDN)**: 쇼핑몰 페이지에 삽입된 스크립트가 `widget_code`를 기반으로 API를 호출하여 리뷰/상품 데이터를 가져온다.
3. **Strategy 패턴**: V2 아키텍처에서는 위젯 타입별로 Strategy 클래스가 매핑되어 있어, 데이터 조회/필터링/캐싱/시리얼라이징을 각 위젯 특성에 맞게 처리한다.

### V1 / V2 구조

- **V1** (`widget/views.py`, `widget/serializers.py`): "TO BE DEPRECATED"로 표기된 레거시 코드. 직접적인 쿼리셋 조합 방식.
- **V2** (`widget/v2/`): Feature/Manager/Strategy 계층 분리 패턴을 적용한 현행 아키텍처.

## 2. 데이터 모델

### 2.1 WidgetSet (위젯 세트)

위젯의 최상위 컨테이너. 하나의 쇼핑몰(`shop_id`)에 여러 위젯 세트가 존재할 수 있다.

| 필드 | 타입 | 설명 |
|------|------|------|
| `shop_id` | IntegerField | 쇼핑몰 고유 ID |
| `widget_code` | CharField(20) | 위젯 세트 고유 코드 (UUID 기반 8자리, 자동 생성) |
| `location_type` | IntegerField | 위젯 설치 위치 (`WidgetSet__LocationType` choices) |
| `title` | CharField(50) | 위젯 세트 제목 |
| `custom_id` | CharField(50) | 커스텀 HTML id |
| `custom_class` | CharField(50) | 커스텀 HTML class |
| `custom_js` | TextField | 위젯 세트용 커스텀 JS |
| `custom_css` | TextField | 위젯 세트용 커스텀 CSS |
| `is_admin_only` | BooleanField | 관리자 전용 위젯 세트 여부 (키워드위젯, 작성유도버튼) |
| `is_preview` | BooleanField | 프리뷰 위젯 여부 |

**`WidgetSet__LocationType` (위젯 설치 위치)**

| 값 | 라벨 |
|----|------|
| `1` | 전체 |
| `2` | 상세페이지 |
| `3` | 메인페이지 |
| `4` | 리뷰게시판 |

### 2.3 WidgetOption (위젯 옵션)

위젯의 상세 설정을 키-값(key-value) 쌍으로 저장한다. **DB에는 기본값과 다른 옵션만 저장**되며, 조회 시 `WIDGET_TYPE_TO_DEFAULT_OPTION_MAPPER`의 기본값과 병합된다.

| 필드 | 타입 | 설명 |
|------|------|------|
| `widget` | ForeignKey(Widget) | 상위 위젯 (related_name: `widget_option_set`) |
| `title` | CharField(50) | 옵션명 (key) |
| `value` | TextField | 옵션값 (value) |

### 2.4 WidgetLabelRelation (위젯-라벨 관계)

위젯에서 노출하거나 제외할 리뷰 라벨을 지정한다.

| 필드 | 타입 | 설명 |
|------|------|------|
| `widget` | ForeignKey(Widget) | 상위 위젯 (related_name: `widget_label_relation`) |
| `label` | ForeignKey(Label) | 대상 라벨 |
| `is_included` | BooleanField | `True` = 포함, `False` = 제외 |

### 2.5 WidgetProductRelation (위젯-상품 관계)

위젯에서 노출하거나 제외할 상품을 지정한다.

| 필드 | 타입 | 설명 |
|------|------|------|
| `widget` | ForeignKey(Widget) | 상위 위젯 (related_name: `widget_product_relation`) |
| `product_id` | PositiveIntegerField | 상품 고유 ID |
| `is_included` | BooleanField | `True` = 포함, `False` = 제외 |

### 2.6 WidgetCategoryRelation (위젯-카테고리 관계)

위젯에서 노출하거나 제외할 카테고리를 지정한다.

| 필드 | 타입 | 설명 |
|------|------|------|
| `widget` | ForeignKey(Widget) | 상위 위젯 (related_name: `widget_category_relation`) |
| `category_id` | PositiveIntegerField | 카테고리 고유 번호 (deprecated 예정) |
| `category_platform_id` | CharField(100) | 플랫폼별 카테고리 고유 번호 |
| `is_included` | BooleanField | `True` = 포함, `False` = 제외 |

### 2.7 WidgetTarget (위젯 타겟 - 비정규화 테이블)

`WidgetProductRelation`과 `WidgetCategoryRelation`을 통합하여 최종 product_id 목록을 비정규화(denormalize)한 모델이다. 카테고리 관계의 경우 하위 카테고리까지 확장하여 소속 상품을 모두 WidgetTarget에 기록한다.

| 필드 | 타입 | 설명 |
|------|------|------|
| `widget` | ForeignKey(Widget) | 상위 위젯 (related_name: `widget_target_set`) |
| `product_id` | PositiveIntegerField | 상품 고유 ID |
| `widget_product_relation` | ForeignKey(WidgetProductRelation) | 원본 상품 관계 (nullable) |
| `widget_category_relation` | ForeignKey(WidgetCategoryRelation) | 원본 카테고리 관계 (nullable) |
| `is_included` | BooleanField | `True` = 포함, `False` = 제외 |

## 3. 비즈니스 규칙

### 3.1 위젯 코드 생성 규칙

- `widget_code`는 UUID 기반으로 생성된 8자리 문자열이다.
- 생성 시 기존 코드와의 중복 검사를 수행한다.
- WidgetSet과 Widget 모두 독립적인 `widget_code`를 가진다.

### 3.2 위젯 옵션 기본값 시스템

위젯 옵션은 DB에 **기본값과 다른 항목만 저장**하는 델타(delta) 저장 방식을 사용한다.

- **저장 시**: 입력된 옵션 중 `WIDGET_TYPE_TO_DEFAULT_OPTION_MAPPER[widget_type]`의 기본값과 동일한 항목은 저장하지 않고, 다른 항목만 `WidgetOption` 레코드로 생성한다.
- **조회 시**: DB에 저장된 옵션을 먼저 읽고, 기본값과 병합하여 완성된 옵션 딕셔너리를 반환한다 (`WidgetOption__Utils.get_widget_options_with_default_value()`).
- **재구성 시**: 기존 옵션을 전체 삭제(`bulk_delete`) 후 재생성(`bulk_create`)하는 방식으로 갱신한다.

### 3.8 관리자 위젯 자동설치

키워드 위젯과 작성유도버튼은 `is_admin_only=True`로 설정되어 대시보드 위젯 세트 목록에서 제외된다. 이 위젯들은 별도의 자동설치 로직(`WidgetSet__Feature.get_manual_code()`)을 통해 생성/관리된다.

- **키워드 위젯**: `ShopKeywordReview__Manager`의 `widget_auto_install` 플래그로 활성화 여부 판단
- **작성유도버튼**: `ShopDesign`의 `review_write_btn_show` 플래그로 활성화 여부 판단
- 자동설치가 OFF인 경우 `WIDGET_AUTO_INSTALL_NOT_ALLOWED` 오류 발생

### 3.9 캐싱 전략

#### 위젯 데이터 캐시

- **캐시 단위**: 100개 리뷰/상품(`CACHE_SIZE=100`)을 하나의 캐시 페이지로 관리
- **캐시 키**: `widget_{type}_{code}_{product_no}_{page}_{sort}_{device}` 등 조합
- **캐시 무효화**: `CACHE_OFF` 설정 또는 `is_dummy=True` 더미 모드 시 캐시 비활성화
- **캐시 분할 조회**: 요청된 페이지가 여러 캐시 페이지에 걸칠 경우, while 루프로 순회하며 필요한 만큼 데이터를 수집

#### CDN 캐시 제어 (스크립트 뷰)

| 용도 | Cache-Control TTL |
|------|-------------------|
| 민감 데이터 (좋아요 등) | 30초 |
| 위젯 데이터 | 180초 |
| 상세 데이터 | 300초 |
| 설정 데이터 | 300초 |
| 5xx 에러 응답 | 86400초 (1일) |

### 3.10 리뷰 최소 개수 체크 (`_check_review_minimum`)

1페이지 요청 시에만 동작하며, `review_none` 옵션 값에 따라:

- `message` (리뷰작성유도메시지 노출): 리뷰가 0개이면 `RESULT_EMPTY_SHOW_MESSAGE` 예외 발생
- `hide` (위젯 미노출): 리뷰가 `review_minimum`보다 적으면 `RESULT_EMPTY_HIDE` 예외 발생

검색어, 평점, 설문, 옵션, 미디어 필터가 적용된 경우에는 검사하지 않는다.

## 4. 상태 전이

### 4.1 위젯 세트 생명주기

```
[생성] --> [활성] --> [수정] --> [활성]
                  |
                  +--> [삭제]
```

- **생성**: `WidgetSet__Feature.create_or_update(shop_id, data, id=None)` -- 새 WidgetSet + 하위 Widget 생성
- **수정**: `WidgetSet__Feature.create_or_update(shop_id, data, id=widget_set_id)` -- 기존 WidgetSet 업데이트 + Widget 재구성
- **삭제**: `WidgetSet__Manager.delete(shop_id, id)` -- CASCADE로 하위 Widget, WidgetOption, 관계 모델 모두 삭제

### 4.2 위젯 생성/수정 프로세스

`Widget__Feature.update_or_create()` 호출 시:

```
1. Widget 생성 또는 수정
   ├── id 없음 → Widget__Manager.create() (widget_code 자동 생성)
   └── id 있음 → Widget__Manager.patch()
2. WidgetOption 재구성 (전체 삭제 후 기본값과 다른 항목만 재생성)
3. WidgetLabelRelation 재구성 (전체 삭제 후 재생성)
4. WidgetTarget 전체 삭제
5. WidgetProductRelation 재구성 → WidgetTarget에 상품 동기화
6. WidgetCategoryRelation 재구성 → WidgetTarget에 카테고리 상품 동기화
7. [브리핑위젯] WidgetSpecificFilter 재구성
```

### 4.3 프리뷰 위젯 설치 프로세스

```
1. 클라이언트가 WidgetSetInstall__View.post() 호출
   - shop_id (query param)
   - widget_sets: [{widgetset_type, widgetset_title}, ...]
2. 각 widgetset_type을 WidgetSet__PreviewType으로 변환
3. WIDGET_PREVIEW_MAPPER에서 포함 위젯 목록 조회
4. WidgetSet__Feature.create_or_update() 재사용하여 실제 쇼핑몰에 생성
5. 생성된 위젯 세트 목록 반환
```

### 4.4 프리셋 위젯 생성 프로세스

```
1. WidgetSetInitial__DashBoardView.post() 호출
   - data: {review_tab: true, simple_photo: false, ...}
2. True인 항목만 WidgetSet__PresetType으로 변환
3. WIDGET_PRESET_MAPPER에서 포함 위젯 목록 조회
4. WidgetSet 생성 + Widget bulk_create
5. {isOk: True} 반환
```

## 5. Usecase 흐름

### 5.1 위젯 데이터 렌더링 (스크립트 호출)

```
[스크립트] → Widget__View.get(widget_code, page, product_no, ...)
    │
    ├── Widget__Feature.get_widget_data()
    │   ├── Widget__Manager.get_by_widget_code() → Widget 인스턴스
    │   ├── WIDGET_TYPE_TO_STRATEGY_MAP[widget_type] → Strategy 클래스
    │   ├── Strategy 인스턴스 생성 (widget, page, page_size, product_no, ...)
    │   │
    │   ├── [캐시 확인] cache.get(cache_id)
    │   │   ├── 캐시 HIT → cached_data 반환
    │   │   └── 캐시 MISS → strategy.flow() 호출
    │   │       ├── validate_params()
    │   │       ├── get_data() → 리뷰/상품 데이터
    │   │       └── build_serializer_fields() → 허용 필드 목록
    │   │
    │   └── (result, serializer_fields, context) 반환
    │
    └── [응답] 시리얼라이즈 후 Cache-Control 헤더와 함께 반환
```

### 5.6 기타 위젯 기능

#### ShopDesign 플레이스홀더 치환

`WidgetShopDesign__Feature.get_shop_design_with_reward_placeholder()`에서 ShopDesign의 다음 필드에 대해 `{최대포인트}` 문자열을 SNS 리뷰 제외 총 리워드 금액으로 치환한다:

- `revisit_text`
- `empty_review_text`
- `empty_review_custom_pc`
- `empty_review_custom_mo`

#### 리뷰 작성 URL 생성

`WidgetModule__Feature.get_review_write_url_data()`에서 다음 우선순위로 리뷰 작성 URL을 결정한다:

1. `order_item_code` 제공 시 -- 해당 주문 상품의 작성 URL
2. `order_code` + `product_no` 제공 시 -- 주문 코드 기반 작성 URL
3. 그 외 -- 일반 작성 URL (`get_normal_write_url()`)
4. 위 모두 실패 시 -- 미검증 리뷰 URL + 추가 정보 반환

#### 재방문 팝업 (`WidgetExtra__Feature.get_revisit_popup_data()`)

1. `member_id`로 고객 옵션 조회
2. `shop_design.revisit_show` 비활성화 시 `REVISIT_POPUP_NOT_ALLOWED` 예외
3. 빠른리뷰 작성 가능한 마지막 주문 상품 조회
4. 없으면 `{has_pending_reviews: False}` 반환
5. 있으면 상품 정보 + 플로팅 디자인 정보 + 리뷰 작성 URL + 텍스트(`{최대포인트}`, `{고객명}` 치환) 반환

#### 위젯 권한 확인 (`WidgetPermission__Feature.get_permission()`)

다음 항목의 활성화 여부를 반환한다:

- `active_keyword_widget` -- 키워드 위젯 자동설치 여부
- `active_write_button` -- 작성유도버튼 노출 여부
- `active_revisit_popup` -- 재방문 팝업 활성화 여부
- `active_ticker` / `active_ticker_main` / `active_ticker_product` -- 세일즈 티커 활성화 여부

#### 설문 필터 (`WidgetSurvey__Feature`)

게시판 위젯에서 설문 필터링 기능에 필요한 설문 질문/답변 목록을 반환한다. 특수 처리:

- `survey_question_id = 10` (키): 150cm 이하 ~ 190cm 이상 범위 선택지
- `survey_question_id = 11` (몸무게): 40kg 이하 ~ 100kg 이상 범위 선택지
- 그 외: DB에 저장된 선택지 그대로 반환

# 외부 플랫폼 연동

## 1. 도메인 개요

외부 플랫폼 연동은 Instagram, 네이버 스마트스토어, 네이버 블로그 등 외부 채널의 리뷰/게시물을 알파리뷰 시스템으로 수집하고 관리하는 도메인이다. 수집된 외부 리뷰는 알파리뷰 위젯에서 통합 노출되며, SNS 리뷰의 경우 해시태그 검증 기반의 인증/해제 시스템과 혜택 지급 로직이 함께 동작한다.

### 지원 플랫폼 (`Review__Platform`)

| 값 | 설명 |
|---|---|
| `알파리뷰` | 자체 플랫폼에서 작성된 리뷰 |
| `CAFE24` | Cafe24 게시판에서 작성된 리뷰 |
| `스마트스토어` | 네이버 스마트스토어에서 수집된 리뷰 |
| `지그재그` | 지그재그에서 수집된 리뷰 |
| `인스타그램` | Instagram에서 수집된 리뷰 |
| `네이버블로그` | 네이버 블로그에서 수집된 리뷰 |

### 외부 연동 관련 서비스 타입 (`Service__Type`)

| 값 | 코드 | 설명 |
|---|---|---|
| `스마트스토어리뷰연동` | 131 | 스마트스토어 리뷰 연동 서비스 (별도 구독 필요) |

## 2. 데이터 모델

### 2.1 InstagramMedia

Instagram 게시물 정보를 저장하는 모델. Facebook Graph API를 통해 수집한다.

| 필드 | 타입 | 설명 |
|---|---|---|
| `shop_id` | IntegerField | 쇼핑몰 고유 ID |
| `is_tagged` | BooleanField (default=True) | `True`: 태그된 게시물, `False`: 내 게시물 |
| `media_id` | CharField(50) | Meta(Instagram) 미디어 고유 ID |
| `caption` | TextField | 게시글 내용 |
| `permalink` | URLField | 게시글 링크 |
| `comments_count` | IntegerField | 댓글 수 |
| `like_count` | IntegerField | 좋아요 수 |
| `username` | CharField(127) | 작성자 아이디 |
| `timestamp` | DateTimeField | 게시 일자 |

**유니크 제약조건**: `(shop_id, media_id, is_tagged)` -- 동일 쇼핑몰에서 같은 미디어를 태그/내 게시물 별로 각각 한 건만 허용

**인덱스**: `(shop_id)`, `(shop_id, media_id)`

### 2.2 InstagramMediaPhoto

InstagramMedia에 포함된 개별 이미지/동영상 썸네일 정보를 저장한다.

| 필드 | 타입 | 설명 |
|---|---|---|
| `ig_media` | FK -> InstagramMedia | 상위 미디어 (related_name: `photos`) |
| `is_photo` | BooleanField (default=True) | `True`: 이미지, `False`: 동영상 썸네일 |
| `photo_url` | URLField | 이미지/썸네일 URL (상품 연동 후에는 S3 경로로 업데이트됨) |

### 2.3 ReviewInstagramRelation

Instagram 게시물과 알파리뷰 Review를 연결하는 중간 테이블이다. 하나의 InstagramMedia가 여러 상품(Product)과 연동될 수 있으므로 N:M 관계를 관리한다.

| 필드 | 타입 | 설명 |
|---|---|---|
| `review` | FK -> Review | 생성된 리뷰 (related_name: `review_instagram_set`) |
| `ig_media` | FK -> InstagramMedia | 원본 Instagram 미디어 (related_name: `review_instagram_set`) |

**유니크 제약조건**: `(review, ig_media)`

### 2.4 SmartstoreReview

스마트스토어 리뷰 연동 설정을 관리하는 모델이다. 쇼핑몰별로 하나의 레코드가 존재한다.

| 필드 | 타입 | 설명 |
|---|---|---|
| `shop_id` | IntegerField | 쇼핑몰 고유 ID |
| `update_at` | DateTimeField | 마지막으로 크롤링한 날짜 (다음 크롤링 시 시작일로 사용) |
| `identify_name` | CharField(50) | 스마트스토어 상점 식별 이름 (어드민 선택 시 사용) |
| `url` | TextField | 스마트스토어 URL |
| `status` | CharField(50) | 연동 상태 |
| `status_text` | CharField(200) | 연동 상태 설명 |

**미사용 필드**: `type`, `user_id`, `password`, `allow_product_option`, `auto_product_name`

### 2.5 SmartstoreProduct

알파리뷰 상품과 스마트스토어 상품 번호 간의 매핑 테이블이다.

| 필드 | 타입 | 설명 |
|---|---|---|

| `product_id` | IntegerField | 알파리뷰 상품 고유 ID |

### 2.7 ShopInstagram (shopreviewpolicy 앱)

| 필드 | 타입 | 설명 |
|---|---|---|

**refresh_status 값** (`Common__ProgressType`):

| 값 | 코드 | 설명 |

| `실패` | 400 | 새로고침 실패 |

**refresh_error_code 값** (`ShopInstagram__RefreshErrorCode`):

| 값 | 코드 | 설명 |
|---|---|---|

| `FACEBOOK_UNKNOWN_ERROR` | 1 | Facebook 알 수 없는 에러 |
| `FACEBOOK_INVALID_PARAMETER` | 100 | 잘못된 파라미터 |
| `FACEBOOK_INVALID_OAUTH` | 190 | Access Token 자체가 유효하지 않음 |
| `FACEBOOK_PERMISSION_ERROR` | 200 | Access Token은 유효하나 권한 없음 |
| `FACEBOOK_OTHER_ERROR` | 999 | 기타 에러 (위에 해당하지 않는 모든 에러) |

## 3. 비즈니스 규칙

### 3.1 Instagram 리뷰 연동 규칙

#### 3.1.1 소셜 리뷰 (관리자 수동 연동)

관리자가 Instagram 게시물을 직접 선택하여 상품과 연동하는 방식이다.

**수집 방식 (Facebook Graph API `/tags` 엔드포인트)**:
1. `GET https://graph.facebook.com/{user_id}/tags` API를 호출하여 해당 쇼핑몰 Instagram 계정에 태그된 게시물을 수집
2. 요청 필드: `id, caption, comments_count, like_count, username, timestamp, permalink, media_url, thumbnail_url, children{media_url, thumbnail_url}`
3. 최근 1년 이내 게시물만 필터링
4. 최대 수집 건수: 기본 2,000건 (특수 업체별 별도 제한 존재, 예: shop_id 1397은 300건)

**새로고침 규칙**:
1. `ShopInstagram.check_last_refresh`: 마지막 새로고침 후 **1시간 이상** 경과해야 재실행 가능
2. `ShopInstagram.check_media_expired`: 이미지 만료일자가 유효한 경우에만 정상 갱신 진행
   - 이미지 만료일자가 유효하고, 갱신주기 유효 -> 정상 갱신 진행
   - 이미지 만료일자가 유효하고, 갱신주기 미유효 -> 실패 처리 (에러코드: `INVALID_LAST_REFRESH`)
   - 이미지 만료일자가 만료된 경우 -> 갱신주기 무관하게 진행 (임시 만료일자 = 현재 + 7일)
3. 갱신 실패 시: 1년 이내가 아니고 상품 연동되지 않은 InstagramMedia를 삭제
4. 갱신 성공 후: 인스타그램에서 삭제된 게시물에 해당하는 InstagramMedia 삭제 (단, 이미 상품 연동된 건은 보존)

**이미지 만료일자 관리**:
- Instagram 미디어 URL에 포함된 `oe` 파라미터(16진수 Unix timestamp)를 파싱하여 만료일 추출
- 수집한 모든 미디어 URL 중 가장 빠른 만료일을 `ShopInstagram.media_expired`에 저장
- Facebook error code == 1 발생 시: `limit=10`으로 줄이고 2초 대기 후 재시도

**상품 연동 프로세스** (`InstagramMedia__Feature.connect`):
1. 관리자가 `ig_media_ids`(Instagram 게시물 목록)와 `product_ids`(상품 목록)를 지정
2. 기존 연동 상태와 비교하여 신규 연동 / 해제를 결정:
   - 새로 추가할 상품: `product_ids - existing_product_ids`
   - 해제할 상품: `existing_product_ids - product_ids`
3. 신규 연동 시 `Review` 생성:
   - `review_type`: `소셜` (4)
   - `creater_type`: `시스템` (6)
   - `platform`: `인스타그램`
   - `ratings`: 고정값 5
   - `posting_status`: `게시됨` (20)
   - `content`: Instagram 게시물의 caption
   - `social_link`: Instagram permalink
4. `InstagramMediaPhoto`의 이미지를 ReviewPhoto로 변환:
   - 이미 S3에 저장된 경우(`review/thumbnail`로 시작): 기존 ReviewPhoto를 복제
   - 그렇지 않으면: 이미지를 다운로드하여 S3 업로드 후 ReviewPhoto 생성, InstagramMediaPhoto의 `photo_url`을 S3 경로로 업데이트
   - 만료된 photo_url인 경우 이미지 없이 리뷰 생성
5. 해제 대상: 해당 상품 + social_link가 동일한 Review를 일괄 삭제

#### 3.1.2 SNS 리뷰 (웹훅 자동 연동)

고객이 Instagram에 해시태그를 포함한 게시물을 올리면 웹훅으로 자동 감지되어 리뷰가 생성되는 방식이다.

**Facebook 웹훅 수신** (`Review__WebhookFacebookView`):
1. GET 요청: `hub.challenge` 값을 그대로 반환 (구독 검증용)
2. POST 요청: Instagram mentions/comments 변경 이벤트 수신
   - `entry[].changes[].field`가 `mentions` 또는 `comments`인 경우만 처리
   - `entry[].id`로 Instagram 사용자 ID를 식별하고, Core API를 통해 shop_id를 조회
   - 항상 HTTP 200 반환 (Facebook 재전송 방지)

**웹훅 데이터 처리 흐름** (`Facebook__Feature.process_webhook_data`):
1. `entry[].id`(Instagram ID)로 Core API에서 shop 정보 조회 (`Core__Feature.fetch_instagram_id`)
2. `facebook_profile`에서 `facebook_access_token` 획득
3. `mentioned_media.media_id({media_id})` 형식으로 Facebook Graph API v15.0 호출
4. 응답에서 미디어 데이터(media_url, username, caption, permalink, like_count, comments_count, children 등) 추출
5. caption에서 해시태그 패턴 추출: `#[^#@\n ]*`

**SNS 리뷰 생성** (`Facebook__Feature.create_facebook_review`):
1. `sns_review_id`(media_id)로 기존 SNS 리뷰 존재 여부 확인
2. 해시태그에서 `short_code` 매칭:
   - 게시물의 각 해시태그에서 `#` 제거 후 `OrderItem.short_code`와 매칭
   - `OrderItem.permit_sns == True`인 주문상품만 대상
3. 필수 해시태그 검증:
   - `OrderItem.hashtag_all`에서 `@`로 시작하는 항목 제외
   - 나머지 해시태그가 게시물에 모두 포함되어야 통과 (`set(required) - set(post_hashtags) == empty`)
4. SNS 리뷰 생성/업데이트:
   - `Review.review_type`: `SNS` (3)
   - `Review.creater_type`: 고객 유형에 따라 회원(1)/비회원(2)/NPAY(3)/주문생성(4)/관리자(5)/시스템(6)
   - `Review.platform`: `인스타그램`
   - `Review.ratings`: 고정값 5
   - `sns_completed_at`: 현재시간 + `ShopOption.maintenance_period_sns`일 (기본 3일)
   - `OrderItem.is_review_sns_written`을 `True`로 변경

**혜택 지급 규칙**:
1. 이미 지급 완료된 리워드가 있으면 추가 지급하지 않음
2. 지급 제한 조건 확인:
   - `OrderItem.is_permit_reward`가 `False`이면 지급 제한
   - 상품 품절(`sold_out`) + 정책에서 품절 시 미지급 설정 시 제한
   - 상품 미진열(`display_out`) + 정책에서 미진열 시 미지급 설정 시 제한
   - 상품 미판매(`selling_out`) + 정책에서 미판매 시 미지급 설정 시 제한
3. 지급 금액 계산:
   - SNS 리뷰 기본 혜택 (`ShopReward__Title.SNS리뷰`, 코드: 150)
   - 추가 혜택: 회원최초 SNS리뷰 (`ShopReward__Title.회원최초_SNS리뷰`, 코드: 700), 선착순 SNS리뷰 (`ShopReward__Title.선착순_SNS리뷰`, 코드: 220)
   - `%` 단위 설정 시: `상품가격 * 비율 / 100` (최소 포인트 이상)
   - 적립금 상한선 적용: `use_max_total_point_range` 설정에 따라 기본/추가 혜택 합산 전후 적용 결정
   - 주문당 혜택 제한: `use_number_reward` 설정에 따라 동일 주문의 기존 SNS 리뷰 지급 수 확인
4. 지급 상태 결정 (`Review__RewardStatus`):
   - `reward_option == "admin"`: `지급대기` (11) - 관리자 검수 후 수동 지급
   - `reward_option == "immediate"`: `지급예정` (10) - 즉시 지급 예정
   - `reward_option == 숫자(일)`: `지급예정` (10) - N일 후 지급 예정
   - 지급 불가: `지급제한` (23) 또는 `지급포인트없음` (25)
5. 비회원 처리: `message_check` 5번째 자리를 `1`로 설정하여 SNS 혜택 제공 완료 플래그 기록

**게시 상태 결정**:
- `posting_option == "immediate"`: 즉시 게시됨 (20)
- `posting_option == "admin"`: 게시대기 (11) - 관리자 승인 필요
- `posting_option == 숫자(일)`: 게시예정 (10) - N일 후 자동 게시
- 키워드 알림 대상이고 `posting_option_keyword` 활성화 시: 강제로 게시대기 (11)

**대체어 처리**: `AlternativeKeyword`에 등록된 원본/대체어 쌍으로 caption 텍스트 치환

**챌린지 연동**: 인스타그램 게시물 업로드 시간(`instagram_created_at`) 기준으로 진행중인 챌린지가 있으면 `ReviewChallengeRelation`을 생성

**미디어 파일 처리**:
- `children.data`가 있는 경우 (다수 미디어): 각 child의 `media_type`에 따라 IMAGE/VIDEO 분류
- `children` 없는 경우 (단일 미디어): `media_url`과 `media_type`으로 분류
- VIDEO는 ReviewVideo로, IMAGE는 ReviewPhoto로 생성 (VIDEO의 노출 순서가 IMAGE보다 앞)

### 3.2 SNS 인증 완료/해제 규칙

1시간 주기 배치 커맨드 `every_1hour_review_facebook`에서 실행된다.

#### 3.2.1 SNS 인증 완료 처리

**조건**:
- `review_type == SNS`
- `is_sns_completed == False`
- `sns_completed_at <= 현재시간` (유지기간 경과)
- `sns_deauthed_at IS NULL` (인증 해제되지 않은 상태)
- `to_be_deleted_at IS NULL` (삭제 예정 아님)

**처리**: `is_sns_completed = True`로 업데이트, "SNS리뷰 완료" 로그 기록

#### 3.2.2 SNS 인증 유지 확인 (진행중)

**대상 리뷰** (두 그룹의 union):
- 그룹 1: `is_sns_completed == False` + `sns_completed_at > 현재` + `sns_deauthed_at IS NULL` + `to_be_deleted_at IS NULL`
- 그룹 2: `is_sns_completed == False` + `sns_completed_at > 현재` + `sns_deauthed_at >= (현재-60일)` + `to_be_deleted_at IS NULL`

**확인 프로세스** (Facebook Graph API v16.0 `mentioned_media` 조회):
1. `sns_review_id`(media_id)로 Instagram API 호출
2. API 에러 발생 시 -> **인증 해제**: 해시태그 삭제, 비공개, 보관 등의 사유로 판단
3. 정상 응답 시 -> caption에서 해시태그 추출하여 `OrderItem.hashtag_all`과 비교:
   - 필수 해시태그가 모두 존재: **인증 유지** - `sns_deauthed_at = None`, 소셜 링크/좋아요 수/작성자 정보 업데이트
   - 필수 해시태그 누락: **인증 해제**

#### 3.2.3 인증 해제 처리 (`Review__Manager.deauthorize_sns`)

- `rewarded_at = None` (지급 예정일 초기화)
- `total_point = 0` (총 포인트 초기화)
- `sns_deauthed_at = 현재시간`
- 로그: "SNS리뷰 인증해제 - {N}일째 인증해제됨"

#### 3.2.4 인증 유지 로그

- 마지막 "SNS리뷰 유지중" 로그 이후 24시간 초과 시에만 새 로그 기록
- 형식: "SNS리뷰 유지중 : 좋{like_count}/댓{comment_count}"

### 3.3 스마트스토어 리뷰 연동

#### 3.3.1 구독 확인

스마트스토어 리뷰 연동은 별도 서비스 구독이 필요하다. `Service__Type.스마트스토어리뷰연동` (131)의 `main_status`가 `라이브`인 업체만 대상이다.

#### 3.3.2 크롤링 방식

Selenium 기반 웹 크롤러로 네이버 커머스 어드민에 로그인하여 리뷰를 엑셀로 다운받은 후 파싱한다. 두 가지 커맨드가 존재한다.

**`daily_06am_smartstore`**: 단일 업체 대상 (ID 하드코딩)
- 최대 크롤링 기간: 28일 단위

**`daily_06am_smartstore_remote`**: 전체 구독 업체 대상 (EC2에서 실행, IP 제약 때문)
- `ThreadPoolExecutor` 사용 (최대 6개 워커)
- 최대 크롤링 기간: 14일 단위
- 기본 소급 기간: 2일 (`DEFAULT_REVIEW_SCRAPING_BACKWARD_WINDOW`)
- 실패 시 최대 3회 재시도 (`ERROR_MAX_RETRY`)
- `SmartstoreReview.update_at`이 없으면 현재 - 2일로 설정

**크롤링 프로세스**:
1. 네이버 커머스 어드민 로그인 (2단계 인증 처리 가능)
2. 대상 스토어 선택 (`SmartstoreReview.identify_name` 기반)
3. 리뷰 작성일 범위 지정 후 검색 -> 엑셀 다운로드 (40초 대기)
4. 엑셀 파싱하여 리뷰 업로드
5. 완료 후 `SmartstoreReview.update_at`을 `end_date + 1일`로 업데이트

#### 3.3.3 리뷰 업로드 규칙

엑셀에서 추출하는 필드:
- A열: `product_no` (스마트스토어 상품 번호)
- B열: `product_name` (상품명)
- D열: `ratings` (별점)
- E열: `attach_file_urls` (첨부파일 URL, 쉼표 구분)
- F열: `content` (리뷰 내용, HTML 태그 제거/unescape 처리)
- H열: `user_id` (작성자 ID)
- I열: `created_at` (작성일시)
- K열: `article_no` (리뷰 고유번호)
- N열: `display` (노출 상태: "정상" -> 게시, 그 외 -> 비게시)

**리뷰 생성 규칙**:
- `Review__Manager.update_or_create` 사용:
  - 필터: `review_type=일반(1)`, `creater_type=시스템(6)`, `platform=스마트스토어`, `external_product_option=article_no`
  - 동일한 article_no의 리뷰가 이미 존재하면 업데이트, 없으면 신규 생성
- 대체어 처리 적용 (`CreateReview__Features.get_text`)
- 상품 매핑: `SmartstoreProduct`에서 `smartstore_product_no`로 조회하여 알파리뷰 상품에 연결
  - 매핑되지 않은 경우 `product_fk=None`, `product_id=None`으로 저장
- 미디어 처리: 첨부파일 URL에서 이미지(`.jpg`, `.jpeg`, `.png`, `.gif`, `.webp`)/비디오(`.mp4`, `.mov`) 분류 후 다운로드 및 저장
- 신규 생성 시에만 자동댓글 생성 + 게시 설정 적용

#### 3.3.4 스마트스토어 상품 매핑 갱신 (`refresh_smartstore_product`)

크롤링 완료 후 실행되며, `SmartstoreProduct`에 등록된 매핑 정보를 기반으로 `product_id`가 비어있는 기존 리뷰들을 일괄 업데이트한다.

#### 3.3.5 스마트스토어 상품 연동/해제 API

**연동** (`SmartstoreProduct__Feature.create`):
- `smartstore_product_no`로 기존 레코드 조회 (`get_or_create`)
- 이미 다른 `product_id`와 연동된 경우 "이미 연동된 상품이에요." 에러 반환
- 동일 상품이면 업데이트

**해제** (`SmartstoreProduct__Manager.delete`):
- `shop_id`와 `id`로 조회하여 삭제

### 3.4 네이버 블로그 리뷰 수집

관리자가 개별 블로그 게시글 URL을 입력하여 리뷰를 생성하는 수동 방식이다.

#### 3.4.1 URL 검증 (`NaverBlogUrl__Feature`)

- URL은 반드시 `https://blog.naver.com/`으로 시작해야 함
- gRPC를 통해 웹 크롤러 서비스에 크롤링 가능 여부 확인 요청 (`is_possible_to_crawling`)

#### 3.4.2 리뷰 생성 (`NaverBlog__Feature`)

**입력 파라미터**:
- `product_id`: 연동할 상품 ID (필수)
- `user_name`: 작성자 이름 (필수)
- `user_id`: 작성자 아이디 (선택, `CustomerOption.member_id`와 매칭)
- `url`: 블로그 게시글 URL (필수)

**처리 흐름**:
1. gRPC로 웹 크롤러 서비스 호출하여 블로그 크롤링 (`do_naver_blog_crawling`)
   - 크롤링 결과: `sns_user_name`, `content`, `img_urls`, `like_count`, `comments_count`
2. `user_id`가 입력된 경우 `CustomerOption`에서 회원 조회:
   - 회원 매칭 성공: 임시 `Order`, `OrderItem` 생성 (혜택 지급 목적) -> 회원정보 기반 리뷰 생성
   - 회원 매칭 실패: 입력된 `user_name`, `user_id`로 리뷰 생성
3. 리뷰 생성 속성:
   - `review_type`: `소셜` (4)
   - `creater_type`: `시스템` (6)
   - `platform`: `네이버블로그`
   - `ratings`: 고정값 5
   - `posting_status`: `게시됨` (20)
   - `social_link`: 블로그 URL
4. 이미지 다운로드:
   - URL에서 `?type=w773` 파라미터 추가 후 http로 변환하여 다운로드
   - 이미지가 있으면 `is_photo = True`

### 3.5 Facebook 웹훅 처리 (Cafe24 리뷰 웹훅)

Cafe24 게시판 리뷰 변경 이벤트를 처리하는 별도의 웹훅 경로이다 (Instagram과 다른 경로).

**허용 이벤트** (`EVENT_NO_ALLOWED`):
- `게시물생성` (90033)
- `게시물삭제` (90036)
- `게시물수정` (90039)

**필터링 규칙** (`filter_webhook`):
1. `event_no`, `resource` 필드 존재 확인
2. `board_no == 4`인 경우만 허용 (리뷰 게시판)
3. 작성자 블랙리스트 필터: "카톡luxky7", "카톡boqi88", "카톡:XXKK", "One race", "royal race", "한미차집"

**처리 방식**:
- 유효한 웹훅 데이터를 SQS로 전송 (`Webhook__Feature.push_webhook_to_sqs`, 기본 60초 지연)
- SQS 소비자가 `Webhook__Processor`를 통해 `review_refresh`(생성/수정) 또는 `review_delete`(삭제) 처리

## 5. Usecase 흐름

### 5.1 관리자가 Instagram 게시물을 상품에 연동

```
1. 관리자 -> POST /external/instagram (새로고침 요청)
   -> InstagramMedia__Feature.update(shop_id)
   -> ShopInstagram 갱신주기/만료일자 검증
   -> Facebook Graph API로 태그된 게시물 수집 (최대 2000건, 최근 1년)
   -> InstagramMedia + InstagramMediaPhoto 생성/업데이트

2. 관리자 -> GET /external/instagram (게시물 목록 조회)
   -> 정렬: 1st 게시일자(date) 내림차순, 2nd (좋아요+댓글) 합산 내림차순
   -> 필터: 태그/내 게시물, 기간, 검색어, 상품연동 여부

3. 관리자 -> POST /external/instagram/connection
   -> { ig_media_ids: [...], product_ids: [...] }
   -> 각 ig_media에 대해:
      a. 기존 연동 상품 vs 새 상품 비교
      b. 신규: Review + ReviewPhoto 생성
      c. 해제: 해당 Review 삭제
```

### 5.2 고객이 Instagram SNS 리뷰를 작성

```
1. 고객 -> 주문 완료 (OrderItem.permit_sns=True, short_code/hashtag_all 할당)

2. 고객 -> Instagram에 게시물 업로드 (쇼핑몰 계정 태그 + 해시태그 포함)

3. Facebook -> POST /webhook/facebook (웹훅 전송)
   -> Facebook__Feature.process_webhook_data()
      a. entry.id로 shop 조회
      b. Graph API v15.0으로 미디어 상세 조회
      c. caption에서 해시태그 추출
   -> Facebook__Feature.create_facebook_review()
      a. 해시태그에서 short_code 매칭
      b. 필수 해시태그 전체 포함 여부 검증
      c. SNS Review 생성 (update_or_create)
      d. 혜택 계산 및 ReviewReward 생성
      e. 게시 상태 설정
      f. 미디어 파일 다운로드 및 ReviewPhoto/ReviewVideo 생성

4. 배치 (1시간마다) -> every_1hour_review_facebook
   -> 유지기간 만료: is_sns_completed = True
   -> 미완료 리뷰: Graph API로 해시태그 유지 여부 확인
      a. 유지 -> sns_deauthed_at = None, 정보 업데이트
      b. 미유지 -> deauthorize_sns (인증 해제)
```

### 5.3 스마트스토어 리뷰 수집

```
1. 관리자 -> PATCH /external/smartstore
   -> SmartstoreReview 설정 수정 (identify_name, url 등)

2. 관리자 -> POST /external/smartstore/product
   -> SmartstoreProduct 생성 (알파리뷰 상품 <-> 스마트스토어 상품번호 매핑)

3. 배치 (매일 06:00 AM) -> daily_06am_smartstore_remote
   -> 구독 업체 목록 조회 (Service__Type.스마트스토어리뷰연동 == 라이브)
   -> 업체별 크롤링 (ThreadPoolExecutor, max_workers=6):
      a. 네이버 커머스 어드민 로그인
      b. 스토어 선택
      c. 리뷰 엑셀 다운로드 (start_date ~ end_date)
      d. 엑셀 파싱 -> Review 생성/업데이트
      e. 스마트스토어 상품 매핑 갱신
   -> 실패 시 최대 3회 재시도
```

### 5.4 네이버 블로그 리뷰 등록

```
1. 관리자 -> POST /external/blog/url
   -> { url: "https://blog.naver.com/..." }
   -> gRPC로 크롤링 가능 여부 확인

2. 관리자 -> POST /external/blog
   -> { product_id, user_name, user_id?, url }
   -> gRPC로 블로그 크롤링 (content, img_urls, like_count, comments_count)
   -> user_id가 있으면 CustomerOption 조회:
      a. 회원 매칭: 임시 Order/OrderItem 생성 -> 혜택 지급 가능
      b. 미매칭: 입력값으로 리뷰 생성
   -> Review + ReviewPhoto 생성 (platform=네이버블로그)
```

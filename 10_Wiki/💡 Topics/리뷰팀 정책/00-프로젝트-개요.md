# Alpha Review 프로젝트 개요

## 1. 프로젝트 소개

Alpha Review는 이커머스 쇼핑몰을 위한 **리뷰 관리 플랫폼**이다. 온라인 쇼핑몰에 리뷰 위젯을 설치하여 고객 리뷰를 수집, 관리, 분석하고, 리뷰 작성에 대한 보상 지급, KakaoTalk 메시지 발송, AI 기반 리뷰 분석 등의 기능을 제공한다.

### 핵심 가치
- 쇼핑몰별 리뷰 수집 및 관리 (멀티테넌시)
- 다양한 채널의 리뷰 통합 (자체, Instagram, 네이버블로그, 스마트스토어, CAFE24 등)
- AI 기반 리뷰 분석 (감성분석, 키워드 추출, VOC 태그)
- 리뷰 작성 보상 시스템 (포인트, 쿠폰)
- 위젯 기반 리뷰 노출 시스템
- KakaoTalk 알림톡을 통한 리뷰 작성 유도

## 3. 아키텍처 구조

### 3.1 서비스 역할 분리
Django 서버는 환경변수 `SERVICE_ROLE`에 따라 두 가지 모드로 동작한다:

- **admin**: 관리자 대시보드 API (전체 미들웨어 스택 - 세션, 인증, CSRF 포함)
- **widget**: 위젯 렌더링 API (경량 미들웨어 - 캐싱 최적화를 위해 세션/인증 미들웨어 제거)

### 3.2 마이크로서비스 구조
```
┌─────────────────────────────────────────────────┐
│              Django Main Server                  │
│  (admin / widget 모드)                           │
│  - 23개 Django 앱                                │
│  - REST API (V1 + V2)                            │
└──────────┬──────────┬───────────┬────────────────┘
           │ gRPC     │ gRPC      │ SQS/HTTP
    ┌──────▼──┐  ┌────▼────┐  ┌──▼──────────────┐
    │Keyword  │  │Web      │  │AI Review        │
    │Service  │  │Crawler  │  │Analyzer         │
    │:50051   │  │Service  │  │(OpenAI/Gemini/  │
    │(kiwipiepy)│ │:50052   │  │Bedrock)         │
    └─────────┘  │(Selenium)│  └─────────────────┘
                 └─────────┘
    ┌─────────────────┐  ┌──────────────────┐
    │Review Persistence│  │Init Transfer     │
    │Service (Worker)  │  │Worker            │
    └─────────────────┘  └──────────────────┘
```

### 3.3 데이터베이스 구조
- **Primary (Write)**: `DB_WRITE_HOST` - 쓰기 작업
- **Replica (Read)**: `DB_READ_HOST` - 읽기 작업
- **스키마**: `DB_SCHEMA` 환경변수로 지정
- **드라이버**: psycopg2

## 5. URL 라우팅 구조

### 5.1 V1 API (Shop 스코프)
모든 V1 엔드포인트는 `/shops/<int:shop_id>/` 하위에 위치하여 멀티테넌시를 구현한다.

```
/healthcheck                          - 헬스체크
/shops/<shop_id>/                     - 샵 설정/통계
/shops/<shop_id>/reviews              - 리뷰 관리
/shops/<shop_id>/policy               - 리뷰 정책
/shops/<shop_id>/products             - 상품 관리
/shops/<shop_id>/orders               - 주문 관리
/shops/<shop_id>/widgets              - 위젯 관리
/shops/<shop_id>/alimtalk             - 알림톡 설정
/shops/<shop_id>/mypage               - 마이페이지
/shops/<shop_id>/external             - 외부 연동
/shops/<shop_id>/feed                 - Instagram 피드
/shops/<shop_id>/seeds                - 시드 캠페인
/shops/<shop_id>/ticker               - 세일즈 티커
/shops/<shop_id>/alarms               - 알림
/shops/<shop_id>/reports              - 리포트
/webhook                              - 웹훅 (외부 리뷰, Facebook)
```

### 5.2 V2 API (신규)
```
/v2/shops/<shop_id>/reviews           - 리뷰 API V2
/v2/shops/<shop_id>/widgets           - 위젯 대시보드 V2
/v2/shops/<shop_id>/products          - 상품 API V2
/v2/shops/<shop_id>/stats             - 분석/통계 V2
/v2/api-widget                        - 위젯 스크립트 API V2
/v2/widget                            - 위젯 프리뷰 API
```

### 5.3 공통 엔드포인트 (Shop 스코프 외부)
```
/alimtalk                             - 알림톡 공통
/widget                               - 위젯 공통
/review                               - 리뷰 공통
/api_widget                           - 레거시 위젯 뷰
/feed                                 - Instagram 피드 위젯
```

## 8. 핵심 코드 패턴

### 8.1 멀티테넌시
- 모든 모델에 `shop_id` 필드가 있어 쇼핑몰별 데이터 격리
- URL 경로에 `<int:shop_id>`로 테넌트 식별
- `ShopDataAPIViewMixin`이 요청 시 shop 데이터를 가져와 request 객체에 첨부

### 8.2 뷰 아키텍처
- `BaseAPIViewMixin + APIView` 조합 (ViewSet 미사용)
- `PaginationAPIViewMixin`으로 페이지네이션 처리
- 클래스 네이밍: `Feature__ViewType` (예: `Badge__ListView`, `Shop__View`)

### 8.3 시리얼라이저
- `DynamicFieldsMixin`: 동적 필드 필터링 지원
- 위젯 시리얼라이저: Strategy 패턴으로 20+ 위젯 타입별 처리
- 중첩 시리얼라이저로 복합 데이터 구조 표현

### 8.4 데이터 패턴
- 소프트 삭제: `is_deleted`, `deleted_at` 필드
- 감사 필드: `created_at`, `updated_at` 타임스탬프
- 비정규화: Product, OrderItem에 집계 카운트 직접 저장
- 복합 유니크 제약조건 활용

### 8.5 비즈니스 타입 정의
- `common/types.py`에 모든 비즈니스 열거형(IntegerChoices, TextChoices) 정의
- 한국어 레이블 사용 (예: `Review__ReviewType.일반`, `Review__PostingStatus.게시됨`)

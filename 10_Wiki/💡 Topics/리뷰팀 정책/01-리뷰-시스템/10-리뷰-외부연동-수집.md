# 리뷰 외부연동/수집

## 1. 개요

외부연동 시스템은 다양한 외부 플랫폼(Instagram, 네이버 스마트스토어, 네이버 블로그, CAFE24 등)에서 리뷰를 수집하여 Alpha Review 시스템으로 통합하는 기능이다. 각 플랫폼별로 수집 방식이 다르며, Facebook/Instagram 웹훅, API 연동, 웹 크롤링 등의 방식을 사용한다.

---

## 2. 플랫폼 유형 (Review__Platform)

Review 모델의 `platform` 필드는 리뷰가 작성된 플랫폼을 나타낸다.

| 플랫폼 값 | 설명 |
|-----------|------|
| `알파리뷰` | Alpha Review 자체 위젯에서 작성된 리뷰 (기본값) |
| `CAFE24` | CAFE24 플랫폼에서 수집된 리뷰 |
| `스마트스토어` | 네이버 스마트스토어에서 수집된 리뷰 |
| `지그재그` | 지그재그 플랫폼에서 수집된 리뷰 |
| `인스타그램` | Instagram을 통해 수집된 SNS 리뷰 |
| `네이버블로그` | 네이버 블로그에서 수집된 리뷰 |

## 3. Instagram 리뷰 연동

### 3.1 데이터 모델

#### InstagramMedia (인스타그램 미디어)

| 필드 | 타입 | 설명 |
|------|------|------|
| `shop_id` | Integer | 쇼핑몰 고유 ID |
| `is_tagged` | Boolean | 태그된 게시물 여부 (True=태그된 게시물, False=내 게시물) |
| `media_id` | String(50) | Instagram Meta 미디어 ID |
| `caption` | Text | 게시글 내용 |
| `permalink` | URL | 게시글 링크 |
| `comments_count` | Integer | 댓글 수 |
| `like_count` | Integer | 좋아요 수 |
| `username` | String(127) | 작성자 아이디 |
| `timestamp` | DateTime | 게시일자 |

- **Unique Constraint**: `shop_id` + `media_id` + `is_tagged` 조합이 유일해야 한다.

#### InstagramMediaPhoto (인스타그램 미디어 이미지)

| 필드 | 타입 | 설명 |
|------|------|------|
| `ig_media` | FK(InstagramMedia) | 인스타그램 미디어 참조 |
| `is_photo` | Boolean | 이미지 여부 (True=이미지, False=동영상 썸네일) |
| `photo_url` | URL | 이미지/썸네일 링크 |

#### ReviewInstagramRelation (리뷰-인스타그램 매핑)

| 필드 | 타입 | 설명 |
|------|------|------|
| `review` | FK(Review) | 리뷰 참조 |
| `ig_media` | FK(InstagramMedia) | 인스타그램 미디어 참조 |

- **Unique Constraint**: `review` + `ig_media` 조합이 유일해야 한다.

### 3.2 수집 방식

Instagram 리뷰는 **Facebook Graph API 웹훅**을 통해 수집된다.

#### Facebook 웹훅 수신 흐름 (`Review__WebhookFacebookView`)

```
1. Facebook 웹훅 검증 (GET 요청)
   - hub.challenge 파라미터를 그대로 반환하여 검증 완료

2. 인스타그램 멘션/코멘트 수신 (POST 요청)
   - FacebookWebhookResponseSerializer로 데이터 검증
   - Facebook__Feature.process_webhook_data()로 데이터 처리
   - Facebook__Feature.create_facebook_review()로 리뷰 생성

3. 항상 HTTP 200 반환 (에러 발생 시에도)
   - Facebook 웹훅 스펙에 따라 200 외 응답 시 재전송이 발생하므로 항상 200 반환
```

#### 리뷰 생성 시 처리 (`Facebook__Feature.create_facebook_review`)

```
1. 웹훅 데이터에서 media_id 추출
2. 쇼핑몰의 SNS 허용 주문상품(order_item) 조회
3. Review 생성 또는 업데이트:
   - review_type = SNS
   - platform = 인스타그램
   - ratings = 5 (SNS 리뷰는 기본 5점)
   - sns_review_id = media_id
4. sns_completed_at 설정 (현재 시점 + 유지 기간)
5. 주문상품의 is_review_sns_written = True 업데이트
6. ReviewLog 생성 ("리뷰작성 : SNS리뷰")
```

## 4. SNS 인증 완료/해제 규칙

SNS 리뷰는 일정 기간 동안 게시글을 유지해야 인증이 완료되는 구조이다.

### 4.1 관련 필드 (Review 모델)

| 필드 | 타입 | 설명 |
|------|------|------|
| `is_sns_completed` | Boolean | SNS 인증 완료 여부 (기본값: False) |
| `sns_completed_at` | DateTime | SNS 인증 완료 예정 일시 |
| `sns_deauthed_at` | DateTime | SNS 인증 해제 일시 |

### 4.2 인증 상태 판별 로직

| 상태 | 조건 | 설명 |
|------|------|------|
| 인증 진행중 | `is_sns_completed=False`, `sns_completed_at > now`, `sns_deauthed_at=NULL` | 유지 기간 진행 중 |
| 인증 완료 | `is_sns_completed=True` | 유지 기간을 충족하여 인증 완료 |
| 인증 해제 | `sns_deauthed_at IS NOT NULL` | 해시태그 제거, 비공개 전환, 보관 등의 사유로 인증 해제 |

### 4.3 매시간 검증 크론 (`every_1hour_review_facebook`)

```
1. 인증 완료 처리:
   - 조건: review_type=SNS, is_sns_completed=False, sns_completed_at <= now, sns_deauthed_at=NULL
   - 동작: is_sns_completed=True로 업데이트

2. 진행 중 리뷰 검증 (2가지 케이스):
   a) 정상 진행 중: sns_completed_at > now, sns_deauthed_at=NULL
   b) 인증 해제 후 60일 이내: sns_completed_at > now, sns_deauthed_at >= (now - 60일)

3. 각 리뷰에 대해 Facebook Graph API 호출:
   - mentioned_media 조회 (media_url, username, children, like_count, caption, permalink 등)

4. API 응답에 따른 처리:
   - 에러 응답: sns_deauthed_at 설정 (인증 해제)
   - 정상 응답: 해시태그 포함 여부 검증
     - 필수 해시태그 누락 시: 인증 해제 (deauthorize_sns)
     - 해시태그 정상 포함 시: social_link, sns_like_count, sns_user_id 업데이트
```

### 4.4 인증 해제 사유
- Instagram 게시글 삭제
- 게시글 비공개 전환
- 게시글 보관 처리
- 필수 해시태그 제거
- Instagram 계정 비활성화

## 5. Smartstore 연동

### 5.1 데이터 모델

#### SmartstoreReview (스마트스토어 리뷰 연동 설정)

| 필드 | 타입 | 설명 |
|------|------|------|
| `shop_id` | Integer | 쇼핑몰 고유 ID |
| `update_at` | DateTime | 마지막 업데이트 일시 |
| `identify_name` | String(50) | 식별 이름 |
| `url` | Text | 스마트스토어 URL |
| `status` | String(50) | 연동 상태 |
| `status_text` | String(200) | 상태 설명 텍스트 |

> 참고: `type`, `user_id`, `password`, `allow_product_option`, `auto_product_name` 필드는 현재 **미사용** 상태이다.

#### SmartstoreProduct (스마트스토어 상품 매핑)

| 필드 | 타입 | 설명 |
|------|------|------|
| `shop_id` | Integer | 쇼핑몰 고유 ID |
| `product_id` | Integer | 상품 고유 ID |
| `product_fk` | FK(Product) | Alpha Review 상품 참조 |
| `smartstore_product_no` | String(255) | 스마트스토어 상품 번호 |
| `update_at` | DateTime | 마지막 업데이트 일시 |

#### SmartstoreReviewLink (스마트스토어 리뷰 링크)

| 필드 | 타입 | 설명 |
|------|------|------|
| `shop_id` | Integer | 쇼핑몰 고유 ID |
| `product` | FK(Product) | Alpha Review 상품 참조 |
| `type` | String(50) | 연동 유형 |
| `status` | String(50) | 연동 상태 |
| `status_text` | String(200) | 상태 설명 텍스트 |
| `link` | Text | 리뷰 페이지 링크 |
| `link_product_no` | String(200) | 연동 상품 번호 |
| `link_product_name` | String(200) | 연동 상품명 |
| `link_product_img` | String(200) | 연동 상품 이미지 |
| `update_at` | DateTime | 마지막 업데이트 일시 |

### 5.2 수집 방식
- 스마트스토어 리뷰 페이지 URL 기반으로 리뷰를 수집한다.
- `SmartstoreProduct`를 통해 스마트스토어 상품과 Alpha Review 상품을 매핑한다.
- `SmartstoreReviewLink`를 통해 특정 상품의 리뷰 링크를 관리한다.

## 6. CAFE24 웹훅 연동

### 6.1 수집 방식 (`Review__WebhookView`)

CAFE24 플랫폼에서 게시물(리뷰) 변경이 발생하면 웹훅으로 이벤트를 수신한다.

#### 허용 이벤트 (EVENT_NO_ALLOWED)

| 이벤트 | 설명 |
|--------|------|
| `게시물생성` | 리뷰 생성 (article - create) |
| `게시물삭제` | 리뷰 삭제 (article - destroy) |
| `게시물수정` | 리뷰 수정 (article - update) |

#### 필터링 규칙

```
1. event_no가 허용 이벤트 목록에 포함되어야 한다.
2. resource에 board_no가 포함된 경우 board_no=4(리뷰 게시판)만 허용한다.
3. 작성자 블랙리스트 필터링:
   - 스팸으로 식별된 특정 작성자명은 차단된다.
```

#### 처리 흐름

```
1. 웹훅 데이터 검증 (filter_webhook)
2. Webhook__ReviewSerializer로 데이터 직렬화
3. SQS로 웹훅 데이터 전송 (Webhook__Feature.push_webhook_to_sqs)
   - event_type, mall_id, shop_no, article_no, writer 정보 포함
4. Worker에서 비동기로 리뷰 데이터 처리
```

## 7. 네이버 블로그 연동

### 7.1 수집 방식
- 네이버 블로그 리뷰는 **웹 크롤링**을 통해 수집된다.
- `web_crawler_grpc` 서비스를 활용한 gRPC 기반 크롤링이다.
- 수집된 리뷰의 `platform`은 `네이버블로그`로 설정된다.

---

## 8. 플랫폼별 수집 방식 비교

| 플랫폼 | 수집 방식 | 실시간성 | 비고 |
|--------|-----------|----------|------|
| **알파리뷰** | 위젯 직접 작성 | 실시간 | 자체 플랫폼 |
| **Instagram** | Facebook Graph API 웹훅 | 준실시간 | 멘션/태그 기반, 주기적 인증 검증 |
| **CAFE24** | 플랫폼 웹훅 | 준실시간 | SQS 비동기 처리 |
| **스마트스토어** | API/URL 기반 수집 | 배치 | 상품 매핑 필요 |
| **네이버블로그** | 웹 크롤링 (gRPC) | 배치 | web_crawler_grpc 서비스 |
| **지그재그** | 외부 연동 | 배치 | 플랫폼 API |

## 9. Review 모델의 외부연동 관련 필드

| 필드 | 설명 |
|------|------|
| `platform` | 리뷰 작성 플랫폼 (Review__Platform choices) |
| `sns_review_id` | SNS 리뷰 고유 아이디 (Instagram media_id 등) |
| `sns_user_id` | SNS 리뷰 유저 아이디 |
| `social_link` | SNS 게시글 퍼머링크 |
| `sns_like_count` | SNS 좋아요 수 |
| `is_sns_completed` | SNS 인증 완료 여부 |
| `sns_completed_at` | SNS 인증 완료 예정 일시 |
| `sns_deauthed_at` | SNS 인증 해제 일시 |
| `external_product_name` | 외부연동 임시 상품명 |
| `external_product_option` | 외부연동 임시 상품옵션 |
| `external_product_code` | 외부연동 임시 상품코드 |
| `external_content` | 외부연동 임시 리뷰내용 |

`external_*` 필드는 외부 플랫폼에서 수집한 데이터가 아직 Alpha Review 상품과 매핑되지 않은 상태에서 임시로 저장되는 필드이다.

# 리뷰 게시 정책 (Shop Review Policy)

> **앱**: `shopreviewpolicy`
> **최종 갱신**: 2026-02-20

---

## 1. 도메인 개요

리뷰 게시 정책은 쇼핑몰별로 리뷰 작성/게시/보상에 관한 규칙을 정의하는 도메인이다. 크게 다음 네 가지 영역으로 나뉜다.

| 영역 | 핵심 모델 | 역할 |
|------|-----------|------|
| **검수 옵션** | `ShopConfirmOption` | 리뷰 게시 시점(자동/수동) 및 보상 지급 시점 결정 |
| **리뷰 작성 제한** | `ShopPermit`, `ShopPermitProduct`, `ShopPermitCategory` | 리뷰 유형별 ON/OFF 및 상품/카테고리별 세부 제한 |
| **리뷰 정책** | `ShopPolicy` | 작성 가능 고객, 작성 기한, 보상 한도, 최소 콘텐츠 기준 등 |
| **리뷰 유형별 설정** | `ShopQuickReview`, `ShopKeywordReview`, `ShopInstagram`, `ShopReward` | 빠른리뷰 문구, 키워드리뷰 카테고리, 인스타그램 연동, 혜택 금액 |

이 도메인의 설정은 리뷰 작성 시점, 게시 판단 시점, 보상 지급 시점 등 리뷰 라이프사이클 전반에 걸쳐 참조된다.

---

## 2. 데이터 모델

### 2.1 ShopConfirmOption (게시/보상 검수 옵션)

쇼핑몰 단위(`shop_id` unique)로 리뷰 게시와 보상 지급의 자동/수동 판단 기준을 설정한다.

```
ShopConfirmOption
├── shop_id: int (unique)
│
├── [게시 옵션]
│   ├── posting_option: str          -- "immediate" = 자동 게시, 그 외 = 수동 게시
│   ├── posting_option_ratings: int  -- N점 이하 리뷰 수동 게시 (0 = 비활성)
│   ├── posting_option_keyword: bool -- 알림 키워드 포함 시 수동 게시
│   ├── posting_option_photo: bool   -- 이미지 포함 시 수동 게시
│   ├── posting_option_video: bool   -- 동영상 포함 시 수동 게시
│   ├── posting_option_duplicatedMedia: bool -- 중복 이미지 포함 시 수동 게시
│   └── posting_option_sentimentNegative: bool -- 부정적 리뷰 수동 게시
│
└── [보상 옵션]
    ├── reward_option: str           -- "admin" = 관리자 수동, "immediate" = 즉시, 숫자 = N일 후
    ├── reward_option_ratings: int   -- N점 이하 리뷰 보상 수동 지급
    ├── reward_option_keyword: bool  -- 키워드 포함 시 보상 수동 지급
    ├── reward_option_photo: bool    -- 이미지 포함 시 보상 수동 지급
    ├── reward_option_video: bool    -- 동영상 포함 시 보상 수동 지급
    └── reward_option_duplicatedMedia: bool -- 중복 이미지 포함 시 보상 수동 지급
```

**기본값 정책**: 게시는 `"immediate"` (자동 게시), 보상은 `"admin"` (관리자 수동 지급). 모든 부가 플래그는 기본 비활성(`False` / `0`).

### 2.2 ShopPermit (리뷰 작성 제한)

쇼핑몰 단위로 리뷰 유형별 작성 허용 여부를 관리한다.

```
ShopPermit
├── shop_id: int (unique)
├── permit_normal: bool   -- 일반 리뷰 (기본: True)
├── permit_quick: bool    -- 빠른 리뷰 (기본: True)
├── permit_survey: bool   -- 설문 리뷰 (기본: False)
├── permit_video: bool    -- 동영상 리뷰 (기본: True)
├── permit_sns: bool      -- SNS 리뷰 (기본: False)
├── permit_keyword: bool  -- 키워드 리뷰 (기본: False)
├── permit_unverified: bool -- 미검증 리뷰 (기본: False)
└── updated_at: datetime
```

**참고**: `ShopPermit` 변경 시 `ShopWritePage__Feature.sort(shop_id)`가 호출되어 위젯 내 리뷰 작성 페이지의 정렬 순서가 자동으로 갱신된다.

### 2.3 ShopPermitProduct / ShopPermitCategory (상품/카테고리별 제한)

`ShopPermit`의 하위 관계로, 특정 상품이나 카테고리에 대해 세부적인 제한 유형을 지정한다.

```
ShopPermitProduct                    ShopPermitCategory
├── shop_permit: FK(ShopPermit)      ├── shop_permit: FK(ShopPermit)
├── product_id: int                  ├── category_id: int (deprecated 예정)
└── type: int (ShopPermitRelation__Type)  ├── category_platform_id: str
                                     └── type: int (ShopPermitRelation__Type)
```

**ShopPermitRelation__Type (제한 유형)**:

| 코드 | 라벨 | 의미 |
|------|------|------|
| 100 | `reward` | 포인트 지급 제외 |
| 200 | `normal` | 전체 리뷰 작성 제외 |
| 210 | `quick` | 빠른 리뷰 작성 제외 |
| 220 | `keyword` | 키워드 리뷰 작성 제외 |
| 230 | `survey` | 설문 리뷰 작성 제외 |
| 240 | `video` | 동영상 리뷰 작성 제외 |
| 250 | `sns` | SNS 리뷰 작성 제외 |
| 260 | `unverified` | 미검증 리뷰 작성 제외 |
| 270 | `ticker` | 세일즈 티커 제외 |

### 2.4 ShopPolicy (리뷰 정책)

쇼핑몰 단위(`shop_id` unique)로 리뷰 정책의 전반적인 설정을 관리한다.

```

│
├── [작성 가능 고객]
│   ├── use_posting_nonmember: bool     -- 비회원 구매자 (기본: True)
│   ├── use_posting_npay: bool          -- 네이버페이 구매자 (기본: True)
│   ├── use_posting_zpay: bool          -- 지그재그페이 구매자 (기본: True)
│   ├── use_posting_kakaopay: bool      -- 톡 체크아웃 구매자 (기본: True)
│   └── use_posting_marketplus: bool    -- 기타 마켓플러스 구매자 (기본: True)
│
├── [리뷰 작성 기한]
│   ├── review_write_timing: int        -- 작성 시작 시점 (400=배송중, 500=배송완료)

│   └── use_reward_unverified: bool     -- 미검증 리뷰 보상 허용 (기본: False)
│
├── [포인트 지급 후 동작]
│   ├── use_review_revise: bool         -- 포인트 지급 후 수정 허용 (기본: True)
│   └── use_review_delete: bool         -- 포인트 지급 후 삭제 허용 (기본: True)

├── [포인트 한도]
│   ├── use_min_point: bool             -- 최소 포인트 한도 설정 (기본: False)
│   ├── min_point: int                  -- 최소 포인트(정액) (기본: 10)
│   ├── use_max_total_point: bool       -- 최대 포인트 한도 설정 (기본: False)
│   ├── use_max_total_point_fixed: bool -- 정액제 최대 한도 활성화 (기본: True)
│   ├── max_total_point_fixed: int      -- 정액제 최댓값 (기본: 20,000)
│   ├── use_max_total_point_percentage: bool -- 정률제 최대 한도 활성화 (기본: False)
│   ├── max_total_point_percentage: str -- 정률제 최댓값 (기본: "10%")
│   ├── use_max_total_point_range: bool -- 추가 포인트를 한도에서 제외 (기본: False)

│
├── [최소 콘텐츠 기준]
│   ├── use_min_text: bool              -- 글자수 제한 설정 (기본: True)
│   ├── min_text: int                   -- 최소 글자 수 (기본: 10)
│   ├── min_photo: int                  -- 최소 사진 수 (기본: 1)
│   └── min_video: int                  -- 최소 동영상 길이(초) (기본: 5)
│
├── [SNS 리뷰 정책]
│   ├── use_maintenance_sns: bool       -- 최소 유지 기한 설정 (기본: True)

│   ├── reward_condition_price: int          -- 최소 구매 금액 (기본: 0)
│   ├── use_reward_condition_delivery: bool  -- 수령 후 보상 불가 기간 제한 (기본: False)
│   ├── reward_condition_delivery: int       -- 불가 기간(일) (기본: 90)
│   ├── use_reward_condition_soldout: bool   -- 품절 상품 지급 여부 (기본: True=지급)
│   ├── use_reward_condition_display: bool   -- 미진열 상품 지급 여부 (기본: True=지급)

│   └── use_refund_point_cancel: bool   -- 취소/교환/반품 시 자동 회수 (기본: True)
```

### 2.6 ShopQuickReview (빠른리뷰 설정)

쇼핑몰별로 빠른리뷰 문구를 관리한다.

```
ShopQuickReview
├── shop_id: int
├── evaluation: str   -- "만족"(5,4점) / "보통"(3점) / "별로"(2,1점)
├── review_text: str  -- 빠른리뷰 문구 (최대 300자)
└── is_custom: bool   -- True=사용자 정의, False=프리셋
```

**빠른리뷰 타입** (`ShopPolicy.quick_review_type`):
- `1` (커스텀): 쇼핑몰이 직접 정의한 빠른리뷰 문구 사용
- `2` (프리셋): 시스템 기본 프리셋 빠른리뷰 문구 사용

### 2.7 ShopKeywordReview (키워드리뷰 설정)

```
ShopKeywordReview
├── shop_id: int (unique)
├── init_keyword: bool          -- 키워드 초기화 여부
├── selected_category: str      -- 카테고리 선택 (패션의류/반려동물/식품/뷰티/디지털/...)
├── widget_auto_install: bool   -- 키워드 요약 위젯 ON/OFF
└── widget_display_limit: int   -- 위젯 표시 기준 (기본: 10)
```

### 2.8 ShopInstagram (인스타그램 연동 관리)

```
ShopInstagram
├── shop_id: int (unique)
├── last_refresh: datetime      -- 마지막 새로고침 실행일
├── last_update: datetime       -- 마지막 조회 게시물 게시일
├── media_expired: datetime     -- 가장 빠른 이미지 만료일
├── refresh_status: int         -- 진행(100) / 실패(200) / 완료(300)
├── refresh_error_message: str  -- 에러 메시지
└── refresh_error_code: int     -- 에러 코드
```

**속성(Property)**:
- `check_last_refresh`: 마지막 갱신일로부터 1시간 이상 경과해야 `True` 반환 (갱신 주기 제한)
- `check_media_expired`: 이미지 만료일이 현재보다 미래이면 `True`, 만료되었으면 `False`

### 2.9 보조 모델

| 모델 | 역할 |
|------|------|
| `AlertKeyword` | 리뷰에 특정 키워드가 포함될 때 관리자 알림 발생 대상 단어 관리 |

## 3. 비즈니스 규칙

### 3.1 리뷰 게시 판단 규칙 (ShopConfirmOption)

리뷰가 작성되면 `ShopConfirmOption`을 참조하여 자동 게시 여부를 결정한다.

**자동 게시 조건** (`posting_option == "immediate"`인 경우):
- 다음 조건 중 하나라도 해당되면 **수동 게시로 전환**:

| 조건 | 필드 | 판단 기준 |
|------|------|-----------|
| 저평점 | `posting_option_ratings` | 리뷰 별점이 설정값 이하 (0이면 비활성) |
| 키워드 포함 | `posting_option_keyword` | `AlertKeyword`에 등록된 키워드가 리뷰에 포함 |
| 이미지 포함 | `posting_option_photo` | 리뷰에 이미지가 첨부됨 |
| 동영상 포함 | `posting_option_video` | 리뷰에 동영상이 첨부됨 |
| 중복 이미지 | `posting_option_duplicatedMedia` | 기존 리뷰와 동일한 이미지가 감지됨 |
| 부정적 감정 | `posting_option_sentimentNegative` | 감정 분석 결과 부정적으로 판별됨 |

위 조건에 해당하지 않으면 리뷰는 즉시 자동 게시된다.

### 3.2 보상 지급 판단 규칙 (ShopConfirmOption)

**보상 지급 시점 결정** (`reward_option` 값에 따라):
- `"admin"`: 관리자가 수동으로 보상 지급
- `"immediate"`: 리뷰 게시 즉시 보상 자동 지급
- 숫자(예: `"7"`): 리뷰 게시 후 N일 후 자동 지급

**보상 자동 지급 시 수동 전환 조건**:
- `reward_option_ratings`: 별점이 설정값 이하
- `reward_option_keyword`: 알림 키워드 포함
- `reward_option_photo`: 이미지 포함
- `reward_option_video`: 동영상 포함
- `reward_option_duplicatedMedia`: 중복 이미지 감지

### 3.3 리뷰 작성 허용 규칙 (ShopPermit)

**쇼핑몰 전체 수준**: `ShopPermit`의 `permit_*` 플래그로 리뷰 유형별 ON/OFF를 결정한다.

| 필드 | 기본값 | 대상 리뷰 유형 |
|------|--------|----------------|
| `permit_normal` | True | 일반 (텍스트/포토) 리뷰 |
| `permit_quick` | True | 빠른리뷰 (별점 + 선택 문구) |
| `permit_survey` | False | 설문 리뷰 |
| `permit_video` | True | 동영상 리뷰 |
| `permit_sns` | False | SNS(인스타그램) 리뷰 |
| `permit_keyword` | False | 키워드 리뷰 |
| `permit_unverified` | False | 미검증 리뷰 |

**상품/카테고리 수준**: `ShopPermitProduct`와 `ShopPermitCategory`를 통해 특정 상품 또는 카테고리에 대해 `ShopPermitRelation__Type`에 해당하는 제한을 적용한다.

- 예: `product_id=123`에 `type=100(포인트지급_제외)` -> 해당 상품 리뷰에 포인트 미지급
- 예: `category_platform_id="45"`에 `type=250(SNS리뷰작성_제외)` -> 해당 카테고리 상품은 SNS리뷰 작성 불가

**카테고리 제한의 하위 전파**: `ShopPermitCategory__Features.group_by_category()`는 지정된 카테고리의 하위(child) 카테고리까지 자동으로 포함하여 제한을 적용한다. 하위 카테고리 조회 결과는 5분간 캐시된다.

### 3.4 리뷰 작성 자격 규칙 (ShopPolicy)

**작성 가능 고객 유형**:
- `use_posting_nonmember`: 비회원 구매자
- `use_posting_npay`: 네이버페이 구매자
- `use_posting_zpay`: 지그재그페이 구매자
- `use_posting_kakaopay`: 톡 체크아웃 구매자
- `use_posting_marketplus`: 기타 마켓플러스 구매자

각 플래그가 `False`이면 해당 유형의 구매자는 리뷰 작성이 불가하다. 기본값은 모두 `True`이다.

**리뷰 작성 기한**:
- `review_write_timing`이 `400(배송중)`이면 배송이 시작된 시점부터 리뷰 작성 가능
- `review_write_timing`이 `500(배송완료)`이면 배송이 완료된 시점부터 리뷰 작성 가능
- `use_review_write_limit`이 `True`이면, 작성 시작 시점으로부터 `review_write_limit`일 이내에만 작성 가능 (기본 365일)

**미검증 리뷰**:
- `write_unverified_day`: 1일 최대 작성 가능 수 (기본 5건)
- `use_reward_unverified`: 미검증 리뷰에도 보상을 지급할지 여부 (기본 False)

### 3.5 콘텐츠 최소 기준 (ShopPolicy)

리뷰 유형별 보상을 받기 위한 최소 조건이다.

| 리뷰 유형 | 설정 | 기본값 | 설명 |
|-----------|------|--------|------|
| 텍스트 리뷰 | `use_min_text` / `min_text` | True / 10자 | 글자수 제한 활성화 시 최소 글자 수 |
| 포토 리뷰 | `min_photo` | 1장 | 최소 사진 첨부 수 |
| 동영상 리뷰 | `min_video` | 5초 | 최소 동영상 길이(초) |
| SNS 리뷰 | `use_maintenance_sns` / `maintenance_sns` | True / 3일 | 게시물 최소 유지 기간 |

### 3.6 포인트 지급 규칙 (ShopPolicy + ShopReward)

**포인트 금액 결정**: `ShopReward`에서 `title`(리뷰 종류) + `value`(작성자 유형) 조합으로 지급 포인트를 결정한다.
- 정액제: `reward` 필드에 숫자만 입력 (예: `"100"`)
- 정률제: `reward` 필드에 `%` 포함 입력 (예: `"10%"`)

**포인트 한도**:
- `use_min_point=True`일 때: 정률제 계산 결과가 `min_point`보다 작으면 `min_point` 보장
- `use_max_total_point=True`일 때:
  - `use_max_total_point_fixed=True`: 정액 최대 한도 적용 (`max_total_point_fixed`)
  - `use_max_total_point_percentage=True`: 정률 최대 한도 적용 (`max_total_point_percentage`)
- `use_max_total_point_range=True`: 추가 포인트를 최대 한도 계산에서 제외

**주문당 보상 횟수 제한**:
- `use_number_reward=True`일 때, 한 주문에서 `number_reward`회까지만 보상 지급
- `use_number_reward_quick=True`이면 제한 초과 시에도 빠른리뷰(별점리뷰) 보상은 허용

**추가 적립금 제한**:
- `use_general_extra_point_limit=True`: 일반 리뷰의 추가 적립금(선착순, 장문 등)을 1회로 제한
- `use_sns_extra_point_limit=True`: SNS 리뷰 추가 적립금을 제한

### 3.7 포인트 지급 제외 조건 (ShopPolicy)

| 조건 | 필드 | 기본값 | 설명 |
|------|------|--------|------|
| 최소 구매 금액 | `use_reward_condition_price` / `reward_condition_price` | False / 0 | 상품 구매 금액이 기준 미달 시 보상 제외 |
| 수령 후 기한 | `use_reward_condition_delivery` / `reward_condition_delivery` | False / 90일 | 배송 완료 후 N일 경과 시 보상 제외 |
| 품절 상품 | `use_reward_condition_soldout` | True | False 시 품절 상품 리뷰 보상 제외 |
| 미진열 상품 | `use_reward_condition_display` | True | False 시 미진열 상품 리뷰 보상 제외 |
| 미판매 상품 | `use_reward_condition_selling` | True | False 시 미판매 상품 리뷰 보상 제외 |

### 3.8 포인트 회수 규칙

- `use_refund_point_cancel=True` (기본): 주문 취소/교환/반품 시 지급된 포인트를 자동 회수
- `use_review_revise`: 포인트 지급 후에도 리뷰 수정 허용 여부
- `use_review_delete`: 포인트 지급 후에도 리뷰 삭제 허용 여부

### 3.9 보상 금액 검증 규칙 (ShopReward__DynamicSerializer)

`ShopReward`의 `reward` 필드는 다음과 같이 검증된다:
- **정액제**: 0 이상의 정수만 허용
- **정률제**: `%`로 끝나야 하며, 0 이상 100 이하의 실수만 허용

### 3.10 인스타그램 새로고침 규칙 (ShopInstagram)

- 갱신 주기: 마지막 새로고침(`last_refresh`) 이후 **최소 1시간** 경과해야 재갱신 가능
- 이미지 만료: `media_expired`가 현재 시간을 지나면 만료 상태로 판단

### 3.11 SNS 리뷰 보상 특수 규칙 (ShopReward__Manager)

- SNS 리뷰는 미검증 리뷰로 작성할 수 없다 (초기 ShopReward 생성 시 SNS리뷰 + 미검증리뷰 조합 제외)
- `customer_type`이 `"생성됨"`인 경우 `"비회원"`으로 자동 처리

## 5. Usecase 흐름

### 5.1 관리자가 게시 검수 옵션을 변경하는 흐름

```
1. 관리자 → PATCH /confirm-option
2. ShopConfirmOption__Manager.patch(shop_id, data)
   - 변경 필드만 동적으로 업데이트 (update_fields)
   - audit_log 데코레이터를 통해 변경 이력 기록
3. 변경된 필드만 응답으로 반환 (DynamicSerializer)
```

### 5.2 관리자가 리뷰 작성 제한을 변경하는 흐름

```
1. 관리자 → PATCH /shop-permit
2. ShopPermit__Features.patch(shop_id, data)  [트랜잭션]
   2a. shop_permit_categories가 포함된 경우:
       - 기존 카테고리 ID와 요청 카테고리 ID 비교
       - 교집합: 수정(유지), 차집합(기존-요청): 삭제, 새 항목: 생성
       - bulk_create / bulk_update / bulk_delete 실행
   2b. shop_permit_products가 포함된 경우:
       - 동일한 비교 로직으로 생성/수정/삭제 분류
   2c. ShopPermit 본체 필드 업데이트
3. ShopWritePage__Feature.sort(shop_id)
   - 위젯 내 리뷰 작성 페이지 정렬 순서 갱신
4. 응답 반환
```

### 5.3 특정 type 기준 상품/카테고리 제한 일괄 교체 흐름

```
1. 관리자 → POST /shop-permit/product?type=100 (또는 /category?type=100)
2. 해당 type을 가진 기존 데이터 전체 삭제
3. 요청된 새로운 데이터로 bulk_create
4. 생성된 결과 반환
```

이 방식은 "기존 삭제 → 새로 생성"의 교체(replace) 패턴을 사용한다.

### 5.4 관리자가 리뷰 정책을 변경하는 흐름

```
1. 관리자 → PATCH /shop-policy
2. ShopPolicy__DynamicSerializer로 요청 필드만 검증
3. ShopPolicy__Manager.patch(shop_id, data)
   - audit_log 데코레이터를 통해 변경 이력 기록
4. {"isOk": True} 응답 반환
```

**참고**: `use_reward_unverified` 변경 시 기존 미검증 리뷰에 대한 `ShopReward`가 없으면 회원 기준 혜택을 복사하여 생성해야 하는 로직이 TODO로 남아 있다.

### 5.5 관리자가 혜택 설정을 변경하는 흐름

```
1. 관리자 → PATCH /shop-reward
2. ShopReward__DynamicSerializer로 데이터 검증
   - reward 필드: 정액제(0 이상 정수) 또는 정률제(0~100% 실수) 검증
3. ShopReward__Features.patch(shop_id, data)  [트랜잭션]
   - 기존 ID와 요청 ID 비교:
     - id가 없는 항목 → 생성
     - 기존에 있고 요청에도 있는 id → 수정(유지)
     - 기존에 있지만 요청에 없는 id → 삭제 대상
   - 단, 초기 데이터(기본 리뷰 종류 6개)는 삭제 목록에서 제외
   - bulk_create / bulk_update / bulk_delete 실행
   - audit_log로 변경 이력 기록
4. {"isOk": True} 응답 반환
```

### 5.6 카테고리별 제한 적용 시 하위 카테고리 포함 조회 흐름

```
1. ShopPermitCategory__Features.group_by_category(shop_id) 호출
2. ShopPermitCategory 전체 조회 후 type별 그룹화
   예: {100: ["32", "52"], 200: ["45"]}
3. 각 그룹의 카테고리에 대해 Core__Feature.fetch_categories_sub() 호출
   - 하위(child) 카테고리 ID까지 포함하여 반환
   - 캐시 키: "category_sub_{shop_id}_{category_ids}"
   - 캐시 TTL: 5분
4. 최종 결과: {100: ["32", "52", "32-1", "52-1", ...], 200: ["45", "45-1", ...]}
```

### 5.7 인스타그램 정보 조회 흐름

```
1. 클라이언트 → GET /shop-instagram?selected_fields=last_refresh,refresh_status
2. ShopInstagram__Manager.get_or_create(shop_id)
   - 존재하지 않으면 기본값으로 자동 생성 (get_or_create 패턴)
3. DynamicSerializer로 요청된 필드만 직렬화하여 응답
```

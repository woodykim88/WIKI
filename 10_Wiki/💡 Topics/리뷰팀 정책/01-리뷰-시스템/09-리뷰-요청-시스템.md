# 리뷰 요청 시스템

## 1. 개요

리뷰 요청 시스템은 쇼핑몰이 주문 데이터를 기반으로 대량의 리뷰 작성을 유도하는 배치 처리 기능이다. 주문 건에 대한 리뷰 작성 요청을 일괄 발송하고, 발송 상태를 추적하며, 스태프 리뷰 대량 업로드 기능도 함께 제공한다.

---

## 2. 데이터 모델

### 2.1 ReviewRequest (리뷰 요청)

| 필드 | 타입 | 설명 |
|------|------|------|
| `shop_id` | Integer | 쇼핑몰 고유 ID |
| `name` | String | 요청 이름 |
| `status` | Choice | 요청 상태 (진행 / 완료) |
| `order_item_data_count` | Integer | 대상 주문상품 수 |
| `completed_at` | DateTime | 완료 일시 |

### 2.2 ReviewRequestOrder (리뷰 요청 주문)

| 필드 | 타입 | 설명 |
|------|------|------|
| `request` | FK(ReviewRequest) | 리뷰 요청 참조 |
| `order` | OneToOne(Order) | 주문 참조 (1:1 관계) |
| `order_platform_id` | String | 플랫폼 주문 ID |
| `customer_name` | String | 고객 이름 |
| `customer_cellphone` | String | 고객 전화번호 |
| `is_cancelled` | Boolean | 발송 취소 여부 |
| `is_failed` | Boolean | 발송 실패 여부 |
| `failure_reason` | String | 실패 사유 |

### 2.3 ReviewRequestOrderItemRelation (요청 주문-주문상품 관계)

| 필드 | 타입 | 설명 |
|------|------|------|
| `order` | FK(ReviewRequestOrder) | 리뷰 요청 주문 참조 |
| `order_item` | OneToOne(OrderItem) | 주문상품 참조 (1:1 관계) |
| `product_code` | String | 상품 코드 |
| `product_name` | String | 상품명 |
| `product_option` | String | 상품 옵션 |
| `product_price` | Integer | 상품 가격 |

## 3. 비즈니스 규칙

### 3.1 리뷰 요청 구조

```
ReviewRequest (리뷰 요청 배치)
├── ReviewRequestOrder (주문 1)
│   ├── ReviewRequestOrderItemRelation (주문상품 1-1)
│   ├── ReviewRequestOrderItemRelation (주문상품 1-2)
│   └── ...
├── ReviewRequestOrder (주문 2)
│   ├── ReviewRequestOrderItemRelation (주문상품 2-1)
│   └── ...
└── ...
```

- 각 리뷰 요청은 **여러 주문**을 포함한다.
- 각 주문은 **여러 주문상품**을 포함한다.
- 주문과 주문상품은 각각 **OneToOne** 관계로 원본 데이터와 연결된다.

### 3.2 발송 상태 추적

| 상태 | 필드 | 설명 |
|------|------|------|
| 정상 발송 | `is_cancelled=False`, `is_failed=False` | 리뷰 작성 요청이 정상 발송됨 |
| 발송 취소 | `is_cancelled=True` | 관리자가 발송을 취소함 |
| 발송 실패 | `is_failed=True` | 발송 중 오류 발생, `failure_reason`에 사유 기록 |

### 3.3 요청 완료
- 모든 주문에 대한 발송 처리가 끝나면 `status`가 완료로 변경된다.
- `completed_at`에 완료 시점이 기록된다.

## 4. 스태프 리뷰 대량 업로드

### 4.1 StaffReviewUploadStatus

| 필드 | 타입 | 설명 |
|------|------|------|
| `shop_id` | Integer | 쇼핑몰 고유 ID (**unique**) |
| `status` | Choice | 업로드 진행 상태 |

- 쇼핑몰당 **하나의 업로드 상태**만 존재한다 (`shop_id` unique).
- 스태프 리뷰 대량 업로드 작업의 진행 상태를 관리한다.
- 동시에 여러 업로드 작업이 실행되는 것을 방지한다.

---

## 5. 리뷰 데이터 다운로드

### 5.1 RawDataRequestLog (엑셀 다운로드 요청 로그)

| 필드 | 타입 | 설명 |
|------|------|------|
| `shop_id` | Integer | 쇼핑몰 고유 ID |
| `email` | String | 다운로드 요청자 이메일 |
| `start_date` | Date | 조회 시작일 |
| `end_date` | Date | 조회 종료일 |
| `data_fields` | JSON/String | 포함할 데이터 필드 목록 |
| `file_url` | URL | 생성된 파일 다운로드 URL |
| `file_password` | String | 파일 암호 |

- 리뷰 데이터를 엑셀 파일로 내보내는 요청을 추적한다.
- 생성된 파일은 보안을 위해 암호가 설정되며, `file_url`을 통해 다운로드할 수 있다.

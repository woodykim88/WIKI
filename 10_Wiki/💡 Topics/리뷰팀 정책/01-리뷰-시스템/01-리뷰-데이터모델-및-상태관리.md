> **모듈 경로**: `projectreview/review/models.py`
> **타입 정의**: `projectreview/common/types.py`

## 1. 도메인 개요

리뷰(Review)는 알파리뷰 서비스의 핵심 도메인으로, 쇼핑몰(Shop)의 상품(Product)에 대해 고객이 작성하는 평가 데이터를 관리한다.

### 1.1 핵심 개념

| 개념 | 설명 |
|------|------|
| **리뷰** | 상품 구매 후 고객이 남기는 평가. 텍스트, 별점, 사진, 동영상, 설문, 키워드 등 다양한 형태를 지원한다. |
| **게시 상태(posting_status)** | 리뷰가 위젯에 노출되는 상태를 제어한다. 게시예정 → 게시대기 → 게시됨/숨김 흐름을 따른다. |
| **혜택 상태(reward_status)** | 리뷰 작성에 대한 포인트 지급 상태를 관리한다. 지급예정 → 지급완료, 또는 지급취소/회수 등의 흐름을 따른다. |
| **작성자 유형(creater_type)** | 리뷰 작성자의 분류. 회원/비회원/NPAY는 실구매 리뷰, 관리자/시스템은 관리 목적 리뷰이다. |
| **리뷰 유형(review_type)** | 일반(텍스트+사진), 빠른(별점만), SNS(인스타그램), 소셜(소셜링크) 네 가지 유형이 있다. |
| **원본-편집본 관계** | 리뷰 수정 시 기존 리뷰를 원본(creater_type=100)으로 보관하고, 새 편집본을 생성하여 `review_origin` FK로 연결한다. |

### 1.2 도메인 간 관계

```
Shop (1) ─── (N) Review (N) ─── (1) Product
                   │
                   ├── (1) OrderItem
                   ├── (N) ReviewPhoto
                   ├── (N) ReviewVideo
                   ├── (N) ReviewMediaIndex
                   ├── (N) ReviewLog
                   ├── (N) ReviewComment
                   ├── (N) ReviewViews
                   ├── (N) ReviewReport
                   ├── (N) ReviewReward
                   ├── (1) ReviewAI
                   ├── (N) ReviewNegativePhrase
                   ├── (N) ReviewBadgeRelation ─── Badge
                   ├── (N) ReviewLabelRelation ─── Label
                   ├── (N) ReviewKeywordRelation ─── Keyword
                   ├── (N) ReviewChallengeRelation ─── ReviewChallenge
                   └── (1) ReviewSurveyFormRelation ─── SurveyForm
```

#### 인덱스 설계

복합 인덱스가 대량으로 정의되어 있으며, 주요 조회 패턴은 다음과 같다:

- **기본 조회**: `(shop_id)`, `(shop_id, product_fk)`, `(shop_id, product_id)`, `(shop_id, review_no)`, `(shop_id, review_no_new)`
- **시간순 조회**: `(shop_id, created_at)`, `(shop_id, is_alert, created_at)`, `(shop_id, ratings, created_at)`, `(shop_id, like_count, created_at)`
- **작성자 유형별 조회**: `(shop_id, creater_type, is_deleted, created_at)`
- **위젯 필터 조회**: `(shop_id, product_fk/product_id, is_visible, is_photo, is_video, is_survey, review_type, sort, created_at)` -- 정렬 기준(created_at/ratings/like_count)별로 각각 인덱스 존재

## 3. 비즈니스 규칙

### 3.1 Review.save() 로직

리뷰 저장 시 다음의 전처리 및 후처리 로직이 실행된다.

```python
# projectreview/review/models.py

def save(self, *args, **kwargs) -> None:
    # [전처리] ratings 값 범위 클램핑
    if self.ratings > MAX_RATINGS:     # MAX_RATINGS = 5
        self.ratings = MAX_RATINGS
    if self.ratings < 0:
        self.ratings = MIN_RATINGS     # MIN_RATINGS = 1

    super().save(*args, **kwargs)

    # [후처리 1] SNS 리뷰 → sns_like_count 갱신
    if self.review_type == Review__ReviewType.SNS:
        self.sns_like_count = self.like_count + self.comment_count

    # [후처리 2] Product.is_review_changed 플래그 설정
    if (self.product_id is not None) and self.product_fk:
        product_instance = self.product_fk
        product_instance.is_review_changed = True
        product_instance.save(update_fields=["is_review_changed"])

    # [후처리 3] OrderItem.quick_written_rating 갱신
    if (self.order_item_id is not None) and self.order_item_fk:
        order_item_instance = self.order_item_fk
        order_item_instance.quick_written_rating = self.ratings
        order_item_instance.save(update_fields=["quick_written_rating"])
```

**주의사항:**
- `ratings`가 0 이하이면 `MIN_RATINGS(1)`로, `MAX_RATINGS(5)` 초과이면 5로 보정된다.
- `sns_like_count` 갱신은 `super().save()` 이후에 이루어지지만, 변경된 `sns_like_count`가 다시 DB에 저장되지 않는 점에 주의한다 (인스턴스 레벨에서만 변경). 별도의 벌크 업데이트나 후속 저장이 필요할 수 있다.
- `Product.is_review_changed` 플래그는 상품의 리뷰 통계 캐시 갱신 트리거로 사용된다.

### 3.2 Review.delete() 로직

```python
# projectreview/review/models.py

def delete(self, *args, **kwargs):
    # Product.is_review_changed 플래그 설정
    if self.product_id and self.product_fk:
        product_instance = self.product_fk
        product_instance.is_review_changed = True
        product_instance.save(update_fields=["is_review_changed"])

    # 연관된 ReviewInstagramRelation cascade 삭제
    from external.managers import ReviewInstagramRelation__Manager
    instagram_relations = ReviewInstagramRelation__Manager.get_list(
        shop_id=self.shop_id, filters={"review_id": self.id}
    )
    instagram_relations.delete()

    return super().delete(*args, **kwargs)
```

**주의사항:**
- 코드 주석에 `is_deleted = True` 변경으로의 전환이 예정되어 있다 (소프트 삭제로의 마이그레이션).
- 현재는 실제 하드 삭제(`super().delete()`)가 수행된다.
- Instagram 연관 데이터는 Django CASCADE가 아닌 수동 삭제로 처리된다.

### 3.3 is_verified (실구매 리뷰 판별)

```python

        Review__CreaterType.회원,      # 1
```

### 3.4 reward_target_review (혜택 대상 리뷰)

```python
@property
def reward_target_review(self):
    if self.creater_type == Review__CreaterType.원본:
        return Review.objects.get(shop_id=self.shop_id, review_origin=self)
    return self
```

리뷰 수정 시 원본 리뷰가 보관되고 편집본이 생성된다. 혜택 관련 처리는 항상 편집본(최신 리뷰)에서 수행되므로, 원본 리뷰에서 호출하면 편집본을 반환한다.

## 5. Usecase 흐름

### 5.1 일반 리뷰 작성 흐름

```
1. 고객이 리뷰 작성 폼을 제출한다
2. Review 인스턴스를 생성한다
   - review_type = 일반(1)
   - creater_type = 회원(1) / 비회원(2) / NPAY(3)
   - posting_status = 게시대기(11) (기본값)
   - reward_status = 지급대기(11) (기본값)
3. Review.save() 가 호출된다
   - ratings 범위 클램핑 (1~5)
   - Product.is_review_changed = True 로 갱신
   - OrderItem.quick_written_rating 에 별점 기록
4. 사진이 있으면 ReviewPhoto를 생성한다
   - 원본 이미지 저장
   - 100x100 WEBP 썸네일 자동 생성
   - 900px 너비 WEBP 대형 이미지 자동 생성
5. 동영상이 있으면 ReviewVideo를 생성한다
   - 원본/고화질/중간화질/저화질 CompressVideoField에 저장
6. ReviewMediaIndex로 미디어 순서를 기록한다
7. 설문이 있으면 ReviewSurveyFormRelation / ReviewSurveyFormAnswer를 생성한다
8. ReviewLog에 상태 변경 이력을 기록한다
```

### 5.2 빠른 리뷰 (별점만) 작성 흐름

```
1. 고객이 별점만 제출한다
2. Review 인스턴스를 생성한다
   - review_type = 빠른(2)
   - content = "" (빈 텍스트)
   - is_photo = False, is_video = False
3. Review.save() 시 OrderItem.quick_written_rating 갱신
```

### 5.3 SNS 리뷰 (인스타그램) 등록 흐름

```
1. 고객이 인스타그램 게시물 URL을 제출한다
2. Review 인스턴스를 생성한다
   - review_type = SNS(3)
   - platform = "인스타그램"
   - sns_review_id, sns_user_id 설정
3. Review.save() 시 sns_like_count = like_count + comment_count 계산
4. SNS 유지기간 검증
   - 미달성: reward_status = SNS지급대기(13), sns_deauthed_at 기록
   - 달성: is_sns_completed = True, sns_completed_at 기록
5. ReviewInstagramRelation 생성 (외부 모듈)
```

### 5.4 리뷰 수정 흐름

```
1. 고객/관리자가 리뷰 수정을 요청한다
2. 기존 리뷰의 creater_type을 원본(100)으로 변경한다
3. 새 편집본 Review 인스턴스를 생성한다
   - review_origin = 원본 리뷰 (OneToOneField)
4. 원본 리뷰에서 reward_target_review를 호출하면 편집본을 반환한다
5. 혜택 관련 처리는 항상 편집본 기준으로 수행된다
```

### 5.5 리뷰 삭제 흐름

```
1. 관리자가 리뷰 삭제를 요청한다
2. Review.delete() 가 호출된다
   - Product.is_review_changed = True 설정
   - ReviewInstagramRelation 수동 삭제
   - super().delete() 로 하드 삭제 수행
3. CASCADE로 연관 모델 자동 삭제:
   - ReviewPhoto, ReviewVideo, ReviewMediaIndex
   - ReviewLog, ReviewComment, ReviewViews
   - ReviewReport, ReviewReward, ReviewAI
   - ReviewBadgeRelation, ReviewLabelRelation
   - ReviewKeywordRelation, ReviewChallengeRelation
   - ReviewSurveyFormRelation, ReviewNegativePhrase
```

### 5.6 혜택 지급 흐름

```
1. 리뷰 게시 후 혜택 지급 조건을 확인한다
2. 조건 충족 시:
   - ReviewReward 인스턴스 생성 (point, to_be_pay_date 설정)
   - Review.reward_status = 지급예정(10)
3. 지급 실행:
   - ReviewReward.payed_status = "지급완료"
   - ReviewReward.payed_date 기록
   - Review.reward_status = 지급완료(20)
   - Review.is_reward = True
   - Review.rewarded_at 기록
4. 비회원인 경우:
   - Review.reward_status = 비회원지급대기(12) 로 전환
   - 회원 전환 후 지급 진행
5. SNS 리뷰인 경우:
   - SNS 게시물 유지기간 미달 시 reward_status = SNS지급대기(13)
   - 유지기간 달성 시 지급 진행
```

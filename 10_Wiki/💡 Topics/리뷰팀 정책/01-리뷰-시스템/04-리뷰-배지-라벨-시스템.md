# 리뷰 배지/라벨 시스템

## 1. 개요

리뷰 배지(Badge)와 라벨(Label) 시스템은 리뷰에 시각적 마커를 부여하여 특정 리뷰를 강조하거나 분류하는 기능이다. 배지는 자동/수동 부여 방식을 지원하며, 라벨은 리뷰를 카테고리화하는 데 사용된다.

---

## 2. 데이터 모델

### 2.1 Badge (배지)

| 필드 | 타입 | 설명 |
|------|------|------|
| `shop_id` | Integer | 쇼핑몰 고유 ID |
| `badge_name` | String | 배지 이름 |
| `badge_description` | String | 배지 설명 |
| `design` | - | 디자인 설정 (border, background, color) |
| `badge_type` | Choice | 배지 유형 (`Badge__Type`: 자동 / 수동) |
| `is_active` | Boolean | 활성화 여부 |

### 2.2 BadgeAutoOption (자동 배지 부여 조건)

자동 배지가 부여되는 조건을 정의하는 모델이다. 하나의 배지에 여러 조건을 조합할 수 있다.

| 필드 | 타입 | 설명 |
|------|------|------|
| `is_sns` | Boolean | 인스타그램 리뷰 여부 |
| `is_naver_blog` | Boolean | 네이버블로그 리뷰 여부 |
| `is_fixed` | Boolean | 상단고정 리뷰 여부 |
| `is_repurchase` | Boolean | 재구매 회원 여부 |
| `manager_write` | Boolean | 스태프 리뷰 여부 |
| `like_count` | Integer | 도움수(좋아요) 기준값 |
| `is_review_keyword` | Boolean | 키워드 지정 여부 |
| `review_newly_count` | Integer | 최근 n일 이내 등록 기준 |
| `order_date_count` | Integer | 주문일 기준 일수 |
| `order_date_info` | Choice | 주문일 기준 방향 (`BadgeAutoOption__OrderDateInfo`: 이내 / 이후) |
| `is_category_product_label` | Boolean | 카테고리/상품/라벨 지정 여부 |
| `is_customer_group` | Boolean | 회원등급 지정 여부 |

### 2.3 BadgeAutoOption 하위 조건 모델

#### BadgeAutoOptionKeyword (키워드 조건)
- 자동 배지의 키워드 매칭 조건을 정의한다.
- `is_review_keyword`가 활성화된 경우에 사용된다.

#### BadgeAutoOptionCustomerGroup (회원 그룹 조건)
- 특정 회원등급에 속한 고객의 리뷰에만 배지를 부여하는 조건이다.
- `is_customer_group`이 활성화된 경우에 사용된다.

#### BadgeAutoOptionConnect (카테고리/상품/라벨 연결)
- `connect_type` 필드(`BadgeAutoOptionConnect__ConnectType`)로 연결 대상을 구분한다.
  - 카테고리 연결
  - 상품 연결
  - 라벨 연결
- `is_category_product_label`이 활성화된 경우에 사용된다.

### 2.4 ReviewBadgeRelation (리뷰-배지 매핑)

| 필드 | 타입 | 설명 |
|------|------|------|
| `review` | FK(Review) | 리뷰 참조 |
| `badge` | FK(Badge) | 배지 참조 |
| `is_immutable` | Boolean | 불변 조건 (True이면 자동 재계산 시에도 유지) |

---

### 2.5 Label (라벨)

| 필드 | 타입 | 설명 |
|------|------|------|
| `shop_id` | Integer | 쇼핑몰 고유 ID |
| `value` | String | 라벨 텍스트 |
| `label_type` | Choice | 라벨 유형 (`Label__LabelType`: preset / custom / 특수) |

### 2.6 ReviewLabelRelation (리뷰-라벨 매핑)

| 필드 | 타입 | 설명 |
|------|------|------|
| `review` | FK(Review) | 리뷰 참조 |
| `label` | FK(Label) | 라벨 참조 |

## 3. 비즈니스 규칙

### 3.1 배지 유형

| 유형 | 설명 | 동작 |
|------|------|------|
| **자동** | 조건 기반 자동 부여 | `BadgeAutoOption`에 정의된 조건이 충족되면 시스템이 자동으로 배지 부여 |
| **수동** | 관리자 수동 부여 | 관리자가 대시보드에서 직접 리뷰에 배지를 부여 |

### 3.2 자동 배지 조건 조합

자동 배지는 다양한 조건을 **AND 조합**으로 결합하여 사용할 수 있다.

```
예시: 키워드 AND 회원등급 AND 카테고리/상품/라벨
- is_review_keyword=True + BadgeAutoOptionKeyword 설정
- is_customer_group=True + BadgeAutoOptionCustomerGroup 설정
- is_category_product_label=True + BadgeAutoOptionConnect 설정
```

모든 활성화된 조건이 동시에 충족되어야 배지가 부여된다.

### 3.3 라벨 유형별 규칙

| 유형 | 설명 | 삭제 가능 여부 |
|------|------|---------------|
| **preset** | 시스템 기본 라벨 (초기화 시 자동 생성) | 삭제 불가 |
| **custom** | 쇼핑몰 관리자가 생성한 커스텀 라벨 | 삭제 가능 |
| **특수** | 시스템 내부 용도 (예: "베스트 SNS리뷰" 등) | 시스템 관리 |

### 3.4 라벨 초기화

`Label__Features.initial()` 메서드를 통해 쇼핑몰 최초 설정 시 preset 라벨이 자동으로 생성된다. `Label__Features.delete()` 메서드는 preset 라벨 삭제 시도를 차단한다.

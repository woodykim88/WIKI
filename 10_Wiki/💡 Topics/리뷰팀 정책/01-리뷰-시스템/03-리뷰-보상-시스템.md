# 리뷰 보상 시스템

## 1. 개요

리뷰 보상 시스템은 고객이 리뷰를 작성했을 때 포인트(적립금)를 자동으로 산정하고 지급하는 핵심 비즈니스 로직이다. 리뷰 유형(별점, 텍스트, 포토, 동영상, 설문)과 고객 유형(회원, 비회원, 네이버페이 등)에 따라 차등 보상을 지급하며, 선착순/장문/조기작성 등 다양한 추가 보상 조건을 지원한다.

쇼핑몰 관리자는 보상 정책(ShopPolicy), 검수 옵션(ShopConfirmOption), 보상 항목(ShopReward)을 통해 보상 규칙을 유연하게 설정할 수 있다.

## 2. 관련 모델

### 2.1 ReviewReward (보상 내역)

리뷰에 대해 산정된 개별 보상 항목을 저장하는 모델이다. 하나의 리뷰에 여러 개의 보상 항목(별점리뷰, 텍스트리뷰, 선착순 등)이 생성될 수 있다.

| 필드 | 타입 | 설명 |
|------|------|------|
| `shop_id` | IntegerField | 쇼핑몰 고유 ID |
| `review` | FK (Review) | 혜택 관련 리뷰 (reward_target_review) |
| `order_fk` | FK (Order) | 연관 주문 |
| `point` | PositiveIntegerField | 지급 포인트 (기본값: 0) |
| `payed_status` | CharField | 지급 상태 ("지급대기", "자동지급" 등) |
| `payed_reason` | CharField | 지급 상태 변경 사유 |
| `to_be_pay_date` | DateTimeField | 지급 예정일 (nullable) |
| `payed_date` | DateTimeField | 실제 지급일 (nullable) |
| `reason` | CharField | 지급 항목 ("별점리뷰", "텍스트리뷰", "선착순" 등) |
| `is_custom_payed` | BooleanField | 추가(수동) 지급 여부 |
| `is_completed` | BooleanField | 지급 완료 여부 |

> **참고**: `review` 필드는 원본 리뷰가 아닌 `reward_target_review` (혜택 관련 리뷰)를 참조한다. 원본(`creater_type == 원본`) 리뷰인 경우 `review_origin`으로 연결된 사본 리뷰를 반환하고, 그 외에는 자기 자신을 반환한다.

**소스 위치**: `projectreview/review/models.py` (503행)

## 4. 핵심 비즈니스 로직 (ReviewReward__Util)

`ReviewReward__Util` 클래스는 리뷰 보상 산정의 전체 프로세스를 담당한다. 리뷰 작성 시 4단계의 순차 처리를 통해 보상을 산정하고 저장한다.

**소스 위치**: `projectreview/review/utils/review_reward.py`

### 4.1 초기화

```python
ReviewReward__Util(review: Review, review_data: dict = {})
```

초기화 시 다음을 수행한다:
1. 쇼핑몰 정책(`ShopPolicy`)과 검수 옵션(`ShopConfirmOption`)을 로드한다
2. 리뷰의 현재 상태(`review_status`)를 계산한다:
   - `text`: 텍스트 글자 수 (`len(content_origin)`)
   - `photo`: 첨부 사진 수 (`review_photo_set` count)
   - `video`: 총 동영상 길이 (초 단위, `Review__Feature.get_video_duration()`)
   - `survey`: 설문 답변 완료 여부 (모든 answer 항목이 비어있지 않은지 확인)
   - `alert_keyword`: 알림 키워드 포함 여부 (`"keyword" in review.alert_reason`)
3. `reward_target_review`를 결정한다 (원본 리뷰 vs 사본 리뷰)

### 4.2 처리 흐름

```
ReviewReward__Util 초기화
    |
    v
[1단계] check_review_minimum()  -- 리뷰 최소 조건 확인
    |                               (이미 보상 지급된 경우 중단)
    v
[2단계] check_reward_limit()    -- 지급 제한 조건 확인
    |                               (제한 해당 시 중단)
    v
[3단계] check_reward_waiting()  -- 지급 대기/예정 판단
    |                               (update_data 갱신)
    v
[4단계] generate_reward_list()  -- 보상 리스트 생성 및 저장
```

---

### 4.3 1단계: check_review_minimum() - 리뷰 최소 조건 확인

리뷰가 각 유형별 최소 조건을 충족하는지 확인하고, `review_status_with_min_condition` 플래그를 설정한다.

**반환값**: `bool` (프로세스 중단 여부)

#### 최소 조건 판정 로직

| 리뷰 유형 | 조건 | 비고 |
|-----------|------|------|
| 텍스트리뷰 | `text >= min_text` (use_min_text=True일 때) 또는 `text >= 1` (False일 때) | `min_text`는 ShopPolicy 설정값 (기본값: 10) |
| 포토리뷰 | `photo > 0` AND `photo >= min_photo` | `min_photo`는 ShopPolicy 설정값 (기본값: 1) |
| 동영상리뷰 | `video > 0` AND `video >= min_video` | `min_video`는 ShopPolicy 설정값 (기본값: 5초) |
| 설문리뷰 | 모든 설문 answer 항목이 비어있지 않아야 함 | survey 데이터의 answer 배열 검사 |

#### 중단 조건

- `reward_target_review.is_reward == True` (이미 보상이 지급된 리뷰)인 경우 → `stopped = True`를 반환하여 이후 프로세스를 중단한다

#### 로그 기록

- 리뷰 유형 목록을 슬래시(`/`)로 구분하여 기록한다 (예: "리뷰작성 : 별점리뷰/텍스트/포토")
- 이미 보상 지급된 경우 "혜택지급 완료", 그 외 "혜택지급 대기"로 기록한다

### 4.4 2단계: check_reward_limit() - 지급 제한 확인

주문 상품 및 상품 상태에 따른 보상 지급 제한 여부를 확인한다. 제한 조건에 해당하면 `reward_status`를 `지급제한`으로 설정하고 `total_point`를 0으로 만든다.

**반환값**: `bool` (프로세스 중단 여부)

#### 지급 제한 조건

다음 조건에 하나라도 해당하면 지급이 제한된다. 각 조건은 독립적으로 평가되며 누적 적용된다:

| 순서 | 조건 | 제한 사유 (reward_status_error) |
|------|------|------|
| 1 | `order_item.permit_reward_reason`이 존재 | "지급제한:{사유}" |
| 2 | 상품 품절(`sold_out`) + `use_reward_sold_out=False` | "지급제한:상품 품절" |
| 3 | 상품 미진열(`display_out`) + `use_reward_display_out=False` | "지급제한:상품 미진열" |
| 4 | 상품 미판매(`selling_out`) + `use_reward_selling_out=False` | "지급제한:상품 미판매" |
| 5 | 미검증 리뷰 + `use_reward_unverified=False` | "지급제한:미검증리뷰" |

#### 부가 동작

- 지급 제한 시 `is_alert=True`, `alert_reason`에 `"reward_disabled"` 추가 (permit_reward_reason 제외)
- 기존 보상 내역을 초기화한다: "선착순", "회원최초", "추가혜택" reason을 **제외**한 기존 ReviewReward 레코드를 일괄 삭제한다
- 지급 제한인 경우 리뷰를 즉시 업데이트하고 `stopped = True`를 반환한다

---

### 4.5 3단계: check_reward_waiting() - 지급 대기/예정 판단

`ShopConfirmOption.reward_option` 값에 따라 보상 지급 시점과 방식을 결정한다.

**반환값**: 없음 (`update_data`를 내부적으로 갱신)

#### 기본 지급 시점 결정

| reward_option 값 | reward_status | rewarded_at | payed_status | 설명 |
|------------------|---------------|-------------|--------------|------|
| `"admin"` | 지급대기 | None | "지급대기" | 관리자가 수동 검수 후 지급 |
| `"immediate"` | 지급예정 | 현재시각 + 2시간 | "자동지급" | 2시간 후 자동 지급 |
| 숫자 (예: `"7"`) | 지급예정 | 현재시각 + n일 | "자동지급" | n일 후 자동 지급 |

#### 추가 대기 조건

기본 지급 시점이 결정된 후, 다음 조건에 해당하면 `지급대기`로 변경된다 (관리자 검수 필요):

| 조건 | 검사 대상 | reward_status_info |
|------|-----------|-------------------|
| 미검증 리뷰 | `order_item.order_status_sub == "unverified"` | "지급대기:미검증 리뷰" |
| 낮은 별점 | `review.ratings <= confirm_option.reward_option_ratings` | "지급대기:낮은 별점" |
| 포토리뷰 포함 | `review_status["photo"]` + `reward_option_photo=True` | "지급대기:포토리뷰" |
| 동영상리뷰 포함 | `review_status["video"]` + `reward_option_video=True` | "지급대기:동영상리뷰" |
| 키워드 알림 | `review_status["alert_keyword"]` + `reward_option_keyword=True` | "지급대기:키워드 알림" |

> **참고**: 추가 대기 조건에 해당하면 `rewarded_at`이 `None`으로 재설정되고 `payed_status`가 "지급대기", `payment_type`이 "검수완료시 지급"으로 변경된다.

### 4.6 4단계: generate_reward_list() - 보상 리스트 생성

실제 보상 금액을 계산하고, 보상 내역을 저장하는 핵심 로직이다.

#### 4.6.1 기본 보상 계산

기본 보상은 리뷰 유형별로 순서대로 계산된다. 각 항목은 `ShopReward__Util.get_reward(title, customer_type)`을 통해 해당 쇼핑몰의 보상 설정에서 금액을 조회한다.

| 순서 | 보상 항목 | 조건 | 비고 |
|------|-----------|------|------|
| 1 | 별점리뷰 | 항상 지급 | 모든 리뷰의 기본 보상 |
| 2 | 텍스트리뷰 | `review_status_with_min_condition["text"] == True` | 최소 글자 수 충족 시 |
| 3 | 포토리뷰 | `review_status_with_min_condition["photo"] == True` | 최소 사진 수 충족 시 |
| 4 | 동영상리뷰 | `review_status_with_min_condition["video"] == True` | 최소 영상 길이 충족 시 |
| 5 | 설문리뷰 | `review_status_with_min_condition["survey"] == True` | 설문 답변 완료 시 |

#### 4.6.2 정률제/정액제 보상 계산

`ShopReward__Util.get_reward()`에서 보상 금액을 계산할 때, 설정값에 `%`가 포함되어 있으면 정률제로 계산한다:

```
정률제: calc_reward = product_price * ratio / 100
        최종값 = max(calc_reward, min_point)   ← use_min_point가 True일 때만 min_point 적용
정액제: calc_reward = int(설정값)
```

#### 4.6.3 추가 보상 계산

추가 보상은 기본 보상 조건이 충족된 경우에만 계산된다. 각 추가 보상 항목에는 `value`(조건 값)와 `reward`(보상 금액)가 설정되어 있으며, 여러 단계가 있을 수 있다.

##### 텍스트리뷰 충족 시 추가 보상

| 추가 보상 | 조건 | 선택 방식 |
|-----------|------|-----------|
| 장문리뷰 | `text >= value` (n자 이상) | value 내림차순 정렬 후 첫 번째 충족 항목 (가장 높은 단계) |
| 조기작성 | `(오늘 - 배송완료일).days < value` (n일 이내) | value 내림차순 정렬 후 첫 번째 충족 항목, `delivered_date`가 있어야 함 |

##### 포토리뷰 충족 시 추가 보상

| 추가 보상 | 조건 | 선택 방식 |
|-----------|------|-----------|
| 회원최초_포토리뷰 | 해당 고객의 첫 포토리뷰 + `customer.review_photo_total == 0` | 조건 충족 시 지급 |
| 선착순_포토리뷰 | 해당 상품의 포토리뷰 수 < value (n번째 이내) | value 오름차순 정렬 후 첫 번째 충족 항목 |
| 많은이미지 | `photo >= value` (n장 이상) | value 내림차순 정렬 후 첫 번째 충족 항목 |

> **회원최초 포토리뷰 특수 처리**: 수정된 리뷰에서 이미 대기 중인 회원최초 리워드가 있으면 삭제 후 다시 추가한다.
>
> **선착순 포토리뷰 특수 처리**: 수정된 리뷰에서 이미 대기 중인 선착순 리워드(같은 point)가 있으면 삭제 후 다시 추가한다.

##### 동영상리뷰 충족 시 추가 보상

| 추가 보상 | 조건 | 선택 방식 |
|-----------|------|-----------|
| 선착순_동영상리뷰 | 해당 상품의 동영상리뷰 수 < value (n번째 이내) | value 오름차순 정렬 후 첫 번째 충족 항목 |
| 영상길이 | `video >= value` (n초 이상) | value 내림차순 정렬 후 첫 번째 충족 항목 |

> **주의**: 동영상 선착순의 경우, 수정된 리뷰에 대한 기존 리워드 확인 로직이 포토리뷰와 달리 누락되어 있다. (코드 내 TODO 주석 참조)

### 4.7 보상 한도 규칙

보상 리스트가 생성된 후, 다음 한도 규칙이 순서대로 적용된다.

#### 4.7.1 추가 적립 1회 제한 (`use_general_extra_point_limit`)

- **조건**: `ShopPolicy.use_general_extra_point_limit == True`
- **동작**: 여러 추가 보상 중 **가장 높은 금액의 보상 1개만** 지급한다
- **로직**: `reward_extra_list`에서 `reward` 값이 최대인 항목만 남기고 나머지를 제거한다

#### 4.7.2 최대 포인트 한도 (`max_point`)

`OrderItem.max_point` 값이 0보다 크고 기본 보상 합계(`total_reward`)보다 작을 때 적용된다.

| `use_max_total_point_range` 값 | 한도 적용 방식 | 최종 보상 계산 |
|------|------|------|
| `True` (추가 포인트 제외) | 기본 보상에만 한도 적용 | `max_point + total_reward_extra` |
| `False` (추가 포인트 포함) | 전체에 한도 적용 | `max_point` |

#### 4.7.3 주문당 보상 횟수 제한 (`use_number_reward`)

- **조건**: `ShopPolicy.use_number_reward == True` AND `calc_reward > 0`
- **동작**: 동일 주문 내 보상 대상 리뷰 수가 `number_reward` 횟수를 초과하면 제한을 적용한다

**제한 알고리즘** (`_check_number_reward()`):
1. 같은 주문의 리뷰 목록을 조회한다 (SNS/소셜 리뷰 및 원본 리뷰는 제외)
2. `-is_reward`, `-total_point` 순으로 정렬하여 이미 지급된 리뷰와 높은 포인트 리뷰가 우선한다
3. 현재 리뷰를 제외한 목록을 순회하며:
   - 이미 지급 완료(`is_reward=True`)된 리뷰는 무조건 카운트
   - `total_point >= calc_reward`인 리뷰도 무조건 카운트
   - 그 외 리뷰는 제한 횟수 미만이면 카운트, 초과하면 **기존 리뷰의 보상을 취소**한다
4. 현재 리뷰도 제한에 걸리면:
   - `use_number_reward_quick == True`: 별점리뷰 보상만 지급
   - `use_number_reward_quick == False`: 보상 완전 차단 (`reward_status = 지급제한`)

### 4.9 포인트 지급 사유 문자열 생성 규칙 (`_build_reward_reason`)

외부 포인트 지급 API 호출 시 `reason` 파라미터로 전달되는 문자열 생성 규칙이다.

#### 제약 조건

- **최대 60 bytes** (API 제한)
- UTF-8 기준. 한글 1자 = 3 bytes, ASCII 1자 = 1 byte, 공백 = 1 byte

#### 포맷 규칙

```
{prefix}[:{상품명}[(옵션명)]][ 외 N개]
```

| 파트 | 내용 | bytes |
|------|------|-------|
| prefix | `"리뷰혜택지급"` | 18 bytes (고정) |
| `":"` | 상품명 있을 때만 추가 | 1 byte |
| 상품명 | 최대 3글자, 초과 시 `".."` 추가 | 최대 11 bytes |
| `"()"` | 옵션명 있을 때만 추가 | 2 bytes |
| 옵션명 | 최대 3글자, 초과 시 `".."` 추가 | 최대 11 bytes |
| `" 외 N개"` | 리뷰 2개 이상일 때 추가. N = review_count - 1 | 9~11 bytes |

> `".."` 는 2 bytes (ASCII 2자)

#### 출력 예시

| 입력 | 출력 |
|------|------|
| 상품명 없음 | `"리뷰혜택지급"` |
| 상품명 `"애플워치"` | `"리뷰혜택지급:애플워.."` |
| 상품명 `"애플워치 울트라 2세대"` | `"리뷰혜택지급:애플워.."` |
| 상품명 + 옵션명 `"블랙"` | `"리뷰혜택지급:애플워..(블랙)"` |
| 상품명 + review_count=3 | `"리뷰혜택지급:애플워.. 외 2개"` |
| 상품명 + 옵션명 + review_count=3 | `"리뷰혜택지급:애플워..(블랙) 외 2개"` (47 bytes) |
| 상품명 + 옵션명 + review_count=3 | `"리뷰혜택지급:애플워..(스페이..) 외 2개"` (55 bytes) |

**소스 위치**: `projectreview/review/management/commands/every_1hour_review_tracking.py` (`_build_reward_reason`)

## 7. 보상 계산 예시

### 예시 1: 일반 회원이 텍스트+포토 리뷰 작성

**조건**:
- 고객 유형: 회원(member)
- 텍스트 50자, 사진 3장
- ShopPolicy: min_text=10, min_photo=1
- 장문리뷰 조건: 30자 이상 → 200원
- 많은이미지 조건: 3장 이상 → 300원
- 상품 max_point: -1 (제한 없음)

**계산 과정**:
1. 기본 보상: 별점리뷰(300) + 텍스트리뷰(500) + 포토리뷰(1,000) = 1,800원
2. 추가 보상: 장문리뷰(200) + 많은이미지(300) = 500원
3. 최종 보상: 1,800 + 500 = **2,300원**

### 예시 2: 추가 적립 1회 제한 적용

**조건**: 위와 동일하나 `use_general_extra_point_limit=True`

**계산 과정**:
1. 기본 보상: 1,800원
2. 추가 보상 후보: 장문리뷰(200), 많은이미지(300)
3. 1회 제한 적용: 최대값 = 많은이미지(300)만 지급
4. 최종 보상: 1,800 + 300 = **2,100원**

### 예시 3: 최대 포인트 한도 (추가 포인트 제외)

**조건**: 기본 보상 1,800원, 추가 보상 500원, max_point=1,000, `use_max_total_point_range=True`

**계산 과정**:
1. 기본 보상(1,800) > max_point(1,000) → 기본 보상을 1,000으로 제한
2. 추가 보상(500)은 한도에서 제외
3. 최종 보상: 1,000 + 500 = **1,500원**

### 예시 4: 최대 포인트 한도 (추가 포인트 포함)

**조건**: 동일하나 `use_max_total_point_range=False`

**계산 과정**:
1. 기본 보상(1,800) > max_point(1,000) → 전체를 1,000으로 제한
2. 최종 보상: **1,000원**

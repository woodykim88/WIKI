# 리뷰 AI 분석 시스템

## 1. 개요

AI 분석 시스템은 리뷰 텍스트에 대한 감성 분석, 부정 문장 추출, VOC 태그 분류 등을 수행하는 기능이다. OpenAI, Google Gemini, AWS Bedrock(Claude) 등 다양한 AI 엔진을 활용하며, 배치 분석과 실시간 분석을 모두 지원한다.

---

## 2. 데이터 모델

### 2.1 ReviewAI (리뷰 AI 분석 결과)

| 필드 | 타입 | 설명 |
|------|------|------|
| `review` | OneToOne(Review) | 리뷰 참조 (1:1 관계) |
| `sentiment` | String | 감성 분석 결과 (긍정 / 부정 / 중립) |
| `positive` | Float | 긍정 점수 |
| `negative` | Float | 부정 점수 |
| `neutrality` | Float | 중립 점수 |
| `thoroughness` | Float | 성실도 점수 |
| `review_category` | String | 리뷰 카테고리 분류 결과 |

### 2.2 ReviewNegativePhrase (부정 문장)

| 필드 | 타입 | 설명 |
|------|------|------|
| `review` | FK(Review) | 리뷰 참조 |
| `phrase` | String | 부정으로 판별된 문장 |

### 2.3 ReviewAnalysisConfig (분석 설정)

| 필드 | 타입 | 설명 |
|------|------|------|
| `shop_id` | Integer | 쇼핑몰 고유 ID |
| `analysis_type` | Choice | 분석 유형 (부정리뷰=10, 태그분석=20) |
| `is_active` | Boolean | 활성화 여부 |
| `enabled_at` | DateTime | 활성화 일시 |
| `disabled_at` | DateTime | 비활성화 일시 |

## 3. 태그 시스템

### 3.1 ReviewSharedTag (공유 태그)

| 필드 | 타입 | 설명 |
|------|------|------|
| `tag_name` | String | 태그 이름 |
| `tag_description` | String | 태그 설명 |
| `category` | FK(AlphaCategory) | 알파 카테고리 참조 |

시스템 전역에서 공유되는 태그로, 카테고리별로 분류된다.

### 3.2 ReviewCommonTag (공통 태그, deprecated)

`ReviewSharedTag`로 대체되었다. 하위 호환을 위해 유지된다.

| 코드 | 이름 | 설명 |
|------|------|------|
| `BIZ_PRODUCT` | 상품특징 | 상품 관련 태그 |
| `BIZ_DELIVERY` | 배송/공정 | 배송 및 공정 관련 태그 |
| `BIZ_OPERATION` | 쇼핑몰운영 | 쇼핑몰 운영 관련 태그 |
| `SYS_*` | 시스템 관리용 | 시스템 내부 관리 태그 |

### 3.3 ReviewTag (쇼핑몰별 태그)

| 필드 | 타입 | 설명 |
|------|------|------|
| `shop_id` | Integer | 쇼핑몰 고유 ID |
| `tag_name` | String | 태그 이름 (최대 20자, **수정 불가**) |
| `tag_description` | String | AI 프롬프트용 설명 (최대 100자) |
| `common_tag` | FK(ReviewCommonTag) | 공통 태그 참조 (deprecated) |
| `shared_tag` | FK(ReviewSharedTag) | 공유 태그 참조 |
| `status` | Choice | 상태 (비활성=100, 활성=200, 삭제진행중=300) |
| `is_negative_only` | Boolean | 부정 리뷰 전용 태그 여부 |

### 3.4 ReviewPhrase (리뷰 문장)

| 필드 | 타입 | 설명 |
|------|------|------|
| `shop_id` | Integer | 쇼핑몰 고유 ID |
| `review` | FK(Review) | 리뷰 참조 |
| `phrase` | String | 추출된 문장 |
| `phrase_start_idx` | Integer | 원문에서의 시작 위치 |
| `is_negative` | Boolean | 부정 문장 여부 |

### 3.5 ReviewPhraseTagRelation (문장-태그 매핑)

| 필드 | 타입 | 설명 |
|------|------|------|
| `shop_id` | Integer | 쇼핑몰 고유 ID (파티셔닝 키) |
| `review` | FK(Review) | 리뷰 참조 |
| `phrase` | FK(ReviewPhrase) | 문장 참조 |
| `tag` | FK(ReviewTag) | 태그 참조 |

`shop_id`를 파티셔닝 키로 사용하여 대용량 데이터에 대한 쿼리 성능을 최적화한다.

### 3.6 ReviewTagRelation (태그-리뷰 집계)

| 필드 | 타입 | 설명 |
|------|------|------|
| `shop_id` | Integer | 쇼핑몰 고유 ID |
| `tag` | FK(ReviewTag) | 태그 참조 |
| `review` | FK(Review) | 리뷰 참조 |
| `positive` | Integer | 긍정 문장 수 |
| `negative` | Integer | 부정 문장 수 |

### 3.7 ReviewAnalysisLog (분석 로그)

분석 작업의 실행 이력을 기록하는 모델이다.

### 3.8 BatchAnalysisRequest (배치 분석 요청)

| 필드 | 타입 | 설명 |
|------|------|------|
| `shop_id` | Integer | 쇼핑몰 고유 ID |
| `target_review_count` | Integer | 분석 대상 리뷰 수 |
| `finish_count` | Integer | 완료된 리뷰 수 |
| `status` | Choice | 상태 (PROCESSING / COMPLETED) |
| `retry_count` | Integer | 재시도 횟수 |

## 4. AI 검수 흐름

`ReviewAI__Feature.process_AI()` 메서드는 리뷰 등록 시 AI 기반 자동 검수를 수행한다.

### 처리 순서

```
1. 중복 이미지 검수 조건 확인
   - posting_option_duplicatedMedia (게시 옵션) OR
   - reward_option_duplicatedMedia (보상 옵션)

2. 조건 충족 시 SQS로 AI 작업 전송

3. AI 작업 중 리뷰 임시 숨김 처리:
   - is_deleted = True
   - is_visible = False
   - posting_status = 게시대기
   - reward_status = 지급대기

4. AI 작업 완료 후 원래 상태 복원
```

AI 작업 중 리뷰를 숨기는 이유는 중복 이미지 등 부적절한 콘텐츠가 위젯에 노출되는 것을 방지하기 위해서이다.

## 5. 태그 비즈니스 규칙

### 5.1 커스텀 태그 제한

| 제한 항목 | 수량 |
|-----------|------|
| 커스텀 태그 생성 최대 수 | 100개 |
| 동시 활성화 최대 수 | 20개 |

### 5.2 태그 이름 규칙
- `shop_id` + `tag_name` 조합에 대해 **unique constraint**가 적용된다.
- 태그 이름은 생성 후 **수정 불가**하다 (최대 20자).

### 5.3 공통 태그 규칙
- `common_tag` 또는 `shared_tag`를 참조하는 태그는 상태 변경 및 삭제가 불가하다.
- 시스템에서 관리하는 표준 태그이므로 쇼핑몰 관리자가 임의로 변경할 수 없다.

### 5.4 삭제 규칙
- 삭제진행중(DELETING, status=300) 상태의 태그는 상태 변경이 불가하다.
- 태그 삭제 시 연관된 `ReviewPhraseTagRelation`, `ReviewTagRelation` 데이터도 정리된다.

### 5.5 tag_description 역할
- `tag_description` 필드는 AI에게 전달되는 **프롬프트** 역할을 한다.
- AI가 리뷰 문장을 분석할 때 이 설명을 기반으로 해당 태그에 문장을 매핑한다 (최대 100자).

## 6. 배치 분석 (BatchAnalysis__Feature)

### 6.1 분석 대상 조건
- **1년 이내** 작성된 리뷰만 대상
- 한 번에 최대 **1,000개** 리뷰 처리

### 6.2 처리 흐름

```
1. Redis/S3에 분석 대상(target) 저장
2. N개 리뷰에 대한 분석 이벤트 발행
3. AI 서비스에서 분석 수행
4. S3에 완료 파일 저장
5. 완료 파일 카운트로 진행률 계산 (finish_count / target_review_count)
6. 모든 분석 완료 시 status를 COMPLETED로 변경
```

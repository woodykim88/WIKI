# 리뷰 정책 규칙

## 1. 도메인 개요

### 1.1 비즈니스 문제

쇼핑몰 운영자가 리뷰 시스템을 자신의 비즈니스에 맞게 세밀하게 제어할 수 있어야 한다. 이 도메인은 다음 문제를 해결한다.

- **리뷰 게시 통제**: 어떤 리뷰를 즉시 게시하고, 어떤 리뷰를 관리자 검수 후 게시할지 결정
- **혜택(포인트) 지급 통제**: 리뷰 작성자에게 언제, 얼마의 포인트를 지급할지, 어떤 조건에서 제한할지 결정
- **리뷰 유형별 허용/제한**: 쇼핑몰에서 허용하는 리뷰 유형(일반, 빠른, SNS, 설문, 동영상, 키워드, 미검증) 설정
- **상품/카테고리별 세분화된 제한**: 특정 상품이나 카테고리에 대해 리뷰 작성이나 포인트 지급을 개별적으로 제한
- **자동 답글/댓글 관리**: 리뷰 만족도에 따른 자동 답글 템플릿 관리

### 1.2 핵심 개념 및 용어 정의

| 용어 | 정의 |
|------|------|
| **ShopConfirmOption** | 리뷰 게시 및 혜택 지급의 검수 옵션. 즉시/수동 게시 여부와 조건을 결정 |
| **ShopPermit** | 쇼핑몰 단위의 리뷰 유형별 허용/제한 설정 |
| **ShopPermitProduct** | 특정 상품에 대한 리뷰 제한 설정 (ShopPermit의 하위) |
| **ShopPermitCategory** | 특정 카테고리에 대한 리뷰 제한 설정 (ShopPermit의 하위) |
| **ShopPolicy** | 리뷰 작성 조건, 포인트 한도, 지급 제한 등 상세 정책 |
| **ShopReward** | 리뷰 유형별/작성자 유형별 포인트 지급 금액 설정 |
| **ShopCommentReview** | 리뷰에 대한 자동/수동 답글 템플릿 |
| **ShopQuickReview** | 빠른 리뷰(별점 리뷰) 프리셋/커스텀 텍스트 |
| **ShopKeywordReview** | 키워드 리뷰 위젯 설정 |
| **게시 시점(posting_option)** | 리뷰가 위젯에 노출되는 시점 (즉시 게시 vs 수동 게시) |
| **혜택 지급 시점(reward_option)** | 포인트가 지급되는 시점 (즉시 지급 vs 관리자 승인 후 지급) |
| **미검증 리뷰** | 구매 확인 없이 작성된 리뷰 |
| **정액제(P)** | 고정 금액으로 포인트 지급 (예: 1000P) |
| **정률제(%)** | 상품 가격 대비 비율로 포인트 지급 (예: 10%) |

## 2. 데이터 모델

### 2.1 ShopConfirmOption (검수 옵션)

리뷰 게시 및 혜택 지급의 자동/수동 처리 기준을 정의한다. 쇼핑몰당 1개 존재한다 (`shop_id` UNIQUE).

| 필드 | 타입 | 기본값 | 비즈니스 의미 |
|------|------|--------|---------------|
| `shop_id` | int (unique) | - | 쇼핑몰 고유 ID |
| `posting_option` | varchar(31) | `"immediate"` | 리뷰 게시 시점. `"immediate"` = 즉시 게시, 그 외 = 수동 게시 |
| `posting_option_ratings` | int | `0` | 수동 게시로 전환되는 별점 기준 (이하일 때 수동 게시) |
| `posting_option_keyword` | bool | `False` | 알림 키워드 포함 시 수동 게시 여부 |
| `posting_option_photo` | bool | `False` | 이미지 포함 리뷰를 수동 게시로 전환할지 여부 |
| `posting_option_video` | bool | `False` | 동영상 포함 리뷰를 수동 게시로 전환할지 여부 |
| `posting_option_duplicatedMedia` | bool | `False` | 중복 이미지 포함 시 수동 게시 여부 |
| `posting_option_sentimentNegative` | bool | `False` | 부정적 리뷰를 수동 게시로 전환할지 여부 |
| `reward_option` | varchar(31) | `"admin"` | 혜택 지급 시점. `"admin"` = 관리자 승인 후 지급 |
| `reward_option_ratings` | int | `0` | 수동 지급으로 전환되는 별점 기준 |
| `reward_option_keyword` | bool | `False` | 알림 키워드 포함 시 혜택 수동 지급 여부 |
| `reward_option_photo` | bool | `False` | 이미지 포함 시 혜택 수동 지급 여부 |
| `reward_option_video` | bool | `False` | 동영상 포함 시 혜택 수동 지급 여부 |
| `reward_option_duplicatedMedia` | bool | `False` | 중복 이미지 포함 시 혜택 수동 지급 여부 |
| `reward_option_sentimentNegative` | bool | `False` | 부정적 리뷰의 혜택 수동 지급 여부 |

> **참고**: DynamicSerializer 응답에서 `posting_option_sentimentNegative`와 `reward_option_sentimentNegative` 필드는 제외되어 있다. API 레벨에서는 노출되지 않으나 모델에는 존재한다.

### 2.2 ShopPermit (리뷰 유형별 허용/제한)

쇼핑몰에서 허용하는 리뷰 유형을 관리한다. 쇼핑몰당 1개 존재한다 (`shop_id` UNIQUE).

| 필드 | 타입 | 기본값 | 비즈니스 의미 |
|------|------|--------|---------------|
| `shop_id` | int (unique) | - | 쇼핑몰 고유 ID |
| `permit_normal` | bool | `True` | 일반 리뷰 허용 여부 |
| `permit_quick` | bool | `True` | 빠른 리뷰(별점 리뷰) 허용 여부 |
| `permit_survey` | bool | `False` | 설문 리뷰 허용 여부 |
| `permit_video` | bool | `True` | 동영상 리뷰 허용 여부 |
| `permit_sns` | bool | `False` | SNS 리뷰 허용 여부 |
| `permit_keyword` | bool | `False` | 키워드 리뷰 허용 여부 |
| `permit_unverified` | bool | `False` | 미검증 리뷰 허용 여부 |
| `updated_at` | datetime | auto | 수정 날짜 |

### 2.3 ShopPermitProduct (상품별 리뷰 제한)

특정 상품에 대해 세분화된 리뷰 제한을 적용한다.

| 필드 | 타입 | 비즈니스 의미 |
|------|------|---------------|
| `shop_permit` | FK -> ShopPermit | 상위 리뷰 제한 설정 (CASCADE 삭제) |
| `product_id` | int | 대상 상품 고유 ID |
| `type` | int (ShopPermitRelation__Type) | 제한 유형 코드 |

### 2.4 ShopPermitCategory (카테고리별 리뷰 제한)

특정 카테고리에 대해 세분화된 리뷰 제한을 적용한다.

| 필드 | 타입 | 비즈니스 의미 |
|------|------|---------------|
| `shop_permit` | FK -> ShopPermit | 상위 리뷰 제한 설정 (CASCADE 삭제) |
| `category_id` | int (nullable) | 카테고리 고유 번호 (코어의 `category_no`). **deprecated 예정** |
| `category_platform_id` | varchar(100) (nullable) | 플랫폼별 카테고리 고유 번호 (코어의 `ProductCategory.platform_id`) |
| `type` | int (ShopPermitRelation__Type) | 제한 유형 코드 |

> **참고**: `category_id`에서 `category_platform_id`로의 마이그레이션이 진행 중이다. 시리얼라이저에서 두 필드 간 자동 변환 로직이 존재한다. `category_platform_id`가 없으면 `category_id`를 문자열로 변환하여 사용하고, 반대의 경우도 마찬가지이다.

| 필드 | 타입 | 비즈니스 의미 |

| `reward` | varchar(128) | 지급 포인트. 정액제이면 숫자만, 정률제이면 `%` 접미사 포함 |

### 2.7 모델 간 관계도

```
ShopConfirmOption (1:1 shop_id)
    - 게시 시점, 혜택 지급 시점의 자동/수동 처리 기준

ShopPermit (1:1 shop_id)
    |
    +-- ShopPermitProduct (1:N, FK -> ShopPermit, CASCADE)
    |       - 상품별 리뷰 제한 (type별 분류)
    |
    +-- ShopPermitCategory (1:N, FK -> ShopPermit, CASCADE)
            - 카테고리별 리뷰 제한 (type별 분류)

ShopPolicy (1:1 shop_id)
    - 작성 조건, 포인트 한도, 지급 제한 등 상세 정책

ShopReward (1:N shop_id)
    - 리뷰 유형별/작성자별 포인트 금액

ShopCommentReview (1:N shop_id)
    - 리뷰 답글 템플릿 (자동/수동)

ShopQuickReview (1:N shop_id)
    - 빠른 리뷰 텍스트 프리셋

ShopKeywordReview (1:1 shop_id)
    - 키워드 리뷰 위젯 설정

AlertKeyword (1:N shop_id)
    - 키워드 알림 대상 단어

AlternativeKeyword (1:N shop_id)
    - 키워드 필터 대체어 (origin -> alternative)
```

## 3. 비즈니스 규칙

### 3.1 게시 정책 (ShopConfirmOption)

#### 3.1.1 게시 시점 규칙

`posting_option` 필드에 의해 리뷰의 게시 방식이 결정된다.

- **`"immediate"` (즉시 게시, 기본값)**: 리뷰 작성 즉시 위젯에 노출
- **그 외 값**: 관리자가 수동으로 승인해야 게시됨

#### 3.1.2 수동 게시 전환 조건

`posting_option`이 수동 게시인 경우, 다음 조건에 해당하는 리뷰는 수동 게시 대기 상태로 전환된다.

| 조건 | 필드 | 기본값 | 설명 |
|------|------|--------|------|
| 낮은 별점 | `posting_option_ratings` | `0` | 이 별점 이하인 리뷰는 수동 게시 |
| 키워드 포함 | `posting_option_keyword` | `False` | 알림 키워드 포함 리뷰를 수동 게시 |
| 이미지 포함 | `posting_option_photo` | `False` | 이미지가 포함된 리뷰를 수동 게시 |
| 동영상 포함 | `posting_option_video` | `False` | 동영상이 포함된 리뷰를 수동 게시 |
| 중복 이미지 | `posting_option_duplicatedMedia` | `False` | 다른 리뷰와 중복된 이미지 포함 시 수동 게시 |
| 부정적 감성 | `posting_option_sentimentNegative` | `False` | AI 감성 분석 결과 부정적 리뷰를 수동 게시 |

#### 3.1.3 혜택 지급 시점 규칙

`reward_option` 필드에 의해 포인트 지급 방식이 결정된다.

2. 새로운 데이터를 일괄 생성

**카테고리 그룹화 (group_by_category)**:
- type별로 카테고리를 그룹화하고, 하위(child) 카테고리까지 포함하여 반환
- 코어 서비스의 `fetch_categories_sub` API를 호출하여 하위 카테고리를 가져옴
- 결과는 5분 동안 캐시됨 (캐시 키: `category_sub_{shop_id}_{category_ids}`)

쇼핑몰 최초 설정 시 다음 규칙으로 ShopReward가 생성된다.

1. **6가지 리뷰 유형** x **7가지 작성자 유형**의 조합으로 생성
   - 리뷰 유형: 별점, 텍스트, 포토, 동영상, 설문, SNS
   - 작성자 유형: 회원, 비회원, 네이버페이, 지그재그, 카카오페이, 마켓플러스, 미검증

3. 각 리뷰 유형의 기본 포인트 값은 `SHOPREWARD_REWARD_PRESET`에 정의됨
4. 이미 존재하는 조합은 중복 생성하지 않음

#### 3.3.4 보상 수정 규칙 (ShopReward__Features.patch)

1. 트랜잭션 내에서 처리
2. 요청 데이터에 `id`가 포함되면 수정, `id`가 없으면 신규 생성
3. 기존에 존재하지만 요청 데이터에 포함되지 않은 항목은 삭제 대상
4. **초깃값(initial에서 생성된 6가지 기본 리뷰 유형)은 삭제에서 제외** (`filter_initial_reward` 메서드)
5. 감사 로그(audit_log)가 자동 기록됨

#### 3.3.5 보상 값 유효성 검증

`ShopReward__DynamicSerializer.validate_reward`에서 다음 규칙으로 검증한다.

- **정액제**: 0 이상의 정수만 허용 (예: `"1000"`)
- **정률제**: `%`로 끝나는 문자열이며, 0 이상 100 이하의 실수만 허용 (예: `"10%"`)

#### 3.3.6 SNS 리뷰 포인트 조회 규칙

### 3.4 리뷰 정책 상세 규칙 (ShopPolicy)

#### 3.4.1 리뷰 작성 기한

리뷰 작성 가능 시점은 `ShopReviewPolicy__ReviewStartTimingType`에 의해 결정된다.

| 값 | 이름 | 설명 |
|----|------|------|
| `400` | `배송중` | 배송 시작(송장 입력) 시점부터 리뷰 작성 가능 |
| `500` | `배송완료` | 배송 완료 시점부터 리뷰 작성 가능 |

- `use_review_write_limit = True`이면 `review_write_limit` (기본 365일) 경과 후 리뷰 작성 불가
- 시작 시점과 종료 기한으로 리뷰 작성 가능 구간이 결정됨

#### 3.4.2 미검증 리뷰 제한

- `write_unverified_day`: 사용자당 하루 최대 미검증 리뷰 작성 수 (기본 5건)
- `use_reward_unverified`: 미검증 리뷰에 대한 혜택 지급 여부 (기본 비허용)
- 위젯 모듈에서 미검증 리뷰 관련 리다이렉트 코드:
  - `UNVERIFIED_REVIEW_RESTRICTED_PRODUCT`: 해당 상품은 미검증 리뷰 작성이 제한됨
  - `UNVERIFIED_REVIEW_TODAY_WRITE_POSSIBLE`: 일일 미검증 리뷰 작성 가능
  - `UNVERIFIED_REVIEW_TODAY_LIMIT_EXCEEDED`: 일일 미검증 리뷰 작성 제한 소진

#### 3.4.3 포인트 최소/최대 한도

**최소 한도**:
- `use_min_point = True`이면, 정률제 포인트가 `min_point` (기본 10P)보다 적을 경우 `min_point`로 보정

**최대 한도**:
- `use_max_total_point = True`이면, 총 포인트에 상한선 적용
- 정액제 한도: `use_max_total_point_fixed = True`일 때, `max_total_point_fixed` (기본 20,000P) 적용
- 정률제 한도: `use_max_total_point_percentage = True`일 때, `max_total_point_percentage` (기본 10%) 적용
- `use_max_total_point_range = True`이면, 추가 포인트(선착순, 많은이미지 등)를 최대 한도 계산에서 제외

#### 3.4.4 포인트 지급 제한

- `use_number_reward = True`이면, 동일 상품에 대해 `number_reward` (기본 1건)까지만 포인트 지급
- `use_number_reward_quick = True`이면, 지급 제한 시에도 별점 리뷰(빠른 리뷰)에는 기본 포인트를 지급

#### 3.4.5 포인트 지급 제외 조건

다음 조건에 해당하면 포인트를 지급하지 않는다.

| 조건 | 사용 플래그 | 상세 필드 | 설명 |
|------|-------------|-----------|------|
| 최소 구매 금액 미달 | `use_reward_condition_price` | `reward_condition_price` | 구매 금액이 기준 미만이면 미지급 |
| 수령 후 기간 초과 | `use_reward_condition_delivery` | `reward_condition_delivery` (기본 90일) | 수령 후 해당 기간 초과 시 미지급 |
| 품절 상품 | `use_reward_condition_soldout` | - | `False`이면 품절 상품 리뷰에 미지급 |
| 미진열 상품 | `use_reward_condition_display` | - | `False`이면 미진열 상품 리뷰에 미지급 |
| 미판매 상품 | `use_reward_condition_selling` | - | `False`이면 미판매 상품 리뷰에 미지급 |

> **주의**: `use_reward_condition_soldout`, `use_reward_condition_display`, `use_reward_condition_selling`은 기본값이 `True`이므로, 기본적으로는 품절/미진열/미판매 상품에도 혜택이 지급된다.

#### 3.4.6 포인트 회수

- `use_refund_point_cancel = True`이면, 취소/교환/반품 시 이미 지급된 포인트를 자동 회수

#### 3.4.7 빠른 리뷰 타입 (ShopPolicy__QuickReviewType)

| 값 | 이름 | 레이블 | 설명 |
|----|------|--------|------|
| `1` | `커스텀` | `"custom"` | 쇼핑몰이 직접 작성한 빠른 리뷰 텍스트 사용 |
| `2` | `프리셋` | `"preset"` | 시스템 기본 제공 텍스트 사용 |

### 3.5 자동 답글 정책 (ShopCommentReview)

#### 3.5.1 답글 분류

- `auto = True`: 자동 답글 (시스템이 랜덤 선택하여 자동 부착)
- `auto = False`: 수동 답글 (관리자가 직접 작성)

#### 3.5.2 평가(evaluation)별 답글 매칭

`ShopQuickReview__Evaluation` 기준으로 매칭된다.

| 값 | 이름 | 대응 별점 |
|----|------|-----------|
| `"만족"` | 만족 | 5, 4점 |
| `"보통"` | 보통 | 3점 |
| `"별로"` | 별로 | 2, 1점 |

#### 3.5.3 기본 답글 템플릿

초기화 시 다음 6개의 기본 답글이 생성된다.

- **만족 (4개)**: 긍정적인 감사 메시지, `{회원이름}`, `{쇼핑몰이름}` 변수 포함
- **보통 (1개)**: 개선 의지를 보이는 중립 메시지
- **별로 (1개)**: 사과와 개선 약속 메시지

#### 3.5.4 답글 리셋 규칙

- 리셋 시 `auto = True`인 답글만 전체 삭제 후 기본 템플릿으로 재생성
- `auto = False`인 수동 답글은 리셋의 영향을 받지 않음

#### 3.5.5 답글 텍스트 선택 규칙

`ShopCommentReview__Manager.create_review_comment_text`에서:
- 해당 evaluation에 매칭되는 댓글 목록에서 `random.choice()`로 랜덤 선택
- `{포인트}` 키워드를 포함하는 댓글은 필터링하여 제외 (`get_list_with_filter` 메서드)

### 3.6 키워드 관련 정책

#### 3.6.1 알림 키워드 (AlertKeyword)

- 리뷰 내 특정 단어(`origin`)를 다른 단어(`alternative`)로 대체하여 표시
- CRUD 및 일괄 삭제(bulk_delete) 지원

#### 3.6.3 키워드 리뷰 설정 (ShopKeywordReview)

| 필드 | 기본값 | 설명 |
|------|--------|------|
| `init_keyword` | `False` | 키워드 초기화 여부 |

## 4. 상태 전이

### 4.1 리뷰 게시 상태 흐름

ShopConfirmOption의 `posting_option`과 각 조건 플래그에 의해 결정된다.

```
리뷰 작성 완료
    |
    +-- [posting_option == "immediate" AND 수동 게시 조건 미해당]
    |       -> 게시됨 (PostingStatus: 20)
    |
    +-- [posting_option != "immediate" OR 수동 게시 조건 해당]
            -> 게시대기 (PostingStatus: 11)
                |
                +-- [관리자 승인] -> 게시됨 (PostingStatus: 20)
                +-- [관리자 숨김] -> 숨김 (PostingStatus: 21)
```

### 4.2 혜택 지급 상태 흐름

ShopConfirmOption의 `reward_option`과 ShopPolicy의 각 지급 조건에 의해 결정된다.

```
리뷰 작성 완료
    |
    +-- [지급 제외 조건 해당 (상품/카테고리 제한, 최소 금액 미달 등)]
    |       -> 지급제한 (RewardStatus: 23)
    |
    +-- [reward_option == "admin" OR 수동 지급 조건 해당]
    |       -> 지급대기 (RewardStatus: 11)
    |           |
    |           +-- [관리자 승인] -> 지급완료 (RewardStatus: 20)
    |           +-- [관리자 취소] -> 지급취소 (RewardStatus: 21)
    |
    +-- [자동 지급]
            -> 지급예정 (RewardStatus: 10)
                |
                +-- [예정 시간 도래] -> 지급완료 (RewardStatus: 20)
                    |
                    +-- [취소/교환/반품 + use_refund_point_cancel == True]
                            -> 회수예정 (RewardStatus: 30)
                                |
                                +-- 회수완료 (RewardStatus: 31)
                                +-- 회수실패 (RewardStatus: 32)
```

### 4.3 SNS 리뷰 유지 기간 상태

```
SNS 리뷰 작성 완료
    |
    +-- [use_maintenance_sns == True]
    |       -> SNS지급대기 (RewardStatus: 13)
    |           |
    |           +-- [maintenance_sns (기본 3일) 경과 후 게시물 확인]
    |                   |
    |                   +-- [게시물 유지됨] -> 지급완료 (RewardStatus: 20)
    |                   +-- [게시물 삭제됨] -> 지급취소 (RewardStatus: 21)
    |
    +-- [use_maintenance_sns == False]
            -> 즉시 지급 처리
```

## 5. Usecase 흐름

### 5.1 쇼핑몰 초기 설정

```
1. ShopConfirmOption 생성 (기본값: 즉시 게시, 관리자 승인 혜택)
2. ShopPermit 생성 (기본값: 일반/빠른/동영상 허용, 설문/SNS/키워드/미검증 비허용)
3. ShopPolicy 생성 (기본값으로 각 정책 초기화)
4. ShopReward__Features.initial(shop_id) 호출
   - 6종 리뷰 x 7종 작성자 = 최대 41개 ShopReward 생성 (SNS+미검증 제외)
   - 기본 포인트: 별점 300P, 텍스트 500P, 포토 1000P, 동영상 1000P, 설문 500P, SNS 2000P
5. ShopCommentReview__Features.initial(shop_id) 호출
   - 6개 기본 답글 템플릿 생성
6. ShopQuickReview__Feature.initial(shop_id) 호출
   - 빠른 리뷰 기본 텍스트 생성
```

### 5.2 리뷰 작성 시 정책 적용

```
1. ShopPermit 확인
   - 해당 리뷰 유형이 permit_{type} == True인지 확인
   - False이면 리뷰 작성 거부

2. ShopPermitProduct/ShopPermitCategory 확인
   - 해당 상품/카테고리가 리뷰 작성 제외(type 200~270) 목록에 있는지 확인
   - 미검증 리뷰인 경우 type 260 확인
   - 해당하면 리뷰 작성 거부

3. ShopPolicy 확인
   - 작성 가능 고객 확인 (비회원, 네이버페이 등)
   - 리뷰 작성 기한 확인 (review_write_timing ~ review_write_limit)
   - 미검증 리뷰인 경우 일일 작성 한도(write_unverified_day) 확인
   - 텍스트 최소 글자 수, 포토 최소 사진 수, 동영상 최소 길이 확인

4. ShopConfirmOption 적용
   - 게시 시점 결정 (즉시 게시 vs 수동 게시)
   - 혜택 지급 시점 결정 (즉시 지급 vs 수동 지급)
```

### 5.3 포인트 지급 시 정책 적용

```
1. ShopPermitProduct/ShopPermitCategory 확인
   - 해당 상품/카테고리가 포인트지급_제외(type 100) 목록에 있는지 확인
   - 해당하면 포인트 미지급

2. ShopPolicy 지급 조건 확인
   - 최소 구매 금액 확인 (reward_condition_price)
   - 수령 후 기간 확인 (reward_condition_delivery)
   - 품절/미진열/미판매 상태 확인
   - 상품당 지급 횟수 확인 (number_reward)

3. ShopReward에서 해당 리뷰 유형 + 작성자 유형의 포인트 조회
   - title(리뷰 유형) + value(작성자 유형) 조합으로 조회

4. 포인트 한도 적용
   - 최소 한도: 정률제 계산 결과 < min_point 이면 min_point 적용
   - 최대 한도: 정액제/정률제 한도 중 해당하는 것 적용
   - use_max_total_point_range 여부에 따라 추가 포인트 포함/제외

5. 포인트 지급 또는 대기 상태 전환
```

### 5.4 리뷰 허용 설정 변경

```
1. 관리자가 ShopPermit 수정 요청 (PATCH)
2. ShopPermit__Features.patch 호출 (트랜잭션 내)
   a. ShopPermitCategory 처리 (있으면)
      - 기존/요청 ID 비교 -> 생성/수정/삭제 분류
      - bulk_create, bulk_update, bulk_delete 실행
   b. ShopPermitProduct 처리 (있으면)
      - 동일한 생성/수정/삭제 분류 로직
   c. ShopPermit 필드 업데이트
3. ShopWritePage__Feature.sort(shop_id) 호출
   - 변경된 permit 설정에 맞게 작성 페이지 리뷰 유형 순서 재정렬
4. 감사 로그 기록 (@audit_log 데코레이터)
```

### 5.5 혜택 설정 변경

```
1. 관리자가 ShopReward 수정 요청 (PATCH, 리스트 형태)
2. ShopReward__Features.patch 호출 (트랜잭션 내)
   a. 기존 reward 목록 수집 (로깅 목적)
   b. 요청 ID와 기존 ID 비교 -> 생성/수정/삭제 분류
   c. 초깃값(기본 6종) 삭제 보호 (filter_initial_reward)
   d. bulk_create, bulk_update, bulk_delete 실행
   e. 변경 후 reward 목록 수집 (로깅 목적)
3. 감사 로그 기록 (@audit_log 데코레이터)
```

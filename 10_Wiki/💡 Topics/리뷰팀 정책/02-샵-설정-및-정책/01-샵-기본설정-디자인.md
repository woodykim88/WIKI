# 샵 기본설정 및 디자인

## 1. 도메인 개요

### 1.1 비즈니스 문제
알파리뷰 서비스를 이용하는 각 쇼핑몰(Shop)은 고유한 설정값과 디자인 커스터마이징이 필요하다. 이 도메인은 다음 문제를 해결한다:

- **멀티테넌시 기반 쇼핑몰 설정 관리**: 각 쇼핑몰이 독립적으로 기본설정, 디자인, 리뷰 작성 페이지를 커스터마이징할 수 있도록 한다.
- **리뷰 위젯 디자인 커스터마이징**: 상세페이지, 리뷰 없을 때 유도화면, 재방문 팝업, 배너 버튼 등 다양한 리뷰 위젯의 시각적 설정을 쇼핑몰별로 관리한다.
- **데이터 이관 상태 추적**: Cafe24 등 외부 플랫폼으로부터의 상품/주문/리뷰 데이터 이관 진행 상태를 관리한다.
- **AI 분석 컨텍스트 관리**: 쇼핑몰별 AI 분석 성능 향상을 위한 프롬프트(소개글)를 관리한다.

### 1.2 핵심 개념 및 용어 정의

| 용어 | 정의 |
|------|------|
| **Shop** | 알파리뷰 서비스에 가입한 하나의 쇼핑몰 단위. `shop_id`로 식별. |
| **ShopOption** | 쇼핑몰의 기본 운영 설정 (자동 댓글, 해시태그, 스태프리뷰 설정, 해지 관련 옵션 등). |
| **ShopDesign** | 리뷰 위젯의 시각적 디자인 설정 (색상, 폰트, 상세페이지, 재방문 팝업, 배너 등). |
| **ShopWritePage** | 리뷰 작성 페이지의 디자인 및 구성 설정 (테마 색상, 리뷰 유형별 정렬 순서 등). |
| **ShopAutocrawling** | Cafe24로부터의 데이터 이관(상품, 주문, 리뷰 등) 진행 상태 추적 모델. |
| **스태프리뷰** | 관리자가 직접 작성하는 리뷰. 작성자 아이디/이름/SNS ID를 별도로 설정 가능. |
| **재방문 팝업** | 구매 이력이 있는 사용자가 재방문 시 리뷰 작성을 유도하는 팝업/플로팅 버튼. |
| **리뷰 작성 유도** | 리뷰가 없는 상품 페이지에서 첫 리뷰 작성을 유도하는 화면. |
| **QR 코드** | 송장에 포함되는 QR 코드로, 리뷰 작성 페이지로 바로 연결. |
| **short_code** | 쇼핑몰별 고유 코드. QR 코드 URL 생성에 사용. |

## 2. 데이터 모델

### 2.1 Shop (쇼핑몰 기본 정보)

| 필드명 | 타입 | 기본값 | 설명 |
|--------|------|--------|------|
| `shop_id` | IntegerField (unique) | - | 쇼핑몰 고유 ID. 전체 시스템의 멀티테넌시 키. |
| `short_code` | CharField(20) | null | 쇼핑몰 고유 코드. QR URL 등에 활용. |
| `service_sns` | BooleanField | `False` | SNS 서비스 사용 여부. |
| `introduce` | TextField | null | 쇼핑몰 소개글. AI 분석 키워드 기능과 함께 추가. 고객사가 AI 분석 성능 향상을 위해 입력하는 프롬프트. |

**비즈니스 의미**: Shop은 시스템 전체에서 최상위 테넌트 역할을 하며, 모든 하위 모델이 `shop_id`를 기준으로 분리된다. `introduce` 필드는 "맞춤형 AI 컨텍스트"라는 이름으로 대시보드에 노출되며, 최대 500자까지 입력 가능하고 빈 문자열은 허용되지 않는다.

### 2.7 기타 관련 모델

#### ShopPopup (쇼핑몰 팝업)
- `shop_id` (unique), `target`, `content`, `is_visible`, `is_verified`, `expired_at`
- 특정 대상에게 노출되는 팝업 관리. 만료일(`expired_at`) 기반 제어.

#### ShopOptionActiveFunction (기능 활성화 옵션)
- `shop_id` (unique), `stat_simple`, `stat_simple_status`, `popup_report`, `email_report`
- 간단 통계 표시 상태: `OFF(0)` = 사용자 강제 비활성화, `AUTO(1)` = 자동 결정, `ON(2)` = 사용자 강제 활성화
- `stat_simple`은 월간 실적에 따라 자동 변경됨.

#### ShopStatDaily (쇼핑몰 일간 통계)
- `shop_id` + `created_at` 복합 유니크 제약조건
- 리뷰(전체/알파/회원/비회원/NPAY/지그재그/외부/빠른/일반/포토/동영상/설문/SNS), 주문, 포인트, PV/UV, 매출 등 일간 누적 통계 저장

#### ShopAdminLog (어드민 로그)
- 어드민 접속 기록. IP 주소 추출 시 `X-Forwarded-For` 헤더 우선 사용(프록시 환경 대응).

---

### 2.8 모델 간 관계

```
Shop (shop_id) ----1:1---- ShopOption (shop_id)
      |
      +--------1:1---- ShopDesign (shop_id)
      |
      +--------1:1---- ShopWritePage (shop_id)
      |
      +--------1:1---- ShopAutocrawling (shop_id)
      |
      +--------1:N---- ShopAutocrawlingReviewTarget (shop_id)
      |
      +--------1:1---- ShopPopup (shop_id)
      |
      +--------1:1---- ShopOptionActiveFunction (shop_id)
      |
      +--------1:N---- ShopStatDaily (shop_id + created_at)
      |
      +--------1:N---- ShopAdminLog (shop_id)
```

**중요**: 모든 모델은 `shop_id`를 IntegerField로 독립 저장하며, Django ForeignKey가 아닌 논리적 관계로 연결되어 있다. 이는 멀티테넌시를 별도의 FK 제약 없이 운영하는 레거시 설계이다.

### 3.2 쇼핑몰 초기화 (Shop__Feature.initial)

쇼핑몰이 서비스에 가입하면 `Shop__Feature.initial(shop_id)`를 호출하여 모든 기본 설정을 생성한다.

**1단계 - 생성(create) 대상**:
1. `Shop` - 기본 쇼핑몰 레코드

9. `InstagramFeedDesignSetting` - 인스타그램 피드 디자인

**2단계 - 초기화(initial) 대상**:
1. `ShopWritePage` - 리뷰 작성 페이지 (기본 로그인 메시지 포함)
2. `ShopDesign` - 디자인 설정 (기본 유도 텍스트, 재방문 텍스트, 배너 텍스트 포함)

### 3.5 리뷰 작성 페이지 정렬 로직 (ShopWritePage__Feature.sort)

리뷰 작성 페이지에서 리뷰 유형의 노출 순서를 결정하는 복합 로직:

1. **리뷰 비허용 시**: `ShopPermit.permit_normal`이 `False`이면 모든 정렬값을 `0`으로 설정 (모든 유형 비활성화).
2. **모든 정렬값이 0일 때**: 기본 순번(1~6)을 자동 지정.
3. **quick_sort**: 항상 `1`번 (최우선 고정).
4. **keyword_sort**: `ShopPermit.permit_keyword`가 `True`이면 `2`번, `False`이면 `0` (비활성화).
5. **나머지 유형** (`text_sort`, `photo_sort`, `video_sort`, `survey_sort`):
   - 기존 순서 유지하되, `0`인 항목은 후순위로 밀림.
   - `text_sort`, `photo_sort`: 항상 순번 할당 (필수 유형).
   - `video_sort`: `permit_video`가 `True`일 때만 순번 할당.
   - `survey_sort`: `permit_survey`가 `True`일 때만 순번 할당.
   - 비허용된 유형은 `0`으로 설정.

### 3.6 자동 이관 설정 (ShopAutoCrawling__Feature.set_auto_transfer)

트랜잭션(`@transaction.atomic`)으로 보호되는 자동 이관 설정 로직:

| 조건 | 필수 입력값 | 저장 필드 |
|------|------------|-----------|
| `review_auto=True` + `use_full_transfer=True` | `review_auto`, `use_full_transfer` | `review_auto`, `use_full_transfer` |
| `review_auto=True` + `use_full_transfer=False` | `review_auto`, `use_full_transfer`, `interval_start_date` | `review_auto`, `use_full_transfer`, `interval_start_date` |
| `review_auto=False` | `review_auto` | `review_auto`만 |

**유효성 검증 규칙**:
- `review_auto=True`일 때 `use_full_transfer`는 필수 (null 불가).
- `use_full_transfer=False` (구간 이관)일 때 `interval_start_date`는 필수.

### 3.9 배치 분석 자격 조건 (BatchAnalysisEligibility__Feature)

## 4. 상태 전이

### 4.1 ShopAutocrawling 이관 진행 상태

각 이관 항목(product, order, review, install, widget, review_number, promotion, review_upload)은 동일한 상태 전이를 따른다:

```
[대기 (none)]
     |
     v
[설치중 (install)]
     |
     +----------> [완료 (complete)]
     |
     +----------> [실패 (stop)]
```

- **대기 -> 설치중**: Task 실행 시 `patch_on_running`을 통해 전환.
- **설치중 -> 완료**: 이관 작업이 성공적으로 끝났을 때.
- **설치중 -> 실패**: 이관 중 에러 발생 시. 에러 내용은 `*_error_code` 필드에 기록.
- `is_complete`는 모든 항목이 완료되었을 때 `True`로 설정.

### 4.2 ShopAutocrawlingReviewTarget 진행 상태

```
[대기 (100)] -> [진행 (200)] -> [완료 (300)]
                     |
                     +---------> [실패 (400)]
```

실패 시 `error_info` 필드에 에러 내용이 기록된다.

## 5. Usecase 흐름

### 5.1 쇼핑몰 최초 가입 (초기화)

```
1. 외부 시스템(Cafe24 등)에서 가입 요청 수신
2. POST /shop/{shop_id}/initial 호출
3. Shop__Feature.initial(shop_id) 실행
   3.1. Shop, ShopOption, ShopConfirmOption 등 9개 모델 create
   3.2. ShopWritePage, ShopDesign 등 9개 모델 initial (기본 텍스트 포함)
   3.3. 각 단계의 성공/실패/소요시간을 Datadog에 로깅
4. 쇼핑몰 사용 가능 상태
```

### 5.2 공통 디자인 수정

```
1. 관리자가 대시보드에서 디자인 설정 변경
2. PATCH /shop/{shop_id}/design/common 호출
   - body: { primary_color, font_family, font_size, ... }
3. DesignCommon__Serializer로 유효성 검증
4. ShopDesign__Manager.patch(shop_id, data) 실행
   4.1. @audit_log 데코레이터가 변경 전 상태 캡처
   4.2. 필드별 setattr + save
   4.3. 변경 내역을 감사 로그에 기록
5. 변경된 ShopDesign 인스턴스를 직렬화하여 응답
```

### 5.3 리뷰 작성 페이지 정렬 순서 갱신

```
1. ShopPermit(리뷰 허용 정책) 변경 발생
2. ShopWritePage__Feature.sort(shop_id) 호출
3. ShopPermit, ShopWritePage 현재 상태 조회
4. 정렬 순서 재계산
   4.1. permit_normal=False -> 모든 sort를 0으로
   4.2. quick_sort는 항상 1
   4.3. keyword_sort는 permit_keyword에 따라 결정
   4.4. 나머지는 기존 순서 유지 + permit 확인
5. ShopWritePage__Manager.patch로 갱신
```

### 5.4 리뷰 자동 이관 설정

```
1. 관리자가 이관 설정 페이지에서 자동 이관 옵션 선택
2. PATCH /shop/{shop_id}/auto-crawling/set 호출
   - body: { review_auto: true, use_full_transfer: true/false, interval_start_date?: "2024-01-01" }
3. ShopAutoCrawling__SetSerializer 유효성 검증
   3.1. review_auto=true이면 use_full_transfer 필수
   3.2. use_full_transfer=false이면 interval_start_date 필수
4. ShopAutoCrawling__Feature.set_auto_transfer (트랜잭션)
5. ShopAutocrawling 레코드 업데이트
```

### 5.5 QR 코드 디자인 조회

```
1. 관리자가 QR 코드 설정 페이지 접근
2. GET /shop/{shop_id}/design/collect/qr-page 호출
3. ShopDesign__Feature.get_qrcode_data(shop_id)
   3.1. ShopDesign 조회 (qr_text_1, qr_text_2, qr_text_3)
   3.2. Core-Redis에서 쇼핑몰 정보 조회 -> short_code 획득
   3.3. QR URL 생성: {REVIEW_WRITE_PAGE_HOST}/s{short_code}
   3.4. 동적으로 qr_url 속성 추가
4. DesignQRcodePage__GetSerializer로 직렬화하여 응답
```

### 5.6 배치 분석 자격 확인

```
1. 관리자가 배치 분석 기능 접근 시도
2. GET /shop/{shop_id}/batch-analysis/eligibility 호출
3. BatchAnalysisEligibility__Feature.check_eligibility(shop_id)
   3.1. ReviewTag 활성 개수 확인 -> 0개여야 함
   3.2. 최근 3개월 일반리뷰 수 확인 -> 200개 이상이어야 함
4. 결과: { result: true/false }
```

### API 엔드포인트 요약

| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | `/shop/{shop_id}/initial` | 쇼핑몰 초기 설정 생성 |
| GET/PATCH | `/shop/{shop_id}/context` | 맞춤형 AI 컨텍스트 조회/수정 |
| GET/PATCH | `/shop/{shop_id}/comment/auto-setting` | 댓글 자동 설정 조회/수정 (deprecated) |
| GET/PATCH | `/shop/{shop_id}/shop-option` | 쇼핑몰 기본설정 조회/수정 |
| GET/PATCH | `/shop/{shop_id}/design/common` | 공통 디자인 조회/수정 |
| GET/PATCH | `/shop/{shop_id}/design/detail-page` | 상세페이지 디자인 조회/수정 |
| GET/PATCH | `/shop/{shop_id}/design/write-page` | 리뷰 작성 페이지 조회/수정 |
| GET/PATCH | `/shop/{shop_id}/design/collect` | 리뷰 수집 메인 설정 조회/수정 |
| GET/PATCH | `/shop/{shop_id}/design/collect/empty-page` | 리뷰 없을 때 디자인 조회/수정 |
| GET/PATCH | `/shop/{shop_id}/design/collect/revisit-page` | 재방문 디자인 조회/수정 |
| GET/PATCH | `/shop/{shop_id}/design/collect/banner-page` | 배너 디자인 조회/수정 |
| GET/PATCH | `/shop/{shop_id}/design/collect/qr-page` | QR 코드 디자인 조회/수정 |
| GET | `/shop/{shop_id}/auto-crawling` | 이관 상태 조회 |
| PATCH | `/shop/{shop_id}/auto-crawling/set` | 자동 이관 설정 |
| POST | `/shop/{shop_id}/admin-log` | 어드민 로그 생성 |
| GET | `/shop/{shop_id}/batch-analysis/eligibility` | 배치 분석 자격 확인 |

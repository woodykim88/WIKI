### 💡 결론 먼저

분석 정확도 개선이 선결 과제. 품질 기준 미달 상태에서의 추가 세일즈는 보류. 특정 카테고리(의류 등) 타겟팅 전략으로 방향 선회.

### 🔍 핵심 문제 진단

#### 초기 기획 가치 vs 실제 제공 가치 괴리

기획 당시 제시한 두 가지 가치

리소스 절감 (정기 리포트 → 명확한 액션 아이템 제공)

골든타임 확보 (치명적 이슈 발생 시 알림)

실제 제공된 것

리포트: 대시보드 내용을 그대로 복붙한 수준. 명확한 액션 아이템 없음 → 고객 반응 "그래서 이걸 보고 어쩌라는 거지?"

알림: 3~4월 치명적 리뷰 알림 224건 수동 검토 결과, 분류 정확도 58.5% 수준

대시보드: 고객 업무 루틴상 접속 자체를 거의 안 함. 접속한 고객도 "너무 복잡하고 이해하기 어렵다" 피드백

#### 고객 피드백 요약 (무료체험 12개사 중 7개사 응답)

| 반응 유형 | 주요 내용 |
| --- | --- |
| 담당자 부재/보류 | 실무 담당자 아닌 직원이 전화 받음, 내부 논의 후 회신 예정 |
| 활용도 저조 | 대시보드 접속 거의 없음, 법인폰 알림만 수신 중 |
| 비용 효용성 부족 | 서비스 자체는 나쁘지 않으나, 유료 결제할 만큼 강한 필요성 못 느낌 |
| 체험 환경 부재 | 회사 이사 등 내부 사정으로 체험 기간 내 미사용 |
| 기능 불만 | 부정 분석 정확도 체감 안 됨, 대시보드 복잡 |

### 🛠️ 기술적 문제 상세

#### 분류 정확도 저하 원인

모델이 프리트레인된 일반 도메인 외, 세밀한 카테고리(특수 식품, 신선식 등)에서 정확도 급락

상품 정보 분석(크롤링 → 이미지 분석 → 특성 추출) 성공률: 전체의 8% (6만 1천 건 시도 중 5천 건 성공)

실패 원인: 프로덕션 환경에서 웹페이지 과부하, 컴퓨팅 리소스 문제

의류 등 정형화된 카테고리에서는 정확도 90% 가능성 있음

상품 분석 성공률이 낮을수록 리뷰 분류 정확도도 낮아지는 구조

#### 원가 문제

초기 시뮬레이션: 리뷰 1건당 태그 분석 비용 0.34원

실제 발생 비용: 리뷰 1건당 2.4~2.5원 (약 7배 초과)

원인: 네이버페이/스마트스토어 연동 리뷰에 본문 없는 케이스 多 → 아무 내용 없는 리뷰를 3회 재시도하며 토큰 낭비

현재 마진율: 약 1.5% (목표 대비 현저히 낮음)

버그 수정 및 필터링 로직 개선 시 마진율 80~90% 수준 회복 가능한 것으로 추정

### 💬 주요 논의

#### 개선이냐 홀드냐

품질이 최소 기준에 미달한 상태에서 가격 조정만으론 의미 없다는 데 대체로 동의

정확도 90% 미만에서는 "이 기능을 제공한다"고 말하기 어렵다는 판단

한편 "니즈 자체가 있는지 아직 검증 안 됐다"는 근본적 의문도 제기됨

#### 카테고리 타겟팅 전략 (주요 제안)

전체 카테고리 정확도 개선은 난이도 높음

의류 등 정형화된 카테고리에서만 먼저 분류 정확도 90% 달성 가능

해당 카테고리 업체에 집중 어프로치 → 30개 유료 확보 재시도

크레마(경쟁사)가 의류에 집중하는 이유와 맥락 같음

#### 가격 조정 방향

현재 가격이 알파리뷰 기본 구독료보다 높아 세일즈 허들로 작용

두 가지 안 논의:

AI 인사이트Pro 단독 가격 인하

알파리뷰 기본 구독료 자체를 올리고 AI 인사이트 기능을 번들로 포함 (프라이싱 리빌딩)

가격 최종 결정은 대표님과 별도 논의 필요

Phase 2 (브랜드 평가, 상품별 고도화 인사이트) 이후 가격 재조정 가능성 열어둠

#### 수동 보완 아이디어

무료 체험 중인 업체의 상품 DB를 수동으로 채워 정확도를 인위적으로 높인 후 어프로치하는 방식 제안

소수 업체 대상 빠른 실험 가능 (24시간 내 작업 가능 수준)

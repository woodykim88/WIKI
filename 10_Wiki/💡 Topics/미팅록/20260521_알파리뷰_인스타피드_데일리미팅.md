# 미팅록: 알파리뷰 인스타피드앱 개발 데일리미팅 (0521)

* **일자**: 2026년 5월 21일 (목)
* **주최자**: 김종우
* **참여자**: 정태성, 최규백, 최민수, 송용협, 안동혁, 배준수

---

## 1. 디자인 및 기획 리뷰

### 1.1. 요금제 화면 개선 (유료 전환 플로우)
* **기존**: 요금제 목록을 한꺼번에 여러 개 나열하여 유저에게 불필요한 정보 노출 및 혼란을 유발함.
* **개선 방향**: 해당 쇼핑몰 업체의 월 주문수를 백엔드에서 계산해 플랜 등급을 사전에 판별한 뒤, **딱 맞는 요금 1개만 화면에 표시**한다.
* **결정**: 주문수 판별 로직이 기존 시스템에 이미 구현되어 있어 추가 개발 공수가 들지 않는 것으로 파악됨. 기획/디자인 의도대로 그대로 진행.
* **우려 포인트**: 업체 주문량 초과 시 플랜 업그레이드 안내 과정에서 "처음엔 8만 9천이라더니 갑자기 플랜이 바뀌었다"라는 식의 고객 혼선이 생길 수 있음.
* **대응**: 플랜 한도 초과 시 자동으로 발송되는 알림톡 및 이메일 정책이 이미 시스템에 구성되어 있어, 실시간 대응이 가능한 것으로 최종 확인됨.

### 1.2. 계정 정보 미입력 시 사용 차단 모달 정책
* 스태프 리뷰 등록 전에 계정 연동을 강제하는 '계정 연동 요청 모달'을 무조건 띄운다. (기존에는 미입력 상태에서도 리뷰 등록이 가능했던 예외 케이스 존재)
* 인스타그램 피드 위젯도 동일한 로직을 적용한다.
  * 쇼핑몰 정보 미입력 상태로 '선택 완료' 클릭 시, 계정 입력 요청 차단 모달을 띄운다.

### 1.3. 복수 계정 선택 및 쿼터 초과 처리
* **무료 버전**: 오직 단일 계정 연결만 허용함.
* **복수 선택 시**: '쿼터 초과 경고 모달'을 출력하고, 확인을 누르면 프로세스를 불가 처리함.
* **결정**: 유료 업그레이드를 자연스럽게 넛지(유도)하는 것이 목적이므로, 선택 단계 자체를 막기보다 초과하여 적용하려고 할 때 모달로 차단하고 전환을 제안하는 방향으로 최종 결정함.

## 2. 개인별 업무 현황

* **규백 (디자이너)**:
  * 홈 화면 케이스 추가 디자인 완료 및 Figma 링크 공유 완료.
  * 상세 페이지 초안은 내일까지 공유 예정.
  * 다음 주에 용수 PO 피드백 반영 후 최종 확정 예정.
* **연재**:
  * 특이사항 없음.
* **용협**:
  * 인스타그램 설치 완료 배너 추가 작업 진행 중.
  * `iframe` 내부에서 CSS 조절을 통해 배너 내부에 자연스럽게 삽입 가능한 방안을 파악 완료함.
  * 로딩 UI 구현 작업 예정.
* **동혁**:
  * 메타 토큰 만료-갱신 테스트 방법 논의 필요 (용현님과 별도 싱크 예정).
  * 스크립트태그 전환 실패 케이스 발생 시 처리 정책 수립 담당.
  * 전환 완료 및 성공 시의 알림톡 템플릿 시스템 등록 담당.
* **민수**:
  * 변경 사항 없음.
  * 시연용 데이터 생성 완료 (약 20개 상품 / 1명의 실사용자 / 2개의 주문).
  * 기존 데이터 이관 프로세스 조정 진행 중.

## 3. 기술 이슈 논의

### 3.1. 스크립트태그 전환 (제거 ➔ 설치) 실패 시 예외 처리 정책
* **옵션 1 (기각)**: 설치 요청 자체를 에러/실패로 처리함. ➔ 유저에게 불필요한 실패 팝업을 노출하여 혼란을 유발할 가능성이 매우 높음.
* **옵션 2 (채택/최종 결정)**: 우선 스크립트 전환 실패 시 로그만 조용히 남겨 두고, 코어팀이 데이터독(Datadog) 경고 알람으로 감지하여 직접 대응한다.
  * *유료 ➔ 무료 다운그레이드 케이스*: TP팀 작업 진행 후 스크립트 태그를 최종 전환함.
  * *무료 ➔ 유료 업그레이드 케이스*: 유료 스크립트 태그를 먼저 헤더에 올린 뒤, 백그라운드에서 TP팀 작업을 순차적으로 연계 진행함. (유력 이슈 상황 예방)

### 3.2. 알림톡 정책 (유/무료 전환 요청 알림)
* 현재 설치 및 제거 요청 시에는 알림톡 발송 정책이 존재하나, 무료 ↔ 유료 전환 요청 케이스에 대한 템플릿 및 정책이 미비한 상태임.
* **최소 필수 항목 확정**: `[유료 전환 성공]` 알림톡 템플릿 우선 구축.
  * 템플릿 문구: *"유료 전환에 성공했어요! 이제부터 더욱 강력한 유료 버전을 사용해 보세요!"*
* **추가 고려 시나리오**: 다운그레이드 케이스 시 알림톡 발송 여부, 등급별 분기 발송 조건 등을 검토 예정 (규백 PO 판단).
* **후속**: 코어 PO와 위성스쿼드 PD가 문구를 공동 결정한 뒤, 동혁님이 시스템에 최종 등록함.

### 3.3. 코드 구조 개선 제안
* **제안 내용**: `Cafe24WebhookEvent__Type` 스펙을 공통 규격인 `QUEUE_MESSAGE__TYPE`으로 명칭 및 명세를 통일하는 구조 제안 (동혁 ➔ 민수).
* **스펙 규격**:
  * 요구 `group_id` = `<mall_id>:<QUEUE_MESSAGE__TYPE>` 형태로 정의.
  * 코어팀 내부 논의 후 내일까지 민수님에게 피드백 공유 필요.

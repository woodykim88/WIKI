# AI 추천탭 개발 방향성 미팅록

**일시:** 2026-05-20  
**참여:** 리뷰 개발팀, 기획, (민수님, 태성님 언급)

---

## 배경 및 논의 출발점

### 논의 중이던 두 가지 안

| 구분 | 내용 |
|------|------|
| **1안** | AI 추천이라는 **별도 탭**을 두고, 자동 정렬된 리뷰 목록을 표시 |
| **2안** | 대표님 제안. **위젯 속 위젯(미젯 in 미젯)** 구조. 기존 게시판 위젯 안에 "추천 리뷰" 슬롯이 고정 노출되는 형태 |

지난 회의까지는 2안 구조를 기준으로 세부 정책을 수립 중이었음.

---

## 핵심 쟁점

### 쟁점 1. 2안의 기술적 문제 (민수님 제기)

- 위젯을 불러올 때 **상단 보정(상단 고정)** 로직이 이미 적용되어 있음
- 위젯마다 옵션(포토리뷰 포함 여부 등)이 다르게 적용됨
- 여기에 AI 추천 영역까지 추가 호출하면 **로딩 지연 + 비용 증가** 우려
- 결론: 2안은 구현 난이도가 1안보다 **유의미하게 높음**

### 쟁점 2. 1안(AI 추천 탭)의 정렬 정책 구체화

AI 추천 탭에서의 정렬 기준 (미팅 중 논의된 방향):

```
① 유니스코어 4점 이상  →  상단 (자동 보정과 동일 효과, 고점수순 정렬)
② 4점 미만 일반 리뷰  →  중간 (최신순)
③ 유니스코어 1점대    →  하단 고정 (역순 배치)
```

- **상단 보정 슬롯 노출 없음**: AI 추천 탭에서는 상단 고정된 리뷰를 별도 슬롯으로 강조 표시하지 않음
- **세 개만 표시 옵션**: 4점 이상 리뷰 중 N개만 먼저 보이는 설정 가능 (정책 결정 사항)

> **핵심 발견:** 1안에서도 "4점 이상 먼저, 상위 N개만 표시"로 가면  
> 사용자 경험 측면에서 **1안과 2안의 차이가 거의 없어짐**

### 쟁점 3. 구현 방식 논쟁 (개발 관점)

**개발자 A (1안 지지)**
- 기존 위젯 최적화 로직은 그대로 두고, AI 추천 리뷰만 가져오는 별도 API를 끼워 넣는 방식
- 1안과 2안의 구현 난이도 차이가 크지 않다고 봄

**개발자 B (2안 난이도 우려)**
- 상단 보정이 적용된 모든 위젯 API를 수정해야 함
- 포토리뷰 등 위젯별 조건도 모두 처리해야 함 → 개발 공수 현저히 높음

**모듈화 제안 → 결론: 현실적으로 어려움**
- 위젯마다 옵션이 얽혀 있고, 중복 리뷰 노출 방지 정책까지 고려하면 완전한 분리가 어려움

### 쟁점 4. 위젯 확장성 이슈 (포토리뷰 등)

- 게시판 위젯에는 탭(최신순/추천순)이 있어 사용자가 선택 가능
- **포토리뷰, 미니리뷰 등 다른 위젯에는 정렬 탭 UI가 없음**
- AI 추천순을 디폴트 정렬로 쓰지 않으면, 탭 없는 위젯에서는 AI 추천이 사실상 작동하지 않음

| 해결 방법 | 결론 |
|----------|------|
| 탭 UI 추가 | 포토리뷰 등엔 탭 자체가 어색 |
| 위젯 설정에서 기본값 변경 | **현실적인 방향으로 수렴** |
| 모든 위젯 기본값 = AI 추천순 | 최종 논의 방향 |

## 추가 논의 (미팅 후반)

### 쟁점 5. 수동 상단고정을 AI 추천 탭에 포함할 것인가

**문제 제기**
- 관리자가 직접 지정한 수동 상단고정 리뷰를, AI 추천 탭에서도 최상단에 보여줄 것인가
- 현재 정렬 스킴(①~④)에 수동 고정이 명시되어 있지 않음

**논의 내용**
- 포함 시: 관리자 의도가 반영되어 실무 수용성 높아짐. 단, "AI 추천"이라는 브랜딩과 충돌
- 제외 시: 순수 AI 정렬로 일관성 확보. 단, 이벤트/캠페인 리뷰 등 관리자 니즈 반영 불가

---

### 쟁점 6. 4단계 정렬 스킴의 복잡도 문제

**현행 스킴 구조적 문제 지적**

기준이 세 가지(관리자 의지 / 점수 임계값 / 시간)가 혼재되어 있음:

| 레이어 | 기준 | 누가 결정 |
|--------|------|----------|
| ① 수동 상단고정 | 관리자 의지 | 사람 |
| ② 4점↑ 상단 | 점수 임계값 | 규칙 |
| ③ 최신순 | 시간 | 시스템 |
| ④ 1점대 하단 | 점수 임계값 | 규칙 |

- "AI 추천"이라는 탭 이름과 실제 로직이 불일치 (사실상 규칙 기반 + 수동 개입)
- 중간 구간(③)이 품질 공백: 3점짜리 최신 리뷰가 5점짜리 오래된 리뷰 위에 올 수 있음
- 수동 고정이 유지되면 자동화 목적이 반감되고, 고객사 설명이 어려워짐

---

### 쟁점 7. 핵심 딜레마: 순수 스코어 추천 vs 자동정렬(관리자 고정 포함)

> **"AI 추천순이 디폴트 정렬로 어디서든 쓰여야 임팩트가 크다.  
> 그런데 관리자 의도를 반영 못하면, 디폴트로 쓸 수 없는 위젯이 생긴다."**

#### 순수 스코어 추천 (관리자 고정 없음)

| | 내용 |
|--|------|
| **장점** | AI 추천 브랜딩과 완전히 일치. 단순 로직으로 설명 용이. 고객 신뢰도 높음. 조작 불가 |
| **단점** | 이벤트/캠페인 리뷰 Featured 불가. 관리자 저항 → 디폴트 채택 어려울 수 있음 → **임팩트 반감** |

#### 자동정렬 (관리자 수동 고정 포함)

| | 내용 |
|--|------|
| **장점** | 관리자 의도 반영 → 디폴트 채택 가능성 높음. 더 넓은 위젯에서 디폴트 사용 → **임팩트 최대화** |
| **단점** | AI 추천 브랜딩 순수성 훼손. 복잡한 정렬 로직. 관리자가 저품질 리뷰를 고정할 경우 소비자 신뢰 리스크 |

## 종합 의견 및 권장 방향

### 핵심 판단 기준

> AI 추천탭의 목적은 "순수한 AI"가 아니라 **"운영 효율화 + 좋은 리뷰 노출 자동화"** 다.  
> 따라서 **디폴트로 얼마나 넓게 쓰이느냐**가 임팩트를 결정한다.

### 권장안: 자동정렬 채택 + 관리자 고정 제한적 허용

**정렬 순서 (단순화)**
```
① 관리자 수동 고정 (최대 3개 이내로 제한)
② 유니스코어 기반 내림차순 (임계값 구분 없이 단일 기준)
③ 유니스코어 일정 점수 미만 → 하단 배치
```

**핵심 설계 원칙 3가지**

1. **관리자 고정은 상한선을 둔다** (예: 최대 3개)  
   → 소수의 고정으로 "AI 추천" 경험의 대부분은 자동 유지

2. **고정 가능한 리뷰에 최소 품질 조건을 건다** (예: 유니스코어 3점 이상만 고정 허용)  
   → 저품질 리뷰를 관리자가 상단에 올리는 신뢰 리스크 차단

3. **임계값(4점) 구분을 제거하고 연속 정렬로 단순화**  
   → "4점 이상이냐 아니냐"의 단절 대신, 점수 순 연속 정렬로 중간 공백 해소

**이 방향의 이유**
- 관리자가 신뢰하는 정렬이어야 디폴트로 채택됨 → 채택 범위가 임팩트를 결정
- 수동 고정 리뷰 수는 실제로 1~3개로 적어 → AI 경험 훼손 최소
- 기술적으로 기존 상단고정 로직 재사용 가능 → 구현 부담 낮음

### 추가 고려: 투명성 확보 방안

- 수동 고정 리뷰에 "관리자 추천" 뱃지 노출 여부는 정책 결정 사항
- 단, 내부적으로는 수동 고정 리뷰를 로그로 관리해 어뷰징 모니터링 가능하게 설계

## 최종 결정사항

### 결정 1. AI 스코어 기반 상하단 자동정렬 (기본값 OFF)

- AI가 판단한 유니스코어 기준으로 **좋은 리뷰는 상단, 별로인 리뷰는 하단 고정** 가능
- **기본값은 OFF**, 관리자가 직접 ON 해서 사용하는 방식
- AI 추천순 탭 신설 등 복잡한 UX 변경 없이, 기존 정렬 설정 내에서 옵션으로 제공

### 결정 2. AI 자동정렬 프로젝트 홀드

- 오늘 논의된 1안(AI 추천 탭) 및 관련 정렬 스킴 전체 개발은 **홀드**
- 이유: 개발 공수 대비 비즈니스 임팩트가 아래 우선순위 기능보다 낮음

### 결정 3. 개발 로드맵 조정

| 시기 | 작업 | 비고 |
|------|------|------|
| **~2주 내** | 결정 1의 상하단 자동정렬 옵션 개발 완료 | 최소 공수로 진행 |
| **6월 내** | **AI 리뷰 요약 위젯** 개발 | 신규 프로젝트 착수 |

> **AI 리뷰 요약 위젯**: 상품에 달린 리뷰 전체를 AI가 분석해 핵심 내용을 요약문으로 제공하는 위젯.  
> 구매 전환율 직접 기여, 프리미엄 티어 과금 명분, 영업 경쟁력 확보 측면에서 자동정렬보다 우선순위 높음으로 판단.

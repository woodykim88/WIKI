# [[20260519_알파리뷰_인스타피드_데일리미팅]]

## 📌 한 줄 통찰 (The Karpathy Summary)

> 인스타피드 무료 구독 해지 UI 및 쿼터 제한 UI 셋업 R&R을 정의하고, 위젯 자동 설치 실패 시 TP팀 수동 리소스를 투입하는 대신 셀프 가이드 페이지 리디렉션 정책으로 노선을 통일한 데일리 스크럼.

## 📐 도메인 모델 개요 (Domain Model Overview)

| # | 도메인 | 핵심 개념 |
| - | ------------ | ------------------- |
| 1 | 해지 및 온보딩 프론트 개선 | 리뷰 작성 더보기 삭제, 해지 설문 조정 및 중간 리뷰 이관 페이지 과감한 삭제 R&R. |
| 2 | 소셜 리뷰 복수 상품 매핑 스펙 | 인스타 게시글 1개당 단건 연동 구조로의 기능 제약 스펙 확정. |
| 3 | 숏컷 랜딩 & 렌더링 애니메이션 | fade-in 애니메이션 설계 R&R 및 CSM 제공 셀프 가이드 링크 연계. |
| 4 | 자동 설치 감지 & 로그 가드레일 | 프론트 레벨 푸터 미탐지 식별 및 모니터링 분석용 스크립트 설치 로그 영구 보존(삭제 주기 연장). |

## 📖 핵심 도메인별 상세 (Domain Deep-Dive)

### 📖-1. 해지 및 온보딩 프론트 개선 (UX Refactoring)
- **리뷰 요청 알림톡 대응**: 알림톡이 이미 고객 액션 단위별 복수 건으로 개별 발송되므로 프론트의 `작성 가능한 리뷰 더보기` 링크는 불필요하여 삭제하기로 합의.
- **무료 구독 해지 플로우 축소**:
  - 해지 이탈율 방어용 `중간 리뷰 이관 페이지`는 사용자 경험 저하 유발 판단에 따라 과감히 **삭제**.
  - 해지 설문조사 항목 중 솔루션사 경쟁사 관련 민감 항목은 삭제 또는 변경하기로 결정 (프론트 단독 0.5일 리소스 예상).

### 📖-2. 소셜 리뷰 다중 상품 연동 제한
- **스펙 제약 (Anti-Complexity)**:
  - 하나의 인스타그램 게시글 원본이 다수의 쇼핑몰 등록 상품에 매핑되는 복수 연동 구조를 배제하고, **단건 상품 매핑 구조**로 기능을 스코핑하여 코어 시스템의 오버헤드를 예방.
- **쿼터 제한 제어**: 무료 플랜 쿼터 도달 시, 기능 자체는 유지하되 상단에 디자인 가이드 및 업그레이드 배너를 노출하는 방식으로 타협.

### 📖-3. 숏컷 랜딩페이지 모션 추가
- **첫인상 강화**: 무료 온보딩 성공 시 최초로 도달하는 숏컷 페이지에 로딩 스피너 및 첫 진입 시에만 작동하는 **fade-in 트랜지션 애니메이션**을 적용하여 고급스러운 사용 경험(Wow-factor) 제공.
- **R&R**: CSM 부서에서 다음 주까지 셀프 가이드 링크를 최종 인계하고 용협님이 관련 프론트 인수인계 작업을 마무리하기로 지정.

### 📖-4. 위젯 자동 설치 검수 및 로그 모니터링 가드레일
- **자동설치 실패 대응 정책**:
  - 만약 스크립트 엔진이 고객사의 쇼핑몰 메인 템플릿 혹은 푸터(Footer) 영역을 찾지 못해 자동설치가 실패하는 경우, **TP(기술지원)팀이 수동으로 붙여주는 리소스를 투입하지 않기로 결정.**
  - **대안**: 즉시 설치 실패를 UI 단에서 감지하여 `수동 설치 가이드 페이지`로 랜딩시키고 직접 설치 또는 유료 설치(결제)로 유도하도록 노선을 완전히 단순화함.
- **코어 시스템 로그 가드레일**:
  - 기존 2주 주기로 삭제되던 **스크립트 설치 로그 히스토리를 삭제하지 않고 영구 보관**하도록 DB 정책 변경. 배포 초기 정합성 검증 및 설치 성공률 트래킹을 위한 핵심 모니터링 리소스 확보 목적.

## ⚠️ 모순 및 업데이트 (Contradictions & RL Update)

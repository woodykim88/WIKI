# TL;DR

Rule Base 로 상위노출 되는 리뷰를 찾아내는 정확도는 57% 입니다. 이 정확도는 AI 를 이용해 개선하는 방법을 찾아보았습니가.

기존 연구 리뷰를 통해 **UniScore** 라는 단일 점수 체계로 리뷰 퀄리티를 숫자화하는 방법을 적용했고, 이때 8개의 척도를 LLM 이 평가하고 각 척도에 대해 점수를 기존 고객사가 설정해놓은 상단고정, 하단고정 리뷰를 학습하여 가중치를 적용하도록 했습니다.

비즈니스 이상의 플랜을 가진 고객사 중 상위/하위 노출 리뷰가 충분히 많은 32개 고객사를 대상으로 실험한 결과:

1. **상위 vs 하위 노출 리뷰 한 쌍을 비교했을 때 86% 의 확률로 정답 순서를 맞춥니다** (Cohen d 1.65, very large effect)
2. 미정렬(normal) 리뷰까지 포함한 3-class 분류 정확도는 **60%** 입니다.
    - 1번과 2번을 종합하면 — 본 방법론은 **양 끝(상위/하위) 구분은 명확히 잘하지만, 미정렬 리뷰와의 경계는 모호함** 을 알 수 있습니다 (normal 의 본질적 회색지대 특성).
3. 고객사 카테고리별 평균 가중치보다 **각 고객사마다 따로 학습된 가중치** 가 더 정확합니다.
4. AI 가 상위 노출로 분류한 (UniScore 높은) 리뷰의 점수와 **실제 인간이 정한 상위 노출 순서는 거의 무관** 합니다 (Cohen d 0.09).

이상의 인사이트를 기반으로 현재 방법론은:

- **상위/하위 노출 후보군 추천에는 탁월합니다**
- 그러나 후보군 안의 정확한 정렬 순서를 맞추는 데는 약합니다

→ 따라서 현재는 **AI 가 후보군만 추천하고, 추천된 후보군의 실제 정렬은 고객사가 직접 선택** 하는 분업 모델을 권장합니다.
→ 이는 운영 부담은 최소로 두고 (사람의 정렬 통제권 유지), 가장 큰 가치 (좋은 후보 발굴 자동화) 만 우선 확보하는 **가성비 좋은 첫 도입 형태** 입니다.

## 2. 실험설계

### 2.1 방법론

각 고객사가 이미 상단/하단 고정 해놓은 리뷰를 아래의 척도별로 점수화하고, 이 점수의 가중치를 계산하여 UniScore 라는 것을 만듭니다.

이 점수가 리뷰의 퀄리티 점수가 됩니다.

$$
\text{UniScore}(x^*) = \sum_{k=1}^m \tilde{w}_k \cdot \hat{s}_k(x^*)
$$

그런데 상단/하단 고정해둔 리뷰가 없는 업체는 어떻게 하느냐?

그런 업체들은 고객사가 취급하는 상품별로 카테고리에 기반한 카테고리 기반 가중치를 이용합니다.

- **AI 가 판단한 척도를 점수화하는 방법론 (JSD-derived AHP, 딥한 내용)**
    
    ## 1. 핵심 개념: UniScore (JSD-derived AHP)
    
    ### 1.1 직관적 설명
    
    “이 리뷰는 좋은 리뷰인가?” 를 평가하려면 보통 **여러 지표** (구체적인지, 진정성이 있는지, 정보가 있는지…) 를 함께 본다. 그런데:
    
    - **지표마다 중요도가 다르다** → 가중치 필요
    - **가중치를 사람이 직접 매기면 주관적·편향됨**
    - **단순 평균은 모든 지표가 동등하다고 가정** → 부정확
    
    UniScore 는 다음 흐름으로 해결한다:
    
    | 단계 | 작업 | 산출 |
    | --- | --- | --- |
    | 1 | 리뷰를 미리 알려진 두 그룹으로 나눔 (상단, 하단) | $G_{\text{low}}$, $G_{\text{high}}$ |
    | 2 | LLM 이 각 리뷰를 여러 지표로 1~5점 채점 (scorer 역할) | 점수 행렬 $S$ |
    | 3 | 두 그룹의 점수 분포 차이 측정 (JSD) — 차이 큰 지표일수록 변별력 ↑ | 변별력 $d_k$ |
    | 4 | 변별력 차이로 쌍대비교 행렬 구성 → AHP 주고유벡터 방법으로 가중치 산출 | 가중치 $w_k$ |
    | 5 | 부호 부착 (상단 평균이 높으면 +, 낮으면 −) | signed weight $\tilde{w}_k$ |
    
    ### 1.2 클래식 AHP 와의 차이 — LLM 의 위치
    
    본 방법은 AHP 의 **수학적 기계장치 (positive reciprocal matrix + 주고유벡터 + Consistency Ratio)** 를 그대로 적용하지만, 클래식 AHP 의 **판단 elicitation 방법 (다중 전문가 주관적 쌍대비교)** 은 사용하지 않는다.
    
    | 구성 요소 | 클래식 Saaty AHP | 본 분석 (UniScore) |
    | --- | --- | --- |
    | 쌍대비교 ratio 출처 | 사람 전문가 주관 (“A 가 B 보다 몇 배 중요?”) | JSD 통계량에서 기계적으로 산출 ($a_{ij} = 1 + 8\Delta_{ij}$) |
    | 다중 전문가 통합 | 기하평균 | 해당 없음 (LLM 은 쌍대비교 안 함) |
    | LLM 의 역할 | — | **LLM-as-scorer**: 개별 리뷰 × criterion 의 1~5 점수만 생성 |
    | Positive reciprocal matrix | ✓ | ✓ (Saaty 1~9 ratio 척도 보존) |
    | 주고유벡터 방법 | ✓ | ✓ |
    | Consistency Ratio | ✓ | ✓ |
    
    **핵심**: LLM 은 “criterion A 가 B 보다 중요한가?” 같은 judge 판단을 하지 않는다. LLM 의 출력은 리뷰별 raw score 이고, 그 score 의 **분포 통계(PMF → JSD)** 가 AHP 쌍대비교 자리에 대신 들어간다.
    
    ### 1.3 왜 이 방법을 쓰는가?
    
    | 대안 | 문제점 | UniScore 의 해결 |
    | --- | --- | --- |
    | **단순 평균** | 모든 지표 동등 취급 | 데이터 기반으로 가중치 자동 산출 |
    | **선형 회귀 가중치** | LLM 점수는 중간값에 몰리는 편향 (central tendency bias) → 회귀 가중치가 왜곡됨 | JSD 는 절댓값이 아닌 **분포 형태 차이** 만 보므로 편향 무관 |
    | **클래식 AHP (사람 전문가)** | 주관적, 1·9 척도 추정 어려움, 비용 큼 | JSD 분포 통계로 객관적으로 ratio 자동 산출 |
    | **LLM-as-judge (쌍대비교를 LLM 에 직접 질의)** | 모델 편향이 가중치에 직접 전파, central tendency 영향 | LLM 은 scoring 만 담당 → 가중치는 통계량 기반 |
    | **검은 상자 ML 모델** | 해석 불가능 | 가중치가 명시적으로 출력됨 (interpretable) |
    
    ---
    
    ## 2. 수학적 산식
    
    > **방법론 명명 재확인**: 아래 산식은 (a) 분포 차이를 정량화하는 **JSD distance**, (b) JSD 차이를 ratio 척도 쌍대비교 행렬로 변환하는 **difference mapping**, (c) AHP 의 **주고유벡터 방법**, (d) **Consistency Ratio** 검정으로 구성된다. (c)·(d) 는 클래식 AHP 의 핵심 수학이며 그대로 사용. 단, (a)·(b) 가 클래식 AHP 의 “사람 전문가 쌍대비교” 자리를 대체한다. 따라서 정확한 명칭은 **“Data-Driven AHP”** 또는 **“JSD-derived AHP”** (Bang 2026 의 UniScore 명명).
    > 
    
    ### 2.1 그룹 분할
    
    각 shop_id 의 리뷰 집합을 state 기준으로 분할:
    
    $$
    G_{\text{low}} = \{x_i \mid s_i = \text{"bottom"}\}, \quad G_{\text{high}} = \{x_i \mid s_i = \text{"top"}\}
    $$
    
    샘플 크기: $|G_{\text{low}}| = |G_{\text{high}}| = \min(\text{SAMPLES\_PER\_SIDE}, \text{available})$. SAMPLES_PER_SIDE=100, MIN_PER_SIDE=30.
    
    ### 2.2 LLM 채점 (LLM-as-Scorer)
    
    각 리뷰 $x_i$ 에 대해 모든 criterion $c_k$ 의 점수를 LLM 으로 채점:
    
    $$
    s_{ik} = \text{LLM}(P_k, x_i, \text{category}_i) \in \{1, 2, 3, 4, 5\}
    $$
    
    - LLM 의 역할은 “**scorer**” — 개별 리뷰의 criterion 점수만 산출. 쌍대비교 판단(judge) 은 하지 않음.
    - 8개 criterion 을 단일 프롬프트로 동시에 채점하여 비용 절감
    - temperature=0 으로 deterministic 출력
    - 모델: **Qwen-Flash** (DashScope OpenAI 호환 엔드포인트, Alibaba Cloud Model Studio 유료 plan)
    - CONCURRENCY=15 (429 rate limit 트레이드오프)
    
    ### 2.3 분포 추정 (PMF)
    
    각 그룹 $G_g$, criterion $c_k$, Likert 값 $v \in \{1,2,3,4,5\}$ 에 대해:
    
    $$
    n_k^g(v) = \sum_{i \in G_g} \mathbf{1}[s_{ik} = v]
    $$
    
    Laplace smoothing $\epsilon = 10^{-6}$ 적용:
    
    $$
    P_k^g(v) = \frac{n_k^g(v) + \epsilon}{\sum_{u \in \mathcal{L}} (n_k^g(u) + \epsilon)}
    $$
    
    ### 2.4 Jensen-Shannon Distance
    
    $$
    \text{JSDiv}(P \,\|\, Q) = \tfrac{1}{2} D_{\text{KL}}(P \,\|\, M) + \tfrac{1}{2} D_{\text{KL}}(Q \,\|\, M), \quad M = \tfrac{1}{2}(P + Q)
    $$
    
    $$
    d_k = \sqrt{\text{JSDiv}(P_k^{\text{low}} \,\|\, P_k^{\text{high}})} \in [0, 1]
    $$
    
    ### 2.5 Pairwise Comparison Matrix (JSD difference mapping)
    
    $$
    \Delta_{ij} = d_i - d_j \in [-1, 1]
    $$
    
    $$
    a_{ij} = \begin{cases} 1 + 8\Delta_{ij} & \text{if } \Delta_{ij} \geq 0 \\ \frac{1}{1 + 8|\Delta_{ij}|} & \text{if } \Delta_{ij} < 0 \end{cases}
    $$
    
    ### 2.6 주고유벡터 → 정규화 가중치 (AHP 본체)
    
    $$
    A \mathbf{v} = \lambda_{\max} \mathbf{v}, \quad \mathbf{w} = \frac{\mathbf{v}}{\mathbf{1}^\top \mathbf{v}}, \quad \sum_{k=1}^m w_k = 1
    $$
    
    ### 2.7 Sign 부착 → Signed Weight
    
    $$
    \text{sign}_k = \text{sign}(\bar{s}_k^{\text{high}} - \bar{s}_k^{\text{low}}), \quad \tilde{w}_k = \text{sign}_k \cdot w_k
    $$
    
    ### 2.8 Consistency Ratio
    
    $$
    \text{CR} = \frac{(\lambda_{\max} - n)/(n-1)}{\text{RI}_n}
    $$
    
    본 분석 모든 shop **CR < 0.024** — 매우 안정적.
    
    ### 2.9 추론: UniScore
    
    $$
    \text{UniScore}(x^*) = \sum_{k=1}^m \tilde{w}_k \cdot \hat{s}_k(x^*)
    $$
    
    ### 2.10 카테고리 평균 가중치 (운영 적용)
    
    shop-specific 가중치를 카테고리로 평균 → 일반화:
    
    $$
    \bar{w}_k^{cat} = \frac{1}{|S_{cat}|} \sum_{s \in S_{cat}} \tilde{w}_k^s
    $$
    

### 2.2 주요가설

이번 실험을 통해

1. 알파리뷰 고객사마다 리뷰를 상하단 고정하는 매커니즘이 다를 것이다.
    1. 리뷰 상하단 고정 분류 패턴을 확인한다.
2. 알파리뷰 고객사에게 유효한 리뷰 퀄리티 판단 척도는 기존 연구와 다를 것이다.
    1. 북미와 대한민국은 문화가 많이 다르기 때문
    2. 다양한 척도를 실험하고 가장 명확하게 상하단 고정을 나누는 시그널이 있는 척도를 선별한다.
3. 가중치를 가진 UniScore 계산을 통해 리뷰 정렬순서를 `80% 이상` 정확하게 맞출 수 있을 것이다.
    1. 적어도 80%는 나와야 실제 쓸만하지 않을까?
    2. 그런데 인간이 큐레이션해둔 결과가 AI 가 큐레이션한 결과보다 객관적으로 정렬이 잘못되었다면 정렬순서 일치도가 낮게 나올 수도 있겠다.
4. AI 가 계산한 UniScore 가 높을 수록 더 상위에 노출되는 리뷰일 것이다.
- **최종적으로 선별한 8개의 척도:**

| 코드 | 한국어 명 | 타입 | 정의 |
| --- | --- | --- | --- |
| `expertise` | 전문성 | Likert 1~5 | 제품·도메인 지식 (성분·소재·사양·용어·비교) 이 드러나는 정도 |
| `specificity` | 구체성 | Likert 1~5 | 구체적 수치·사례·사용 상황·결과 묘사의 풍부함 |
| `polarity` | 감정 명확성 | Likert 1~5 | 만족/불만족 입장의 **분명함** (방향과 무관). “최악이에요”도 5점 |
| `consistency` | 내적 일관성 | Likert 1~5 | 리뷰 내부 평가가 모순 없이 논리적으로 이어지는지 |
| `sentiment_direction` | 감정 방향 | Likert 1~5 | 1=매우 부정, 3=중립, 5=매우 긍정. polarity 와 별개 |
| `buy_decision` | 행동 의도 | Likert 1~5 | 1=환불·후회, 5=강한 재구매·추천 의사. 감정과 분리된 행동 시그널 |
| `opinion_with_reason` | 근거 있는 의견 | bool | 단순 감상이 아닌 “왜” 가 함께 제시? |
| `improvement_desire` | 개선 요청 | bool | “이 부분 개선됐으면” 같은 **구체적·건설적** 비판 (단순 불만 아님) |
- **폐기한 척도 3개의 척도:**

### 2.4 리뷰 상/하단 고정 정확도 검증 방법

### 2.4.1 정확도 검증

분류 정확도 검증 고객사로 선정된 32개 고객사의 리뷰를 100개를 아래와 같은 비율로 샘플링합니다.

- 상단 노출 리뷰 30개 / 하단 노출 리뷰 30개 / 순서가 부여되지 않은 일반 리뷰 40개

이렇게 샘플링한 데이터에 대해사 아래의 지표를 산출하여 검증합니다.

1. **Accuracy(정확도)**: 각 state 를 얼마나 잘 맞추는지 평가합니다.
    
    Confusion Metrics 기반의 Accuracy 를 의미하며, UniScore 로 리뷰를 정렬하여 상위 30%가 상단정렬, 하위 30%가 하위 정렬이 된것으로 평가합니다.
    
    → “UniScore 를 통해 이산적인 차원에서 상위 노출, 하위 노출 리뷰를 얼마나 잘 구분할 수 있을까?” 에 대한 대답
    
2. **Spearmans r (순위 상관 계수)**: 두 변수의 **순위(rank)** 가 얼마나 일치하는지 −1 ~ +1 로 측정.
    
    → “UniScore 를 통해 연속적인 차원에서 상위 노출, 하위 노출 리뷰를 얼마나 잘 구분할 수 있을까?” 에 대한 대답
    

$$
\text{Spearman r} = \text{corr}(\text{rank}(X), \text{rank}(Y))
$$

정확도 검증에서는:

$$
\text{Spearman r} = \text{corr}(\text{rank}(\text{UniScore}), \text{rank}(\text{state}))
$$

여기서 state 는 ordinal: 하단 정렬=0 < 미정렬=1 < 상단 정렬=2

|  | r |
| --- | --- |
| ≥ 0.7 | 매우 강함 |
| 0.5 ~ 0.7 | 강함 |
| 0.3 ~ 0.5 | 중간 |
| 0.1 ~ 0.3 | 약함 |
| < 0.1 | 무관 |
1. **Pearson r (상관 계수)**: 리뷰가 정렬되는 sort 값과 UniScore 의 정렬강도 측정

→ “UniScore 를 통해 상위 노출 리뷰의 정렬 순서를 잘 맞출 수 있을까?” 에 대한 대답

| 양 끝 (top↔︎bottom) 정반대 오분류 | — | — | **0건** |

→ UniScore 가중치는 한 모델로 학습해도 다른 모델로 추론 가능. 운영에 어느 모델을 써도 결론 동일.

다만 criterion 별로는 모델별 채점 편향이 존재합니다 (예: Haiku 가 polarity·opinion_with_reason·expertise 에는 박하게, consistency 에는 후하게 평가). 그러나 가중 합산(UniScore) 후에는 상쇄되어 최종 결과는 안정적.

### ⭐ 3.1.4 UniScore 와 상위 노출 순서 (sort) 의 상관관계

**가설 4 검증**: “AI 가 계산한 UniScore 가 높을수록 더 상위에 노출되는 리뷰일 것이다”

각 shop 의 top 리뷰에는 sort 값 (큰 값 = 더 위 노출) 이 부여되어 있습니다. 검증 셋의 top 리뷰 954건 (32 shop × 30) 에 sort 값을 매핑한 후 UniScore 와의 상관을 다양한 각도에서 측정.

### ⭐ 3.1.5 노출 알고리즘의 “이중 구조” 발견

이상의 결과를 종합하면 노출 알고리즘은 **두 단계로 분리**됨이 확인됩니다:

| 단계 | 측정 | 결과 | 의미 |
| --- | --- | --- | --- |
| ① **Coarse** (3-class 분류) | 3-class accuracy | 59.3% (random 33% 대비 +26%p) | 상위 vs 일반, 일반 vs 하단 분류는 잘 못한다. |
| ② **Mid** (pair-wise) | top > bottom 순서 | **86.2%** | 상위 노출 과 하위 노출을 분리하는건 86%의 정확도를 가진다. |
| ③ **Fine** (within-top 순위) | sort vs UniScore | **Cohen d 0.094** | 상위 노출 내에서 정렬순서를 맞추는건 못하낟. |

**해석**: 노출 알고리즘 = 이중 구조

```
┌──────────────────────────────────────────┐
│ Stage 1: top 후보 선정                    │
│   (top vs not-top)                        │
│   → UniScore criteria 와 강한 일치        │
│   → Cohen d 1.65, pair-wise 86%          │
│   → "어떤 리뷰가 큐레이션 영역에 들어가나"  │
└──────────────────────────────────────────┘
              ↓
┌──────────────────────────────────────────┐
│ Stage 2: top 안의 미세 순서              │
│   (sort 1, 2, 3, ...)                     │
│   → UniScore 와 거의 무관 (Cohen d 0.09) │
│   → ❓ 시간·클릭·공감·수동 배치 등 다른 신호│
└──────────────────────────────────────────┘
```

→ AI 는 **top 후보 발굴까지는 강력**, **최종 순서까지 대체하려면 추가 신호 필요**.

### 3.2 가설의 입증

1. **[부분 기각]** 알파리뷰 고객사마다 리뷰를 상하단 고정하는 매커니즘이 다를 것이다.
    1. 일부 건강기능식품, 신발과 같이 사실위주이 정보 전달이 중요한 카테고리는 있지만 **대부분의 카테고리에서 “일관된 긍정신호” 를 보내는 리뷰가 일반적으로 상위 노출될 가능성이 높았습니다.**
    2. 하지만 사실위주의 정보전달이 더 필요한 업체도 있기 때문에 AI 페르소나를 분리하는 방법도 가능할것으로 보입니다.
    3. **일부 공통 Factor 는 존재하지만 각 고객사마다 선호하는 리뷰의 패턴이 다른것은 사실이므로, AI 모델이 활용하는 가중치를 각 모델마다 따로 이용하는게 적절할것으로 보임**
2. **[채택]** 알파리뷰 고객사에게 유효한 리뷰 퀄리티 판단 척도는 기존 연구와 다를 것이다.
    1. 기존 연구에서는 word_count 가 중요한 변수였고 , sentiment_direction 이라는 변수는 존재하지 않았습니다. 하지만 유용한 리뷰가 아니라 고객사의 취향에 맞는 리뷰는 sentiment_direction 라는 변수가 너무 중요했고, word_count 의 가치는 높지 않았습니다.
3. **[부분 기각]** 가중치를 가진 UniScore 계산을 통해 리뷰 정렬순서를 80% 이상 정확하게 맞출 수 있을 것이다.
    1. 3-class accuracy 는 평균 **59.3%** 로 목표치 80% 미달. 다만 random baseline (33%) 대비 +26%p 개선.
    2. **그러나 정렬용 신호로는 매우 강함** — Cohen d 1.655 (very large effect), pair-wise top > bottom 86.2%.
    3. **shop 별 편차가 매우 큼** — accuracy 0.33 (3530) ~ 0.89 (2184) 까지 약 2.7배 차이. 일부 shop (2184, 3588 등) 은 80%+ 달성.
    4. 80% 미달 원인 분석: ① normal 의 본질적 모호성으로 3-class 양자화가 제약, ② 일부 shop (43829, 50592) 의 큐레이션 자체가 객관 점수와 일치하지 않음.
    5. 사용 사례에 따라 재해석:
        - “정렬용 시그널” 로는 충분히 강함 (Cohen d 1.65, **pair-wise 86%**)
        - “정확한 분류” 로는 부족 (60%)
        - 운영에는 **연속 점수로 정렬 후 임계값 분류** 가 적합
4. **[부분 기각]** AI 가 계산한 UniScore 가 높을수록 더 상위에 노출되는 리뷰일 것이다.
    1. UniScore 와 sort (큰 값 = 위 노출) 의 상관: **shop 내 정규화 후 Pearson r = 0.038, Cohen d = 0.094**. 양의 방향이지만 매우 약함.
    2. **이중 구조 발견**: UniScore 는 **top 영역 선정** 에는 강하지만 (Cohen d 1.65), **top 안의 1위·2위 미세 순서** 는 거의 잡지 못함 (Cohen d 0.09).
    3. 결론: **“AI = top 후보 발굴, 사람·추가 신호 = 안쪽 정렬”** 의 분업 모델이 적합.
    4. shop 별 양상은 다양 — 6031 패션 (Spearman r 0.627) 은 우리 모델로 안쪽 순서까지 잡힘, 3588 뷰티 (Spearman r −0.329) 은 안쪽 순서가 다른 logic 으로 결정.

### ⭐ 3.3 기능 개발 핵심 인사이트

1. **개별 고객사 맞춤 모델이 정확도 최대** — 개별 고객사마다 상단고정 리뷰, 하단고정 리뷰를 직접 선택하게 하고 최적화된 AI 모델을 만드는게 가장 정확도가 높다.
    - 학습 데이터가 충분한 shop 은 자체 가중치, 부족한 shop 은 카테고리 평균 fallback
    - 카테고리 평균만 적용해도 정확도 +1.5%p 개선 + 신규 카테고리 +6.4%p 개선 확인
2. **연속 Score 정렬이 분류보다 강함** — 3-class 양자화 시 60% 정확도이지만, 연속 점수로 보면 Cohen d 1.65 의 매우 강한 분리력
    - 운영 정렬용으로는 raw UniScore 가 최선
    - 분류 라벨이 필요할 때만 임계값 적용
    - Threshold 산출은 percentile 기반 (calibrate.py)
3. **AI 의 강점 영역과 약점 영역이 명확** — 노출 알고리즘은 “이중 구조” 로 작동
    - **Stage 1 (top 후보 발굴)**: AI 강력 (Cohen d 1.65, pair-wise 86%)
    - **Stage 2 (top 안 미세 순서)**: AI 약함 (Cohen d 0.09, Spearman r 평균 0.04)
    - 운영 모델: **AI = 후보 발굴, 사람·추가 신호 = 최종 정렬** 의 분업이 합리적
4. **shop 단위 진단이 카테고리 단위보다 결정적** — 같은 카테고리 안에서도 큐레이션 효율 23배 차이
    - 뷰티/화장품 11개 shop 안에 displacement 3.3% (모범) ~ 76.7% (실패) 공존
    - 카테고리 단위 일률 정책 부적절, shop 단위 차등 도입이 합리적
5. **객관 점수 ≠ 사람 큐레이션 의 격차가 비즈니스 가치 영역**
    - 32 shop 평균 displacement ratio 33% — 현 top 의 33% 가 normal 풀의 더 좋은 후보로 교체 가능
    - shop 43829 같은 큐레이션 실패 케이스에 즉시 displaced champions 보고서 전달 가능
    - shop 50592 같은 알고리즘 반전 케이스는 운영 정책 인터뷰부터 필요
6. **Model-agnostic 검증 완료** — Qwen-Flash 와 Bedrock Haiku 4.5 의 prediction agreement 93.6%
    - 운영 시 LLM 공급자 lock-in 회피 가능
    - 비용·속도에 따라 자유롭게 모델 교체

### 4.2 **demoted_top 프로파일 — “감정-only 리뷰” 가 강등됨**

- displacement: 사람이 정렬한 리뷰가 AI 정렬 결과 원래 위치에 있지 않은 현상
- demoted_top: 원래 상위 노출 리뷰였지만 AI 분류결과 상위 노출에서 탈락함
- promoted_normal: 원래 미분류 리뷰였지만 AI 분류결과 상위 노출 리뷰로 올라감

| 지표 | demoted_top | kept_top | promoted_normal | (promoted − demoted) |
| --- | --- | --- | --- | --- |
| **buy_decision** | **3.42** | 4.59 | 4.36 | **+0.94** ★ |
| **polarity** | **3.92** | 4.86 | 4.83 | **+0.91** ★ |
| **sentiment_direction** | **3.98** | 4.89 | 4.86 | **+0.88** ★ |
| specificity | 1.53 | 2.87 | 2.18 | +0.65 |
| consistency | 3.80 | 4.59 | 4.39 | +0.59 |
| expertise | 1.83 | 2.75 | 2.31 | +0.48 |
| opinion_with_reason | 0.37 | 0.82 | 0.65 | +0.28 |
| improvement_desire | 0.01 | 0.02 | 0.02 | +0.01 |
- demoted_top 의 프로파일: 감정 표시는 있지만 (polarity 3.92, sd 3.98) 정보 빈약 (expertise 1.83, specificity 1.53) + 구매결정 도움 없음 (buy_decision 3.42). = “좋아요” 수준의 감정-only 리뷰.
- kept_top vs demoted_top 의 결정적 차이: specificity (+1.34), buy_decision (+1.17). 정보 풍부 + 구매결정 직결 두 축이 사람 큐레이션의 implicit 기준과 일치.

**즉, 현재 척도에서 AI 를 이용해서 정렬할 경우 단순히 긍정적인 리뷰보다 구체적으로 긍정적인 리뷰가 상위 노출이 될 확률이 올라간다는 뜻임**

# [[알파스킨(AlphaSkin) - 막힘 없는 커스텀 경험 구축]]

## 📌 한 줄 통찰 (The Karpathy Summary)
> 기존 수백만 원짜리 외주 제작에 의존하거나, 낡은 카페24 HTML 편집창(스마트디자인)에서 헤매던 초기 창업자에게, 앱 진입 시 3개의 질의응답만으로 메인 스펙을 자동 조립하고 노코드(No-code)로 수정하는 '원스톱 스킨 빌더 경험'을 제공한다.

## 📐 도메인 모델 개요 (Domain Model Overview)

| # | 도메인 | 핵심 로직 및 정책 |
|---|--------|-----------|
| 1 | 타겟 고객 | 카페24 베이스가 전혀 없는 쇼핑몰 최초 구축 창업자 ("무엇을 바꿀 때 무엇이 적용되는지 모르는" 계층) |
| 2 | 가치 제안 (USP) | ① 노코드 편집 ② 목적 맞춤형 프리셋 ③ 템플릿 무상 교체(1년) ④ 알파앱스 N개월 통합 제공 |
| 3 | 코어 액션 플로우 | 템플릿 구매(디자인센터) → 덫(스마트디자인 경고) → 알파스킨 앱 진입 → 온보딩(Q&A) → 노코드 에디터 → 라이브(적용) |
| 4 | No-Gos (개발 제외) | 기존 알파앱스(업셀/리뷰) 기능 중복 개발 금지, 과도한 자유도 설계 금지, 이전 버전 복구 등 고급 기능 생략 |
| 5 | 성공 지표 (Funnel) | 스킨 구매수 → 앱 진입률 → 온보딩 완료률 → 에디터 저장률 → 대표 디자인(Live) 적용률 |

## 📖 핵심 도메인별 상세 (Domain Deep-Dive)

### 📖-1. 타겟 유저 및 페인 포인트 (Pain Points)

- **상황:** 쇼핑몰 초보자는 스킨 구매 후에도 무지함으로 인해 무엇을 어디에 배치할지 모르며, 간단한 텍스트 하나를 수정하기 위해서도 HTML 구조를 파악해야 하는 장벽에 부딪힘.
- **기존 대안의 실패:** 외주 대행사에게 150만 원~300만 원의 비용을 들여 "깔끔하게 알아서 해주세요"라고 요청. 결국 조잡한 디자인과 낮은 만족도를 경험함.
- **해결 방안:** 쇼핑몰 기획을 고객이 직접 할 필요 없도록 목적에 맞는 '완성형 프리셋 섹션'들을 조립식으로 쥐어주고, HTML 접근 지식 없이 직관적인 UI 위젯 형태로 패치워크할 수 있도록 함.

---

### 📖-2. 유저 저니 맵 및 제품 스펙 (User Scenario)

#### Step 0. 스킨 구매 및 트래픽 하이재킹 (The Trap)
- **액션:** 카페24 디자인센터에서 템플릿 결제 후 샵에 스킨이 생성됨.
- **설계:** 스킨 자체를 수정하려 카페24 기본 수정기능인 '스마트디자인(HTML 편집창)'으로 진입하는 유저를 차단해야 함.
- **장치 로직:** 스마트디자인 진입 시 가장 먼저 노출되는 `index.html` 상단에 주석이나 강제 큰 텍스트로 *"디자인 수정은 알파스킨 노코드 빌더로 진행하세요. 앱을 설치하세요"* 라는 경고문을 삽입.

#### Step 1. 온보딩 마법사 (Auto-Assembly Onboarding)
- **액션:** 스킨 구매자가 '알파스킨' 앱에 최초 진입.
- **질의 스펙 (UX):**
  - `Q0.` 브랜드명은 무엇인가?
  - `Q1.` 메인 컬러 파레트 (프리셋 선택 지원)
  - `Q2.` 메인 화면 최상단 강제 노출 목적: [베스트셀러 / 리뷰 / 브랜드스토리 / 신상품] 중 택 1.
  - `Q3.` 전역 글꼴 속성: [깔끔 단정(고딕) / 고급(명조) / 트렌디(빅 타이포)]
- **시스템 동작:**
  - `Q1, Q3`: 모든 버튼, 뱃지, 타이틀의 전역 토큰(Global CSS Variable)에 실시간 주입.
  - `Q2`: 구매한 원본 스킨 내에 숨겨진 여러 컴포넌트 중 선택된 목적의 섹션을 '메인 히어로 배너' 바로 아래로 순서를 끌어올리고 `Active` 처리.

#### Step 2. UX 중심의 노코드 에디터 (No-Code Patchwork)
- **원칙:** 패널을 복잡하게 타고 들어가는 뎁스(Depth) 디자인을 극도로 지양. **미리보기 내에서 직접 컨트롤(Inline Editing)** 하는 방식.
- **동작 범위:**
  - 전역 디자인 요소 수정.
  - 로고 데이터 업로드, 배너 사진 교체.
  - 메인 텍스트 커스텀.
  - 필수 위젯 원클릭 Add/Remove.

#### Step 3. 템플릿 교체 및 사후 관리
- Typeform / Tally 폼으로 접수하여 시스템 개발 전까지 슬랙 인입을 통해 운영팀이 매뉴얼 대응. (교체 요청 트래픽이 임계점을 넘으면 Phase 2에서 백엔드 자동화 착수)

---

### 📖-3. 정책 및 No-Gos 제약 (Constraints)

초기 리소스 타격 방지 및 알파앱스 생태계 유입을 위한 강력한 Product Principle.

1. **과도한 자유도 제공 금지:** 
   - 사용자가 레이아웃 구조를 마음대로 분해하거나 마진/패딩 값을 픽셀 단위로 커스텀할 수 없음. QA 범위의 폭발을 방지하며, 정해진 텍스트/이미지만 갈아 끼울 수 있도록 통제.
2. **알파앱스 연계 기능 중복 개발 금지:** 
   - 스킨 자체에 리뷰 기능이나 업셀/객단가 상승 기능을 네이티브하게 박아 넣지 않음. 
   - 알파리뷰, 알파업셀, 알파푸시의 무료 체험 쿠폰과 기간을 지급하여 코어 앱 생태계 안에서 문제를 해결하도록 넛지.
3. **고급 에디터 기능 개발 후순위:** 
   - 실행 취소(Ctrl+Z), 이전 버전 단위 복구 등의 복잡한 State 관리 기능은 Phase 1 범위에서 탈락.

## ⚠️ 모순 및 업데이트 (Contradictions & RL Update)

- **[스킨 가격 vs 앱 프라이싱 BM 전환]** 초기에는 템플릿 구매 비용만 BM이 적용되며, 노코드 빌더 앱 이용에 대한 무료 기간 종료 후 정기구독(BM) 전환 허들 정책이 아직 추상적인 상태(Unknown).
- **[코어 샵 정보 동기화 타이밍]** 알파앱스 스토어가 아닌 카페24 디자인센터에서 다이렉트로 유입된 트래픽의 경우 해당 샵 데이터베이스의 회원정보 동기화 타이밍과 생성 로직(알파앱스 통합 API 연동 여부)에 대한 백엔드 FSD 분석이 추가 필요함.

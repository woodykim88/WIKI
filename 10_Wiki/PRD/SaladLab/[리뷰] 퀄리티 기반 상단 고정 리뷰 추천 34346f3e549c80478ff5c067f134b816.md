# 1. Problem

리뷰 퀄리티 점수 로직([[리뷰] 리뷰 퀄리티 판단로직 만들기](https://www.notion.so/34346f3e549c806fbf3edc03cbdf6f8c?pvs=21) )이 완성되면, 
각 리뷰에는 상/중/하 등급이 자동으로 부여된다. 
그러나 이 등급 정보가 관리자의 상단 고정 행동으로 연결되지 않으면 
**로직의 실질적인 가치가 발휘**되지 않는다.

## 핵심 문제

**리뷰가 쌓이는 속도를 관리자가 따라가지 못하고, 계속 리뷰를 일일히 읽어가며 좋은 리뷰와 부정적인 리뷰를 수동으로 관리해줘야한다.** 

알파리뷰를 사용하는 쇼핑몰 관리자 중 **1/4(데이터 필요)만이 매주 접속해 좋은 리뷰를 수동으로 선별**합니다. 
나머지 3/4는 알파리뷰에 접속하지 않아서 구매 전환의 기회를 놓치고 있는 상황입니다. 

## 구체적인 문제 상황

- **관리자 A (적극 사용자)**
    
    매주 알파리뷰에 접속하여 유입된 리뷰를 일일이 읽으며 양질 리뷰와 부정 리뷰를 수동으로 선별합니다. 상단고정을 적극적으로 활용하는 1,602개 업체 중 1위 업체(ASHROLEN)는 3,631개를 상단고정했을 정도로 매주 실질적인 수작업을 하고 있습니다. 이들에게는 수동 작업 시간을 줄여주는 것이 필요합니다.
    
- **관리자 B (비접속 사용자)**
    
    접속하지 않으므로 리뷰 진열에 아무런 관리가 이루어지지 않고 있습니다. 
    단순 등록순으로 노출되어 있어 구매 전환에 유리한 리뷰가 묻힙니다.

## PRD가 문제를 해결하는 방식

이번 PRD는 **접속하는 관리자의 수동 선별 부담**을 줄이는 것을 목표로 한다. 다음 단계인 자동 정렬 PRD가 완성되면, 한 번이라도 접속해서 ON을 켠 업체는 이후 접속 없이도 퀄리티 기반 노출이 유지된다. 두 단계가 함께 완성될 때 비접속 88.3% 업체 문제까지 커버할 수 있다.

## 경쟁 업체 동향

![스크린샷 2026-04-24 오전 11.38.15.png](%5B%EB%A6%AC%EB%B7%B0%5D%20%ED%80%84%EB%A6%AC%ED%8B%B0%20%EA%B8%B0%EB%B0%98%20%EC%83%81%EB%8B%A8%20%EA%B3%A0%EC%A0%95%20%EB%A6%AC%EB%B7%B0%20%EC%B6%94%EC%B2%9C/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2026-04-24_%E1%84%8B%E1%85%A9%E1%84%8C%E1%85%A5%E1%86%AB_11.38.15.png)

크리마는 2025년 12월 'AI 리뷰 매니저'를 정식 출시했다. 딥러닝으로 리뷰 본문을 분석해 우수 리뷰를 자동 선별하고, 원클릭 상단 고정까지 연결하는 구조다.

- 신규 리뷰 중 AI가 좋은 리뷰를 골라서 보여줌
- 리뷰마다 AI 추천 점수를 수치로 표시
- 업계 평균 대비 우수 리뷰 비율을 벤치마킹해서 보여줌
- "추천 리뷰 지정" 버튼으로 원클릭 상단 고정 연결

**→ 알파리뷰와 구조가 거의 동일하다. 차이는 크리마는 리뷰 텍스트를 AI가 직접 읽어서 판단하고, 알파리뷰는 텍스트 길이·포토 수 같은 형식 요소로 판단하는 룰베이스 방식이다.**

## 경쟁 업체 대비 알파리뷰의 차별화 포인트

크리마의 분석 대상은 최근 90일 이내 신규 리뷰로 제한된다.

관리자 입장에서 설명하면 이렇다. 쇼핑몰을 운영하다 보면 한 달 전, 두 달 전에 작성된 리뷰가 오히려 더 상세하고 믿음직스러운 경우가 많다. 실제로 데이터를 보면 관리자들은 53~127일 된 리뷰를 가장 앞에 고정하는 패턴을 보인다. "최신 리뷰 중 좋은 것"이 아니라 **"전체 리뷰 중 가장 좋은 것"** 을 원하는 것이다.

알파리뷰는 전체 리뷰 풀을 대상으로 추천하고, 여기서 한 발 더 나아가 **"지금 1위에 고정된 리뷰보다 더 좋은 리뷰가 생기면 교체를 제안"** 한다. 크리마는 새로 고정할 리뷰를 추천하는 데서 끝나지만, 알파리뷰는 현재 고정된 것과 더 좋은 후보를 나란히 보여주며 교체 여부를 관리자가 직접 판단하게 한다.

# 3. Solution

## 핵심 아이디어

- 퀄리티 점수(상/중/하 등급 + 최신순)를 기반으로, 관리자 어드민 내 별도 섹션에 
'상단 고정 추천 리뷰'를 모아서 보여준다.
- 관리자는 추천 리뷰를 보고 원하는 리뷰를 직접 상단 고정하면 된다.
- 시스템은 제안만 하고, 최종 결정은 관리자가 한다.
- 추천 섹션은 서비스 차원에서 항상 노출된다.

## 사용자 시나리오

### 관리자 A — 매주 접속하는 적극 사용자

> "오늘도 리뷰 관리해야지" 하고 월요일 오전에 어드민에 접속한다.
지난주에 새로 들어온 리뷰가 47개다.
어떤 리뷰를 상단에 올릴지 판단하려면 47개를 하나씩 열어서 읽어야 한다.
포토가 있는지, 내용이 충분한지, 부정적인 내용은 없는지 눈으로 확인한다.
30분쯤 지나서 괜찮아 보이는 리뷰 3개를 골라 상단 고정한다.
이 작업을 매주 반복한다.
> 

**문제:** 리뷰가 많아질수록 읽어야 할 양이 늘어난다.
어떤 기준으로 골라야 하는지도 명확하지 않아서, 매번 감으로 판단한다.
이미 상단에 고정된 리뷰보다 더 좋은 리뷰가 들어와도 모르고 지나치는 경우가 생긴다.

### 관리자 B — 접속하지 않는 비활성 사용자

> 알파리뷰를 도입했지만 어드민에 잘 들어오지 않는다.
리뷰가 쌓이고 있다는 건 알지만, 뭘 어떻게 해야 할지 모른다.
위젯은 기본 설정인 등록순으로 노출되고 있다.
최근 들어온 리뷰 중 포토도 있고 내용도 좋은 리뷰가 몇 개 있지만,
2년 전에 작성된 짧은 리뷰들 뒤에 묻혀 소비자 눈에 잘 띄지 않는다.
> 

**문제:** 접속하지 않으면 좋은 리뷰가 쌓여도 활용되지 않는다.
리뷰 관리를 시작하려 해도 어디서부터 손을 대야 할지 몰라 진입 장벽이 높다.

**지금 이 소비자가 경험한 것:** 상단에 노출된 리뷰가 좋았기 때문에 구매 결정이 빨라졌다.

> 비슷한 상황에서 다른 쇼핑몰에 들어간다.
리뷰가 180개인데, 맨 위 리뷰가 "배송 빠름"이라는 한 줄짜리다.
두 번째는 별점만 있고 내용이 없다.
세 번째는 2년 전에 작성된 리뷰다.
좋은 리뷰가 있는지 찾아보려고 스크롤을 내린다.
찾기 귀찮아서 그냥 페이지를 닫는다.
> 

**지금 이 소비자가 경험한 것:** 리뷰는 많았지만 좋은 리뷰를 찾지 못해서 구매를 포기했다.

### 정렬 처리 방식 (26.04.21 회의 결정사항 반영)

기존 단건 요청 방식에서 **벌크(bulk) 처리 + 반영 버튼** 방식으로 변경.

- 프론트에서 정렬을 완료한 후 **"반영" 버튼 한 번**으로 일괄 처리
- DB 커넥션 횟수 감소, 타임아웃 위험 감소
- 단건 처리는 **최상단/최하단 이동**만 유지
- 현재 단건 요청은 동시에 여러 변경 발생 시 정렬이 꼬이고, Datadog 기준 수 초 이상 소요되는 성능 이슈 확인됨

# 4. 인수 조건 (Acceptance Criteria)

> 아래 조건이 충족되면 운영 환경 배포 가능한 것으로 본다.
> 
- [ ]  관리자 어드민에 '상단 고정 추천 리뷰' 섹션(=드로어)이 별도로 노출된다.
- [ ]  기존 리뷰 목록 페이지에서 신규 페이지로 진입하는 UX가 추가된다.
- [ ]  추천 리뷰는 `상 등급 → 중 등급 순, 같은 등급 내 최신순`으로 표시된다.
- [ ]  상단 고정된 리뷰 전체(추천 경유 + 직접 고정 모두)에 '쇼핑몰 추천 리뷰' 뱃지가 자동 노출된다. (관리자가 뱃지 설정에서 ON으로 활성화한 경우에 한함)
- [ ]  이미 상단 고정된 리뷰는 추천 목록에 노출되지 않는다.
- [ ]  삭제·비노출 리뷰는 추천 목록에 노출되지 않는다.
- [ ]  추천 섹션에서 상단 고정 버튼을 누르면 해당 리뷰가 상단 고정 목록에 추가된다.
- [ ]  추천 가능한 리뷰(상/중 등급)가 없을 경우 '추천할 리뷰가 없습니다' 메시지가 노출된다.
- [ ]  추천 섹션은 쇼핑몰 위젯의 리뷰 노출 순서(등급 기반 정렬)에 영향을 주지 않는다.
- [ ]  현재 상단 고정 상위 리뷰보다 높은 등급의 리뷰가 존재할 경우,
추천 섹션에 교체 추천 카드가 노출된다.
- [ ]  상단 고정 목록에서 순서 변경은 프론트에서 정렬 완료 후 "반영" 버튼 클릭 시 벌크로 처리된다.
- [ ]  최상단/최하단 이동은 단건 즉시 처리된다.
- [ ]  상단 고정 목록은 최대 25개 내외로 표시된다.
- [ ]  위젯 설정에서 상단고정 순서 반영이 꺼져있을 경우, 상단고정 관리 페이지 내에 경고 메시지와 위젯 설정 페이지 이동 링크가 노출된다. (26.05.13 추가)

## No-Gos (이번 버전 제외)

- 퀄리티 높은 리뷰 풀을 자동으로 롤링 노출하는 완전 자동화 → 자동 정렬 PRD에서 고려할 예정 (관리자가 선택한 상단 고정 리뷰와 충돌 없이 동작하는 설계가 선행되어야 함.)
- 카테고리별 추천 기준 세분화
- 관리자가 추천 기준을 직접 수정하는 기능
- 상품 간 리뷰 연결 상태에서 상품별 독립 상단 고정 적용
(현재 A상품 상단 고정 시 연결된 B상품에도 강제 고정되는 구조 개선은 이번 범위 제외)
- 뱃지 신규 개발 — 기존 '쇼핑몰 추천 리뷰' 뱃지 기능을 그대로 활용
- 슬롯 제한 기능 신규 개발 — 기존 위젯 설정 기능 활용.
관리자 인지 제고를 위한 안내 문구 추가만 이번 범위에 포함.
- 교체 추천 시 순위 지정 삽입 기능 (3위, 5위 등 특정 순위로 넣기) — 현재 상단 고정 구조 제약으로 이번 범위 제외.

# 5. 킥오프 전 개발자와 협의 필요한 항목

> PD가 정의한 러프한 솔루션을 기반으로, 아래 디테일은 개발자가 이어받아 구체화한다.

- [ ]  추천 갱신 시점: 신규 리뷰 등록 시 실시간 처리 vs 배치 처리 여부
- [ ]  **[미확정 ]** 상 등급 내 AI 세부 소팅 처리 방식 — 실시간 처리(리뷰 등록 시마다 즉시 분석) vs 주기 처리(배치, 여러 리뷰를 묶어 상대 평가) 중 어느 쪽이 정확도와 비용 면에서 유리한지 논문 검토 후 결정 예정.
- [ ]  추천 수용 피드백 수집 방식 — DB 테이블 vs S3 저장 vs 외부 도구 중 결정 필요. 별도 테이블을 파기보다 S3 저장 후 주기적 확인 방식이 비용 효율적일 수 있음.
- [ ]  추천 노출 최대 개수 (N): 데이터 기반으로 적정 수량 결정 필요
- [ ]  추천 카드에 노출할 정보 항목 (본문 미리보기 글자수, 포토 썸네일 여부 등)
- [ ]  벌크 처리 구현 방식 — 프론트 정렬 완료 후 반영 버튼 클릭 시 변경된 항목만 전송할지, 전체 순서를 전송할지 개발팀과 협의 필요
- [ ]  상단 고정 추가 시 기존 순서 처리 방식 — 추천 경유 고정은 등급 기반 자동 삽입(추천 리뷰 등급보다 낮은 등급이 처음 나오는 자리 앞에 삽입)으로 확정. 단, 현재 상단 고정 구조상 추가 시 맨 위로만 올라가는 기술적 제약 있어 구현 방식 개발팀과 협의 필요. 순위 지정 삽입(3위, 5위 등)은 이번 범위 제외.
- [ ]  동일 리뷰어가 여러 리뷰를 남긴 경우 중복 추천 처리 정책
- [ ]  슬롯 제한: 추천 경로로 상단 고정 가능한 최대 슬롯 수 정책 결정 필요
- [ ]  연결 상품 정보 표시 방식: 상품명 텍스트 나열 vs 링크 vs 툴팁 등 UI 방식 협의 필요
- [ ]  연결 상품 리뷰 제외 처리 시점: 연결 해제 후 다음 추천 갱신 시 반영할지, 즉시 반영할지
- [ ]  교체 추천 기준: 상위 몇 순위까지 비교할지 (1위만 vs 1~5위)
- [ ]  대용량 업체(3,000~3,500건) 대상 정렬 화면 표시 범위 세부 스펙 — 필터 조합 시 결과 건수가 달라지는 케이스 처리 방식 포함
- [ ]  위젯별 상단고정 순서 반영 설정 연동 방식 — 여러 위젯 중 어느 위젯 기준으로 설정 꺼짐 여부를 체크할지, 경고 메시지 노출 조건 및 위치 협의 필요. 위젯별 최대 노출 개수 설정값도 상단고정 관리 페이지에서 인지할 수 있도록 안내 방식 결정 필요.
- [ ]  상품별 상단 고정 페이지의 신규 페이지 통합 범위
- [ ]  **[미결] 신규 기능 유입 퍼널 방안** — 상단 고정 자체를 잘 활용하지 않는 업체가 많은 상황에서, 이 기능으로의 접속을 어떻게 유도할지 방안 필요. 알림톡 발송 / 1회성 툴팁 / 배너 등 옵션 중 결정 필요. (최규백님 코멘트 26.04.27)

# 7. FAQ

**Q. 관리자가 이미 상단 고정한 리뷰가 추천에도 뜨면?**
중복 제외. 이미 상단 고정된 리뷰는 추천 대상에서 자동 제외한다.

**Q. 추천 리뷰를 상단 고정하면 기존 상단 고정 순서는 어떻게 되나?**
A. 추천 리뷰의 등급에 맞는 위치에 자동 삽입된다. 현재 고정 목록에서 추천 리뷰 등급보다 낮은 등급이 처음 나오는 자리 앞에 들어가며, 동일 등급 내에서는 최신순을 유지한다. 고정 목록 전체가 동일 등급이거나 비어 있을 경우 맨 아래에 추가된다. 관리자가 이후 전용 페이지에서 순서를 직접 조정할 수 있다.

**Q. 하 등급 리뷰가 전부인 업체는 추천이 아예 안 뜨나?**
그렇다. 추천 가능한 상/중 등급 리뷰가 없으면 섹션에 '추천할 리뷰가 없습니다' 상태 메시지를 노출한다.

**Q. 추천 갱신은 얼마나 자주 되나?**
신규 리뷰 등록 시 실시간 반영, 재구매 점수 등 배치성 데이터는 일 1회 새벽 배치 후 갱신된다.

Q. 상단고정에 대한 내용을 위젯에서 노출하는 방식이 왜 뱃지 기능활용으로 되었을까요? 뱃지를 선택하신 이유와 기준과 보여주고 싶은 내용이 궁금합니다.

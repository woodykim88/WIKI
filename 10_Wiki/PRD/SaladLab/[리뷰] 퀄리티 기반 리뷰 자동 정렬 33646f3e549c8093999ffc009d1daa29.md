# 1. Problem

## 관리자 side

<aside>

퀄리티 점수 로직과 상단 고정 추천 기능이 완성되어도,
관리자가 추천을 보고 직접 고정하지 않으면 비활성 업체에는 아무 변화가 없다.

</aside>

실제로 최근 10일간 상단 고정을 전혀 관리하지 않은 업체는 전체의 **88.3%** 다.
이 업체들은 추천 기능이 생겨도 어드민에 접속하지 않으면 여전히 혜택을 받지 못한다.

> 💡 관리자가 한 번이라도 접속해서 자동 정렬을 켜두면,
이후 접속 없이도 퀄리티 기반으로 리뷰가 자동 노출된다.
이 단계가 완성될 때 비접속 88.3% 업체 문제까지 커버할 수 있다.
> 

## 엔드유저 side

<aside>

현재 상단 고정 리뷰는 모든 정렬 옵션(최신순/추천순/별점순)에서 최우선으로 노출된다. 
이 구조 때문에 최신순을 선택해도 오래된 고정 리뷰가 먼저 보이고, 별점순을 선택해도 별점과 무관한 리뷰가 상단에 뜬다. 정렬 옵션이 존재하지만 사용자의 탐색 의도가 실제로 반영되지 않는 셈이다. 

</aside>

- 정렬 옵션 간 차별성이 사라져 리뷰 영역 전반에 대한 신뢰도가 낮아지는 문제가 발생한다
- 리뷰 탐색 과정에서 기대한 기준(최신 / 평점 / 추천)에 맞는 정보를 빠르게 찾지 못한다.

> 💡 정렬 옵션은 사용자가 정보를 탐색하는 명확한 기준이 되어야 하지만, 현재 구조에서는 판매자(관리자)의 상단 고정 설정이 이를 덮어쓰면서 사용자 의도가 우선되지 못하는 문제가 발생하고 있다.
>

<aside>

8. 내가 직접 상단 고정해둔 리뷰, 또는 미리보기에서 직접 순서를 바꾼 리뷰는 그대로 최상단에 유지된다.

# 3. Solution (러프한 솔루션)

퀄리티 등급(상/중/하)을 기반으로 리뷰 노출 순서를 자동으로 정렬한다.
관리자가 ON/OFF로 자동정렬을 제어하고, 
소비자는 위젯에서 '추천순'을 클릭해, 
자동정렬된 결과를 확인할 수 있다.

어드민 온보딩 튜토리얼

> 알파리뷰 어드민에 처음 접속하는 신규 관리자는 
온보딩 튜토리얼을 통해 자동정렬 기능이 켜져 있음을 인지한다.
> 

**목적:** 기능 인지. 신규 업체 기본값이 ON이므로 "이미 켜져 있고, 이런 기능이에요"를 온보딩 시점에 전달한다. 관리자가 기능을 끄지 않고 유지할 수 있도록 정렬 기준과 효과를 함께 안내한다.

**위치:** 기존 온보딩 튜토리얼의 **권장 세팅** 영역에 스텝 추가

**스텝에서 노출할 정보:**

- 자동정렬이 ON 상태임을 시각적으로 확인
- 정렬 기준 안내 — 포토 포함 여부, 텍스트 길이, 재구매 여부 등 퀄리티 기준을 간결하게 설명
- 구매전환율 기여 메시지 — 좋은 리뷰가 먼저 노출될수록 구매 결정이 빨라진다는 맥락 전달
- 구체적인 카피와 디자인은 디자인 단계에서 확정

> 💡 온보딩 시점에는 아직 리뷰 이관이 완료되지 않은 상태이므로 실제 미리보기는 제공하지 않는다. 자동정렬 기준과 효과를 설명하는 것에 집중한다.
> 

**스킵 처리:**

- 관리자가 스텝을 스킵해도 신규 업체 기본값 ON은 유지된다.
- 스킵한 업체는 이후 어드민 내 자동정렬 토글에서 언제든 상태를 확인하고 변경할 수 있다.

## 자동 정렬 로직

자동정렬 ON 상태에서 리뷰 노출 순서는 **"등급 내 최신순"**으로 동작
즉, 최신 리뷰라도 등급이 낮으면 상위 등급의 오래된 리뷰보다 아래에 노출

ex. 예시 (상단고정 리뷰 없을때, 리뷰 정렬):

# 4. 인수 조건 (Acceptance Criteria)

## 방향 1 — 어드민 자동 정렬 ON/OFF

**위치:** 상단 고정 리뷰 목록 페이지 내 토글

**기본값:**

- 기존 업체: OFF (기존 정렬 방식 유지, 강제 변경 없음)
- 신규 업체: ON (처음부터 퀄리티 기반 정렬 경험)

**신규 업체란? →** 현재 라이브 대기 중인 업체들

**동작 방식:**

- ON 토글 클릭 시 바로 반영되지 않고, 자동 정렬이 적용된 **미리보기** 화면이 먼저 노출된다.
- 미리보기 상태에서 관리자는 특정 리뷰의 순서를 직접 변경할 수 있다.
    - 순서를 변경한 리뷰는 **수동 상단고정으로 저장**되며, 이후 자동 정렬 갱신 시 해당 리뷰의 순서는 변경되지 않는다.
    - 자동 정렬은 수동 고정되지 않은 리뷰 영역에서만 작동한다.
- 미리보기에서 '반영하기'를 누르면 위젯에 실제 적용된다.
- 미리보기에서 '취소하기'를 누르면 ON 이전 상태로 돌아간다.
- 반영 후 **1회 되돌리기**가 가능하다. 되돌리기 시 반영 직전 상태로 복귀한다.
- OFF: 기존 정렬 방식 유지 (등록순 또는 관리자 설정 정렬)
- 관리자가 수동으로 상단 고정한 리뷰는 ON/OFF 무관하게 항상 최상단 유지

> ⚠️ 미리보기에서 순서를 변경한 리뷰가 누적될수록, 자동 정렬이 실질적으로 작동하는 영역이 줄어들 수 있다. 수동 고정 리뷰가 많아지는 패턴은 킥오프 이후 모니터링 지표로 추가 검토가 필요하다.
> 

### **자동 정렬 ON/OFF 인수 조건:**

- [ ]  상단 고정 리뷰 목록 페이지에 자동 정렬 ON/OFF 토글이 노출된다.
- [ ]  기존 업체의 기본값은 OFF다.
- [ ]  신규 업체의 기본값은 ON이다.
- [ ]  ON 토글 클릭 시 미리보기 화면이 노출된다. 위젯에 즉시 반영되지 않는다.
- [ ]  미리보기 상태에서 관리자가 리뷰 순서를 변경할 수 있다.
- [ ]  미리보기에서 순서를 변경한 리뷰는 수동 상단고정으로 저장된다.
- [ ]  미리보기에서 '반영하기'를 누르면 위젯에 실제 적용된다.
- [ ]  미리보기에서 '취소하기'를 누르면 ON 이전 상태로 복귀한다.
- [ ]  반영 후 1회 되돌리기가 가능하다. 되돌리기 시 반영 직전 상태로 복귀한다.
- [ ]  ON 상태에서 위젯의 리뷰 노출 순서가 퀄리티 등급 기반으로 정렬된다.
- [ ]  OFF 상태에서 기존 정렬 방식이 유지된다.
- [ ]  ON/OFF 상태와 무관하게 관리자가 수동으로 상단 고정한 리뷰는 항상 최상단에 노출된다.

### **온보딩 튜토리얼 인수 조건:**

- [ ]  온보딩 튜토리얼 권장 세팅 영역에 자동정렬 안내 스텝이 추가된다.
- [ ]  스텝에서 자동정렬 ON 상태, 정렬 기준(포토, 텍스트 길이, 재구매 여부), 구매전환율 기여 메시지가 함께 노출된다.
- [ ]  스텝을 스킵해도 신규 업체 기본값 ON은 유지된다.

## 방향 2 — 위젯 '추천순', ‘최신 순’ 재정의

정렬 옵션별 동작 방식을 명확히 분리한다.

| 정렬 옵션 | 변경 전 | 변경 후 |
| --- | --- | --- |
| AI 추천순 (신규) | 상단 고정 리뷰 전체 최우선 노출 + 좋아요 많은 순 | 관리자 수동 고정 리뷰 최상단 + 하단은 퀄리티 등급 내 최신순 |
| 최신순 | 상단 고정 리뷰 전체 최우선 노출 + 최신 작성된 순 | 상단 고정 리뷰 최대 3개만 허용, 이후는 작성일 기준 최신순 |
| 별점순 | 상단 고정 리뷰 전체 최우선 노출 + 별점 높은 순 | 기존 방식 유지 |
- **상단 고정 개수는 왜 3개인가?**
    
    
    | 구분 | 수치 |
    | --- | --- |
    | 활성 상단 고정 보유 업체 수 | 1,709개 |
    | 평균 고정 리뷰 수 | 52.5개 |
    | 중앙값 | 11개 |
    | 최대 | 3,620개 |
    
    상단 고정 평균 개수가 52개인데 중앙값이 11개라는 건, 
    일부 업체가 수백~수천 개씩 고정해서 평균을 끌어올리고 있다는 것
    
    | N값 | 이 범위 내 업체 비율 |
    | --- | --- |
    | N=3 | 26% |
    | N=5 | 34.6% |
    | N=10 | 49.3% |
    
    어떤 N을 잡아도 절반 이상의 업체는 상단 고정이 N개를 초과. 
    즉, **N을 낮게 잡을수록 실제로 체감하는 변화가 큰 업체가 많아짐**
    
    중앙값이 11개인 걸 고려하면 **N=3~5** 정도가 "최신순 본연의 의도를 살리면서, 관리자의 핵심 고정 리뷰는 유지"하는 균형점에 가까워 보임. N=10을 넘어가면 절반 업체에서 기존이랑 사실상 같은 경험이 되어버려서 변경 의미가 약해짐
    
    다만 51개 이상 고정하는 업체가 332개(19%)나 있어서, 이 업체들은 변경에 민감하게 반응할 가능성이 있음. 
    
- **왜 잘쓰고 있는 최신순을 변경하는가?**
    
    최신순은 “가장 최근 정보를 확인하려는 사용자 의도”를 반영하는 정렬이다따라서 상단 고정으로 인해 최신 정보 탐색이 방해되지 않도록 한다.
    

> 💡 AI 추천순은 "등급 내 최신순"으로 동작한다. 최신 리뷰라도 등급이 낮으면 상위 등급의 오래된 리뷰보다 아래에 노출된다. 최신순은 "작성일 기준 순수 최신순"을 의미하며, 두 정렬의 결과는 다르다.
> 

**동작 방식:**

- 관리자가 위젯 설정에서 기본 정렬을 'AI 추천순'으로 설정 가능
- 소비자가 위젯에서 직접 'AI 추천순'으로 변경 가능
- 추천순 = 퀄리티 등급 기반 자동 정렬 로직과 동일

# 8. FAQ

**Q. 자동 정렬을 ON하면 바로 위젯에 반영되나?**
아니다. ON 토글을 켜면 미리보기 화면이 먼저 노출된다. 관리자가 결과를 확인하고 '반영하기'를 눌러야 실제 위젯에 적용된다.

**Q. 미리보기에서 순서를 바꿀 수 있나?**
가능하다. 미리보기 상태에서 특정 리뷰의 순서를 직접 변경할 수 있다. 단, 변경한 리뷰는 수동 상단고정으로 저장되며 이후 자동 정렬이 해당 리뷰의 순서를 덮어쓰지 않는다. 자동 정렬은 수동 고정되지 않은 리뷰 영역에서만 계속 작동한다.

**Q. 반영 후 마음에 안 들면?**
반영 직후 1회 되돌리기가 가능하다. 되돌리기 시 반영 직전 상태로 복귀한다.

**Q. AI 추천순과 최신순은 어떻게 다른가?**
AI 추천순은 "등급 내 최신순"으로 동작한다. 상 등급 리뷰가 먼저 노출되고, 같은 등급 안에서 최신 리뷰가 앞에 온다. 오늘 작성된 리뷰라도 중 등급이면 어제 작성된 상 등급 리뷰보다 아래에 노출된다. 최신순은 작성일 기준 순수 최신순으로, 상단 고정 리뷰는 최대 3개까지만 예외 적용된다.

**Q. 모든 정렬에서 상단 고정되었다가, 최신순에서 갑자기 안보이면 기존 고객 경험이 깨지지 않을까?**
최신순은 "가장 최근 리뷰를 확인하려는 사용자 의도"를 반영하는 정렬이다. 이번 변경은 고객 경험을 깨는 것이 아니라, 원래 정렬 옵션이 가져야 할 의미를 되찾는 것이다. 다만 기존에 최신순을 기본 정렬로 사용하던 업체는 상단 고정 리뷰의 노출 방식이 달라지므로, 배포 전 관리자 공지 또는 어드민 내 변경 안내 문구 노출을 검토한다.

**Q. ON으로 켜두면 내가 직접 고정한 리뷰 순서도 바뀌나?**
아니다. 관리자가 수동으로 상단 고정한 리뷰는 자동 정렬과 무관하게 항상 최상단에 고정 유지된다.

**Q. 새 리뷰가 들어오면 순서가 자동으로 다시 바뀌나?**
그렇다. 자동 정렬 ON 상태에서 새 리뷰가 등록되면 등급을 산정한 뒤 자동으로 재정렬된다. 단, 수동 상단고정으로 저장된 리뷰는 재정렬 대상에서 제외된다.

**Q. 기존 '추천순'과 'AI 추천순'은 다른 건가?**
다르다. 기존 추천순은 관리자가 직접 지정한 리뷰 우선 노출 방식이고, AI 추천순은 퀄리티 등급(텍스트 길이, 포토 수, 재구매 여부 등)을 기반으로 자동 정렬하는 방식이다. 구체적인 명칭 구분은 킥오프에서 개발자와 협의한다.

**Q. OFF로 꺼두면 어떻게 되나?**
기존 정렬 방식이 그대로 유지된다. 강제 변경 없음.

**Q. 신규 업체가 ON 기본값인 이유는?**
처음부터 퀄리티 기반 정렬을 경험하게 해서, 알파리뷰의 가치를 빠르게 체감할 수 있도록 하기 위함이다.

**Q. 정렬별로 다르게 처리하면 복잡도 너무 올라가지 않을까?**
정렬별 정책은 필요하지만, 이번 스코프에서는 최신순만 우선 적용하고 별점순은 기존 방식을 유지한다.

# [[에이전틱 알파앱스 PRD]]

## 📌 한 줄 통찰 (The Karpathy Summary)

> 알파앱스 6개 솔루션을 세팅·운영·성장 3축으로 통합하고, AI 에이전트가 이커머스 운영을 자율 수행하여 "매일 로그인할 필요 없는 쇼핑몰"을 구현하는 차세대 플랫폼 PRD.

---

## 1. 문제 정의 (Problem Statement)

### 1.1 고객의 현재 고통

| 고통 | 현상 | 영향 |
|------|------|------|
| **인지 과부하** | 리뷰·업셀·푸시·챗·캔버스·애즈 6개 솔루션이 개별 존재 | 고객이 전체 기능의 30% 미만만 인지·활용 (추정) |
| **도구 피로** | 각 솔루션마다 별도 온보딩·대시보드·설정 | "아무리 편하게 만들어도 어려워함" |
| **UX 불일관** | 제품마다 화이트리스트/블랙리스트 등 체계가 상이 | 학습 비용 중복, 전환율 저하 |
| **수동 운영** | 매일 로그인하여 리뷰 관리·배너 수정·캠페인 확인 | 소규모 셀러일수록 운영 인력 부족 → 성과 미달 |

### 1.2 비즈니스 과제

- 솔루션 수 증가 대비 **ARPU(유저당 매출)가 선형 성장하지 않음**
- "하나만 쓰는 것보다 여러 개를 더 잘 쓸 때 더 큰 성과"를 고객이 체감하지 못함
- 도메인 재설계 없이 기능만 추가하면 **가치 전달이 더 모호해지는 역설**

---

## 2. 제품 비전 (Product Vision)

> **"매일의 운영에서 손을 떼고, 매달의 성장을 에이전트에게."**

알파앱스의 모든 기능은 두 가지로 수렴한다:
1. **매출 향상 그 자체**
2. **매출 향상을 위한 고객의 어려움 해결**

고객은 도구를 배우는 게 아니라, **결과를 받는다.**

---

## 3. 디폴트 프레임: 3축 아키텍처

앞으로의 모든 제품·전략 논의는 이 3축을 디폴트 프레임으로 삼는다.

```
┌─────────────────────────────────────────────────┐
│                마스터 에이전트                      │
│         (모든 API 접근, 자율 의사결정)              │
├────────────┬────────────────┬───────────────────┤
│   세팅      │     운영       │      성장          │
│  (Setup)    │ (Operations)   │    (Growth)        │
│             │                │                    │
│ 10분 완료    │  주1회 보고서   │ 3C→캠페인→A/B     │
│ 브랜드 최적화 │  알림 기반     │  자율 사이클       │
└────────────┴────────────────┴───────────────────┘
```

---

## 4. 축별 요구사항

### 4.1 축 1: 세팅 (Setup)

**목표 경험:** 회원가입 + 카페24 연동 → 10분 내 전 제품 기본 세팅 완료. 스킨·위젯·SEO·가격 구조·알림톡 템플릿 기본값이 내 브랜드에 최적화.

| 요구사항 ID | 요구사항 | 우선순위 | 비고 |
|-------------|----------|----------|------|
| S-01 | 공통 온보딩 레이어: 단일 회원가입 → 전 제품 계정 연동 | P0 | 현재 제품별 개별 온보딩 → 통합 필요 |
| S-02 | 외부 플랫폼 인증 통합: 카페24·쇼피파이·기타 플랫폼 OAuth 일원화 | P0 | 세팅의 전제조건 |
| S-03 | 브랜드 기반 기본값 자동 생성: 카페24 연동 시 상점 데이터 분석 → 위젯·알림톡·SEO 기본값 자동 세팅 | P0 | AI 기반 브랜드 프로파일링 |
| S-04 | 세팅 진행률 대시보드: 남은 세팅 항목 시각화, 자동 세팅 vs 수동 세팅 구분 | P1 | |
| S-05 | 세팅 롤백: 자동 세팅 결과가 부적절할 경우 원클릭 복원 | P1 | 고객 안전장치 |

**성공 지표:**
- 온보딩 완료 시간: 현재 대비 80% 단축 (목표 10분 이내)
- 온보딩 완주율: 현재 대비 50% 개선
- 전 제품 활성화율: 온보딩 완료 사용자의 70%+

---

### 4.2 축 2: 운영 (Operations)

**목표 경험:** 매일 쇼핑몰에 로그인할 필요 없음. 주 1회 보고서만 확인. 1:1 접점이 필요할 때만 알림.

| 요구사항 ID | 요구사항 | 우선순위 | 비고 |
|-------------|----------|----------|------|
| O-01 | 운영 대시보드 통합: Canvas·Review·Chat·Upsell·Push 운영 지표 단일 뷰 | P0 | 현재 각 제품별 분리 |
| O-02 | 자동 운영 에이전트: 리뷰 관리(게시/미게시/답변), 상담 자동 응대, 팝업 자동 노출 | P0 | 고반복 업무 우선 자동화 |
| O-03 | 에스컬레이션 규칙 엔진: "사람 개입이 필요한 순간" 정의 및 알림 | P0 | 부정 리뷰 급증, VIP 문의, 가격 이상 등 |
| O-04 | 주간 운영 보고서 자동 생성: 핵심 지표 요약 + 에이전트 액션 로그 + 이상 탐지 | P1 | |
| O-05 | 좁은 맥락 AI 판단 품질 고도화: 개별 판단(리뷰 답변, 상담 응대)의 정확도 향상 | P1 | 지속적 개선 과제 |

**자동화 대상 우선순위 (4축 매트릭스):**

| 자동화 축 | 대상 업무 | 자동화 전략 |
|-----------|-----------|-------------|
| 고반복 | 상품 진열(추천), 자동화 메시지, 팝업, 리뷰 관리, 상담 | 룰 기반 + AI 보조 (Phase 1) |
| 고시간 | 배너 제작, 상담 | 생성 AI 활용 (Phase 2) |
| 고난도 | CRM 전략(세그먼트·실험·이터레이션), 광고 집행 | 에이전트 판단 아키텍처 (Phase 3) |
| 고의존 | 프로모션/가격, 상담, CRM | Human-in-the-Loop 유지 (장기) |

**성공 지표:**
- 운영자 일평균 로그인 횟수: 현재 대비 70% 감소
- 에이전트 자동 처리율: 전체 운영 이벤트의 80%+
- 에스컬레이션 정확도: 95%+ (false positive < 5%)

---

### 4.3 축 3: 성장 (Growth)

**목표 경험:** 다음 달 매출 목표를 입력하면, 에이전트가 3C 분석 → 캠페인 기획 → 소재 생성 → 집행 → A/B 결과 보고까지 자율 수행. 경영자는 결재와 인사이트 리뷰만.

| 요구사항 ID | 요구사항 | 우선순위 | 비고 |
|-------------|----------|----------|------|
| G-01 | 매출 3제품 완성: 업셀·푸시·애즈 통합 캠페인 인프라 | P0 | 알파애즈 개발 포함 |
| G-02 | 3C 데이터 파이프라인: Company(매출·재고·가격) + Customer(세그먼트·행동·LTV) + Competitor(가격·프로모션·시장) | P0 | 성장 에이전트의 입력 데이터 |
| G-03 | 캠페인 자율 기획 에이전트: 3C 분석 → 타겟 세그먼트 선정 → 캠페인 전략 수립 | P1 | Human 승인 후 집행 |
| G-04 | 소재 자동 생성: 캠페인 전략에 맞는 배너·메시지·푸시 알림 소재 AI 생성 | P1 | |
| G-05 | A/B 자동 실험 + 보고: 집행 중 자동 A/B → 승자 자동 스케일 → 결과 보고서 | P2 | |
| G-06 | 매출 목표 역추적: 목표 매출 입력 → 필요 캠페인 규모·예산 역산 | P2 | |

**성공 지표:**
- 캠페인 기획 → 집행 리드타임: 현재 대비 80% 단축
- 에이전트 추천 캠페인 채택률: 60%+
- 캠페인 ROI: 에이전트 기획 vs 수동 기획 A/B 비교

---

### 4.4 마스터 에이전트 (상위 레이어)

| 요구사항 ID | 요구사항 | 우선순위 | 비고 |
|-------------|----------|----------|------|
| M-01 | 범용 API 접근 계층: 상품·재고·주문·가격 등 모든 코어 API에 대한 에이전트 접근 권한 | P2 | 3축 경계를 넘나드는 액션 가능 |
| M-02 | 자율 의사결정 프레임워크: 에이전트 판단 → 승인 요청 → 실행의 워크플로우 | P2 | Human-in-the-Loop 경계 정의 필수 |
| M-03 | 액션 감사 로그: 마스터 에이전트의 모든 의사결정과 실행을 추적·감사 가능 | P2 | 신뢰 구축의 전제조건 |

⚠️ **미결 사항:** 마스터 에이전트의 자율 범위(어디까지 사람 승인 없이 실행 가능한가)는 별도 논의 필요. 가격 변경·재고 조정 등 비즈니스 크리티컬 액션의 자율 실행 여부가 핵심 쟁점.

## 5. 이커머스 가치사슬과 알파앱스 커버리지

| 가치사슬 단계 | 활동 | 현재 커버 | 에이전틱 목표 |
|--------------|------|-----------|---------------|
| **상품화** | 상세페이지, 썸네일, 광고소재 | Canvas (부분) | Canvas AI 자동 생성 |
| **쇼핑몰 운영** | 가격 변동, 세팅, 배너 수정 | Review·Canvas (부분) | 운영 에이전트 자동 처리 |
| **판매 부스팅** | 씨딩, 캠페인 (애즈·업셀·푸시) | Upsell·Push (프리셋) | 성장 에이전트 자율 캠페인 |
| **채널링** | 외부채널, 글로벌, 3PL, CS | 미지원 | 글로벌 판매·CS 에이전트 (장기) |

## 6. 페이즈 로드맵 (제안)

| Phase | 기간 (제안) | 핵심 딜리버리 | 3축 매핑 |
|-------|-------------|---------------|----------|
| **Phase 0: 기반** | ~ Q3 2026 | 공통 온보딩 레이어, 통합 대시보드 프레임, 인증 통합 | 세팅 |
| **Phase 1: 운영 자동화** | Q3~Q4 2026 | 리뷰·상담·팝업 자동 처리, 에스컬레이션 엔진, 주간 보고서 | 운영 |
| **Phase 2: 성장 에이전트** | Q1~Q2 2027 | 알파애즈 런칭, 3C 파이프라인, 캠페인 자율 기획·집행 | 성장 |
| **Phase 3: 마스터 에이전트** | H2 2027~ | 범용 API 계층, 자율 의사결정, 3축 통합 유즈케이스 | 통합 |

⚠️ 이 로드맵은 컨셉 기반 제안이며, 실제 일정은 리소스·기술 검증·사업 우선순위에 따라 조정 필요.

## 7. 핵심 기술 요구사항

| 영역 | 요구사항 | 비고 |
|------|----------|------|
| **에이전트 아키텍처** | 축별 전문 에이전트 + 마스터 오케스트레이터 패턴 | Multi-agent 시스템 |
| **데이터 파이프라인** | 3C 데이터 실시간 수집·정제·서빙 | Company + Customer + Competitor |
| **판단 품질** | 좁은 맥락 AI 판단 정확도 95%+ | 리뷰 답변, 상담 응대, 캠페인 추천 |
| **Human-in-the-Loop** | 승인 워크플로우, 에스컬레이션, 감사 로그 | 신뢰 구축 핵심 |
| **통합 API 계층** | 카페24·쇼피파이·외부채널 API 일원화 | 세팅·운영·성장 공통 인프라 |

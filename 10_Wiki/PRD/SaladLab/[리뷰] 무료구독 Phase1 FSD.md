# 알파리뷰 무료구독 오픈을 위한 Phase1 _ FSD

# 목적

* PRD v3 변경사항을 반영하여 FSD 구조를 User Story 기반으로 재정리한다.
* 각 기능을 **누가 / 왜 / 무엇을 원하는지** 관점으로 표현한다.
* 각 기능마다 검증 가능한 **Acceptance Criteria (인수조건)** 를 명시한다.
* PRD의 정책/제약/운영 가드레일까지 유지한 상태에서 디자인 / FE / BE / 코어가 바로 협업 가능한 형태로 정리한다.

# Phase1 제품 목표

* 무료구독 사용자가 `설치 → 무료구독 본 온보딩(카드등록 포함, 건너뛰기 가능) → 소셜 리뷰 수집/스태프 리뷰/재방문 배너 → CAFE24 역이관 → 통합 쿼터 도달 → 2차 온보딩 → 무료 체험(14일) → 유료 전환 검토`까지 한 사이클을 경험할 수 있어야 한다.
* 무료구독은 충분한 운영 경험을 제공하되, 알파리뷰 전용 위젯·푸시·업셀 등 심화 기능은 14일 체험/유료 진입 시점에만 노출하여 단계적 가치 제안을 만든다.
* 리뷰 노출은 알파리뷰 전용 리뷰 위젯이 아니라 **CAFE24 역이관(무제한)** 중심으로 해결한다. (단, 인스타그램 피드위젯은 온전히 제공) 
* 알림톡 비용은 **온보딩에서 등록받은 카드로 후불 자동 정산(매월 1일)** 처리한다.

# 공통 원칙

* 기존 기능 최대 재사용, 구독/정책/제한 레이어에 집중
* 무료구독은 통합 쿼터 N건/월 적용 (N값은 검증 후 확정)
* **CAFE24 역이관은 무제한 / 카운트 제외**로 무료구독에서도 제공
* 소셜 리뷰는 자동 역이관, 스태프 / 재방문 / 비구매자 리뷰요청 작성분은 비동기 자동 역이관 (이미지/비디오 정보 포함하여 지속 동기화)
* 대량 업로드 기능은 무료구독에서 사용 불가
* 구매 고객 대상 일반 리뷰 유도 알림톡은 Phase2 때 진행
* 알파리뷰 전용 리뷰 위젯은 Phase1 무료구독 본 온보딩/사용 구간 제외 (2차 온보딩에서 처음 등장)
* 포인트 자동 지급은 제외, 선불 포인트 10,000P 지급 구조도 폐지
* 제한은 프론트 비활성만이 아니라 BE 정책 검증까지 포함해야 한다.

# 핵심 정책 정리 

## 무료구독 / 카드등록 / 체험 / 유료 정책

* 온보딩 중 카드 등록은 **선택 (건너뛰기 가능)** 이다.
* **카드 등록 ≠ 무료체험(14일) 진입**: 카드는 후불 정산용으로만 보관되고, 무료구독은 그대로 유지된다.
* 무료체험(14일) 진입 트리거는 **쿼터 한도 도달 또는 Premium 기능 클릭** 시점이다.
* 카드 미등록 사용자는 **리뷰요청 기능만 잠금**상태가 되며, 클릭 시 카드등록 모달이 노출된다.
* 카드 미등록 사용자도 소셜 리뷰 수집 / 스태프 리뷰 / 재방문 배너는 자유롭게 사용 가능하다.
* 무료구독의 통합 쿼터는 N건/월이며, 코어 구독 월간 시스템 주기에 맞춰 리셋된다. (내부적으로 쿼터 사용에 대한 로그 히스토리를 반드시 기록)
* 유료 전환 시 쿼터 제한이 즉시 제거된다.

## 알림톡 후불 자동 정산 정책

* 비구매자 리뷰요청 알림톡 발송 건당 **12원** (데이터11 검증 후 확정)
* **매월 1일 시스템 주기**에 직전 월 발송 건수 합계 × 단가를 등록 카드로 자동 청구
* 카드 미등록 사용자는 청구 대상에서 자동 제외 (리뷰요청 자체가 잠금이므로)
* 통합 쿼터 N건/월 내에서만 발송 가능 → 월 청구 상한은 자연스럽게 `N × 12원`으로 캡

# User Story + Acceptance Criteria

## 1. 신규 앱 출시 및 진입점 확대: 알파리뷰_인스타피드

### User Story

* 초기 상점 운영자로서, 무료로 시작할 수 있는 별도 진입점(인스타그램 피드)을 통해 부담 없이 알파리뷰를 처음 써보고 싶다.
* 운영팀으로서, 카페24 앱스토어 뿐만 아니라 알파앱스 내부(푸시/업셀) 및 서비스 해지 시점에도 무료구독으로 유도하여 리텐션을 방어하고 싶다.

### Acceptance Criteria

* 카페24 앱스토어에 사용자 시점에서는 '알파리뷰_인스타피드' 앱이 노출된다.
* 무료구독 진입점은 카페24 앱스토어 외에도 **알파푸시/알파업셀 대시보드 배너**, 그리고 **알파리뷰 서비스 해지 시 리텐션 방어 화면**에 포함된다.
* 사용자에게는 별도 앱처럼 보이지만, 실제로는 **알파앱스 통합앱이 설치되고 알파리뷰 무료구독이 활성화** 되는 구조다.
* 앱 소개에 "인스타그램 피드 및 소셜리뷰 수집을 무료로 이용하세요" 가치가 표현된다.
* 설치만으로는 무료구독이 라이브되지 않는다. **무료구독 라이브는 온보딩 완료 시점에 시작**된다.

### 작업영역

* 디자인: 진입점별(앱스토어, 알파푸시/업셀 내부, 해지 화면) 배너 및 안내 화면 설계
* FE: 진입 랜딩 분기 반영
* BE: 설치 이벤트 저장 / 설치 경로 식별값 저장

---

## 2. 무료구독 본 온보딩 플로우 ✅

### User Story

* 사용자는 무료구독의 핵심 가치(소셜 리뷰 수집)를 인스타 연결로 체감한 뒤, 카드 정보를 입력하고 싶다.
* 카드 등록을 건너뛰더라도 무료구독을 바로 경험하고 싶다.

### Acceptance Criteria

* 온보딩 단계: `기본정보 입력 → 계정정보 입력 → 인스타그램 연결 → 카드정보 입력(건너뛰기 가능) → 무료구독 소개 → 완료`
* '리뷰 운영상태', '서비스 소개', '위젯 설치', '푸시/업셀'은 무료 온보딩에서 노출되지 않는다.
* 온보딩 완료 후 쿼터 N건/월이 활성화된다.

---

## 3. 카드 등록 정책 ✅

### User Story

* 알림톡 발송 비용 정산을 위해 카드를 등록받되, 초기 진입 허들은 낮추고 싶다.

### Acceptance Criteria

* 온보딩 중 카드 등록은 **선택(건너뛰기 가능)**.
* 카드 미등록 사용자가 리뷰요청 메뉴 클릭 시 카드등록 모달 노출.
* 쿼터 도달 또는 Premium 기능 클릭으로 체험/유료 전환 시, 카드 미등록 상태면 등록 단계가 자동 삽입.

---

## 4. 구독 상태 흐름 

### User Story

* 구독 상태별(무료구독, 체험, 유료) 정책을 정확히 관리하고 싶다.

### Acceptance Criteria

* 무료구독 사용 중 → 쿼터 도달/프리미엄 클릭 → 유료 전환 검토 → 카드등록/2차 온보딩 → 무료 체험(14일)
* 14일 체험 후 과금 성공 시 '유료 구독', 미결제/선택 시 '무료 구독'.
* 동일 월 내 유료→무료 재전환 시 잔여 쿼터 이어받기.

---

## 5. 소셜 리뷰 수집 및 CAFE24 역이관 

### User Story

* 인스타그램/네이버 리뷰를 가져와서 CAFE24 상품 상세에 표시하고 싶다.

### Acceptance Criteria

* 수집 1건 성공 시 쿼터 1건 차감.
* 수집된 소셜 리뷰는 CAFE24로 **자동 역이관 (무제한/카운트 제외)**.
* 역이관은 비동기 처리 및 실패 재시도 큐 적용.

---

## 6. 인스타그램 피드 위젯

### Acceptance Criteria

* 인스타 피드 위젯은 별도 기능으로 1종 제공.
* 무료구독 사용자에게는 session 측정 스크립트가 설치되지 않음 (트래픽 비용 방지).

---

## 7. 스태프 리뷰 작성 및 역이관

### Acceptance Criteria

* 스태프 리뷰 1건 등록 시 쿼터 1건 차감.
* 대량 업로드(엑셀) 사용 불가.
* 스태프 리뷰는 CAFE24 자동 역이관 대상(무제한). **이미지/비디오 정보도 지속 동기화**된다.

---

## 8. 비구매자 대상 리뷰요청 (카드등록 필수) ✅

### Acceptance Criteria

* 리뷰요청 기능은 카드 등록 사용자에게만 활성화.
* 알림톡 발송은 쿼터 미차감, 리뷰 작성 등록 시 쿼터 1건 차감.
* 발송 건당 12원 후불 자동 정산.

---

## 9. 알림톡 후불 자동 정산 

### Acceptance Criteria

* 매월 1일에 전월 발송건수 합계 자동 청구. 결제 실패 시 큐 대기 및 재시도.
* 청구일 전 사전 알림 메일 발송.

---

## 10. 재방문 배너 ✅

### Acceptance Criteria

* 주문 데이터 연동 기반 배너 1종.
* 경유 등록 시 쿼터 1건 차감. CAFE24 무제한 역이관.

---

## 11. 통합 횟수제한 시스템 ✅

### Acceptance Criteria

* 쿼터 풀 N건/월. 70%, 90%, 100% 도달 시 단계별 안내.
* 로그 히스토리를 데이터베이스에 명확히 적재하여 모니터링.
* 초과 시 API 레벨 차단 및 업그레이드 넛지.

---

## 12. 유료화 넛지 및 제한 레벨 프레임워크 (v3 + 데일리미팅 반영) ✅

### User Story

* 잠금 레벨에서 완전 가리는 것은 피하고, 차라리 메뉴를 보여주되 설명을 잘하여 유료 전환의 설득력을 높인다.

### Acceptance Criteria

* 제한 정책은 기존의 '완전 비노출(L1)'을 폐지하고, L2, L3, L4 세 단계로 구분한다.

| 레벨 | 방식 | UX | 유료화 넛지 |
| -- | -- | -- | -- |
| L2. 노출 + 설명 페이지 | 메뉴만 보이며 접근 시 설명페이지 노출 | 설명 페이지 | 기능 설명 + 업그레이드 안내 |
| L3. 노출 + 인터렉션 | 메뉴와 옵션 표시, 저장/액션 불가 | 부분 체험 가능 | **버튼 액션 시 유료화 팝업** |
| L4. 노출 (수량 제한) | 정해진 수량까지 허용 | 수량 표시 | 한도 도달 시 업그레이드 팝업 |

*(기존 L1으로 지정된 SNS 리뷰 챌린지, 스마트 스토어 연동, 공통 디자인, 등 모든 메뉴는 L2로 변경 적용)*

---

## 13. 리뷰 작성 경로 트래킹

* 재방문 배너 / 리뷰요청 / 스태프 / 소셜 가져오기 경로 식별값 저장 및 집계.

---

## 14. 데이터 마이그레이션 및 보존 정책

### Acceptance Criteria

* 역이관은 스태프/소셜/재방문/리뷰요청 리뷰에 대해 무제한 제공되며, **이미지/비디오 데이터도 무료구독에서 차별 없이 동기화**된다.
* 무료구독 사용 중 백그라운드에서 카페24 기존 리뷰를 미리 이관해두어 체험 전환 시 대기 시간을 제거.

---

## 15. 14일 체험 / 유료 전환 시 2차 온보딩

* `서비스 소개 → 위젯 설치 → 푸시/업셀 연계 유도 → 완료` 형태. 카드 미등록 시 자동 삽입.

# 부록: 데일리 미팅 (26.04.29) 반영 사항 요약

* 튜토리얼 및 FSD 문서 동기화 완료.
* 쿼터 사용에 대한 로그 히스토리 기록 명시.
* 스태프 리뷰 역이관 시 무료구독 상태에서도 이미지/비디오 데이터 지속 동기화 정책 추가 (빈 공간 발생 방지).
* 무료구독 진입점을 카페24 뿐만 아니라 알파푸시/알파업셀, 해지 화면 등으로 확대.
* 제한 레벨에서 완전 비노출(L1) 정책 폐지. 모든 기능을 보여주되, 설명 페이지(L2)를 통해 가치를 소구하는 방향으로 수정.

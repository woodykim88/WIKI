# [[알파앱스 구독개편 PRD]]

## 📌 한 줄 통찰 (The Karpathy Summary)

> 제품별로 파편화된 가격·할인·패키지·후불 체계를 "주문수 기준 5단계 × 올인원 번들 메인" 구조로 단순화하고, 리뷰의 플랫 가격을 주문수 연동 체계로 전환하여 복잡함을 제거하면서 동시에 매출을 확대한다.

---

## 1. 문제 정의 (Problem Statement)

### 1.1 복잡함 — 고객도 영업도 설명할 수 없는 가격표

**현상:** 프로덕트마다 가격 기준·티어 수·패키지·할인 정책이 전부 다르다.

| 항목 | 알파리뷰 (CAFE24) | 알파리뷰 (아임웹) | 알파업셀 | 알파푸시 |
|------|-------------------|-------------------|---------|---------|
| 티어 수 | 6개 | 6개 | **10개** | **1개** |
| 가격 기준 | 주문수 + **UV(방문자수) 혼합** | 주문수 | 주문수 | 없음 |
| 가격 범위 | 79,000 ~ 201,000원 | 79,000 ~ 201,000원 | 39,000 ~ 2,000,000원 | 39,000원 고정 |
| "베이직" 의미 | 100건 **이상** (상한 없음) | ~1,000건 **이하** | ~1,000건 이하 | 없음 |

**구체적 복잡함:**

| 복잡성 원인 | 상세 |
|------------|------|
| 프로덕트별 가격 기준 상이 | 리뷰(CAFE24)는 주문수+UV, 아임웹은 주문수, 업셀은 주문수, 푸시는 없음 |
| 패키지 레이어 존재 | 프로모션팩, 위젯팩(롤링/브리핑), 푸시팩, 세일즈티커 등 → 서비스 비용 = 프로덕트 비용 + 패키지 비용으로 분리 |
| 의미 없는 패키지 | 프리미엄위젯(브리핑) 등 별도 설치 요청까지 필요한 패키지가 독립 과금 → 고객 혼란 |
| 후불 비용 별도 존재 | 알림톡(12원/건), SMS(14원/건), 트래픽(600원/GB) 등이 익월 후불 청구 → 월 구독료와 별개의 청구 흐름 |
| 할인 정책 8가지 | 결합(10%) + 선납 6개월(5%)/12개월(10%) + 복수 쇼핑몰(5~10%) + Win-back + 이관할인 + MSG 단가 조율 + 라이트 예외 |
| 같은 이름이 다른 의미 | "베이직"이 제품마다 다른 주문수 한도와 가격 의미 |

**영향:** 영업 담당자가 계산기 없이 견적을 못 냄. 고객이 요금제 안내 페이지 없이 본인 플랜을 이해할 수 없음. CS 문의의 상당 부분이 "내 요금이 왜 이렇게 나왔는지" 질문.

---

### 1.2 합리성 — 사용량·규모와 가격이 맞지 않는 구조

**핵심 문제: 알파리뷰(CAFE24)의 가격이 규모를 반영하지 않는다.**

```
알파리뷰 CAFE24 현행 가격:
  월 50건 쇼핑몰   → 라이트  79,000원
  월 200건 쇼핑몰  → 베이직  89,000원
  월 5,000건 쇼핑몰 → 베이직  89,000원  ← 동일
  월 50,000건 쇼핑몰 → 베이직  89,000원  ← 동일 (UV < 10만이면)

→ 100건 이상은 전부 89,000원. 규모 50배 차이에 가격 차이 0원.
```

**프로덕트별 가격 합리성 진단:**

| 프로덕트 | 가격 대역 | 배율 | 합리성 |
|----------|----------|------|--------|
| 알파리뷰 (CAFE24) | 79K ~ 201K | **2.5배** | ❌ 규모 무관하게 대부분 89K 구간에 몰림. UV 기준은 주문수와 상관관계 불일치 |
| 알파리뷰 (아임웹) | 79K ~ 201K | **2.5배** | △ 주문수 기준이지만, 10,000건까지만 커버. 상한 낮음 |
| 알파업셀 | 39K ~ 2,000K | **51배** | △ 규모 반영은 하지만 10단계가 과도. 영업팀 없이 셀프 가입 어려움 |
| 알파푸시 | 39K 고정 | **1배** | ❌ 주문 100건이든 50,000건이든 동일 가격. 대형 고객에 과소 청구 |

**지불 용이성 문제:**
- 업셀: 비즈니스(5,000건) 330,000원 → 엔터1(8,000건) 400,000원 → 70,000원 점프가 3,000건 차이에서 발생. 셀프 가입 시 "왜 갑자기 이렇게 뛰지?" 느낌
- 리뷰+업셀 번들: 현행 결합할인 10%는 리뷰+업셀 조합에서만 발동, 푸시 단독 결합은 불가 → 고객에게 설명 어려움

---

### 1.3 매출 확대 — 리뷰가 돈을 못 받고 있다

**알파리뷰 vs 알파업셀 매출 구조 비교:**

월 5,000건 쇼핑몰이 리뷰+업셀+푸시 전체 사용 시:

```
알파리뷰:  120,000원 (24%) ← CAFE24 UV 15만 기준
알파업셀:  330,000원 (68%)
알파푸시:   39,000원 (8%)
──────────────────────────
합계:     489,000원
```

**알파리뷰가 전체 매출의 24%인 이유:**
- 리뷰는 알파앱스에서 가장 많은 고객이 사용하는 핵심 프로덕트
- 그러나 100건 이상은 전부 89,000원 → 규모 커져도 매출이 안 늘어남
- UV 기준 상위 티어도 최대 201,000원으로 천장이 낮음
- **업셀은 같은 규모에서 330,000원을 받는데, 리뷰는 89,000~120,000원만 받는 구조**

**놓치고 있는 매출 규모 추정:**

| 쇼핑몰 규모 | 현행 리뷰 가격 | 주문수 기반 적정 가격 (업셀 비례) | 차이 |
|------------|---------------|-------------------------------|------|
| 월 1,000건 | 89,000원 | 89,000원 | 0원 |
| 월 2,000건 | 89,000원 (UV<10만) | 100,000원 | +11,000원 |
| 월 5,000건 | 89,000원 (UV<10만) | 129,000원 | +40,000원 |
| 월 10,000건 | 100,000원 (UV 15만) | 159,000원 | +59,000원 |
| 월 20,000건 | 120,000원 (UV 25만) | 199,000원 | +79,000원 |

⚠️ 리뷰의 주문수 기반 전환만으로도 **중대형 고객(5,000건+)에서 월 40,000~79,000원/건의 추가 매출** 포착 가능.

**패키지가 매출을 숨기고 있다:**
- 프리미엄위젯(롤링), 프리미엄위젯(브리핑), 세일즈티커가 별도 패키지 과금
- 이 기능들은 본래 해당 플랜의 기능이어야 하는데, 별도 패키지로 빼면서 "기본 플랜 가격은 싸 보이지만 실제로는 패키지까지 더해야 제대로 씀" → 투명성 저하
- 해결: 의미 있는 기능은 플랜에 포함, 의미 없는 패키지는 일몰

## 2. 제품 비전 (Product Vision)

> **"표에 적힌 가격이 최종 가격. 쇼핑몰 규모가 커지면 시스템이 알아서 관리한다."**

**비전의 3축:**

| 축 | 현재 | 목표 |
|----|------|------|
| 단순함 | 8가지 할인, 3종 패키지, 3종 후불을 조합해야 최종 가격 | 가격표 한 장에 최종 가격 표시 |
| 합리성 | 100건이든 50,000건이든 리뷰 89K | 주문수에 비례하는 일관된 5단계 |
| 매출 확대 | 리뷰 가격 천장 201K, 시스템 미개입 | 주문수 연동 자동 업그레이드, 플랜별 가격 차등 |

## 3. 성공 지표 (Success Metrics)

| 지표 | 현재 (추정) | 목표 | 측정 방법 |
|------|-----------|------|-----------|
| **ARPU (유저당 월매출)** | 기준선 측정 필요 | +15% (6개월 내) | 리뷰 플랜 가격 전환 + 번들 전환 효과 |
| **리뷰 플랜 분포** | 베이직(89K)에 고객 80%+ 몰림 | 5단계 고른 분포 | 플랜별 고객 수 분포 |
| **셀프 가입 전환율** | 업셀은 영업 없이 가입 어려움 | 셀프 가입 비율 +20% | 영업 미접촉 가입 건수 |
| **CS 요금 관련 문의** | 기준선 측정 필요 | -30% (3개월 내) | 요금/청구서 관련 CS 티켓 수 |
| **올인원 번들 가입율** | 0% (존재하지 않음) | 신규 가입의 40%+ | 번들 vs 개별 가입 비율 |

## 4. 솔루션 설계 (Solution Design)

### 4.1 설계 원칙

| # | 원칙 | 근거 |
|---|------|------|
| 1 | **주문수 단일 기준** | UV 혼합 기준 제거. 고객이 자기 규모를 직관적으로 판단 가능 |
| 2 | **5단계 통일** | 10단계(업셀)와 1단계(푸시)의 양극단을 하나로. 모든 제품 동일 티어 |
| 3 | **올인원 번들이 메인** | 개별 구독은 유지하되, 번들을 기본 추천 상품으로 전환 |
| 4 | **표시 가격 = 최종 가격** | 결합할인·선납할인을 별도 계산하지 않고, 번들/연간 가격에 내재 |
| 5 | **패키지 → 플랜 흡수** | 프리미엄위젯·세일즈티커 등을 해당 플랜에 포함. 별도 패키지 과금 제거 |
| 6 | **후불 단순화** | 메시지 비용은 기존 후불 유지하되, 무료 크레딧 포함량을 플랜별로 명시 |

---

### 4.2 신규 가격 체계

#### A. 올인원 번들 (리뷰+업셀+푸시) — 메인 상품

| 티어 | 월 주문수 | 월 결제 | 연간 결제(월 환산) | 현행 3개 합산 | 절감율 |
|------|----------|---------|-------------------|--------------|--------|
| **Starter** | ~300건 | **129,000원** | **109,000원** | 157,000원 | -18% |
| **Growth** | ~2,000건 | **249,000원** | **209,000원** | 288,000원 | -14% |
| **Business** | ~5,000건 | **399,000원** | **339,000원** | 489,000원 | -18% |
| **Scale** | ~20,000건 | **799,000원** | **679,000원** | 989,000원 | -19% |
| **Enterprise** | 20,000건+ | **별도 협의** | **별도 협의** | - | - |

**설계 의도:**
- 현행 합산 대비 14~19% 할인이 내재 → 기존 결합할인(10%) + 선납할인(10%)을 **하나의 가격으로 흡수**
- 고객이 보는 건 이 표 하나. 별도 할인 계산 불필요
- 연간 결제는 월 결제 대비 추가 15% 할인 내재
- 무료 크레딧 10,000원/월 포함 (현행 동일)

**올인원에 기본 포함되는 것:**
- 알파리뷰 전 기능 (프리미엄위젯 롤링·브리핑 포함)
- 알파업셀 전 기능 (세일즈티커 포함)
- 알파푸시 전 기능
- 무료 크레딧 10,000원/월
- ⚠️ 후불 메시지 비용(알림톡·친구톡·SMS 등)은 별도 — 단, 가격표에 평균 후불 비용 예시를 함께 표기

#### B. 개별 프로덕트 — "하나만 쓸 때"

| 티어 | 월 주문수 | 알파리뷰 | 알파업셀 | 알파푸시 |
|------|----------|---------|---------|---------|
| **Starter** | ~300건 | 79,000원 | 49,000원 | 39,000원 |
| **Growth** | ~2,000건 | 99,000원 | 169,000원 | 39,000원 |
| **Business** | ~5,000건 | 129,000원 | 349,000원 | 49,000원 |
| **Scale** | ~20,000건 | 199,000원 | 849,000원 | 59,000원 |
| **Enterprise** | 20,000건+ | 별도 협의 | 별도 협의 | 별도 협의 |

**현행 대비 핵심 변경점:**

| 변경 | 현행 | 신규 | 매출 영향 |
|------|------|------|----------|
| **리뷰 주문수 기반 전환** | 100건+ = 전부 89K (CAFE24) | 5단계 차등: 79K → 99K → 129K → 199K | ⬆️ 중대형 고객 매출 +40~110% |
| **업셀 10→5단계 축소** | 엔터1~6 가격 기억 불가 | Scale+Enterprise 2단계로 단순화 | → (기존 매출 유지) |
| **푸시 규모 반영** | 단일 39K | 3단계: 39K → 49K → 59K | ⬆️ 대형 고객 미미한 증가 |
| **패키지 폐지** | 프리미엄위젯(롤링/브리핑), 세일즈티커 별도 과금 | 해당 플랜에 전부 포함 | → (패키지 매출은 플랜에 흡수) |

#### C. 무료 티어

| 프로덕트 | 현행 | 신규 |
|----------|------|------|
| 알파푸시 | 무료 버전 존재 (기능 제한) | 유지 — 변경 없음 |
| 알파리뷰 | 14일 체험 후 유료 강제 | 유지 — 변경 없음 (별도 무료구독 PRD에서 다룸) |
| 알파업셀 | 14일 체험 후 유료 강제 | 유지 — 변경 없음 |

### 4.3 할인 정책 단순화

**현행 8가지 → 신규 3가지:**

| # | 현행 할인 | 처리 |
|---|----------|------|
| ① 결합할인 (10%) | → **올인원 가격에 흡수.** 별도 할인 라인 삭제 |
| ② 선납할인 6개월 (5%) | → **폐지.** 연간 결제 한 가지만 유지 |
| ③ 선납할인 12개월 (10%) | → **연간 결제 가격에 흡수.** 별도 할인 라인 삭제 |
| ④ 복수 쇼핑몰 할인 (5~10%) | → **유지** (신규 할인 #1) |
| ⑤ Win-back 할인 | → **유지** (신규 할인 #2), 단 업셀 한정 → 전 프로덕트 확대 검토 |
| ⑥ 타 솔루션 이관 할인 | → **유지** (신규 할인 #3) |
| ⑦ MSG 단가 조율 | → Enterprise 별도 협의에 포함 |
| ⑧ 라이트 플랜 3개월 선납 예외 | → **폐지.** Starter 티어에 통합 |

**신규 할인 체계:**

| # | 할인명 | 조건 | 할인율 |
|---|--------|------|--------|
| 1 | 복수 쇼핑몰 | 동일 사업자 2개+ 쇼핑몰 | 2~5개: 5%, 6개+: 10% |
| 2 | Win-back | 해지 후 재가입 | 12개월간 10% |
| 3 | 이관 할인 | 타 리뷰 솔루션에서 이관 | 2개월 무료 |

**고객이 이해해야 할 것:** "올인원 가격 or 개별 가격. 연간이면 더 싸다. 끝."

### 4.4 후불 비용 처리

**현행 문제:** 월 구독료(선불) ↔ 메시지 비용(후불)이 별도 청구 흐름 → 고객이 "왜 추가 청구가 나왔지?" 혼란

**단기 해결 (Phase 1):**
- 후불 체계는 유지하되, 가격표에 **"월 평균 후불 비용 예시"를 명시**
- 예: "Starter 기준 월 평균 메시지 비용: 약 15,000~30,000원 (알림톡 1,000건 기준)"
- 무료 크레딧 10,000원/월로 기본 커버되는 범위를 명확히 안내

**중기 해결 (Phase 2 검토):**
- 메시지 비용을 플랜에 일정량 포함하는 "포함 메시지" 모델 검토
- 예: Growth 플랜에 알림톡 3,000건/월 포함, 초과분만 후불
- ⚠️ 이 변경은 과금 엔진 수정 필요 → Phase 2 범위

### 4.5 패키지 정리 (일몰 대상)

| 패키지 | 현재 과금 | 처리 |
|--------|----------|------|
| 프리미엄위젯(롤링) | 별도 패키지 과금 + 설치 요청 필요 | → **Growth 이상 플랜에 기본 포함** |
| 프리미엄위젯(브리핑) | 별도 패키지 과금 + 설치 요청 필요 | → **Business 이상 플랜에 기본 포함** (또는 일몰 검토) |
| 세일즈티커 | 자동 설치, 별도 과금 | → **전 플랜 기본 포함** |
| 프로모션팩 | 별도 패키지 과금 | → **Growth 이상 플랜에 기본 포함** |
| 푸시팩 | 별도 패키지 과금 | → **올인원 번들에 기본 포함** |

**원칙:** "플랜 가격 안에 모든 기능이 포함. 쓸 수 있는 기능은 플랜 등급이 결정한다."

## 5. 스코프 (Scope)

### Phase 1: 가격 안내 단순화 + 리뷰 플랜 전환 (6~8주)

**범위:**

| 항목 | In Scope | Out of Scope |
|------|----------|-------------|
| 가격 | 리뷰 주문수 기반 5단계 전환, 업셀 10→5단계 축소, 올인원 번들 가격 신설 | 과금 엔진 상태머신 변경 |
| 안내 | 요금제 안내 페이지 신설 (올인원 메인 + 개별 서브) | 대시보드 내 구독 관리 UX 전면 개편 |
| 할인 | 올인원/연간 가격에 결합·선납 흡수 → 3가지로 단순화 | 기존 계약구독 고객 할인 체계 변경 |
| 패키지 | 패키지 일몰 선언 + 플랜 포함 전환 | 과금 시스템에서 패키지 엔티티 완전 제거 |
| 청구서 | 청구서 가독성 개선 | 후불 청구 체계 변경 |
| 후불 | 가격표에 평균 후불 비용 예시 표기 | 메시지 포함 모델 전환 |
| 대상 | **신규 가입 고객** 우선 적용 | 기존 고객 강제 마이그레이션 |

**핵심 산출물:**
1. 신규 가격표 (올인원 + 개별)
2. 요금제 안내 페이지
3. 개선된 청구서 UI
4. 패키지 → 플랜 포함 매핑 정의
5. CS 가이드 (신구 체계 비교표)

### Phase 2: 기존 고객 마이그레이션 + 후불 단순화 (8~10주)

| 항목 | 내용 |
|------|------|
| 마이그레이션 | 기존 고객 → 신규 티어 자동 매핑 (Grandfather 옵션: 기존 가격 유지 or 신규 전환 선택) |
| 패키지 완전 제거 | 과금 시스템에서 패키지 엔티티 제거, 플랜 기능으로 완전 흡수 |
| 후불 개선 | 메시지 포함량 플랜별 차등 검토 (Growth: 3,000건/월 등) |
| CAFE24/아임웹 통합 | 리뷰 가격 기준을 플랫폼 무관하게 주문수로 완전 통일 |

### Phase 3: 에이전틱 알파앱스 연동 (에이전틱 MVP 이후)

| 항목 | 내용 |
|------|------|
| 사용량 기반 과금 | 주문수 외 추가 사용량 지표 도입 검토 (API 호출수, 캠페인 수 등) |
| 자동 업/다운그레이드 | 시스템이 주문수 변화를 감지하여 플랜 자동 조정 |
| 기능 단위 과금 | 프로덕트 경계 해체 후, 기능 모듈(리뷰수집/AI분석/추천엔진/캠페인 등) 단위 과금 |

## 7. 리스크 및 대응

| 리스크 | 심각도 | 대응 |
|--------|--------|------|
| **리뷰 가격 인상에 대한 이탈** | 높음 | Grandfather 12개월 유지로 충격 완화. 주문수 300건 미만은 오히려 인하(89K→79K) |
| **업셀 대형 고객 역전** | 중간 | Scale 이상은 Enterprise 별도 협의. 기존 계약구독은 현행 유지 |
| **패키지 일몰 시 매출 감소** | 낮음 | 패키지 비용은 플랜 가격에 흡수. 전체 매출은 유지되면서 복잡성만 제거 |
| **CAFE24 UV → 주문수 전환 정합성** | 중간 | UV-주문수 상관관계 데이터 분석 선행. 과도기 동안 두 기준 병행 가능 |
| **CS 부하 증가** | 중간 | Phase 1에서 신규 고객만 적용 → Phase 2에서 기존 고객 점진 전환 |
| **과금 엔진 변경 범위 초과** | 높음 | Phase 1은 가격표·안내·표시 레이어만. 상태머신 미접촉. Phase 2에서 점진 변경 |

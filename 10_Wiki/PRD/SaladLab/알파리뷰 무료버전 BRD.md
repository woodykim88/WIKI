# [[알파리뷰 무료버전 BRD]]

## 📌 한 줄 통찰 (The Karpathy Summary)

> 초기 쇼핑몰 운영자에게 비용 통제가 수반된 핵심 경험을 맛보여주고, 심리적 손실 프레임 및 엄격한 한도 정책을 통해 유료 전환을 강제하는 전략적 Freemium 비즈니스 요구사항 정의서.

## 📐 도메인 모델 개요 (Domain Model Overview)

| # | 도메인 | 핵심 개념 |
| - | - | - |
| 1 | 배경 및 목적 | 마케팅 비용 대비 무료버전 운영비 효율화 및 에이전틱 알파앱스 연계 |
| 2 | 작성(수집) 레버 | 알림톡, 스태프 리뷰, 소셜미디어 가져오기를 통합한 월 50건 쿼터 및 카드 등록 필수 조건 |
| 3 | 게시(위젯) 레버 | 위젯 1개, 포토/평점/게시판 제한, 동영상 불가 및 가로배열 1 고정 분기 처리 |
| 4 | 관리 및 포인트 레버 | 상하단 고정 제공 여부, 리뷰 연결 1개 제한, 자동 댓글 불가, 포인트 자동지급 불가 |
| 5 | 단계별 진행 계획 | Phase 1(인스타 피드 미끼) → Phase 1.5(수집/관리) → Phase 2(위젯 게시) → Phase 3(작성 개방) |
| 6 | 유료화 넛지 시나리오 | 카드 등록 최종 유도, 70%/90%/100% 한도 사전 경고, 기회 손실 및 경쟁사 대비 수치화 |
| 7 | 모니터링 및 롤백 | 매출/체험수 10% 감소 시 Kill Switch 작동, 평생 Shop ID 이력 보존 및 역이관 정책 |

## 📖 핵심 도메인별 상세 (Domain Deep-Dive)

### 📖-1. 배경 및 목적

*   **Freemium 전략의 본질**: "누구나"가 아니라 "우리의 미래 유료 고객이 될 가능성이 있는" 초기 쇼핑몰 운영자를 타겟으로 무료 플랜을 설계한다. 
*   **비용 등식**: 마케팅 비용 > 무료버전 운영 비용(인프라 + 알림톡 + CS) 구조를 달성하기 위해 타이트한 한도를 설정한다.
*   **타겟 자격 정의**: 
    *   리뷰 관리에 실제 리소스가 소요되기 시작하는 상점.
    *   리뷰가 부재했으나 최초 작성으로 실질적 매출 전환을 체감할 수 있는 상점.
    *   인스타그램, 네이버 블로그 등 외부 SNS 리뷰 자산을 쇼핑몰 마케팅으로 전환하고자 하는 상점.
*   **성장 연계**: 월 주문건이 100건을 초과하면 오히려 무료 플랜 기능이 다운그레이드되도록 설정하여 유료 플랜으로의 업셀을 강제한다.
*   **진짜 지표(North Star Metrics)**: 전체 가입자 수가 아닌, 유료 플랜 가격 페이지 조회 수, 잠긴 유료 기능 클릭 수, 활성 사용자 중 유료 전환율을 비즈니스 판단 기준으로 삼는다.

### 📖-2. 작성(수집) 레버

*   **알림톡 유도 수량 제한**: 
    *   **카드 등록 필수**: 카드 등록이 완료되어야 알림톡 유도 발송 기능이 활성화된다. (유료 전환 시의 심리적 결제 허들을 낮추는 장치)
    *   **비용 할증**: 무료 플랜 알림톡 요청은 건당 100원의 할증 요금이 부과된다. (유료 회원 12원 대비 높은 단가로 비용 리스크 가드)
    *   **최초 지원**: 카드 등록 완료 시 최초 3,000원의 무상 포인트가 제공된다.
*   **스태프리뷰 한도 설정**:
    *   초기 상점이 몰의 활성 상태를 보여주기 위해 스태프리뷰를 몰아서 "깔아두는" 용도로 제한적 허용.
    *   방안 1: 월 20건 상한 / 방안 2: 평생 100건 상한 / 방안 3: 완전 비활성화 중 정책적으로 통합 쿼터 내에서 공유하여 차감하는 방식으로 확정.
*   **통합 횟수 제한 (월 50건)**:
    *   **대상 액션**: `알림톡 유도 + 리뷰 요청 + 스태프 리뷰 + 소셜미디어(인스타/블로그) 가져오기`
    *   리뷰 작성 수 자체에도 카운트를 연계한다. (예: 알림톡 리뷰 요청 발송 1회 + 실제 작성 1회 = 총 2회 쿼터 차감)

### 📖-3. 게시(위젯) 레버

*   **위젯 세트 수량 제한**: 상점당 단 1개의 위젯 세트만 생성 및 설치가 가능하다.
*   **위젯 종류 제한**: `포토리뷰 위젯`, `평점 위젯`, `게시판 리뷰 위젯` 3종만 무료 개방한다. 동영상 리뷰 위젯 및 기타 상세 프리미엄 위젯은 표시 불가 처리한다.
*   **디자인 및 UX 제한**:
    *   **공통 디자인**: 기본 색상만 수정할 수 있도록 제한하며 고급 레이아웃, 세부 폰트, 커스텀 CSS는 불가 처리한다.
    *   **리뷰 상세보기**: 어드민 내 모든 설정 기능 및 배치 조정을 비활성화한다.
    *   **리뷰 작성 화면**: 오직 별점 및 획득 포인트 컬러만 활성화하고 나머지는 고정한다.
    *   **리뷰 작성 유도**: 무료 플랜에서는 전면 사용 불가 처리한다.
    *   **가로 배열 제한**: 포토리뷰 위젯 가로 배열은 '1'로 강제 고정한다. (가로 배열 변경 UI에 Premium 락을 걸어 시각적 제약을 유도한다.)

### 📖-4. 관리 및 포인트 레버

*   **리뷰 상하단 고정**: 무료 버전에서는 상하단 고정 기능을 1개 또는 3개 한도로 허용할지에 대해 방안을 검토하며, 유료 버전은 완전히 자동화된 고정 노출 로직을 제공해 차별화한다.
*   **자동 댓글 기능**: 전면 금지 및 비활성화 처리한다.
*   **리뷰 연결 제한**: 상품과 리뷰를 수동으로 연결하는 기능은 상점당 최대 1개 상품에 한해서만 허용한다. (추가 연결 시 유료 넛지 팝업 트리거)
*   **포인트 자동 지급 제한**:
    *   리뷰 게시 필터링은 오직 '별점'과 '텍스트/포토 키워드'까지만 제공한다. 리워드 조건 및 자동 지급 조건 설정은 전면 차단된다.
    *   일반 리뷰 지급 포인트 UI는 오직 별점, 텍스트, 포토 금액만 수정 가능하도록 열어두고 나머지는 비활성화 고정한다.

### 📖-5. 단계별 진행 계획

*   **Phase 1. 미끼 수준 가치 제공**: 인스타그램 피드 기능만 무료로 단독 앱 형태로 등록 제공한다.
*   **Phase 1.5. 관리 기능 제공 (현재 범위)**:
    *   위젯 설치가 없는 상태에서도 게시물 고정 및 수집 기능 3종을 제공한다.
    *   수집 3종: 리뷰 요청 (알림톡 + 카드 등록), 스태프리뷰, 소셜미디어 가져오기.
    *   재방문 배너 1종 (주문 데이터 연동) 제공을 통한 작성을 시도한다.
*   **Phase 2. 게시/관리 기능 제공**: 포토리뷰/게시판 위젯을 추가 오픈하고, 스마트스토어 연동을 제한적으로 허용(월 20개 수준)한다.
*   **Phase 3. 작성 기능 전면 개방**: 유료화 넛지 시나리오를 극대화하여 본격적인 Freemium 가치 사슬을 구축한다.

### 📖-6. 유료화 넛지 시나리오

*   **손실 현실화(Loss Aversion)**: 업그레이드를 하지 않아 발생한 기회손실을 구체적인 정량적 수치로 가시화한다.
    *   **단계별 경고**: 50건 쿼터 중 70% 도달 시 "이번 달 한도 70% 소진", 90% 도달 시 "10건 남음", 100% 초과 시 기능 자동 차단 및 업셀 유도.
    *   **기회 손실 수치화**: "이번 달 한도 초과로 발송하지 못한 요청 N건. 발송했다면 예상 리뷰 +X건, 예상 매출 Z원 기여" 팝업 표시.
    *   **사회적 비교**: 업종 평균 리뷰 수집량 또는 유료 사용 쇼핑몰의 수집 수치를 대조 노출한다.
*   **완성 충동(Zeigarnik Effect)**:
    *   **셋업 체크리스트**: `① 위젯 설치 → ② 첫 발송 설정 → ③ 카카오채널 연동 → ④ 카드 등록` 진행률(0~100점)을 시각화 노출하여 80% 상태에서 결제를 유도한다.
    *   **카드 등록 배치**: 카드 등록은 저항을 줄이기 위해 온보딩 첫 단계가 아닌, 반드시 체크리스트의 최후반부에 위치시킨다.
*   **사용불가 정책(L3 레벨)**:
    *   설정값을 아예 비활성화하는 대신, UI 옵션은 자유롭게 토글·선택해 볼 수 있도록 체험 상태를 유지하되 **'저장하기' 버튼을 누르는 순간 업셀 모달**을 띄운다.

### 📖-7. 모니터링 및 롤백

*   **Kill Switch (롤백 정책)**: 
    *   무료 버전 출시 후 6주 내 전체 유료 체험 상점 수 10% 이상 감소 시 롤백을 즉각 검토한다.
    *   6주 내 전체 매출액이 10% 이상 감소할 시 롤백 프로토콜을 즉시 트리거한다.
    *   매 2주마다 무료 플랜의 제한 수준 강화를 검토하고 보고한다.
*   **Shop ID 평생 이력 보존**:
    *   무료 혜택 우회를 위한 재설치 방지를 목적으로 `Mall_ID + 멀티쇼핑몰 번호` 기준의 쿼터 사용 이력을 영구 보존 데이터베이스에 아카이빙한다.
*   **이관 및 마이그레이션**:
    *   무료 버전 설치 시 기존 카페24 리뷰 데이터 이관은 차단한다.
    *   무료 버전 상점에서 생성된 스태프리뷰 및 재방문배너 경유 리뷰는 카페24로 정상 역이관하여 상점 자산 가치를 유지해 준다.

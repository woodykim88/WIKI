## 1. Problem

> *지금 우리고객은 어떤 문제를 겪고 있을까. 구체적인 사례와 데이터를 포함한다*

1. **[고객] 초기 운영자는 리뷰기능은 필요하다고 느끼지만, "얼마나/어떻게 해야 성과가 나는지" 모른다.**

   1. 이들은 인스타, 네이버 블로그 등 외부 소셜에서 언급은 조금씩 쌓이지만, 이를 "상품 단위 리뷰 자산"으로 모으는 구조가 없다.
   2. 카페24/자사몰 초기 상점 상당수는 별도 리뷰 솔루션 없이 쇼핑몰 기본 후기 게시판만 쓰거나, 아예 리뷰를 안 쓰는 상태로 운영한다.
   3. 운영자는 "리뷰가 중요하다""포토리뷰가 있어야 전환이 오른다"는 말은 알고 있지만, 실제로는 어떤 채널에서 얼마나 모아야 전환율이 얼마나 바뀌는지 감이 없다.
2. **[회사] 우리 입장에서는 "미래 과금 고객이 될 초기 상점"을 놓치고 있다.**

   1. [경쟁사] 리뷰에이드 등 저가 리뷰 서비스가 확대되고 있어, 초기 상점이 알파리뷰를 접하기도 전에 다른 솔루션에 고착되는 위험이 커지고 있다.
   2. 지금은 과금 구독 기준으로만 제품을 최적화하고 있어, 알파리뷰를 한 번도 써보지 않은 초기 상점의 행동 데이터(리뷰 작성률, 소셜 리뷰 수집량, 위젯 설치율 등)를 충분히 확보하지 못하고 있다.
   3. 이로 인해 "어떤 성장 곡선을 가진 상점이 언제 어떤 기능을 열어줘야 과금로 전환되는지" 넛지를 제대로 못하고 있다.

## 3. Solution

> *문제 해결을 위한 핵심 아이디어를 적는다.*

<aside>

인스타피드 전용 위젯을 무료제공한다. 동시에 리뷰수집(스태프리뷰, 소셜리뷰, 리뷰요청) 가치를 체험할 수 있도록 쿼터제 시스템으로 제공한다.
단, 별도 알파리뷰 위젯을 제공하지는 않기 때문에 CAFE24 쪽으로 리뷰데이터를 이관시켜주고, 카페24전용 리뷰위젯에서 실제 리뷰데이터를 노출하는 방향으로 설계한다.

## 3-A. 사용자 가치

1. 인스타그램 위젯 사용 사용가능 (무제한)
2. 소셜 리뷰 연결(쿼터제)
3. 재방문 배너 사용가능 (쿼터제)
4. 스태프 리뷰 사용가능 (쿼터제)
5. 리뷰요청 사용가능 (후불비용 발생)

## 3-B. 솔루션 설명

**3-B-1. 통합 횟수제한 시스템 `[신규개발]`**

- 리뷰 확보를 위해 사용한 총 액션 수를 하나의 쿼터 풀(Quota Pool) 로 관리한다
- 통합 쿼터는 ‘리뷰를 확보하는 행동’만 카운트한다. 따라서  스태프 리뷰 작성, 소셜 리뷰 수집은 차감 대상이지만, 확보된 리뷰를 CAFE24 리뷰 게시판으로 역이관하는 행위 자체는 차감 대상이 아니다. CAFE24 역이관은 무료구독에서도 무제한 제공한다.
- **한도: 5건/월 (최초 사용시에는 30건 제공)**

  | 액션                                | 쿼터 차감 횟수 | 비고                             |
  | ----------------------------------- | -------------- | -------------------------------- |
  | 리뷰요청                            | 0              | 후불 알림톡 비용발생 (건당 12원) |
  | 스태프 리뷰 작성                    | 1              | 건당                             |
  | 소셜미디어 가져오기 (인스타)        | 1              | 건당                             |
  | 소셜미디어 가져오기 (네이버 블로그) | 1              | 건당                             |
  | 재방문배너                          | 1              | 건당                             |


  > 무료구독 사용자는 설치 완료 후 즉시 통합 쿼터를 부여받아 기능을 사용할 수 있으며, 카드등록 여부와 무료구독 활성화는 분리된다. 단, 리뷰요청  기능은 카드등록 사용자에게만 활성화된다(미등록자는 클릭 시 카드등록 모달)
  > 통합 쿼터 소진 또는 과금구독 기능 진입 시점에 무료체험(14일) 또는 과금 전환을 제안한다. 이 시점에 카드가 미등록 상태였다면 카드등록 후 진입한다.
  >
- **카운터 라이프사이클 :**

  - **기준 주기:** 코어 구독의 월간 시스템(1일) 주기에 맞춰 리셋
  - **과금/차감:** 해당 월의 실제 사용 일수 기준 일할 계산
  - **예시:**
    - 월 1일 설치: 그 달 5건 전량 사용 가능
    - 월 29일 설치: 1일 설치했을 때와 마찬가지로 전량 사용가능
- **구독 상태 변경 시 처리:**

  - 과금구독→무료구독 다운그레이드:
    - 과거 무료구독 이용경험 X 경우 :  다운그레이드 시점에 최초30회/ 이후 5건 쿼터제 시스템 적용
    - 과거 무료구독 이용경험 O 경우 : 과거에 소진했던 쿼터시스템 반영, 적용 (월간 시스템(1일) 주기에 맞춰 리셋)
  - 무료구독→과금구독 업그레이드: 즉시 카운터 비활성화되며, 과금구독으로 라이브 되면 모든 기능 활성화 됨

**Q. 온보딩 단계에서 카드등록을 왜 받나요? 무료구독인데 카드정보를 요구하는 게 이상하지 않나요?**

A. 두 가지 이유 때문입니다.
첫째, **알림톡 후불 정산**입니다. 무료구독에 포함된 ‘구매자자 리뷰요청 ’ 기능은 알림톡 발송 비용(건당 12원)이 실제로 발생합니다. 발송 시점마다 결제를 받는 방식은 사용자 흐름을 끊고 CS 부담이 커지므로, 카드를 미리 등록받아 매월 1회 후불로 자동 정산하는 방식을 선택했습니다.

둘째, **과금 전환 마찰 감소**입니다. 무료구독의 최종 목표는 과금 전환이며, 사용자가 쿼터 소진/Premium 기능 진입 시점에 도달했을 때 카드 정보 입력 단계가 한 번 더 끼면 전환율이 떨어진다는 가설을 가지고 있습니다. 카드를 미리 받아두면 그 시점의 마찰이 사라집니다.

다만 모든 사용자에게 카드를 강제하지는 않습니다. **온보딩 카드등록 단계는 건너뛰기가 가능**하며, 카드를 등록하지 않은 사용자는 리뷰요청  기능만 잠긴 상태(L2)로 무료구독의 다른 기능(소셜 리뷰 수집, 스태프 리뷰, 재방문 배너)을 자유롭게 사용할 수 있습니다. 리뷰요청 을 사용하려고 클릭하면 그 시점에 카드등록 모달이 노출됩니다.

또한 **카드등록은 무료체험(14일) 진입과 분리**되어 있습니다. 카드 정보는 후불 정산용으로만 보관하며, 등록만으로 자동 과금이 시작되지 않습니다. 무료체험 진입은 기존과 동일하게 ‘쿼터 한도 도달’ 또는 ‘Premium 기능 클릭’ 시점에서만 발생합니다.

**Q. 알파리뷰_무료구독(가칭) VS 현재 알파리뷰앱 온보딩 간 차이는 어떻게 되나요?**

A. 현재앱은 사용자가 알파리뷰를 아는지 모르는지 선택(Y/N)에 따라 “알파리뷰 관심 기능 페이지 진입” 또는 “리뷰 사용 솔루션 설문 페이지 진입”으로 분기되는 구조라면, 이번 무료구독은 **앱 설치 → 기본 정보 입력 → 계정 정보 입력 → 인스타그램 계정 연결 → 카드정보 입력(건너뛰기 가능) → 알파리뷰 무료구독 소개 → 완료**의 단선 구조입니다. 카드 입력보다 인스타그램 계정 연결을 먼저 두어, 사용자가 무료구독의 핵심 가치(소셜 리뷰 수집)를 한 번 체감한 뒤 카드 정보를 입력하도록 설계했습니다.

또한 무료구독 사용자가 14일 체험 또는 과금 구독으로 전환할 때, **서비스 소개 → 위젯 설치 → 푸시/업셀 연계 유도화면 → 완료**의 별도 2차 온보딩이 진행됩니다. 무료구독 본 온보딩에서 의도적으로 빠진 ‘서비스 소개’, ‘위젯 설치’, ’푸시/업셀’이 이 2차 온보딩에서 처음 노출되는 구조이며, “무료에서는 CAFE24 역이관으로 충분하고, 체험/과금부터 알파리뷰 전용 위젯과 부가 채널이 열린다”는 단계적 가치 제안을 의도한 설계입니다.

**Q. 여러 액션이 동시에 실행될 때, 쿼터(월 50건)를 초과해서 사용되는 문제는 없나요? cc @최민수**

A. 있습니다. 리뷰요청 과 스태프리뷰는 한 번에 여러 건을 요청할 수 있는 대량 액션이라, 프론트엔드에서만 버튼을 막는 방식으로는 동시 실행 시 쿼터를 초과해 버릴 위험이 있습니다. 그래서 **백엔드에서 현재 사용량과 요청 건수를 기준으로 쿼터를 검사하고, 한도를 넘는 순간 요청을 거절하는 로직**이 필요합니다.

**Q. 기존에 사용하던 상점이 앱을 재설치하면, 설치만으로 바로 무료 구독이 라이브되나요? cc @이태용**

A. 아닙니다. 무료 구독 라이브(리뷰 수집·CAFE24 역이관·월 쿼터 시작)는 온보딩을 모두 완료했을 때 시작되도록 설계합니다. 이렇게 설치 단계(구독 생성)와 온보딩 완료 단계(라이브)를 분리해, 기존 상점이 앱을 다시 설치만 했을 때 의도치 않게 무료 구독 기능이 즉시 활성화되는 상황을 막습니다.

**Q. CAFE24 역이관도 통합 쿼터에 포함되나요?**

A. 아닙니다. 통합 쿼터는 리뷰를 확보하는 액션(구자 리뷰요청 , 스태프 리뷰 작성, 소셜 리뷰 수집)에만 적용합니다.
확보된 리뷰를 CAFE24 리뷰 게시판으로 보내는 역이관 자체는 카운트하지 않으며, 무료구독에서도 무제한 제공할 계획입니다.

50건 한도가 다음달 부터 적용되니, 남은 기간 (28일)은 무료로 제한없이 무료구독이 제공하는 기능들 (스태프리뷰, 리뷰재방문, 리뷰요청 , 소셜리뷰) 를 무제한 쓸 수있는 어뷰징이 발생합니다. 어떻게 대응할까요?**

A. 사용자가 *무료구독을 시작하는 순간*에 바로 월 N건의 쿼터제 룰을 적용시키면 됩니다.

**Q. 과금구독 → 무료(월N건 바로 제공) → 과금구독 → 무료2회차(1회차 때 남은 N건으로 이어서 사용) 자 그럼, 2회차 무료인 경우 어뷰징 막을 수 있나요?**

A. 네 가능합니다. 과금 - 무료 - 과금 - 무료 시나리오가 1달 내 발생했다면 2회차 무료구독 활성화시 전달되는 횟수는 앞선 1회차 무료N건의 소진량을 이어서 사용가능하고, 만약 1달을 넘어서 2회차 무료구독 활성화된 상황이면 월N건 역시 리셋되었으므로 풀카운트로 제공합니다.

1. 과금 → 무료 (최초 전환)
   - 무료 쿼터 50건 **보장** (사용성 보호 목적)
   - 단, 해당 달에 이미 무료 쿼터를 사용한 기록이 있으면 잔여량 이어받음
2. 과금 → 무료 (2회차 이후 전환)
   - 1회차 무료 기간 중 쿼터를 사용하지 않았다면 → 2회차에도 잔여량 유지
   - 한 달 이상 경과 후 재전환 시 → 쿼터 풀 카운트 제공

**Q. 언제 쿼터가 차감되나요?**

A. 쿼터는 크게 **리뷰 등록 시점에** 차감됩니다.

**[리뷰 등록 쿼터 차감]**

최초 리뷰가 생성되고 시스템에 등록되는 순간 1건이 차감됩니다. 아래 경우에 해당합니다

- **재방문 배너**를 통한 리뷰 등록 시 엔드유저가 재방문 배너를 클릭한 후 리뷰를 작성하고 등록을 완료하면 1건이 차감됩니다.
- **스태프 리뷰**는 사용자가 직접 등록할 때 차감되며, 등록에 성공한 경우에만 쿼터가 차감됩니다.(참고로, 대량 업로드 기능은 제공하지 않습니다)
- **소셜 리뷰**의 경우 인스타그램과 네이버 블로그 모두 동일하게, 소셜 플랫폼에서 리뷰를 성공적으로 가져와서 시스템에 등록이 완료되는 순간 1건이 차감됩니다. 가져오기에 실패하면 차감되지 않습니다

PS. 리뷰 요청의 경우 쿼터 차감에 해당하지 않습니다. 등록된 카드정보를 기반으로 알림톡 비용이 후불로 청구됩니다

**Q. 리뷰요청  시에는 카드등록이 필요한가요?**

A. 네, 리뷰요청  기능은 **카드등록 사용자만 사용 가능**합니다. 알림톡 발송 비용(건당 12원)을 매월 1회 후불로 자동 청구하기 위해 카드 정보가 반드시 필요하기 때문입니다.

- 온보딩 단계에서 카드를 등록한 사용자는 리뷰요청  메뉴가 처음부터 활성화되어 있습니다.
- 온보딩에서 카드등록을 건너뛴 사용자가 리뷰요청  메뉴를 클릭하면, 그 시점에 카드등록 모달이 뜨고 등록 완료 후 즉시 발송 가능합니다.
- 무료구독의 다른 기능(소셜 리뷰 수집, 스태프 리뷰, 재방문 배너)은 카드등록 여부와 무관하게 사용할 수 있습니다.

(참고: 기존 카드등록 시 10,000포인트 선불 지급 구조는 폐지했습니다. 정산은 후불 자동 청구 단일 방식으로 정리했습니다.)

**Q. 알림톡 비용은 언제 어떻게 청구되나요?**

A. **매월 1일 시스템 주기에, 직전 월의 발송 건수 합계 × 단가(12원/건)를 등록 카드로 자동 청구**합니다. 코어 구독의 정산 주기와 동일한 시점에 처리되어, 사용자 입장에서 “월 1회 결제일”이라는 단일 기대치를 형성하도록 설계했습니다.

- 통합 쿼터(N건/월) 내에서만 발송 가능하므로, 월 청구 상한은 자연스럽게 `N × 12원`으로 캡(cap)됩니다.

**Q. 구독 상태 전이는(무료↔︎과금↔︎체험) 알파푸시의 플로우와 동일한가요? 아니라면 틀린 부분을 알려주시면 좋겠습니다. (추후 무료버전들은 온보딩 과정을 통합할 생각도 있습니다)**

A. 구독 상태 자체(무료/체험/과금)과, 체험 종료 시점에 자동과금구독으로 전환되는 알파푸시와 동일하게 유지했습니다.

추가로, 이번 설계에서는 **구독 상태(무료/체험/과금)와 온보딩/라이브 상태를 의도적으로 분리**해 두었습니다. 예를 들어 재설치 시에는 단순히 앱을 다시 설치했다고 해서 바로 무료 구독이 라이브되지 않고, 온보딩 완료 시점에만 월 쿼터가 시작되도록 정의했습니다. 이렇게 설치 단계(구독 생성)와 온보딩 완료 단계(실제 라이브)를 나눈 이유는, 향후 다양한 무료버전들에서 온보딩 플로우를 통합하더라도 “언제부터 이 상점을 활성 사용자로 볼 것인가”를 일관되게 관리하기 위해서입니다.

또, 다른 무료버전의 온보딩을 통합하는 것을 미리 고려해, 이번 Phase1에서도 **상태 전이와 트리거를 최대한 일반화된 형태로 정의**했습니다. 방문자 → 앱 설치됨 → 온보딩 진행 중 → 온보딩 완료 → 무료 구독 사용 중 → 과금 전환 검토 중 → 무료 체험 → 과금/무료 구독으로 이어지는 공통 뼈대는 유지하되, 각 단계의 진입 조건(예: 카드등록 시점, 쿼터 한도 도달, Premium 기능 클릭 등)을 모듈형으로 분리해 두었기 때문에, 이후 다른 무료버전이 추가되더라도 동일한 구독 상태 머신 위에서 온보딩 화면만 재구성하는 방향으로 확장하는 것을 목표로 하고 있습니다.

**Q. 무료 구독 업체의 급증으로 이관 프로세스에 문제는 없을까요?**

이관 프로세스는 두 가지로 이루어져 있습니다. 최초이관 프로세스는 상품,주문, 리뷰를 이관하도록 구성할 예정입니다.

웹훅 프로세스도 상품, 주문, 리뷰를 이관하도록 구성할 예정입니다.
단, 해당 프로세스가 별도로 작동하기 위해서는 무료 서비스를 위한 새로운 파이프라인이 필요합니다.

이를 위해 **코어 서비스에서 무료/과금 서비스에 따라 서로다른 파이프라인으로 데이터를 전달할 예정입니다. (= SQS 큐를 분리)**

**Q. 이번 프로젝트에서 “무료버전”을 따로 출시하는 건가요?**

A. 아닙니다. 이번 프로젝트는 알파리뷰 무료 제품이라는 새로운 앱/SKU를 하나 더 만드는 게 아닙니다.  알파앱스 통합앱 안 알파리뷰 무료 구독 개념을 두고, 그 안에 인스타그램피드 온보딩을 통해 쓰게 만드는 것이 핵심입니다.

**Q. 그럼 카페24 스토어에는 어떤 앱이 올라가나요?**

A. 카페24 스토어에는 사용자가 보기에는 '알파리뷰‑인스타피드' 라는 앱이 노출됩니다. 하지만 실제로 설치되는 것은 하나의 알파앱스 통합 앱이고, 이 통합앱 안에서 인스타그램피드를 시작점으로 한 온보딩 플로우가 실행되는 구조입니다.

1. 상품데이터, 주문데이터, 리뷰데이터 이관이 필요합니다.

   **→ #1,#2 대응을 위해 월 30건의 최초 사용 건수를 제한한다. 만약 이렇게 했는데도 비용문제가 해결되지 않는다면 더 극단적으로 사용성을 제한하거나 앱서비스 철회까지 고려할 수있다**

   ◦ 폰트, 상세 컬러, 레이아웃 조합 등 고급 디자인은 Phase 2 이후.
5. **포인트 자동 지급 / 리워드 정교화**
   ◦ 조건별 차등, 이벤트 보너스, 쿠폰 연동 등은 Phase 1 범위에서 제외. 포인트 설정 UI는 L3(체험 가능, 저장 차단).
6. **프라이싱/패키지 전면 개편**
   ◦ 행동 데이터 수집만 해 두고 리디자인은 Phase 2 이후.
7. **경쟁사 분석 기반 차등화**
   ◦ 리뷰에이드 등 경쟁사 분석은 Phase 2부터 리서치.
8. **위젯 연결 기반 리뷰 관리 UI**
   ◦ 위젯이 없으므로 위젯 배치/커스터마이징 UI도 전부 제외
9. **온보딩 카드등록 강제화**
   ◦ 카드등록은 건너뛰기 가능한 선택 단계로 유지. 강제화 검토는 Phase 2 이후 데이터 기반으로 재논의.
10. **선불 포인트(10,000P) 지급 구조**
    ◦ 카드등록 시 선불 포인트 지급은 무료구독에서는 폐지. 정산은 후불 자동 청구 단일 방식

    ◦ 첫주문 후 리뷰를 쌓기 위해 알림톡을 보내는 기능은 너무 강력함. 무료 구독 유형에서 제공하는 기능으로 가져갈지? 과금구독 스펙으로 남겨둘지는 고민이 더 필요함

## 1. Problem

> *지금 우리고객은 어떤 문제를 겪고 있을까. 구체적인 사례와 데이터를 포함한다*
> 
1. **[고객] 초기 운영자는 리뷰기능은 필요하다고 느끼지만, “얼마나/어떻게 해야 성과가 나는지” 모른다.**
    1. 이들은 인스타, 네이버 블로그 등 외부 소셜에서 언급은 조금씩 쌓이지만, 이를 “상품 단위 리뷰 자산”으로 모으는 구조가 없다. (예: 월 주문 50건 내외의 초기 쇼핑몰 운영자가 인스타 후기 10개, 네이버 블로그 3개가 있어도, 상품 상세에는 텅 빈 후기 영역만 보이는 상황이 반복된다) 
    2. 카페24/자사몰 초기 상점 상당수는 별도 리뷰 솔루션 없이 쇼핑몰 기본 후기 게시판만 쓰거나, 아예 리뷰를 안 쓰는 상태로 운영한다.
    3. 운영자는 “리뷰가 중요하다”“포토리뷰가 있어야 전환이 오른다”는 말은 알고 있지만, 실제로는 어떤 채널에서 얼마나 모아야 전환율이 얼마나 바뀌는지 감이 없다.

1. **[회사] 우리 입장에서는 “미래 유료 고객이 될 초기 상점”을 놓치고 있다.**
    1. [경쟁사] 리뷰에이드 등  저가 리뷰 서비스가 확대되고 있어, 초기 상점이 알파리뷰를 접하기도 전에 다른 솔루션에 고착되는 위험이 커지고 있다.
    2. 지금은 유료 플랜 기준으로만 제품을 최적화하고 있어, 알파리뷰를 한 번도 써보지 않은 초기 상점의 행동 데이터(리뷰 작성률, 소셜 리뷰 수집량, 위젯 설치율 등)를 충분히 확보하지 못하고 있다. 
    3. 이로 인해 “어떤 성장 곡선을 가진 상점이 언제 어떤 기능을 열어줘야 유료로 전환되는지” 넛지를 제대로 못하고 있다. 

## 2. Appetite

> *문제를 해결하기 위해 우리는 얼마의 시간을 "베팅"할 것인가.*
> 
- [x]  3주 배팅 (기본) :
    
    목표 : 3**주 안에 ‘설치→첫 리뷰 수집→소셜 리뷰·스태프 리뷰 모으기→기본 위젯 노출→간단 관리’까지 한 사이클이 돌아가게 만드는 것이 목표다. (`리뷰 수집`+ `리뷰 관리`+`소셜 리뷰`+`최소 위젯`)**
    

| 배팅 구간 | 기간 | 범위 | 핵심 목표 |
| --- | --- | --- | --- |
| 1순위 | 1주 | 인스타/네이버 리뷰 수집 + 스태프 리뷰 | 리뷰가 “쌓이는” 경험 |
| 2순위 | 1주 | 리뷰 요청 (알림톡 + 카드등록 플로우) + 통합 횟수제한 | 리뷰가 “요청으로 생기는” 경험 |
| 3순위 | 1주 | 재방문배너 1종 (주문 데이터 연동) + 리뷰 작성 경로 트래킹 | 재방문 루프 완성 |

**제약 사항**

1. 기존 알파리뷰 기능을 최대한 재사용하고, 플랜/정책/제한 레이어에 집중한다.
2. 비용 임팩트가 큰 알림톡은 **카드 등록 + 통합 50건/월 한도** 내에서만 허용한다.
3. 위젯 노출은 Phase 2로 이관. Phase 1에서는 소셜 피드 위젯만 제공한다.
4. 포인트 자동 지급은 전면 막는다 (정상적으로 많은 리뷰가 작성되는 시점부터 제공).

## 3. Solution

> *문제 해결을 위한 핵심 아이디어를 적는다.*

1. **상점 설치 & 온보딩 `디자인`**
    
    (1) 카페24 앱스토어에서 **“인스타/네이버 소셜리뷰”** 앱으로 신규 출시 (무료 진입점 역할)
    (2) 무료버전 전용 온보딩 플로우: 기본정보(도메인, 카테고리, 규모) → 리뷰 운영 상태 확인 → 소셜 계정 연결 유도
    (3) 체험/과금/무료 상태 전이 플로우 설계 필요 (`추가 개발`)
    (4) `체험 → 무료버전`, `과금 → 무료버전`, `무료버전 → 체험` , `무료버전 → 과금(1회차에서 무료체험을 쓴 경우)`_전이 정의
    
2. **소셜 리뷰 연결(인스타·네이버 블로그) 연결 & 피드 구성 
(소셜 리뷰 수집은 통합 횟수제한 카운트에 포함됨)**(50건/월 공유)
    
    (1) 인스타그램  or 네이버블로그 쇼셜리뷰 연동
    
    (2) 상품 상세에 인스타 피드 형태 노출 (그리드/슬라이드 기본 1종)
    
3. **스태프 리뷰 수집 이용가능**
    - 제한: 통합 횟수제한 내에서 사용 (50건/월 공유)
4.  **리뷰 요청(알림톡 기반)**
    
    (1) 선행 조건: 카드 등록 필수
    
    (2) **(Impact)**  카드 등록 시 알림톡 발송 기능 활성화 되면서, 최근 2주 간 최대 주문 30건에 대해서 리뷰 요청을 발송한다.  **`디자인`**
    
    (3) 건당 100원 할증 (유료 시 12원), 최초 포인트 3,000원 제공
    
    (4) 최근 주문 내역 기반 리뷰 요청 알림톡 발송
    
    (5) 채널 설정: 첫 번째 리뷰 요청만 사용 가능 (회원가입 유도, 리마인더 불가)
    
    (6) 작성 유형: 텍스트/포토리뷰만 허용, 나머지 비활성화
    
    (7) 통합 횟수제한 카운트에 포함 (50건/월 공유)
    
5. **리뷰 관리**
    
    - 게시/비게시 처리
    - 기본 필터링 (별점, 키워드)
    - 리뷰 위치 선정 (상/하단 고정) 가능
    - 리뷰 연결 1개
    
    - **제한: `디자인`**
    - 자동 댓글 불가
    - 리뷰 게시 필터링은 별점+키워드만 (리워드 조건 모두 불가)
    - 포인트 자동 지급 불가
    - 포인트 지급 설정: 리뷰 작성 조건 기본값 고정, 별점/텍스트/포토만 수정 가능
    
6. **재방문 배너** 
    
    - 주문 데이터 연동 필요
    - 리뷰 작성 유도를 위한 재방문 배너 기본 1종 제공
    - 리뷰 작성 경로 트래킹에 재방문배너 경로 포함
    

1. **유료 넛지`디자인`**
    
    - 관리 화면 내 주요 기능에 “Premium 기능” 표시 → 클릭 시 유료 버전 모달
    - **잠긴 기능 사용불가 정책:** 옵션은 바꿔볼 수 있되, 저장 시 “유료화 유도 팝업” 노출
    - 접근 자체를 차단하거나 비활성화하지 않음
    - 체험은 가능하되 적용은 유료 전용
    - 한계치 도달 전 단계별 알림 (70% → 90% → 100% 초과 시 기능 축소)
    

**===== 무료 정책 운영을 위한 내부 기능 개발 =====**

(**무료 사용자 식별 → 사용량 측정 → 제한 적용 → 상태 전이 → 유료화 유도 → 비용 통제)**

1. **통합 횟수제한 시스템 `[신규개발]`**
    - 리뷰 확보를 위해 사용한 총 액션 수를 하나의 Quota Pool 로 관리한다
    - **한도:** 50건/월 + 카드 등록 필수
        - **카운터 라이프사이클:**
            - 월간 카운터: 매월 1일 00:00 리셋 (매월 1일 기준 집계 or 배치 리셋)
            - 상태 전이 시: 무료→유료 전환 시 카운터 비활성화 (무제한), 유료→무료 다운그레이드 시 해당 월 잔여분부터 적용
            - **Shop ID 단위 평생 이력:** `mall_id + multi_mall_no` 기준으로 탈퇴 후에도 한도 히스토리 영구 보존 → 재설치로 제한 우회 불가
        - **한도 도달 시 동작:  `디자인`**
            - 70% (35건): 안내 배너 “이번 달 한도 70% 소진”
            - 90% (45건): 경고 알림 “10건 남음” + 업그레이드 넛지 모달
            - 100% 초과: 리뷰 요청·스태프리뷰·가져오기 기능 비활성화 → “이번 달 놓친 리뷰 요청 N건” 손실 프레임 노출 + 다음 달 대기 or 유료 전환

1. **리뷰 작성 경로 트래킹** `신규 개발`

통합 Quota의 정확한 카운트, 그리고 **“어떤 경로에서 리뷰가 가장 쉽게 확보되는지”, “어떤 경로가 유료 전환과 연결되는지”** 판단을 위한 필수 데이터 기반이다. 무료버전에서는 “리뷰가 몇 개 생겼는가”보다 **“어떤 경로에서 생겼는가”**가 중요하다.

- **트래킹 대상 경로 (Phase 1):**
    
    
    | source_type (enum) | 설명 | 현재 구분 가능 여부 |
    | --- | --- | --- |
    | `revisit_banner` | 재방문 배너 경유 작성 | ? |
    | `review_request` | 알림톡 리뷰 요청 경유 | ? |
    | `staff` | 스태프 리뷰 직접 작성 | ✅ 이미 구분 가능  |
    | `social_instagram` | 인스타그램 가져오기 | ✅ 이미 구분 가능  |
    | `social_naver_blog` | 네이버 블로그 가져오기 | ✅ 이미 구분 가능  |
- **데이터 모델 — 리뷰 엔티티 또는 이벤트 로그에 추가할 필드:**
    - `review_source_type` (enum): 위 테이블의 source_type
    - `request_id` (nullable FK): 리뷰 요청 발송 ID → 요청과 작성을 1:1 연결
    - `campaign_id` (nullable): 재방문 배너 종류, 캠페인 구분
    - `import_source` (nullable): 소셜 가져오기 시 원본 URL/ID
    - `staff_written_yn` (boolean): 스태프 작성 여부 (기존 필드 활용 가능 시 생략)

1. **무료버전 플로우 (상태 머신)** `신규 개발` **`디자인`**

무료버전은 기존 체험/과금 플로우에 “얹는” 것이 아니라, **별도의 온보딩과 상태 전이 규칙이 필요한 독립 흐름**이다.

- **A. 상태 머신 설계**
    
    기존 `체험 ↔︎ 과금` 2상태에서 무료가 추가되면서 **상태 머신 수준의 설계**가 필요하다.
    
    ```
    [visitor] → [installed] → [free_onboarding] → [free_active]
                                                        ↓
                                                 [quota_reached] → [paywall_exposed] → [trial] or [paid]
                                                        ↓
                                                 (다음 달 리셋) → [free_active]
    ```
    
    **정규 전이:**
    
    | From | To | 트리거 | 동작 |
    | --- | --- | --- | --- |
    | visitor | installed | 무료 앱 설치 | shop_id에 free_plan 상태 부여 |
    | installed | free_onboarding | 온보딩 시작 | 무료 전용 온보딩 플로우 진입 |
    | free_onboarding | free_active | 온보딩 완료 | Quota 카운터 활성화, 기능 개방 |
    | free_active | quota_reached | 월 50건 소진 | 기능 비활성화 + 업그레이드 넛지 |
    | quota_reached | free_active | 월 리셋 (매월 1일) | 카운터 초기화, 기능 재개방 |
    | free_active/quota_reached | trial | 유료 체험 시작 | 데이터 유지, Quota 해제 |
    | free_active/quota_reached | paid | 결제 완료 | 즉시 모든 기능 해제, 한도 제거 |
    
    **예외 전이 ⚠️** 
    
    | From | To | 트리거 | 고민 포인트 |
    | --- | --- | --- | --- |
    | trial | free_active | 체험 기간 만료 | **가장 위험한 전이.** 허용하면 체험→무료→방치 경로 생김. **복잡하게 생각하지 말고, 일단 막는게 최선으로 보인다.**   |
    | paid | free_active | 결제 실패 확정 / 자발적 다운그레이드 | 기존 리뷰 데이터 유지, 유료 기능(위젯 등) 비활성화.  |
    | free_active | uninstalled | 앱 삭제 | 리뷰 데이터 보존 정책, Shop ID 이력 영구 보존 |
    | uninstalled | free_active | 재설치 (동일 Shop ID) | 이전 Quota 히스토리 복원, 무료 한도 리셋 X |
- **B. 온보딩 플로우 (무료 전용)**
    - 카페24 앱스토어 → 앱 설치 → 무료 전용 온보딩 시작
    - 온보딩 스텝: 기본정보(도메인, 카테고리, 규모) → 리뷰 운영 상태 확인 → 소셜 계정 연결 유도 → 완료
    - 기존 유료 온보딩과 설치 시점에 플랜 판별하여 분기

- **C. 알림톡 / 카드 등록 플로우**
    - 리뷰 요청 기능 사용을 위한 카드 등록 → 알림톡 활성화 과정
    - 플로우: 리뷰 요청 메뉴 진입 → “카드를 등록하면 알림톡으로 리뷰를 요청할 수 있어요” 안내 → 카드 등록 → 최초 포인트 3,000원 충전 → 알림톡 발송 가능 상태
    - **카드 등록 위치:** 셋업 체크리스트의 **마지막 단계**에 배치 (앞 단계에서 가치 체감 후 등록 유도, 첫 단계에 두면 이탈률 급등)

    - 기존에는 “카페24 → 알파리뷰”(초기 이관)가 핵심이었으나, 무료버전에서는 "알파리뷰 → 카페24" 방향의 자동 역이관이 매우 중요하다.

1. **기타 제한 정책 `개발이라기 보다는, 정책정의` `디자인`**

무료버전의 기능 제한은 단순 “비활성화”가 아니라, **유료화 넛지와 연결된 4단계 레벨**로 설계한다.

- **제한 레벨 프레임워크 (v3 - L1 완전 비노출 삭제)**
    
    잠금 레벨에서 메뉴를 완전히 가리지 않고, 차라리 메뉴를 보여주고 설명 페이지를 통해 유료화 가치를 설득한다.
    
    | 레벨 | 방식 | UX | 유료화 넛지 |
    | --- | --- | --- | --- |
    | L2. 노출 + 설명 페이지 | 메뉴만 보이며 접근 시, 기능에 대한 설명 페이지 표시 | 설명 페이지 | 클릭 시 기능 설명 + 업그레이드 안내 |
    | L3. 수정 가능 + 저장 차단 | 옵션을 만져보고 미리보기 가능, 저장 시 차단 | 체험은 가능 | **저장 시 유료화 유도 팝업** (가장 강력) |
    | L4. 제한 수량까지 허용 | 기능 사용 가능하나 한도 존재 | 한도 소진 알림 | 한도 도달 시 업그레이드 넛지 |
- **기능별 제한 레벨 매핑:**
    
    
    | 영역 | 무료 허용 범위 | 제한 레벨 | 차단 범위 |
    | --- | --- | --- | --- |
    | 알림톡 채널 | 첫 번째 리뷰 요청만 | L2 (설명 페이지) | 회원가입 유도, 리마인더 |
    | 리뷰 유형 | 텍스트, 포토 | L2 | 동영상, 프리미엄 리뷰 |
    | 리뷰 게시 필터 | 별점, 키워드 | L2 | 리워드 조건 전체 |
    | 포인트 지급 설정 | 별점/텍스트/포토 금액만 수정 | **L3** (수정 가능, 저장 차단) | 작성 조건 변경, 자동 지급 |
    | 디자인 설정 - 공통 | 기본 색상만 | **L3** | 상세 컬러, 폰트, 레이아웃 |
    | 디자인 설정 - 상세보기 | - | L2 | 전체 |
    | 디자인 설정 - 작성화면 | 별점/포인트 컬러만 | **L3** | 나머지 |
    | 리뷰 작성 유도하기 | - | L2 | 전체 |
    | 자동 댓글 | - | L2 | 전면 차단 |
    | 리뷰 연결 | 1개 | L4 (1개 한도) | 2개 이상 |
    | 위젯 | 인스타 피드만 | L2 (나머지 설명 페이지) | 포토리뷰/평점/게시판 (Phase 2) |
    | 포토리뷰 위젯 가로배열 | 1 고정 | L2 | 변경 불가 |
    | 통합 횟수 | 50건/월 | L4 | 초과 시 기능 비활성화 |
    * 참고: 이전 기획에 있던 L1(완전 비노출) 영역들은 모두 L2(설명 페이지 노출)로 변경하여 잠재적 가치를 소구함.

### 유료 잠식 리스크

무료의 목표는 유료이며, 전체 가입자 수가 아니라 **유료 전환과 직결된 “진짜 지표”**에 집중해야 한다.

**Kill / Rollback Rule — 성패 판단은 이 지표로:**

| 지표 유형 | 지표 | 기준 |
| --- | --- | --- |
| ⚠️ 위험 신호 (Kill Switch) | 6주 내 체험수 감소 | **10% 이상** → 롤백 검토 |
| ✅ 진짜 성공 지표 | 무료→체험 전이율 | 주간 추적 |
| ✅ 진짜 성공 지표 | 무료→과금 전환율 | 주간 추적 |
| ✅ 진짜 성공 지표 | 유료 플랜 가격 페이지 조회수 | 무료 사용자 기준 |
| ✅ 진짜 성공 지표 | 유료 기능 클릭수 (잠긴 기능) | 무료 사용자 기준 |
| 📊 보조 지표 | 과금→무료 다운그레이드 | 주간 10건 초과 시 경고 |
| 📊 보조 지표 | 통합 Quota 소진율 | < 20% → 과다, > 90% → 과소 |
| ❌ 허영 지표 (참고만) | 전체 무료 가입자 수 | 의사결정 기준으로 사용 금지 |

> **핵심:** “사용자는 늘었는데 왜 실패냐” 같은 해석 충돌을 방지하기 위해, Kill/Rollback 기준을 PRD 단계에서 합의한다.
>

| 시나리오 | 트리거 | 영향 | 대응 |
| --- | --- | --- | --- |

### 5. 리스크 대응을 위한 FAQ

> *미래 예상되는 (기술/정책) 리스크 관련, 현재의 대응계획을 미리 적는다*
> 

**Q1. 무료 플랜에서 리뷰 수집·관리가 너무 잘 되어, 유료 플랜으로 전환하지 않아도 “충분히 쓸만한” 상태가 되면 어떡하나?**

A1. 기능 구성을 “운영 경험을 맛보게 하는 수준”에 고정하고, 스케일링/브랜딩/자동화를 유료 플랜에만 둔다. 예를 들어 리뷰 요청·스태프 리뷰·소셜 리뷰 수에 Limit를 두고, 상단 고정 개수/위젯 수/디자인 커스터마이징/포인트/알림톡 자동화는 유료에서만 제공한다. 필요 시 데이터 기반으로 Limit를 상향/하향 조정할 수 있도록 정책 플래그를 둔다

**Q2. 무료 플랜으로 인해 트래픽/메시지/운영 비용이 과도하게 증가하면 어떻게 하나?**

A2. 가장 비용 임팩트가 큰 알림톡·대량 메시지 영역엔 강한 제한을 둔다. 무료 구간에서 사용량이 폭증하면, 특정 임계값 이상일 때 리뷰 요청/스태프 리뷰/소셜 리뷰 연결 Limit를 더 강화하거나, 상위 플랜 업셀을 유도하는 팝업을 도입하는 롤백 플랜을 동작시킨다. 

**Q3. 기존 유료 고객이 “왜 우리는 돈 내는데, 저 기능이 무료냐”고 불만을 제기할 수 있지 않을까?**

A3. 무료 플랜은 “기능 폭은 넓지만 깊이는 얕은 입문 패키지”로 정의하고, 유료 플랜에서만 가능한 고급 기능(자동화, 브랜딩, 무제한, 고급 위젯)을 분명하게 구분한다. 

**Q4. 무료 플랜 고객이 이관요청을 할 경우, 우리는 어떻게 대응해야 하나요? 이관 다 해줘야 하나요? 별도의 비용을 청구해야 할까요?** 

A4.  기존에는 “카페24 → 알파리뷰”(초기 이관)가 핵심이었는데,
무료버전에서 재방문배너·스태프 리뷰 등으로 “알파리뷰 → 카페24” 방향의 데이터가 늘어나기 시작한다는 점이 고민포인트입니다.  리뷰 이관은 단순 CRUD가 아니라 매핑(상품, 회원, 주문), 포맷 변환, 스케줄러 등 인프라 비용이 들어가는 고수준 작업이라, 모든 리뷰/모든 데이터(배너 로그, 세부 메타데이터 등)를 무조건 마이그레이션하면 무료 고객에서도 운영 비용이 과하게 나가는 리스크가 발생합니다. 따라서 무료플랜은, 이관 기능에 제한을 두는게 좋습니다.  지금 생각으로는 월 100건까지 재방문 및 스태프리뷰 이관 가능 단, 수동으로 직접 이관해야 함 (가이드 제공) 이 어떨까 합니다. 

**Q5. 통합 50건/월 한도가 초기 상점에게 부족하거나 과다하면?**

A. 데이터5(기능별 월평균 사용수, 중간값) 확인 후 조정 가능하도록 정책 플래그를 둔다. 라이트 플랜 수준의 70%를 기준으로 설정했으며, 매 2주마다 제한 수준 상향/하향 검토.

### 6. No-Gos

> *이번 버전에서 의도적으로 배제하는 것은 무엇인가? 범위를 명확히 잘라낼수록 성공가능성이 높아진다. (cut back)*
> 
1. **위젯 노출 (Phase 2로 이관)**
    - 포토리뷰 위젯, 평점 위젯, 게시판 리뷰 위젯은 Phase 2 범위. Phase 1에서는 인스타 피드만 제공.
2. **상세 디자인 커스터마이징**
    - 폰트, 상세 컬러, 레이아웃 조합 등 고급 디자인은 Phase 2 이후.
3. **포인트 자동 지급 / 리워드 정교화**
    - 조건별 차등, 이벤트 보너스, 쿠폰 연동 등은 Phase 1 범위에서 제외. 포인트 설정 UI는 L3(체험 가능, 저장 차단).
4. **프라이싱/패키지 전면 개편**
    - 행동 데이터 수집만 해 두고 리디자인은 Phase 2 이후.
5. **경쟁사 분석 기반 차등화**
    - 리뷰에이드 등 경쟁사 분석은 Phase 2부터 리서치.

### 7. Definition of Done (베팅 기간 내 완료조건)

> *이 프로젝트가 성공했음을 어떻게 알 수 있을까?*
> 

**기능 완료 기준:**

1. **1순위 완료 기준 (2주차):**
    - 무료 플랜으로 앱 설치 → 온보딩 완료 → 인스타 피드 연결 또는 소셜/스태프 리뷰 1건 이상 수집 완료한 상점이 최소 **20개** 이상.
2. **2순위 완료 기준 (4주차):**
    - 카드 등록 후 알림톡 리뷰 요청을 최소 1회 발송한 상점이 **10개** 이상.
    - 통합 Quota 시스템이 정상 동작 (50건/월 한도 적용, 카운트 정합성 확인).
3. **3순위 완료 기준 (6주차):**
    - 재방문 배너를 통한 리뷰 작성이 1건 이상 발생한 상점 존재.
    - 리뷰 작성 경로 트래킹에서 경로별 구분이 정상 동작.

**플랫폼 운영 레이어 완료 기준:**

1. **데이터 수집 인프라:**
    - 이벤트 스키마(공통 키 + 3계층 이벤트) 구현 완료.
    - 데이터1~9 사전 분석 완료 → Quota 한도, 제한값 최종 확정.
    - 데이터10~12 산출 완료 → 무료 상점당 월 비용 추정치 도출.
2. **모니터링 대시보드:**
    - 주간 운영 대시보드 구축 완료: 설치수, 온보딩 완료율, 전이율, Quota 소진율, CS 건수.
    - Kill Rule 지표(체험수 감소율, 매출 감소율) 자동 모니터링.
3. **운영 준비:**
    - 채널톡 무료버전 FAQ 지식 추가 완료.
    - CS 시점별(설치직후/카드등록/한도도달) 대응 스크립트 준비 완료.

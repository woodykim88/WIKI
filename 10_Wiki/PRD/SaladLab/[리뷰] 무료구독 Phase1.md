**3-B-1. 통합 횟수제한 시스템 `[신규개발]`**

* 통합 쿼터는 ‘리뷰를 확보하는 행동’만 카운트한다.
  따라서 비구매자 리뷰요청 발송, 스태프 리뷰 작성, 소셜 리뷰 수집은 차감 대상이지만,
  확보된 리뷰를 CAFE24 리뷰 게시판으로 역이관하는 행위 자체는 차감 대상이 아니다.
  CAFE24 역이관은 무료구독에서도 무제한 제공한다.
* **한도: N건/월**
  * **카드등록은 온보딩에서 받는다(건너뛰기 가능).** 무료구독 사용자는 설치 완료 후 즉시 통합 쿼터를 부여받아 기능을 사용할 수 있으며, 카드등록 여부와 무료구독 활성화는 분리된다. 단, 리뷰요청 기능은 카드등록 사용자에게만 활성화된다(미등록자는 클릭 시 카드등록 모달).
  * 통합 쿼터 소진 또는 Premium 기능 진입 시점에 무료체험(14일) 또는 유료 전환을 제안한다. 이 시점에 카드가 이미 등록되어 있으면 즉시 체험/유료 진입이 가능하고, 미등록 상태였다면 카드등록 후 진입한다.
  * **카운터 라이프사이클 :**
    * **기준 주기:** 코어 구독의 월간 시스템(1일) 주기에 맞춰 리셋

    * **예시:**

      * 월 29일 설치: 1일 설치했을 때와 마찬가지로 전량 사용가능

      * 무료→유료 업그레이드: 즉시 카운터 비활성화(무제한)
      * 체험→유료 자동 전환: 체험 중 사용분은 별도 관리, 유료 전환 시 카운터 비활성화
    * **Shop ID 단위 평생 이력:** 탈퇴 후에도 한도 히스토리 영구 보존 → 재설치로 제한 우회 불가

  * 리뷰 데이터 보존 기간: 영구보존 (단 사진 등 미디어 파일은 30일 뒤 하드 삭제 검토)
  * Shop ID 기준 쿼터 히스토리: **영구 보존** (재설치 시 리셋 방지)

  **3-B-6. 알림톡 후불 자동 정산 `신규개발`**

  * **정책:**
    * 비구매자 리뷰요청 알림톡 발송 건당  **12원** (데이터11 검증 후 확정)을 등록 카드로 후불 청구한다.
    * **청구 주기:** 매월 1일 시스템 주기에 직전 월의 발송 건수 합계를 일괄 청구한다(코어 구독 정산 주기와 정렬).
    * **청구 대상:** 카드등록 사용자만 리뷰요청 기능을 사용 가능하므로, 카드 미등록자는 청구 대상에서 자동 제외된다.
  * **결제 실패 시 동작:**
    * 1차 실패 → 재시도 큐 등록 (주기 별도 정의)
    * 최종 실패 확정 시 리뷰요청 기능 비활성화 + 알림 전송
    * 카드 갱신 안내 모달, 카드 갱신 후 즉시 재활성화

**Q. 무료 구독에서 리뷰 수집·관리가 너무 잘 되어, 유료 구독으로 전환하지 않아도 “충분히 쓸만한” 상태가 되면 어떡하나?**

A. 기능 구성을 “운영 경험을 맛보게 하는 수준”에 고정하고, 스케일링/브랜딩/자동화를 유료 구독에만 둔다. 무료는 “CAFE24 네이티브 게시판 수준”에서 멈춥니다.

**Q. 온보딩 단계에서 카드등록을 왜 받나요? 무료구독인데 카드정보를 요구하는 게 이상하지 않나요?**

A. 두 가지 이유 때문입니다.
첫째, **알림톡 후불 정산**입니다. 무료구독에 포함된 ‘비구매자 리뷰요청’ 기능은 알림톡 발송 비용(건당 12원)이 실제로 발생합니다. 발송 시점마다 결제를 받는 방식은 사용자 흐름을 끊고 CS 부담이 커지므로, 카드를 미리 등록받아 매월 1회 후불로 자동 정산하는 방식을 선택했습니다.

둘째, **유료 전환 마찰 감소**입니다. 무료구독의 최종 목표는 유료 전환이며, 사용자가 쿼터 소진/Premium 기능 진입 시점에 도달했을 때 카드 정보 입력 단계가 한 번 더 끼면 전환율이 떨어진다는 가설을 가지고 있습니다. 카드를 미리 받아두면 그 시점의 마찰이 사라집니다.

다만 모든 사용자에게 카드를 강제하지는 않습니다. **온보딩 카드등록 단계는 건너뛰기가 가능**하며, 카드를 등록하지 않은 사용자는 리뷰요청 기능만 잠긴 상태(L2)로 무료구독의 다른 기능(소셜 리뷰 수집, 스태프 리뷰, 재방문 배너)을 자유롭게 사용할 수 있습니다. 리뷰요청을 사용하려고 클릭하면 그 시점에 카드등록 모달이 노출됩니다.

또한 **카드등록은 무료체험(14일) 진입과 분리**되어 있습니다. 카드 정보는 후불 정산용으로만 보관하며, 등록만으로 자동 과금이 시작되지 않습니다. 무료체험 진입은 기존과 동일하게 ‘쿼터 한도 도달’ 또는 ‘Premium 기능 클릭’ 시점에서만 발생합니다.

**Q. 기존 유료 고객이 “왜 우리는 돈 내는데, 저 기능이 무료냐”고 불만을 제기할 수 있지 않을까?**

A. 무료 구독은 “기능 폭은 넓지만 깊이는 얕은 입문 패키지”로 정의하고, 유료 구독에서만 가능한 고급 기능(알파리뷰 전용 위젯, 대량 리뷰 관리, 구매 고객 대상 자동 알림톡, 통계 인사이트, 무제한)을 분명하게 구분합니다.

**Q. 통합 N건/월 한도가 초기 상점에게 부족하거나 과다하면?**

**Q. 수집 된 리뷰를 보여주는 ’리뷰 목록’은 무료구독에서 제공되나요?**

A. 무료구독에서는 ’리뷰 관리’에 해당하는 기능은 제공하지 않습니다. 스태프 리뷰, 재방문리뷰, 리뷰요청, 소셜리뷰 모두 리뷰목록에서 관리하는 기능을 주지 않습니다.

**Q. 튜토리얼 관련해서 무료구독 / 무료체험일 때 UIUX 가 어떻게 다를까요?**

A. 무료구독일 때 튜토리얼에서는 제공되는 기능이 소셜리뷰 수집 기능, 인스타그램 피드기능 제공, 스태프리뷰, 재방문 배너, 리뷰요청까지임을 인지시켜야 합니다 (UIUX). + 쿼터 할당제로 월 N건까지 카운트 리밋이 있다는 것도 알려줘야 합니다. + 유료서비스 대표기능을 엿볼 수 있도록 홍보하는 부분도 필요해보입니다. + 카드를 등록한 사용자에게는 ‘리뷰요청 알림톡 후불 정산’ 안내 카피도 노출해야 합니다.

Q. 유저로부터 위젯 **설치/제거 요청을 받게 되면, 유료로 비용을 받고 진행해줘야 하나요?**

A. Phase 1에서는 설치되는 위젯이 없으므로 별도의 위젯 설치/제거 요청 이슈가 없을 것 같습니다.

---

**Q. ‘알파리뷰-무료구독’ 서비스는 어떻게 설치되나요?**

A. 깡통 앱(소셜 리뷰 등)을 설치하면 통합 앱이 함께 설치되는 기존 방식 활용합니다. 어떤 경로(어떤 깡통 앱)로 설치했는지는 시스템에서 식별 가능하며, 식별된 설치 경로에 따라 랜딩 페이지 및 온보딩 화면을 다르게 노출하고, 온보딩 완료 시점에 알파리뷰 무료 구독 활성화할 계획입니다.

---

**Q. 월 쿼터로 제공받은 회차가 다 소진되면, 기능 제한은 어떻게 진행되나요?**

A. 백엔드(API 차단) 레벨에서 처리하는 것을 원칙으로 합니다. 프론트엔드 disabled 처리만으로는 우회 가능성 존재하기 때문입니다. 단, 백엔드 처리 공수가 많이 들 수 있어 프론트 처리 범위와 병행 검토 필요합니다.

---

**Q. 스태프리뷰 대량 업로드 기능이 있는데요! 엑셀파일에 최대 100개의 리뷰 데이터를 넣어서 생성 요청을 하면 최대 100개의 스태프리뷰가 생깁니다. 요건 통합횟수제한에서 업로드한 엑셀 파일의 수로 계산할 예정인가요? 아니면 실제로 생성되는 스태프리뷰의 갯수로 계산할 예정인가요?**

A. 결론부터 말하면 무료구독에서는 대량 업로드 기능을 사용하지 못한다, 라고 못박는게 중요할 것 같습니다.
(추가로, 엑셀 대량 업로드는 사실상 “스태프 리뷰 n건을 한 번에 만드는 배치 인터페이스”일 것 같습니다. 비즈니스 관점에서는 “리뷰 1건”이 비용과 가치의 최소 단위가 되는 게 더 직관적일 것 같아요. 만약 같은 100건을 만들었는데, 어떤 사람은 하나씩 쓰고 어떤 사람은 엑셀로 올렸다고 해서 소모 쿼터가 다르게 느껴지면, 무료구독 정의에서 말하는 “무료 상점당 월 인프라 원가 < 유료 전환 기대값”이라는 등식이 왜곡될 것 같아요)

**Q. 알파리뷰_무료구독(가칭) VS 현재 알파리뷰앱 온보딩 간 차이는 어떻게 되나요?**

A. 현재앱은 사용자가 알파리뷰를 아는지 모르는지 선택(Y/N)에 따라 “알파리뷰 관심 기능 페이지 진입” 또는 “리뷰 사용 솔루션 설문 페이지 진입”으로 분기되는 구조라면, 이번 무료구독은 **앱 설치 → 기본 정보 입력 → 계정 정보 입력 → 인스타그램 계정 연결 → 카드정보 입력(건너뛰기 가능) → 알파리뷰 무료구독 소개 → 완료**의 단선 구조입니다. 카드 입력보다 인스타그램 계정 연결을 먼저 두어, 사용자가 무료구독의 핵심 가치(소셜 리뷰 수집)를 한 번 체감한 뒤 카드 정보를 입력하도록 설계했습니다.

또한 무료구독 사용자가 14일 체험 또는 유료 구독으로 전환할 때, **서비스 소개 → 위젯 설치 → 푸시/업셀 연계 유도화면 → 완료**의 별도 2차 온보딩이 진행됩니다. 무료구독 본 온보딩에서 의도적으로 빠진 ‘서비스 소개’, ‘위젯 설치’, ’푸시/업셀’이 이 2차 온보딩에서 처음 노출되는 구조이며, “무료에서는 CAFE24 역이관으로 충분하고, 체험/유료부터 알파리뷰 전용 위젯과 부가 채널이 열린다”는 단계적 가치 제안을 의도한 설계입니다.

| 구분                              | 기존 알파리뷰 온보딩                      | 알파리뷰 무료구독 본 온보딩                                               | 무료구독 → 14일 체험/유료 전환 시 2차 온보딩                                |

**Q. 여러 액션이 동시에 실행될 때, 쿼터(월 50건)를 초과해서 사용되는 문제는 없나요? cc @최민수**

A. 있습니다. 리뷰요청과 스태프리뷰는 한 번에 여러 건을 요청할 수 있는 대량 액션이라, 프론트엔드에서만 버튼을 막는 방식으로는 동시 실행 시 쿼터를 초과해 버릴 위험이 있습니다. 그래서 **백엔드에서 현재 사용량과 요청 건수를 기준으로 쿼터를 검사하고, 한도를 넘는 순간 요청을 거절하는 로직**이 필요합니다.

**Q. 기존에 사용하던 상점이 앱을 재설치하면, 설치만으로 바로 무료 구독이 라이브되나요? cc @이태용**

A. 아닙니다. 무료 구독 라이브(리뷰 수집·CAFE24 역이관·월 쿼터 시작)는 온보딩을 모두 완료했을 때 시작되도록 설계합니다. 이렇게 설치 단계(구독 생성)와 온보딩 완료 단계(라이브)를 분리해, 기존 상점이 앱을 다시 설치만 했을 때 의도치 않게 무료 구독 기능이 즉시 활성화되는 상황을 막습니다.

**Q. CAFE24 역이관도 통합 쿼터에 포함되나요?**

A. 아닙니다. 통합 쿼터는 리뷰를 확보하는 액션(비구매자 리뷰요청, 스태프 리뷰 작성, 소셜 리뷰 수집)에만 적용합니다.
확보된 리뷰를 CAFE24 리뷰 게시판으로 보내는 역이관 자체는 카운트하지 않으며, 무료구독에서도 무제한 제공할 계획입니다.

50건 한도가 다음달 부터 적용되니, 남은 기간 (28일)은 무료로 제한없이 무료구독이 제공하는 기능들 (스태프리뷰, 리뷰재방문, 리뷰요청, 소셜리뷰) 를 무제한 쓸 수있는 어뷰징이 발생합니다. 어떻게 대응할까요?**

**Q. 유료구독 → 무료(월N건 바로 제공) → 유료구독 → 무료2회차(1회차 때 남은 N건으로 이어서 사용) 자 그럼, 2회차 무료인 경우 어뷰징 막을 수 있나요?**

A. 네 가능합니다. 유료 - 무료 - 유료 - 무료 시나리오가 1달 내 발생했다면 2회차 무료구독 활성화시 전달되는 횟수는 앞선 1회차 무료N건의 소진량을 이어서 사용가능하고, 만약 1달을 넘어서 2회차 무료구독 활성화된 상황이면 월N건 역시 리셋되었으므로 풀카운트로 제공합니다.

1. 유료 → 무료 (최초 전환)
   * 무료 쿼터 50건 **보장** (사용성 보호 목적)
   * 단, 해당 달에 이미 무료 쿼터를 사용한 기록이 있으면 잔여량 이어받음
2. 유료 → 무료 (2회차 이후 전환)
   * 1회차 무료 기간 중 쿼터를 사용하지 않았다면 → 2회차에도 잔여량 유지
   * 한 달 이상 경과 후 재전환 시 → 쿼터 풀 카운트 제공

**Q. 언제 쿼터가 차감되나요?**

A. 쿼터는 크게 **리뷰 등록 시점에** 차감됩니다.

**[리뷰 등록 쿼터 차감]**

최초 리뷰가 생성되고 시스템에 등록되는 순간 1건이 차감됩니다. 아래 경우에 해당합니다

* **재방문 배너**를 통한 리뷰 등록 시 엔드유저가 재방문 배너를 클릭한 후 리뷰를 작성하고 등록을 완료하면 1건이 차감됩니다.
* **스태프 리뷰**는 사용자가 직접 등록할 때 차감되며, 등록에 성공한 경우에만 쿼터가 차감됩니다.(참고로, 대량 업로드 기능은 제공하지 않습니다)
* **소셜 리뷰**의 경우 인스타그램과 네이버 블로그 모두 동일하게, 소셜 플랫폼에서 리뷰를 성공적으로 가져와서 시스템에 등록이 완료되는 순간 1건이 차감됩니다. 가져오기에 실패하면 차감되지 않습니다

PS. 리뷰 요청의 경우 쿼터 차감에 해당하지 않습니다. 등록된 카드정보를 기반으로 알림톡 비용이 후불로 청구됩니다

**Q. 리뷰요청 시에는 카드등록이 필요한가요?**

A. 네, 리뷰요청 기능은 **카드등록 사용자만 사용 가능**합니다. 알림톡 발송 비용(건당 12원)을 매월 1회 후불로 자동 청구하기 위해 카드 정보가 반드시 필요하기 때문입니다.

* 온보딩 단계에서 카드를 등록한 사용자는 리뷰요청 메뉴가 처음부터 활성화되어 있습니다.
* 온보딩에서 카드등록을 건너뛴 사용자가 리뷰요청 메뉴를 클릭하면, 그 시점에 카드등록 모달이 뜨고 등록 완료 후 즉시 발송 가능합니다.
* 무료구독의 다른 기능(소셜 리뷰 수집, 스태프 리뷰, 재방문 배너)은 카드등록 여부와 무관하게 사용할 수 있습니다.

(참고: 기존 카드등록 시 10,000포인트 선불 지급 구조는 폐지했습니다. 정산은 후불 자동 청구 단일 방식으로 정리했습니다.)

---

**Q. 알림톡 비용은 언제 어떻게 청구되나요?**

A. **매월 1일 시스템 주기에, 직전 월의 발송 건수 합계 × 단가(12원/건)를 등록 카드로 자동 청구**합니다. 코어 구독의 정산 주기와 동일한 시점에 처리되어, 사용자 입장에서 “월 1회 결제일”이라는 단일 기대치를 형성하도록 설계했습니다.

* 통합 쿼터(N건/월) 내에서만 발송 가능하므로, 월 청구 상한은 자연스럽게 `N × 12원`으로 캡(cap)됩니다.

**Q. 구독 상태 전이는(무료↔︎과금↔︎체험) 알파푸시의 플로우와 동일한가요? 아니라면 틀린 부분을 알려주시면 좋겠습니다. (추후 무료버전들은 온보딩 과정을 통합할 생각도 있습니다)**

A. 구독 상태 자체(무료/체험/유료)와, 체험 종료 시점에 자동 유료 전환·무료 전환이 갈리는 구조는 알파푸시와 동일하게 유지했습니다.
온보딩 단계에서 카드등록을 받는다는 점은 알파푸시와 동일합니다. 다만 **알파리뷰 무료구독의 카드등록은 무료체험(14일) 진입과 분리**되어 있다는 점이 다릅니다. 알파푸시의 카드등록은 무료체험 시작 전제이지만, 알파리뷰는 카드등록 = 후불 정산 동의 + 향후 전환 마찰 감소 목적이고, 무료체험 진입은 별도의 트리거(쿼터 한도 도달, Premium 기능 클릭)로 발생합니다.

3-B-2 표 기준으로 보면, 알파푸시에 없던 ‘온보딩 완료 → 무료 구독 사용 중’ 구간과, 쿼터 한도 도달/프리미엄 기능 클릭을 통한 유료화 넛지가 추가된 형태라고 보시면 됩니다.

추가로, 이번 설계에서는 **구독 상태(무료/체험/유료)와 온보딩/라이브 상태를 의도적으로 분리**해 두었습니다. 예를 들어 재설치 시에는 단순히 앱을 다시 설치했다고 해서 바로 무료 구독이 라이브되지 않고, 온보딩 완료 시점에만 월 쿼터가 시작되도록 정의했습니다. 이렇게 설치 단계(구독 생성)와 온보딩 완료 단계(실제 라이브)를 나눈 이유는, 향후 다양한 무료버전들에서 온보딩 플로우를 통합하더라도 “언제부터 이 상점을 활성 사용자로 볼 것인가”를 일관되게 관리하기 위해서입니다.

마지막으로, 추후 다른 무료버전의 온보딩을 통합하는 것을 미리 고려해, 이번 Phase1에서도 **상태 전이와 트리거를 최대한 일반화된 형태로 정의**했습니다. 방문자 → 앱 설치됨 → 온보딩 진행 중 → 온보딩 완료 → 무료 구독 사용 중 → 유료 전환 검토 중 → 무료 체험 → 유료/무료 구독으로 이어지는 공통 뼈대는 유지하되, 각 단계의 진입 조건(예: 카드등록 시점, 쿼터 한도 도달, Premium 기능 클릭 등)을 모듈형으로 분리해 두었기 때문에, 이후 다른 무료버전이 추가되더라도 동일한 구독 상태 머신 위에서 온보딩 화면만 재구성하는 방향으로 확장하는 것을 목표로 하고 있습니다.

**Q. 무료 서비스 사용 업체의 급증으로 이관 프로세스에 문제는 없을까요?**

단, 해당 프로세스가 별도로 작동하기 위해서는 무료 서비스를 위한 새로운 파이프라인이 필요합니다.

이를 위해  **코어 서비스에서 무료/유료 서비스에 따라 서로다른 파이프라인으로 데이터를 전달해주셔야 합니다** .

> 개발자 의견
> 현재의 웹훅 프로세스가 아닌, 새로운 패러다임 필요(람다 등)
> 최초이관 속도 증가방법 고려필요

**Q1. 이번 프로젝트에서 “무료버전”을 따로 출시하는 건가요?**

A1. 아닙니다. 이번 프로젝트는 알파리뷰 무료 제품이라는 새로운 앱/SKU를 하나 더 만드는 게 아닙니다.  알파앱스 통합앱 안 알파리뷰 무료 구독 개념을 두고, 그 안에 인스타그램피드 온보딩을 통해 쓰게 만드는 것이 핵심입니다.

**Q2. 그럼 카페24 스토어에는 어떤 앱이 올라가나요?**

A2. 카페24 스토어에는 사용자가 보기에는 '알파리뷰‑인스타피드' 라는 앱이 노출됩니다. 하지만 실제로 설치되는 것은 하나의 알파앱스 통합 앱이고, 이 통합앱 안에서 인스타그램피드를 시작점으로 한 온보딩 플로우가 실행되는 구조입니다.

**Q3. 사용자가 알파리뷰‑인스타피드를 설치하면, 내부적으로는 어떤 일이 벌어지나요?**

A3. 사용자는 인스타그램 피드 앱을 설치했다고 느끼지만, 시스템 입장에서는 알파앱스 통합앱 설치 → 해당 상점에 알파리뷰 무료 구독 활성화 → 인스타그램피드용 온보딩 시나리오 진입으로 처리됩니다. 온보딩이 끝난 시점부터는 인스타그램 위젯뿐 아니라 스태프리뷰, 재방문리뷰, 소셜리뷰수집, 리뷰요청 기능까지 알파리뷰 무료 구독의 정책 한도 안에서 사용할 수 있습니다.

**Q4. 그렇다면 “알파리뷰 무료 제품”이 따로 존재하는 건 아닌 건가요?**

A4. 그렇습니다. 알파리뷰 무료 제품이라는 별도 앱은 존재하지 않습니다. 존재하는 것은 “알파리뷰라는 서비스 안의 무료 구독(tier)”이고, 이번 Phase1은 그 무료 구독을 인스타그램피드 온보딩을 통해 어떻게 열어줄지를 설계하는 프로젝트입니다.

</aside>

무료구독 → 무료체험 전환시 고려해야 할 것은?

1. 상품데이터, 주문데이터, 리뷰데이터 이관이 필요합니다.
2. 코어에서 상품과 주문데이터는 컨트롤하지만, 리뷰데이터는 리뷰팀에서 자체적으로 관리하고 있습니다.
3. 따라서 무료구독 사용하다가 무료체험으로 100% 14일 동안 모든 기능을 쓴다는 시나리오에서는 ‘리뷰데이터’ 이관에 따른 온보딩 시간이 발생합니다 사용자 사이트에 따라서 최대 1-2일 걸리수도 있습니다.
4. #3이 발생할 경우 사용자입장에서는 갑자기 온보딩 기간 때문에 1-2일 리뷰기능을 못쓰게 될 수 있습니다.

* 무료 상점 CS 우선순위: 유료 고객 하위 (유료 CS 품질 보호)
  * 무료구독의 경우 인간 CS 서비스는 제공하지 않고, 가이드 페이지와 AI 응대로만 진행하고자 합니다.

**롤백 정책:**

1. **즉시 롤백:** 시장 여론 등 정성적 판단으로 부정적 특이사항 시
2. **데이터 기반 롤백:** Kill Rule 기준 도달 시
3. **평가 주기:** 매 주 검토 진행. 롤백여부 등 실시간 리더십 회의체 소집 후 의사결정.

### 6. No-Gos

<aside>

1. **무료구독 본 온보딩에서 알파리뷰 전용 위젯 전면 제외**
   ◦ 게시판 위젯, 포토리뷰 위젯, 평점 위젯, 상품 상세 인스타 피드 위젯 등 무료구독 본 온보딩과 무료구독 사용 구간 전체에서 제외.
   ◦ 무료구독의 리뷰 노출 수단은 CAFE24 네이티브 리뷰 게시판(역이관)으로 통일.
   ◦ 단, 무료 → 14일 체험/유료 전환 시점의 2차 온보딩에서는 위젯 설치 단계가 정상 노출된다. 이는 유료 전환자에 한해 알파리뷰 전용 위젯·푸시/업셀이 열린다는 단계적 가치 제안 설계이며, “Phase1 위젯 없음” 원칙은 무료구독 사용 구간 한정이다.
2. **구매 고객 대상 일반 리뷰 작성 유도 알림톡**
   ◦ 실제 구매 고객 대상 자동 알림톡 유도는 Phase2. Phase1에는 비구매자 대상 리뷰요청만 포함.
3. **상세 디자인 커스터마이징**
   ◦ 폰트, 상세 컬러, 레이아웃 조합 등 고급 디자인은 Phase 2 이후.
4. **포인트 자동 지급 / 리워드 정교화**
   ◦ 조건별 차등, 이벤트 보너스, 쿠폰 연동 등은 Phase 1 범위에서 제외. 포인트 설정 UI는 L3(체험 가능, 저장 차단).
5. **프라이싱/패키지 전면 개편**
   ◦ 행동 데이터 수집만 해 두고 리디자인은 Phase 2 이후.
6. **경쟁사 분석 기반 차등화**
   ◦ 리뷰에이드 등 경쟁사 분석은 Phase 2부터 리서치.
7. **위젯 연결 기반 리뷰 관리 UI**
   ◦ 위젯이 없으므로 위젯 배치/커스터마이징 UI도 전부 제외
8. **온보딩 카드등록 강제화**
   ◦ 카드등록은 건너뛰기 가능한 선택 단계로 유지. 강제화 검토는 Phase 2 이후 데이터 기반으로 재논의.
9. **선불 포인트(10,000P) 지급 구조**
   ◦ 카드등록 시 선불 포인트 지급은 무료구독에서는 폐지. 정산은 후불 자동 청구 단일 방식.
   </aside>

### 7. Definition of Done (완료조건)

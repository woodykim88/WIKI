# 알파솔루션 구독 및 과금 시스템 종합 브리핑

## 📖 핵심 도메인별 상세 (Domain Deep-Dive)

### 📖-1. 시스템 개요 및 상태 관리

계층 구조: **계정 > 쇼핑몰(사이트) > 프로덕트**. 모든 과금·구독 정책은 **사이트** 단위를 기준으로 적용.

**4가지 핵심 상태 정의:**

| 상태 유형 | 값 예시 |
|-----------|---------|
| 사이트 상태 | 라이브, 구독종료, 해지완료, 필수세팅완료 |
| 프로덕트별 상태 | 구독없음, 체험중, 과금중, 서비스중단 |
| 구독 상태 | 예약중, 사용중, 결제실패, 구독중단 |
| 결제 상태 | 성공, 실패, 미수, 실패확정 |

**상태 전이 연쇄 규칙:**
- 결제 실패 → 구독 상태 변경 → 프로덕트 기능 일부 중지 → 사이트 상태 '구독 종료' 전환 가능

**청구서(Invoice) 3종류:**

| 분류 | 종류 | 설명 |
|------|------|------|
| 서비스 청구서 | 정기구독, 계약구독 등 | 고객 서비스 사용 권한에 대한 핵심 청구서 |
| 보조 청구서 | 메시지포인트, 설치/제거 | 부가 서비스 청구서 |
| 임의 청구서 | 기타결제/환불, 예치금, 크레딧 | 슈퍼관리자 수동 생성, 예외 정산 처리 |

### 📖-2. 과금 구조 및 정책

**비용 산출 공식 (순서 엄수):**
```
최종 비용 합계
= (서비스 비용 - 일반 할인) - 계약 할인 - 추가 할인(쿠폰)
= 비용 총계 - 크레딧 사용 - 예치금 사용
```

**알파업셀 플랜별 월간 비용 (원):**

| 플랜 | 라이트 | 베이직 | 그로스 | 비즈니스 | 엔터1 | 엔터2 | 엔터3 | 엔터4 | 엔터5 | 엔터6 |
|------|--------|--------|--------|----------|-------|-------|-------|-------|-------|-------|
| 최대 주문수 | 100 | 1,000 | 2,000 | 5,000 | 8,000 | 10,000 | 20,000 | 30,000 | 50,000 | 5만 초과 |
| 비용(원) | 39,000 | 89,000 | 160,000 | 330,000 | 400,000 | 540,000 | 800,000 | 1,000,000 | 1,600,000 | 2,000,000 |

⚠️ 엔터6: 5만 건 초과 시 1만 건당 400,000원 추가

**할인 정책:**
- **결합 할인:** 2개 이상 동시 구독 → 10%, 3개 이상 → 20% (정책 미확정)
- **연간 구독 할인:** 서비스 비용의 10%
- **적용 순서:** 결합 할인 먼저 → 잔액에 연간 할인 적용
- **쿠폰 3종:**
  - 할인 쿠폰: 일반+계약 할인 후 잔액에 특정 할인율 적용
  - 체험 쿠폰: 기본 14일 추가 연장, 프로덕트별 최초 1회
  - 크레딧 쿠폰: 등록 즉시 크레딧 지급

### 📖-3. 구독 라이프사이클

**구독 종류:**
- **무료 버전 (알파푸시 전용):** 구독료 0원, 무기한 사용. 단, 알림톡·문자 발송 시 건당 발송 비용은 종량제(익월 청구). 기능 제한 있음.
  - 제한 기능: 온사이트 캠페인 1개, 자동화 메시지 캠페인 1개까지만 생성 가능
  - 불가 기능(프리미엄): 초스피드 회원가입, 배너 관리, 탐색 유도 장치, 선물하기, 첫 구매 할인
  - 유료 버전: 월 39,000원
- ⚠️ **알파리뷰·알파업셀은 무료 버전 없음:** 가입 시 14일 무료 체험 제공 후 유료 전환 (알파리뷰 최소 79,000원/월, 알파업셀 최소 39,000원/월)
- **체험 구독:** 최초 1회 자동 생성, 기본 14일, 과금 없음, 플랜 한도 무제한. 종료 시 자동 과금 전환 전제(내부적으로 '예약중' 과금 구독 동시 생성)
- **과금 구독 - 정기:** 월/연 단위 자동 갱신·결제
- **과금 구독 - 계약:** 슈퍼관리자가 직접 설정, 계좌이체 지원, 구독 기간 중 업/다운그레이드 불가

**구독 주기:** 월간, 연간, 계약(3/6/9개월 등). 비용은 실제 일수 무관, 주기 단위 부과.  
**월간 시스템 주기:** 연간/장기 계약이라도 한도 초기화·무료 포인트 지급 등은 최초 과금 시작일 기준 매월 운영.

**구독 변경(업/다운그레이드) 절차:**
1. 기존 구독 환불 → 변경 시점까지 사용분 제외한 잔여 기간 환불 (현금분은 예치금으로 환불)
2. 기존 구독 → '변경완료' 상태로 종결
3. 새 구독 생성 및 결제 → 남은 기간 일할 계산 청구, 1번 환불 예치금·크레딧 우선 사용

**비용 계산 공식:**
```
구독률 = 실제 사용일수 / 전체 구독 기간 일수
사용 금액 = 비용 × 구독률 (할인액도 동일 비율 적용)
```

⚠️ **위약금 (다운그레이드 시):** 기존 구독에 적용된 할인(결합·쿠폰 등)이 취소 → 환불액 감소 → 그 차액이 위약금으로 새 구독 결제 시 합산 청구

**플랜 한도 정책:**
- 100% 초과: 알림 발송
- 105%까지: 유예 기간 (서비스 정상 유지)
- 105% 초과: 위젯 포함 모든 기능 중단
- **안전 한도 옵션:** 105% 초과해도 중단 없이, 다음 월간 주기 시작 시 자동 상위 플랜 업그레이드·결제

**구독 종료 및 해지:**
- **구독 종료:** 해지 요청·슈퍼관리자 조치·결제 실패 확정 시 → 모든 서비스 중단
- **해지 승인 후 데이터 보존:**
  - 즉시 제거: 크레딧, 메시지포인트, 카드 정보
  - 1년 유지: 관리자 기본 정보, 구독 정보
  - 5년 유지: 예치금 잔액
  - 영구 저장: 쇼핑몰명, 도입/해지 일자 (비식별 처리 후)

### 📖-4. 결제 프로세스 및 예외 처리

**결제 시점:**
- **즉시 결제:** 체험 없는 업/다운그레이드, 구독 시작일=오늘인 경우 → 결제 성공 확인 후 구독 변경 완료
- **청구서 대기 후 결제:** 체험 후 첫 과금, 구독 갱신 → 청구서 '대기' 발행 → 구독 시작일 당일 11:00 자동 결제

**정기 구독 결제 실패 처리 (카드):**

| 단계 | 내용 |
|------|------|
| 실패 직후 | 서비스 기능 중지 (위젯 표시 제외) |
| D+1 ~ D+20 | 매일 오전 11시 자동 재시도 (보조카드 등록 시 주카드 실패 → 보조카드 시도) |
| D+21 (미수 확정) | 구독 상태 → 미수, 사이트 상태 → 구독종료 |
| 수동 재결제 | 실패 기간 중 슈퍼관리자·고객사 직접 시도 가능, 성공 시 즉시 정상화 |

**계약 구독 (계좌이체) 실패:**
- 입금 대기 중에도 서비스 정상 제공 (구독 수정/갱신은 제한)
- 계약 기간 후 미입금 → 계좌이체 실패 확정 → 구독 종료 프로세스

**메시지 포인트 자동 충전 실패:** D+3까지 매일 1회 재시도 (총 4회). 최종 실패 시 메시지포인트 일시중지 (다른 서비스 영향 없음)

**미납금:** 결제 실패 상태에서 구독 변경·종료 시 사용 기간 비용이 미납금으로 발생 → 향후 새 과금 구독 시작 시 합산 청구 (슈퍼관리자 강제 제거 가능)

### 📖-5. 금원 및 정산

**차감 우선순위:**
```
무료 메시지포인트 > 유료 메시지포인트 > 크레딧 > 예치금 > 현금(카드/계좌이체)
```

| 금원 | 성격 | 부가세 | 환불 |
|------|------|--------|------|
| 예치금 | 현금성 자산, 미리 충전 | 포함 | 가능 |
| 크레딧 | 비현금성 프로모션 포인트 | 없음 | 불가 |
| 메시지 포인트(유료) | 메시지 발송 전용 | - | 불가 |
| 메시지 포인트(무료) | 매월 초 지급 | - | 월말 소멸 |

**매출 인식 원칙 (발생주의 + 실현주의 혼합):**
- 매출 인식 시점: 청구서 발행 시(결제 대기 상태) → 발생주의
- 연간 구독: ARR → 12 분할 MRR로 내부 성과 측정 → 실현주의적 요소
- 환불 처리: 과거 매출 소급 수정 없이 환불 발생 시점 매출에서 차감

**1원 문제 (부가세 소수점 처리):**
- 여러 금원(예치금+카드) 혼합 결제 시 각 금원별 부가세 합계와 전체 부가세가 1원 차이 발생 가능
- 해결: **마지막 소모 금원이 남은 공급가액과 부가세를 그대로 할당받아** 총액 정합성 유지

### 📖-6. 운영 고려사항 및 슈퍼관리자 FAQ

**의도적 제외 기능 (수동 처리):**
- **카드 결제 취소:** PG사 대시보드에서 직접 취소 → 슈퍼관리자 페이지에서 마이너스 매출 임의 청구서(기타 환불) 수동 발행
- **여러 쇼핑몰 통합 결제:** 구현 보류, 각 쇼핑몰 개별 결제 원칙
- **예치금 환불:** PG사에서 최근 결제 건 카드 부분 취소 원칙

**슈퍼관리자 FAQ 요약:**

| 상황 | 처리 방법 |
|------|-----------|
| 체험 구독 연장 요청 | 원칙상 불가, '무료 제공' 할인 쿠폰 발행으로 대체 |
| 과거 날짜 소급 구독 수정 | 기존 구독 종료 + PG 카드 취소 → 임의 청구서 수동 조정 → 과거 날짜 기준 계약 구독 생성 |
| 일시 중지 요청 | 구독 종료 후 종속 상태를 일시중지로 설정 (해지 승인 없이 데이터 보존) |
| 카드 → 계좌이체 변경 | 기존 구독 종료 + PG 카드 취소 → 동일 조건 계약 구독(계좌이체) 신규 생성 |

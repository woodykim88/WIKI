# 리뷰 종류 요약

## 📌 한 줄 통찰 (The Karpathy Summary)
> 알파리뷰의 모든 리뷰는 `review_type`(리뷰 형식)과 `creater_type`(작성 주체)의 조합으로 완전히 구분되며, 주문 데이터 유무와 platform 값이 포인트 지급 가능 여부를 결정한다.

## 📐 도메인 모델 개요 (Domain Model Overview)

| # | 도메인 | 핵심 개념 |
|---|--------|-----------|
| 1 | 자발적 직접 작성 | 엔드유저가 직접 작성. review_type 1~3, creater_type 1~3 |
| 2 | 관리자 요청 작성 | 알림톡 경유. 미검증 리뷰 / 리뷰 요청 포함 |
| 3 | 관리자 직접 작성 | 스태프 리뷰. 주문 데이터 없음. creater_type=5 |
| 4 | 외부 수집 | 스마트스토어/소셜/네이버블로그. creater_type=6 |
| 5 | 수정/복사 상태 분기 | 수정/복사 시 creater_type이 별도 분기됨 |

## 📖 핵심 도메인별 상세 (Domain Deep-Dive)

### 📖-1. [1] 엔드 유저가 알파리뷰에서 직접 작성 (자발적)

> 가장 기본적인 텍스트 형식의 리뷰를 "텍스트 리뷰"라고 정의

| 리뷰 종류 | [DB] review_type | [DB] creater_type | [DB] platform | 주문 데이터 | createdAt |
| --- | --- | --- | --- | --- | --- |
| 텍스트 리뷰 | 1 (일반리뷰) | 1 (회원) 2 (비회원) 3 (NPAY 비회원) | 알파리뷰 | 있음 | 최초 작성 시점 (별점 클릭) |
| 빠른 리뷰 | 2 (빠른리뷰) | 1 (회원) 2 (비회원) 3 (NPAY 비회원) | 알파리뷰 | 있음 | 최초 작성 시점 (별점 클릭) |
| SNS 리뷰 | 3 (SNS리뷰) | 1 (회원) 2 (비회원) 3 (NPAY 비회원) | 알파리뷰 | 있음 | 해시태그가 인식되어서 admin에 유입된 시점이 createdAt임. 게시 여부와는 별개임 |

**보조 구분 기준**: [회원 / 비회원 / NPAY 비회원] 구분 및 텍스트 리뷰 vs 빠른 리뷰 구분 존재

### 📖-2. [2] 엔드 유저가 알파리뷰에서 직접 작성 (관리자가 엔드 유저에게 요청)

| 리뷰 종류 | [DB] review_type | [DB] creater_type | [DB] platform | 주문 데이터 | createdAt |
| --- | --- | --- | --- | --- | --- |
| 미검증 리뷰 | 1 (일반리뷰) | 11 (미검증 리뷰) | 알파리뷰 | 강제 생성됨 | 최초 작성시점 |
| 리뷰 요청 > 텍스트 리뷰 | 1 (일반리뷰) | 입력한 고객정보(이름, 전화번호)와 일치하는 고객 데이터가 1) 존재하면 ⇒ 해당 고객의 customer_type 값을 그대로 가져옴 2) 존재하지 않으면 ⇒ 4 (주문생성 비회원) | 알파리뷰 | 강제 생성됨 | (경로: 알림톡) 최초 작성 시점 |
| 리뷰 요청 > 빠른 리뷰 | 2 (빠른리뷰) | 입력한 고객정보(이름, 전화번호)와 일치하는 고객 데이터가 1) 존재하면 ⇒ 해당 고객의 customer_type 값을 그대로 가져옴 2) 존재하지 않으면 ⇒ 4 (주문생성 비회원) | 알파리뷰 | 강제 생성됨 | (경로: 알림톡) 최초 작성 시점 |

⚠️ **미검증 리뷰는 "빠른 리뷰"를 쓸 수 없음**

⚠️ **리뷰 요청 관련 예외 케이스 (개선 필요)**
- shop에서 "비회원은 리뷰를 작성할 수 없다"로 설정해놨더라도, 리뷰 요청 생성 시 비회원도 리뷰를 작성하는 것은 가능함
- 적립금 지급 → 해당 업체의 "비회원 관련 설정(지급 수준)"을 따름
- 포인트 설정 화면 정책 → 비회원 작성불가로 설정 시 포인트 설정 영역에서 비회원 미표시
- 단, 화면에서는 노출되지 않더라도 default 값이 있기 때문에, 이 default 값 기반으로 리뷰 요청을 통해 리뷰를 작성한 비회원에게 적립금이 지급됨 → **비회원 포인트가 적용은 되는데 설정 UI는 표시되지 않는 비정합 플로우**
- 작성 제외 상품/카테고리에 속한 상품에 대해서 리뷰 요청 시, 알림톡은 발송되나 리뷰 작성은 불가함 (리뷰 작성 페이지 진입 시 "작성 가능한 리뷰가 없다"라고 뜸)
- 해당 부분 개선 시 리뷰 요청의 처리 속도가 빨라질 것으로 기대됨 (카테고리 정보는 코어(Core) 쪽에 있기 때문에 속도 이슈 있음)

### 📖-5. 수정 / 복사 시 creater_type 상태 전이 규칙

**리뷰 수정 시** (A → B 상태 전이)
- 기존 리뷰 → `creater_type = 100` (원본)으로 변경됨
- 수정된 버전의 리뷰 → 원본 리뷰의 `creater_type` 값을 그대로 따라감

**리뷰 복사 시** (A → B 상태 전이)
- 원본 리뷰 → `creater_type` 그대로 유지
- 복사된 리뷰 → `creater_type = 29` (복사)로 설정됨

⚠️ **리뷰의 종류가 아닌 공통 속성 목록**
- 사진이 포함된 리뷰 / 동영상이 포함된 리뷰 / 설문 리뷰 / 키워드 리뷰
- 위 항목들은 리뷰의 종류가 아니라 **모든 리뷰들이 공통으로 보유하는 속성(Attribute)** 임

# [[에이전틱 알파앱스 서비스 컨셉]]

## 📌 한 줄 통찰 (The Karpathy Summary)

> 알파앱스의 개별 솔루션(리뷰·업셀·푸시·챗·캔버스·애즈)을 **세팅→운영↔성장** 3축으로 통합하고, AI 에이전트가 매일의 운영을 자율 수행하여 "매일 로그인할 필요 없는 쇼핑몰"을 실현하는 전략적 리브랜딩 구상이다.

## 📐 도메인 모델 개요 (Domain Model Overview)

| # | 도메인 | 핵심 개념 |
|---|--------|-----------|
| 1 | 통합의 필연성 (Why) | 솔루션 다변화 → 인지 범위 초과 → 도구(Tool)에서 결과(Outcome) UX로 전환 필요 |
| 2 | 이커머스 운영 활동 분류 | 상품화 → 쇼핑몰 운영 → 판매 부스팅 → 채널링의 4단계 가치사슬 |
| 3 | 자동화 대상 우선순위 | 고반복·고시간·고난도·고의존 4축 매트릭스로 뾰족한 타겟 식별 |
| 4 | 축 1: 세팅 | 회원가입 + 카페24 연동 → 10분 내 전 제품 기본 세팅 완료 |
| 5 | 축 2: 운영 | 매일 로그인 불필요, 주 1회 보고서, 사람 개입 필요 시에만 알림 |
| 6 | 축 3: 성장 | 3C 분석 → 캠페인 기획 → 소재 생성 → 집행 → A/B 보고의 자율 사이클 |
| 7 | 마스터 에이전트 비전 | 3축 전체를 관통하는 상위 에이전트. 모든 API 접근 권한 기반 자율 의사결정 |

## 📖 핵심 도메인별 상세 (Domain Deep-Dive)

### 📖-1. 통합의 필연성 — 왜 에이전틱 알파앱스인가

**문제 인식 3가지:**

| 문제 | 현상 | 근본 원인 |
|------|------|-----------|
| 인지 범위 초과 | 리뷰·업셀·푸시·챗·캔버스·애즈가 개별 솔루션으로 존재 → 고객이 인지하지 못하고 가치 전달 모호 | 프로덕트 다변화 + 기능 다변화 + 제품 간 UX 일관성 부재 (화이트리스트/블랙리스트 체계 차이) |
| 도구 → 결과 전환 필요 | Tools 접근법으로는 아무리 편하게 만들어도 고객이 어려워함 | 완전히 새로운 UX 패러다임이 필요 |
| 도메인 재설계 필요 | 기능들을 고객이 이해할 수 있는 형태로 도메인별 재배치 필요 | 마케팅/세일즈 이해 용이성 + 다기능 시너지 성과 + 편의성 증대 동시 달성 |

**통합 알파앱스의 목표:**
- 모든 기능은 **매출 향상 그 자체** 또는 **매출 향상을 위한 고객의 어려움 해결**에 수렴
- 더 쉬운 이해 → 더 많은 기능 사용 → 더 큰 성과 확보 선순환

---

### 📖-2. 이커머스 운영 활동 분류 — 가치사슬 4단계

| 단계 | 활동 | 세부 내용 |
|------|------|-----------|
| **상품화** | 상세페이지·썸네일·광고소재 제작 | 자사몰 생성, 각종 쇼핑몰 업로드, 디자인 수정 |
| **쇼핑몰 운영** | 가격 변동, 세팅 일괄 처리, 배너 수정 | 자사몰 + 외부채널 + 아마존 가격 관리, 채널톡·리뷰·업셀·푸시·웹사이트 개선 |
| **판매 부스팅** | 씨딩, 캠페인 | 자사몰 중심. 애즈·업셀·푸시 캠페인 |
| **채널링** | 외부채널, 글로벌 확장 | 글로벌 + 3PL + CS. 판매 에이전트(DM→판매→결제), 국가별 메인 유통처 진출, CS 자동 응대 |

**⚠️ 글로벌 확장 비전 (장기):**
- 판매 에이전트: 글로벌 DM 발송 → 1:1 판매 → 딜 → 결제 수금까지 자동화하는 앱
- 국제 배송이 쉬운 국가 우선 선정, 국가별 메인 유통처 1개씩 진출 (한국=쿠팡, 미국=아마존 등)
- CS 에이전트: 해당 서비스에 네이티브한 답변을 자동 생성·응대
- 글로벌e 같은 역할 포지셔닝

---

### 📖-3. 자동화 대상 우선순위 — 4축 매트릭스

| 축 | 대상 업무 | 자동화 난이도 |
|----|-----------|---------------|
| **고반복** | 상품 진열(추천), 자동화 메시지, 팝업, 리뷰 관리, 상담 | 가장 먼저 자동화 가능 |
| **고시간** | 배너 제작, 상담 | 생성 AI 활용 가능 |
| **고난도** | CRM 전략(세그먼트 분리, 실험, 이터레이션 반복), 광고 집행 | 에이전트 판단 아키텍처 필요 |
| **고의존** | 프로모션/가격, 상담, CRM | 사람 판단이 가장 오래 남는 영역 |

---

### 📖-4. 축 1: 세팅 (Setup)

| 구분 | 내용 |
|------|------|
| **현재 상태** | 각 제품의 개별 온보딩이 파편화되어 있음 |
| **목표 상태** | 회원가입 + 카페24 연동 → **10분 내** 알파앱스 전 제품 기본 세팅 완료 |
| **목표 경험** | 회원가입과 카페24 연동만 하면, 10분 안에 쇼핑몰 전체(스킨·위젯·SEO·가격 구조·알림톡 템플릿 기본값)가 내 브랜드에 최적화되어 세팅 완료 |

**핵심 작업:**
1. 공통 온보딩 레이어 설계
2. 외부 플랫폼 인증 통합
3. 브랜드 기반 기본값 자동 생성

---

### 📖-5. 축 2: 운영 (Operations)

| 구분 | 내용 |
|------|------|
| **현재 상태** | Canvas·Review·Chat이 각자 운영 작업을 줄이지만 통합되지 않음. 사람 개입 경계가 불분명 |
| **목표 상태** | "매일 로그인할 필요 없는 쇼핑몰". 사람이 필요한 순간에만 알림 |
| **목표 경험** | 매일 쇼핑몰에 로그인할 필요 없음. 주 1회 보고서만 확인. 1:1 접점이 필요할 때만 알림 |

**핵심 작업:**
1. 운영 대시보드 통합
2. 운영자 개입 필요 순간 정의 (에스컬레이션 규칙)
3. 좁은 맥락(narrow context) AI 판단 품질 고도화

---

### 📖-6. 축 3: 성장 (Growth)

| 구분 | 내용 |
|------|------|
| **현재 상태** | 알파업셀·알파푸시가 프리셋 기반으로 동작. 알파애즈 개발 예정 |
| **목표 상태** | 3C 기반 캠페인 기획·소재 생성·집행·A/B 보고의 자율 사이클 |
| **목표 경험** | 다음 달 매출 목표를 입력하면, 에이전트가 3C 분석 → 캠페인 기획 → 소재 생성 → 집행 → A/B 결과 보고까지 자율 수행. 경영자는 결재와 인사이트 리뷰만 |

**3C 분석 프레임:**
- **Company**: 자사 데이터 (매출 추이, 재고, 가격 구조)
- **Customer**: 고객 데이터 (세그먼트, 행동 패턴, LTV)
- **Competitor**: 경쟁사 데이터 (가격, 프로모션, 시장 동향)

**핵심 작업:**
1. 매출 3제품 완성 (업셀·푸시·애즈)
2. 에이전트 판단 아키텍처 구축
3. 3C 데이터 파이프라인 구축

---

### 📖-7. 마스터 에이전트 비전 — 3축 통합 상위 레이어

- 3축(세팅·운영·성장) 전체가 연관되는 유즈케이스는 아직 구체화되지 않음
- 그러나 **"목적 없는 기능"**이 필요: 상품 정보 수정, 재고 정보 수정, 주문 정보 수정 등
- 모든 API에 접근할 수 있는 권한을 획득한 상태에서 마스터 에이전트가 자율 의사결정 실행
- 이는 세팅·운영·성장의 경계를 넘나드는 범용 에이전트 계층

⚠️ **미결 사항:** 마스터 에이전트의 의사결정 범위와 사람 승인(Human-in-the-Loop) 경계가 아직 정의되지 않음. 매출·가격·재고 등 비즈니스 크리티컬 액션의 자율 수행 범위 결정 필요.

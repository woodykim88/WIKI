# [[알파리뷰 포인트 지급 정책]]

## 📌 한 줄 통찰 (The Karpathy Summary)
> 알파리뷰의 포인트 시스템은 "기본 지급 조건 + 다중 추가 혜택 + 결제 상태/기간 기반 제한 로직"을 결합하여, 리뷰 작성의 양과 질을 동시에 확보하면서도 어뷰징과 오지급을 방지하는 정교한 상태 머신(Status Machine)으로 동작한다.

## 📐 도메인 모델 개요 (Domain Model Overview)

| # | 도메인 | 핵심 개념 |
|---|--------|-----------|
| 1 | 지급 대상 리뷰 | 엔드 유저 직접 작성 리뷰만 대상. 외부 연동/관리자 리뷰는 기본 제외, 네이버블로그 소셜리뷰(beneficiary_id 존재 시) 예외 |
| 2 | 포인트 지급 상태 머신 | 1x(대기계), 2x(완료/취소/제한/실패계), 3x(회수계), 40(추가지급) - 15개 상태 코드 |
| 3 | 포인트 산정 (정액/정률) | 리뷰 종류별 정액/정률 혼용, 회원/비회원 별도 설정, 최소/최대값 제한 |
| 4 | 추가 포인트 (Incentive) | 양질 유도(장문/다이미지/영상), 회원 최초(포토/SNS), 상품 선착순, 조기 작성 |
| 5 | 제한 로직 (Safety Nets) | 특정 상품/카테고리 제외, 주문당 선착순, 금액 미만 제한, N일 경과, 품절/미진열/미판매 |
| 6 | 취소/회수 메커니즘 | 지급완료 → 회수, 지급대기 → 취소. 자동(주문 취소/반품) + 수동(관리자) |
| 7 | 지급 방식 (자동/수동) | 자동(N일 후/즉시), 수동(게시/지급 설정 "수동" 또는 미검증 리뷰) |

## 📖 핵심 도메인별 상세 (Domain Deep-Dive)

### 📖-1. 지급 대상 리뷰

**포인트 지급 가능 리뷰:**
- [1] 엔드 유저가 알파리뷰에서 직접 작성 (자발적): 텍스트, 빠른, SNS 리뷰
- [2] 엔드 유저가 알파리뷰에서 직접 작성 (관리자 요청): 미검증, 리뷰 요청

**포인트 지급 불가 리뷰:**
- 외부 연동 리뷰 (카페24, CSV 이관, 관리자, 스마트스토어, 인스타 소셜)
- ⚠️ **예외:** 네이버블로그 소셜리뷰이며 `beneficiary_id`가 존재할 때는 지급 가능

**혜택 지급 버튼 미표시 조건 (creater_type 기준):**
- 관리자(5), 시스템 수집(6), 복사된 리뷰(29), 원본 리뷰(100)
- ⚠️ **예외:** 네이버블로그 소셜리뷰 + beneficiary_id 존재 시 표시

**수정된 리뷰 관련:**
- 원본 리뷰(100): 포인트 버튼 미표시
- 복사된 리뷰(수정된 리뷰): 원본 리뷰의 포인트 버튼을 대신 표시
- 찐 복사(29): 복사된 리뷰에는 포인트 버튼 미표시

### 📖-2. 포인트 지급 상태 머신

| 코드 | 상태명 | 설명 |
|---|---|---|
| 10 | 지급예정 | 지급 시간이 정해져 있을 때 |
| 11 | 지급대기 | 지급 시간이 정해져있지 않을 때 (default) |
| 12 | 비회원지급대기 | 지급예정 이후 비회원으로 대기 중 |
| 13 | SNS지급대기 | SNS 유지기간 미달성 |
| 20 | 지급완료 | 실제 엔드유저에게 포인트 지급 완료 |
| 21 | 지급취소 | 포인트 지급 완료 전 고객사에 의해 취소 |
| 22 | 자동지급취소 | 취소/반품/교환으로 인한 자동 취소 |
| 23 | 지급제한 | 고객사 설정에 의한 제한 |
| 24 | 지급실패 | 지급 시도했으나 알 수 없는 사유 (ex: 카페24 설정)로 실패 |
| 25 | 지급포인트없음 | 리뷰 작성했으나 지급 포인트가 0원으로 예약 |
| 30 | 회수예정 | 고객사가 회수 선언했으나 미실행 |
| 31 | 회수완료 | 엔드유저 포인트 회수 완료 |
| 32 | 회수실패 | 포인트 잔액 부족 등으로 회수 실패 |
| 40 | 추가지급완료 | 관리자 수동 추가 포인트 지급 완료 |

**상태 전이 규칙:**
- 리뷰 생성 → 지급대기(11) [default]
- 지급대기(11) → 지급예정(10) [트리거: 지급 시간 확정]
- 지급대기(11) → 비회원지급대기(12) [트리거: 비회원 리뷰]
- 지급대기(11) → SNS지급대기(13) [트리거: SNS 유지기간 미달]
- 지급예정(10) → 지급완료(20) [트리거: 지급 실행 성공]
- 지급완료(20) → 회수예정(30) [트리거: 관리자 회수 선언 또는 주문 취소/반품]
- 회수예정(30) → 회수완료(31) [트리거: 포인트 차감 성공]
- 회수예정(30) → 회수실패(32) [트리거: 잔액 부족]
- 지급완료(20) → 추가지급완료(40) [트리거: 관리자 수동 추가 포인트]

### 📖-3. 포인트 산정 (정액/정률)

**기본 산정 방식:**
- **정액제:** 관리자 입력 액수 그대로 지급 (ex: 300 입력 → 300P)
- **정률제:** 상품 가격 x 입력 퍼센트로 계산 (ex: 1% 설정, 상품 10,000원 → 100P)
- **상품 가격 계산 공식:** `product_price + option_price - additional_discount_price`
- 리뷰 종류별/회원 여부에 따라 정액제와 정률제 **혼용 가능**

**회원/비회원 별도 설정:**
- 비회원 종류: 일반 비회원, 네이버페이, 지그재그, 톡 체크아웃, 기타 마켓플러스
- 비회원 지급 방식: 관리자 승인 후 → 비회원지급대기(12) → 회원가입 유도 알림톡 발송 → 지급 신청 페이지에서 ID 입력 → 지급

**리뷰 종류별 포인트 지급 조건:**

| 리뷰 종류 | 지급 조건 |
|---|---|
| 별점리뷰 | 빠른 리뷰 작성 시에도 별점리뷰 포인트 지급 |
| 텍스트리뷰 | 최소 글자 수 이상 작성 필요. "제한 없음" 설정 시 1자 이상 |
| 포토리뷰 | 최소 포토 수 이상 첨부 필요 |
| 동영상리뷰 | 최소 길이 조건 이상 동영상 첨부 필요 |
| 설문리뷰 | 작성 당시 모든 설문 항목에 답변 필요 |
| SNS리뷰 | 별도 조건 없음. 어드민 연동 시 자동 지급. 유지 일수는 포인트와 무관 |

**포인트 최소/최대값:**
- 일반리뷰 최대값: 정액(nP) 또는 정률(상품 금액의 n%P) 중 택 1
- "추가포인트는 한도에서 제외" 설정 가능
- 최소값: 정률제 지급 시 최소 보장 금액 설정 가능 (상품 금액의 n% < 최소값이면 최소값으로 대체)
- **추가 적립금 제한:** 1개 리뷰에 여러 추가 포인트 수령 가능 시 **최대 금액 1개만 지급** 설정 가능
- ⚠️ 리뷰 수정 시(지급완료 전 상태): 수정 시점에 맞게 포인트 재계산

### 📖-4. 추가 포인트 (Incentive)

추가 포인트는 **정액제 전용**, 회원/비회원 무관 공통 적용, 기본 포인트에 누적 지급

**양질의 리뷰 유도:**

| 트리거 | 설정 방식 | 예시 |
|---|---|---|
| 장문 리뷰 혜택 | 다중 조건 가능 | 100자 이상: 1,000P / 200자 이상: 2,000P |
| 많은 이미지 혜택 | 다중 조건 가능 | 3장 이상: 500P |
| 영상 길이 혜택 | 다중 조건 가능 | 10초 이상: 1,000P |

- 다중 조건 중 **최고 조건(더 큰 기준값)** 1개만 적용
- ex) 기본 200P + 200자 이상 추가 2,000P = **2,200P**

**회원 최초:**

| 트리거 | 기준 |
|---|---|
| 포토리뷰 최초 | 알파리뷰 DB상 해당 사이트의 "고객"이 처음 포토리뷰 작성 (쇼핑몰 회원 기준이 아님) |
| SNS리뷰 최초 | 동일 |

**상품 선착순:**
- 포토/동영상/SNS 리뷰에 대해 해당 상품의 작성 등수에 맞는 포인트 지급
- 등수 기준: 알파리뷰에서 "고객" 기준

**조기 작성 혜택:**
- 리뷰 종류와 무관, **배송완료일에서 N일 이내** 작성 시 추가 포인트

### 📖-5. 제한 로직 (Safety Nets)

| 제한 조건 | 설명 |
|---|---|
| 특정 상품/카테고리 | 지정된 상품/카테고리는 지급 대상에서 제외 |
| 주문당 선착순 | 동일 주문 건 내 최대 n건까지만 지급. 작성한 리뷰 중 **가장 큰 포인트**를 받을 수 있는 n건만 지급. 선착순 외 리뷰에 기본 별점 포인트만 지급하는 옵션 있음 |
| 구매 상품 금액 n원 미만 | 상품 금액 < n원이면 지급 제한. 상품 가격 = `product_price + option_price - additional_discount_price` |
| 상품 수령 후 N일 경과 | 혜택 지급 종료일 = `shipped_at + 3일 + 업체 설정 일수` |
| 품절/미진열/미판매 상품 | **리뷰 작성 시점**의 상태를 기준으로 판단 |

- ⚠️ 리뷰 수정 시(지급완료 전 상태): 수정 시점에 맞게 포인트 재계산 및 제한 재판단

---

### 📖-6. 취소/회수 메커니즘

**개념 분리:**
- **취소:** 지급예정(10), 지급대기(11), 비회원지급대기(12) 상태 → 이미 지급되지 않은 경우
- **회수:** 지급완료(20) 상태 → 이미 지급된 포인트를 빼앗는 경우

**자동 취소/회수:**
- 주문 상품이 취소/반품/교환 상태일 때 자동 회수 설정 가능

**수동 취소/회수:**
- 리뷰 목록에서 개별 건에 대해 전체 취소/회수 가능
- 사유 입력 필수

**수동 추가포인트 지급:**
- 관리자가 포인트 액수를 입력하여 수동 추가 지급
- **포인트 상세 내역 모달에서 포인트 지급 상태와 무관하게 항상 가능**
- 추가포인트는 "추가혜택"이라는 문구로 로그에 기록

---

### 📖-7. 지급 방식 (자동/수동)

**자동 지급:**
- N일 후 자동 지급
- 즉시 지급

**수동 지급 (강제):**
- 게시/지급 설정에서 "수동" 지급 사용 시
- "자동" 지급 옵션이라도 예외 조건에 충족된 경우
- **미검증 리뷰: 무조건 수동 지급**

**관리자 수정 리뷰 원칙:**
- 수정 발생 시 이후의 reward 관련 값 핸들링은 전부 **수정된 리뷰**(편집본)에서 처리하고 출력

## ⚠️ 모순 및 업데이트 (Contradictions & RL Update)

- **[2025-11-11] 원본 문서 작성 시작:** 담당자 미지정(비어 있음), 상태 "시작 전". 포인트 지급 플로우 중 "실제 지급까지의 플로우" 섹션이 미완성(ㅁㄴㅇㄹ로 표기).
- **[미확인] 자동 취소/회수 타이밍:** 원본 문서에 "취소/반품/교환 접수??"로 불확실 표기. 정확한 트리거 시점(접수 vs 완료)이 확정되지 않음.
- **[비정합] 비회원 포인트 지급 경로:** 리뷰 요청으로 생성된 비회원의 경우, shop 설정에서 비회원 포인트 UI가 숨겨져 있어도 default 값으로 지급되는 비정합 존재 (리뷰 종류 정리 문서의 ⚠️ 항목과 교차 참조).

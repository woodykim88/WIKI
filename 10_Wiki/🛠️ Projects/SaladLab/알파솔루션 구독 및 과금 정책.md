## 📖 핵심 도메인별 상세 (Domain Deep-Dive)

### 📖-1. 청구서 체계

구독은 고객사의 서비스 사용권 '주문서'이자, 공급자(샐러드랩) 입장의 '청구서(Invoice)'이며, 시스템상 기능 구동 명령/상태의 개념이다.

**청구서 분류 3종:**

| 청구서 분류 | 청구서 종류 | 세부 종류 | 상태 | 비고 |
|---|---|---|---|---|
| 서비스 청구서 | 무료 버전 구독 | 무료 버전 구독 | 사용중, 완료, 취소(구독변경), 구독 중단 | |
| | 체험 구독 | 체험 구독 | 대기중, 사용중, 완료, 취소(구독변경), 구독 중단 | 기간 종료 시 자동 구독 변경 |
| | 과금 구독 | 정기 구독 💳 | 대기중, 사용중, 완료, 취소(구독변경), 결제 실패, 구독 중단 | 사이트에 단 1개만 존재 가능 |
| | | 계약 구독 💳🏦 | 예약중, 대기중, 사용중, 완료, 취소(구독변경), 결제 실패, 구독 중단 | |
| 보조 청구서 | 메시지 포인트 | 자동 충전 💳 | 대기중, 완료 | 잔액 미만 시 자동 충전 |
| | 설치/제거 | 설치 요청 💳 | 접수 전, 접수 완료, 접수 취소, 작업 완료 | 유료 |
| | | 제거 요청 💳 | 접수 전, 접수 완료, 접수 취소, 작업 완료 | 유료, 위젯 유지 가능 |
| | | 직접 설치 | 직접 설치중, 직접 설치 완료, 직접 설치 취소 | |
| | | 직접 제거 | 완료 | |
| 임의 청구서 | 기타 결제/환불 | 기타 결제 💳🏦 | 완료 | 예외적 정산/통계 조정용 |
| | | 기타 환불 🏦 | 완료 | |
| | 예치금 | 예치금 입금 💳🏦 | 완료 | 서비스 구동과 무관 |
| | | 예치금 환불 🏦 | 완료 | |
| | 크레딧 | 크레딧 충전/제거 | 완료 | |
| | 미납금 | 미납금 결제 💳🏦 | 완료 | |
| | | 미납금 제거 | 완료 | |

**서비스 비용 구성:**
- 서비스 비용 = **프로덕트 비용** + **패키지 비용**
- 패키지: 프로모션팩, 위젯팩, 푸시팩 등 자유롭게 추가/제거 가능
- 가격 변동: 주문 수에 따라 플랜이 구분되며, 규모에 따라 프로덕트/패키지 비용이 변동
- **할인:**
  - 일반 할인: 결합할인 + 연간할인 → 서비스비용에서 차감
  - 추가 할인(쿠폰): 일반 할인 후 잔액에서 할인율 시작. **쿠폰 할인 항목별로 가장 높은 할인 1개만 적용**

### 📖-2. 기능 활성화/비활성화

서비스 청구서는 어떤 기능을 켜고 끌 것인가의 근거가 되며, 이를 명시적으로 구현한 것이 "기능 실행/중지" 개념이다.

**4가지 제어 영역:**
1. 대시보드 접근
2. 주요 기능
3. 메시징 서비스
4. 위젯

**슈퍼관리자 오버라이드 규칙:**
- 백오피스 > 쇼핑몰 정보 > 운영에서 기능 실행/중지 팝업으로 조작
- 팝업에서 설정한 값은 **최우선순위 예외**로 별도 저장
- ⚠️ 예외: 온보딩 중 위젯 사용 불가 상태라도, 슈퍼관리자가 강제로 ON 가능
- 프로덕트별 기능 실행/중지는 **구독없음** 및 **서비스중단** 상태에서는 상태고정 유무와 무관하게 무시되고 비활성화되며, 다른 상태로 변경 시 고정 설정이 되살아남

**용어 구분:**
- **구독 종료:** 모든 서비스 청구서를 구독 중단 상태로 만들어 어떠한 종류의 구독도 남기지 않는 것 → "구독 종료" 상태 (해지 전 단계)
- **구독 중단:** 청구서 상태를 '중단'으로 만드는 것. 서비스 청구서 각각에 '구독중단' 상태 존재
- **기능 중지:** 구독과 관계 없이 기능의 동작을 제한하는 행위

### 📖-3. 구독 생성 (시작/수정)

구독 시작 또는 구독 수정 시, 고객사의 온보딩 및 프로덕트 활성화(업그레이드/다운그레이드)에서 구독 생성 명령이 가능하다.

**구독 생성 명령의 하위 기능:**

| 기능 | 고객사 | 슈퍼관리자 |
|---|---|---|
| 과금 구독 생성 | 온보딩/업다운그레이드 시점에 정기 구독 생성 가능 | 정기구독 및 계약구독 생성 가능 (가격조건, 구독 기간 조건 설정) |
| 구독 시작일 지정 | 체험구독 생성 유무에 따라 자동 결정 | 현재/미래/과거 시점 직접 지정 가능 |
| 체험 구독 생성 | 프로덕트별 사이트 기준 최초 1회만 | 미래시점 지정 시 오늘~시작일 사이에 체험구독 생성 |
| 서비스청구서 비활성화 | 해당 없음 | 필수 적용 |
| 예약중 구독변경 불가 | 해당 없음 | 필수 적용, 슈퍼관리자만 수정/중단 가능 |

**프로덕트 활성화 프로세스:**
- 최초 사이트 기준으로 최초 1회만 체험구독 자동 생성
- 설치 요청서가 발생하는 프로덕트라면, "서비스 개시" 또는 "직접설치완료" 시 체험구독 생성
- 현재 대기중이거나 사용중인 모든 서비스 청구서 존재 시 → 수정 시행 날짜에 "취소(구독변경)" 처리
- 예약중인 구독 존재 시 → 해당 예약 명령 제거

**구독 변경 및 결제 시점 옵션 (미래 시작일인 경우):**
- 기본값: A옵션 "즉시 구독 변경 및 구독 변경일 당일 11시 결제"
- B옵션: "즉시 구독 변경 및 즉시 결제" (정기구독 사용 불가, 계약구독만 가능)
- C옵션: "구독 시작일까지 구독 유지 및 변경일 당일 11시 결제"
- ⚠️ 구독 시작일이 "오늘" 또는 "미래"일 때 구독률이 0%로 책정 → 변경 당일 사용비용 부과 안 함

**첫 과금 구독의 결제 실패 정책:**
- 슈퍼관리자의 예약구독 결제 실패 시 → 갱신처럼 매일 결제 재시도
- 체험구독 종료 후 과금구독 결제 실패 시 → 구독 취소 및 무효화 (이전 구독 있으면 이전 구독 복원)

**카드 미등록 상태:**
- 포인트 잔액만큼 메시지 발송 가능 (무료 버전/체험구독)
- 잔액 이상 발송 원할 경우 포인트 충전, 카드 등록 필요
- 체험구독 종료(14일) 후 마지막 시점(15일 자정)에 과금구독 없을 경우 → 프로덕트 "서비스 중단"
- ⚠️ 알파 푸시는 선물하기 기능 때문에 예외 처리

### 📖-4. 구독 주기와 월간 시스템

**구독 주기 종류:**
- 월간 구독: 결제 일자에 맞춰 매월 갱신
- 연간 구독: 결제 일자에 맞춰 매년 갱신
- 계약구독: 3/6/9개월 구독도 가능

**과금 기준:** 일수와 무관하게 결제주기에 따라 비용 부과
- ex) 알파리뷰 89,000원 x 3개월 구독: 91일이든 92일이든 267,000원

**월간 시스템 주기:**
- 구독의 결제 일자부터 발생하며, 연간/계약 구독이더라도 내부적으로 월간 주기가 작동
- 이 주기에 따라 시스템 알람, 한도 초기화, 무료 메시지 포인트 재충전이 진행

**월간 구독 결제일자 결정:**
- 첫번째 과금 구독 생성 시: 해당 월의 마지막 일자까지만 일할계산하여 구독 생성
- 다음 월부터: 1일~말일까지 1개월 구독 생성
- **결제일자는 1일로 고정**
- ex) 15일에 첫 과금구독 시작 → 15일~말일 구독 생성 → 다음 월(M+1)부터 1일~말일로 갱신

**연간 구독 결제일자 결정:**
- 23.03.01 ~ 24.02.29(366일) vs 23.03.01 ~ 24.02.28(365일) → **366일이 맞음**
- ⚠️ **윤달 처리:**
  - 윤달 2월 28일(29일이 아닌 날짜) 결제 시작 시 → xx.2.28 ~ xx.2.27 (366일)
  - 2월 29일 → 29일 결제일 불가 → **28일을 결제일로 지정**
  - 윤년 2월 29일: x년 29일 ~ 2월 28일 365일 → **28일로 결정**

**⚠️ 연간 구독의 마지막 월간시스템주기 예외:**
- 예: 3월 28일 연간결제 시작 → 첫 번째 월간시스템주기: 3월 28일~4월 31일
- 마지막 월간시스템주기: n+1년 3월1일~3월31일이 **아니라** n+1년 3월1일~3월27일로 지정되어야 함

**⚠️ 구독변경 시 월간시스템주기 이슈:**
- 연간구독 중간에 구독변경 시, 해당 날짜로부터 기존 월간시스템의 마지막 종료일까지 1개월 미만의 첫 번째 월간시스템주기 생성

**구독주기 변경 시 결제일자:**
- 월간 → 연간: 변경 당일이 연간구독의 결제 시작일자로 변경

### 📖-5. 플랜 한도

**용어 정의:**

| 명칭 | 내용 |
|---|---|
| 집계 기준 | 주문수 (카페24주문 + 네이버페이 간편주문 포함, 환불수 미감안) |
| 집계 기간 | 한도 사용량의 기준이 되는 기간 (= 월간 시스템 주기) |
| 한도 사용량 | 집계 기간 내 누적 수치의 합 (분자) |
| 플랜 한도 | 해당 플랜에서 제공하는 집계 기준 수치의 한도 (분모) |
| 월간 한도 사용률 | 한도 사용량 / 플랜 한도. 매일 자정 업데이트 |

- ex) 5.1~5.30(집계 기간)의 주문건(집계 기준) 500건 이하(플랜 한도), 현재 350건(한도 사용량) → 사용률 350/500 = 70%

**집계 기간:** 월간 시스템 주기를 따르며, 종료 후 다음 주기 시작 시 한도 0으로 초기화

**⚠️ 구독 변경 시 한도 사용률 일할 환산 공식:**
- 기존 구독의 플랜 한도 사용률 = 구독변경 당일까지의 집계 기간 내에 플랜 한도를 해당 기간의 일수로 일할 계산하여 **반올림** 환산
- ex) 기존 집계 기간 4.1~4.30 → 변경 후 기간 4.1~4.10(10일): 33.333% → 플랜 한도 100개 → **33개**, 한도 사용량 10개 / 33개 = **30.333%**
- 새로운 구독: 집계 기간 4.10~4.30(21일): 70% → 플랜 한도 100개 → **70개**

**한도 초과 정책 (3분기 규칙):**

| 사용률 구간 | 동작 |
|---|---|
| 80% 도달 | 안내 알림 시작 |
| 100% ~ 105% | 기능 유지, 상위 플랜 변경 유도 |
| 105% 초과 | **위젯 포함 모든 기능 중단** |

**안전 한도 옵션 사용 시:**
- 105% 넘어도 **무제한 유지** 혜택 제공
- 다음 주기 전에 알맞는 상위 플랜을 선택하게 유도, 새 결제 진행
- 한도 초과 상태 시 하위 플랜 변경 비활성화

**안전 한도 미사용 시:**
- 한도 초과 상태에서도 구독 변경 시 낮은 플랜 사용 가능
- 기능은 멈추나 구독변경은 가능

**한도 무제한 옵션 (슈퍼관리자 전용):**
- 특별한 니즈 시 한도 고정, 알람 작동 안 함
- 역마진 발생 시 재계산 필요

**연간구독 내 한도 데이터 변경:**
- 현재 활성화된 서비스 구독의 한도 관련 데이터는 집계 기간에 맞춰서 지속적으로 업데이트

### 📖-6. 구독 변경 (Mutation)

**용어:**
- **기존 구독:** 변경 전의 구독
- **변경된 구독:** 기존 구독의 사용기간과 비용을 변경 시점으로 재계산한 구독
- **새로운 구독:** 변경 내용이 반영된 새 구독

**상태 변동:**
- 기존 구독 → "사용중-취소" 상태 (원본으로 보관)
- 변경된 구독 → 청구번호 n-1, "완료" 상태로 생성
- 새로운 구독 → 새 청구번호, 활성화

**구독 변경 원리 (3단계):**
1. "기존 구독"을 **모두 환불** (크레딧, 예치금, 현금 모두 환불, 현금은 예치금으로 환불)
2. "변경된 구독" 가격을 환불한 크레딧/예치금으로 결제 (위약금으로 변경된 구독이 더 비쌀 수 있음 → 위약금은 새로운 구독으로 이월)
3. "새로운 구독" 비용을 결제 (위약금 포함)

**📊 "변경된 구독"의 비용 산출 (구독률 공식):**
- `is_shared_day` 옵션으로 변경된 구독의 하루치 금액 부과 여부 결정
- **구독률 = 사용일수 / 구독 기간 (소수점 6째 자리에서 반올림)**
- ex) ① 기존 구독기간 4.25~5.24(30일) ② 5월 8일에 구독 변경(4.25~5.7, 13일) ③ 구독률 13/30 = 0.43333[...] (소수점 6째 자리 반올림)
- 다수 프로덕트 이용 시: 각각 위 과정 진행하여 전체 서비스 비용 산출. 할인 금액도 가격 재산정에 따라 재산정

**📊 "새로운 구독"의 가격 산출:**
- ex) ① 새 구독 기간 5/8~5/24 ② 17일 / 이전 구독 일수 30일 → 17/30 = 56.666% ③ 정가 x 56.666%

**업그레이드 종류:**

| 종류 | 설명 | 구독내역 표시 |
|---|---|---|
| 프로덕트/패키지 변경 | 프로덕트 또는 패키지 추가 | 프로모션팩 추가 사용(업그레이드) |
| 플랜 변경(수동) | 더 높은 플랜으로 수동 전환 | 라이트 → 베이직 플랜 수동변경(업그레이드) |
| 플랜 변경(한도 안전옵션) | 자동으로 더 높은 플랜 전환 | 라이트 → 베이직 플랜 안전옵션(업그레이드) |
| 구독 주기 변경 | 1→3→6→9→12 방향 | 월간→연간구독 전환(업그레이드) |

**다운그레이드 종류:**

| 종류 | 설명 | 구독내역 표시 |
|---|---|---|
| 프로덕트/패키지 변경 | 사용중 프로덕트/패키지 제거 | 프로모션팩 제거(다운그레이드) |
| 플랜 변경(수동) | 더 낮은 플랜으로 수동 전환 | 베이직 → 라이트 플랜 수동변경(다운그레이드) |
| 구독 주기 변경 | 12→9→6→3→1 방향 | 연간→월간구독 전환(다운그레이드) |
| 해지 (무료버전 전환) | 현재 구독 중단, 새 과금 구독 미생성, 무료 버전 구독 생성 | XX → 무료버전으로 전환(다운그레이드) |
| 해지 (구독 중단) | 현재 구독 중단, 새 과금 구독 미생성 | 해지로 인한 구독 중단 |

**⚠️ 다중 변경 사유:** 다운그레이드 사유가 1개라도 있으면 전체를 다운그레이드로 분류
- ex) 연간→월간(다운) & 패키지 추가(업) → 다운그레이드

**업/다운 아닌 경우:** 쿠폰 추가, 결제 실패 구독 중단, 계약구독 수정

**⚠️ 계좌이체 구독 변경 불가:** 계약구독만 지원하며, 구독 변경 불가능

### 📖-7. 구독 종료 및 해지

**구독 종료 가능 경우:**
1. 고객사의 해지요청
2. 슈퍼관리자의 구독 종료
3. 자동 결제 실패 확정으로 인한 미수 상태

**구독 종료 시 기능 동작:**
- 결제일자 초기화
- 프로덕트별 "서비스 중단"
- 대시보드, 시스템, 메시징서비스, 위젯(A1~A4) 중지
- 리온보딩 접근 필수

**구독 종료 해소 방법:**
- 수동 재결제 성공 → 구독 재개 (결제일자 재정의)
- 구독 시작 → 구독 재개 (결제일자 재정의)
- 해지 승인

**해지 승인 이후 데이터 보존 계층:**

| 보존 기간 | 대상 |
|---|---|
| 즉시 제거 | 크레딧, 메시지 포인트, 카드 정보 |
| 1년 유지 | 관리자 기본 정보, 구독 정보 |
| 5년 유지 | 예치금 잔액 |
| 비식별 처리 | 주문/상품/회원/리뷰 정보 비식별 후 저장 |
| 영구 저장 | 쇼핑몰명, 최초도입일자, 해지일자 |

**이탈 관련 상태:**

| 메인상태 분류 | 메인상태 | 종속 상태 | 설명 |
|---|---|---|---|
| 사이트 상태 | 리드 이탈 | 리드 이탈 | 슈퍼관리자에서만 처리 가능 |
| | 구독 종료/해지 완료 | 일시 중지 | 고객사가 대시보드에서 선택 |
| | | 해지 요청 | 고객사가 대시보드에서 선택 |
| | | 결제 실패 이탈 | 결제실패 확정 또는 미수 시 자동 지정 |
| | | 기타 | 슈퍼관리자만 선택 가능 |
| 프로덕트별 상태 | 서비스 중단 | 과금 중 이탈 | 이탈 시점에 과금구독 중 "사용중" 또는 "완료" 상태가 하나라도 있을 때 |
| | | 과금 전 이탈 | 체험구독 사용중 이탈, 무료버전 사용중 이탈, 과금구독이 "사용중"/"완료" 없는 경우 |

**재도입 vs 재설치 (기준 분리):**
- **재도입:** 사이트 레벨. 기준 = 사이트 메인상태 중 "해지완료" 상태
- **재설치:** 프로덕트 레벨. 기준 = 프로덕트 종속상태 중 체험중/과금중/계약구독중을 한 번이라도 거친 경우

### 📖-8. 청구서 종류별 상세

**서비스 청구서 상태:**

| 상태명 | 설명 |
|---|---|
| 대기중 | 구독 생성됐으나 결제 대기중 |
| 사용중 | 현재 구독기간 내 사용중 |
| 완료 | 구독변경/환불 없이 정상 완료 |
| 취소(구독변경) | 구독 변경 사유 발생 |
| 결제 실패 | 카드결제 실패, 구독 활성화 상태에서 재시도중. 성공 시 사용중 복귀, 실패 확정 시 구독 중단 |
| 구독 중단 | 다운그레이드, 해지 등으로 중단 |

**무료 버전 구독:**
- 구독 주기/기간 없음 (무제한 사용)
- 기능 제약 있음: 메시지 발송 제한, 특정 위젯 제한, 설치요청 불가 등

**체험 구독:**
- 사이트 기준 최초 1회만 제공 (프로덕트별 + 패키지별 각각 생성 가능)
- 구독주기 없음, 과금 없음, 영수증 없음, 플랜 한도 무제한
- 기본 14일, 체험 기간 연장 쿠폰/프로모션으로 유동적 변화 가능
- 체험 종료 시 과금 구독으로 자동 변경
- 체험구독 생성 시 "예약중" 상태의 과금구독이 함께 생성
- ⚠️ 같은 날 2개 이상 프로덕트의 체험 구독 종료 시 → 예약구독 1개로 통합

**⚠️ 체험구독 종료 후 결제 실패 정책 (기존 과금구독 존재 시):**
- 청구서 생성 타입: 1.최초 생성, 2.갱신, 3.구독변경
- 3번(구독변경) 케이스에서 기존 과금구독이 있었다면:
  1. 체험구독 종료 시점 익일 00시 과금구독 생성 → 구독변경 발생
  2. 기존 구독(A, 번호1) → 변경된 구독(B, 번호1-1) → 새로운 구독(C, 번호2) "대기중"
  3. 당일 11시 결제 실패 시 → (C)를 "무효" 상태로 Update → (B)도 "무효" 상태로 Update
  4. (B), (C) 환불 → (A)와 동일 조건으로 구독 복원 (청구서 번호 3, Create)
  5. ⚠️ 복원 과정에서 추가 결제 필요 + 결제 실패 시 → 결제 실패 상태로 매일 재결제

**계약 구독:**
- 구독 주기를 자유롭게 설정 가능 (1년이 아닐 수 있음)
- 구독 중단, 수정만 가능
- 금액 조건 커스텀: 무료 메시지 포인트 규모, 메시지 비용, 프로덕트/패키지 가격, 할인 등
- **계좌이체 방식 특이사항:**
  - 유일하게 계좌이체 지원
  - 결제 전에도 청구서 상태 "사용중"으로 시작 → 모든 기능 정상 사용
  - 영수증만 "입금 대기" 상태
  - 카드 미등록 상태로 구독 시작 가능 (카드 등록 면제 필수)
  - **무제한 한도 옵션 필수**

**⚠️ 계약구독 기간이 1년이 아닐 때:**
- ex) 300일 계약, 1월1일~10월27일, 1,000,000원일 경우:
  - 다음 주기: 300일 → 365일로 변경
  - 비용 변경: 1,000,000 / 300 x 365 = 재산정
  - 결제일자: 1일 고정 (결제 자체는 10월 27일에 발생)
  - 다음 구독 기간: 10월 28일 ~ 익년 10월 27일

**설치/제거 요청 프로세스:**

| | 서비스개시 | 서비스중단 | 구독변경 |
|---|---|---|---|
| 직접 설치 | 설치 완료 후 | | 설치 완료 후 |
| 직접 제거 | | 즉시 | (관계없음) |
| 설치 요청 | 설치 완료 후 | | 설치 완료 후 |
| 제거 요청 | | 제거 작업 완료 후 즉시 | (관계없음) |
| 자동 설치 | 즉시 | | 즉시 |
| 자동 제거 | | 즉시 | (관계없음) |

**프로덕트/패키지별 설치/제거 기능표 (240403 기준):**

| | 리뷰 | 업셀 | 푸시 | 프리미엄위젯(롤링) | 프리미엄위젯(브리핑) | 세일즈티커 |

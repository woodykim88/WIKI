# 알파페이지 마이크로사이트 전략

이 문서는 알파앱스를 "쇼핑몰 최적화 도구"에서 "커머스 진입점"으로 전환하기 위한 마이크로사이트 전략을 정리한 문서입니다.

## 1. 지식 메타데이터
- **문서 분류:** 전략 컨셉 문서
- **작성 일자:** 2026-05-07
- **관련 프로덕트:** 알파앱스 전체 (알파리뷰, 알파업셀, 알파푸시, 알파리뷰 인스타피드)
- **핵심 키워드:** 알파페이지, 마이크로사이트, 브랜드 허브, 카페24 추상화, 온보딩 역전

---

## 2. 배경: 왜 마이크로사이트인가

### 2.1. 현재 문제 — 알파앱스는 "세 번째 생각"

상품력 있는 제조업자가 판매를 시작할 때의 의사결정 순서:

```
1번째 생각: "어디서 팔지?" → 스마트스토어 (트래픽이 있으니까)
2번째 생각: "자사몰도 만들까?" → 카페24 (쇼핑몰 빌더니까)
3번째 생각: "잘 팔려면?" → 알파앱스 (최적화 도구니까)
```

알파앱스는 "이미 팔고 있는 사람의 도구"로 포지셔닝되어 있어, 아직 시작하지 않은 판매자의 머릿속에 떠오르지 않는다.

### 2.2. 목표 — 의사결정 순서 역전

```
되어야 하는 구조:
제조업자 → "어떻게 팔지?" → 알파앱스 → (카페24든 어디든 인프라는 알아서 연결)
```

Shopify가 "호스팅 어디 쓰지? → 플러그인 뭐 깔지?"를 "Shopify 하나면 다 된다"로 바꾼 것과 동일한 전환. 호스팅이 사라진 게 아니라, 판매자의 의사결정에서 보이지 않게 된 것.

### 2.3. 왜 풀 쇼핑몰이 아니라 마이크로사이트인가

| 선택지 | 공수 | 리스크 |
|---|---|---|
| 카페24 독립 (풀 커머스 플랫폼) | 2년+, 수십억 | 카페24를 처음부터 다시 만드는 것 |
| 자체 결제(PG) 직접 연동 | 12~18개월 + 규제 | 전자금융업 신고, PCI-DSS, 정산 시스템 |
| **마이크로사이트 (브랜드 허브)** | **4~6개월** | **기존 위젯 자산 재활용, 결제는 외부 위임** |

마이크로사이트는 카페24와 경쟁하지 않고 카페24 **앞단에 위치**한다. Linktree가 인스타그램과 경쟁하지 않고 인스타 바이오 링크에 앉은 것과 같은 구조.

## 3. 판매자 여정 시나리오

> 페르소나: 김사장님 — 수제 그래놀라 제조업자, 온라인 판매 처음

### 3.1. 진입 (Day 1)

알파앱스에 가입한다. 카페24 계정이 없어도 된다.

```
"어떤 상품을 파시나요?"
→ 상품명, 사진 5장, 가격, 한 줄 소개 입력

"브랜드 분위기를 골라주세요"
→ 템플릿 3~4개 중 선택 (미니멀 / 내추럴 / 볼드 / 프리미엄)
```

**결과물: `granola-kim.alphaapps.io` 즉시 생성.**
가입 후 10분 안에 내 브랜드 URL이 생기는 경험.

### 3.2. 마이크로사이트 구조

한 페이지짜리 브랜드 랜딩페이지. 쇼핑몰이 아니다.

```
┌─────────────────────────────────────┐
│  히어로 섹션                          │
│  "매일 아침, 손으로 굽는 그래놀라"       │
│  [상품 사진 슬라이드]                   │
│  [구매하기 →] (카페24/스마트스토어 링크)  │
├─────────────────────────────────────┤
│  인스타피드 위젯 (알파리뷰 인스타피드)     │
│  실시간 인스타그램 피드 자동 연동          │
├─────────────────────────────────────┤
│  리뷰 위젯 (알파리뷰)                   │
│  "127명이 이 그래놀라를 추천했어요"       │
│  포토리뷰 그리드                        │
├─────────────────────────────────────┤
│  추천 상품 (알파업셀)                   │
│  "이 그래놀라와 함께 많이 사는 제품"      │
├─────────────────────────────────────┤
│  브랜드 스토리 섹션                     │
│  제조 과정, 원재료, 브랜드 철학           │
├─────────────────────────────────────┤
│  CTA 하단 고정 바                      │
│  [카카오톡 문의] [구매하기 →]            │
└─────────────────────────────────────┘
```

기존 알파앱스 위젯이 이미 블록 역할을 한다. 새로 만들 위젯이 많지 않다.

### 3.3. 구매 동선 — 핵심 설계

마이크로사이트에서 결제는 일어나지 않는다. "구매하기" 버튼의 목적지가 분기된다:

| 판매자 상황 | 구매하기 버튼 목적지 | 알파앱스 역할 |
|---|---|---|
| 카페24 쇼핑몰 있음 | 카페24 상품 페이지로 이동 | 트래픽 소스 |
| 스마트스토어 있음 | 스마트스토어 상품 링크로 이동 | 트래픽 소스 |
| 아무것도 없음 | 카카오톡 채널 / DM 주문 | 초기 주문 수집 |
| 나중에 카페24 개설 | 연동 버튼 하나로 자동 전환 | 성장 경로 |

알파페이지는 "쇼핑몰"이 아니라 **"브랜드 허브"**. 어디서 팔든 상관없이, 고객이 처음 만나는 곳은 여기.

### 3.4. 알파앱스 생태계 자연 유입 퍼널

김사장님이 마이크로사이트를 운영하면서 자연스럽게 알파앱스 기능에 노출:

```
Week 1   "인스타피드가 자동으로 올라오네? 편하다"
         → 인스타피드 무료 위젯 사용 중

Week 3   "리뷰가 쌓이기 시작했는데, 관리하고 싶다"
         → 알파리뷰 무료 → 유료 전환 유도

Month 2  "매출이 늘어서 제대로 된 쇼핑몰이 필요하다"
         → "카페24 연동하기" 원클릭
         → 알파앱스가 카페24 셋업 안내
         → 이 시점에서 알파리뷰+업셀+푸시 풀 세트 온보딩

Month 4  "단골 고객한테 신상품 알리고 싶다"
         → 알파푸시 도입
```

**퍼널 요약:**

```
알파페이지(무료) → 인스타피드(무료) → 알파리뷰(유료전환)
                                    → 카페24 연동(쇼핑몰 확장)
                                    → 알파업셀+푸시(풀스택)
```

## 4. 경쟁 포지셔닝

```
             트래픽 있음          트래픽 없음
           ┌──────────────┬──────────────────┐
풀 쇼핑몰   │ 스마트스토어    │  카페24 / 아임웹   │
           │ (마켓플레이스)  │  (자사몰 빌더)     │
           ├──────────────┼──────────────────┤
브랜드 허브  │              │  ★ 알파페이지 ★    │
(랜딩)     │  (비어있음)    │  + 알파앱스 위젯    │
           └──────────────┴──────────────────┘
```

빈 자리를 차지하는 전략. 카페24와 경쟁이 아니라 카페24 **앞단에 위치**.

## 5. 기술 구현 — 필요 구성요소

| 구성요소 | 신규/기존 | 예상 공수 | 비고 |
|---|---|---|---|
| 페이지 빌더 (블록 에디터) | **신규** | 2~3개월 | 위젯 시스템 V2 Strategy 패턴 재활용 |
| 템플릿 시스템 (3~4종) | **신규** | 1개월 | 프리셋 시스템 확장 |
| 서브도메인 호스팅 (`*.alphaapps.io`) | **신규** | 2~3주 | Cloudflare Workers + S3 정적 서빙 |
| 커스텀 도메인 연결 | **신규** | 2~3주 | Let's Encrypt 자동 SSL |
| 인스타피드 위젯 | 진행 중 | 0 | 인스타피드 프로젝트 그대로 |
| 리뷰 위젯 | **기존** | 0 | 현행 위젯 임베드 |
| 업셀 위젯 | **기존** | 0 | 현행 위젯 임베드 |
| 외부 링크 연동 (구매 버튼) | **신규** | 1~2주 | 단순 URL 매핑 |
| 분석 대시보드 (방문자/클릭) | **신규** | 1개월 | 기본 GA 수준 |

**총 예상: 4~6개월**, 기존 위젯 자산을 최대한 활용한다는 전제.

## 6. 장기 로드맵

```
Phase 1 (0~6개월)   → 마이크로사이트 빌더 (랜딩+위젯, 호스팅 래퍼)
Phase 2 (6~12개월)  → 서브몰 방식 결제 연동 (토스 하위가맹점)
Phase 3 (12~18개월) → 풀 쇼핑몰 기능 (상품/주문/재고 자체 관리)
Phase 4 (18개월~)   → 자체 PG 계약 + 카페24 완전 선택화
```

Phase 1만으로도 "알파앱스에서 시작한다"는 경험은 만들 수 있다. 풀 쇼핑몰은 수요가 검증된 후 진행.

## 7. 리스크 및 대응

### 7.1. 정체성 혼란 — "이게 쇼핑몰이야 아니야?"

마이크로사이트가 기능이 늘어나면서 어설픈 쇼핑몰이 되면 카페24보다 못한 것이 된다. **"여기서 결제는 안 됩니다"를 약점이 아니라 정체성으로 만들어야 한다.** "브랜드를 보여주는 곳 / 사는 곳은 따로" — 이 메시지가 흔들리면 안 된다.

- 대응: 기능 추가 시 "이것이 브랜드 허브 정체성에 부합하는가?" 게이트 체크 필수
- 결제 기능은 Phase 2까지 의도적으로 미포함

### 7.2. 카페24와의 관계 — 협력인가 경쟁인가

알파페이지가 커지면 카페24 입장에서 "우리 트래픽을 뺏어가는 것 아닌가?"라는 인식이 생길 수 있다.

- 대응: **알파페이지 → 카페24 유입 데이터**를 처음부터 측정
- "알파페이지를 통해 카페24 신규 가입 N건 발생" 데이터를 파트너십 근거로 활용
- 알파페이지는 카페24의 리드 소스라는 포지션 유지

### 7.3. 초기 트래픽 문제

스마트스토어의 킬러는 네이버 쇼핑 검색 트래픽. 마이크로사이트 자체에는 트래픽 소스가 없다.

- 대응 1: 알파푸시 + 인스타피드 조합으로 자사 트래픽 확보 도구 제공
- 대응 2: 알파앱스 네트워크 내 크로스셀링 (가맹점 간 상호 노출) 검토
- 대응 3: 현실적으로 "스마트스토어도 하고, 브랜드 페이지도 알파앱스로"라는 병행 포지션이 초기 전략

# 알파리뷰 위젯 디자인 및 노출 설정

본 문서는 쇼핑몰 고객에게 수집된 리뷰를 보여주는 프론트엔드 환경인 "위젯(Widget)"의 구조, 디자인 설정, 노출 제어 방법에 대한 지식동기화 문서입니다.

## 1. 지식 메타데이터
- **문서 분류:** 프론트엔드 노출 및 디자인 가이드
- **작성 일자:** 2026-04-19
- **최종 갱신:** 2026-05-06
- **관련 프로덕트:** 알파리뷰(Alpha Review), 알파업셀(Alpha Upsell)
- **핵심 키워드:** 공통디자인, 마스킹, 위젯세트, 위젯타입, Strategy패턴, V2아키텍처, 캐싱, 프리셋

## 2. 아키텍처 개요

### 2.1 V1 / V2 구조

| 구분 | V1 (deprecated) | V2 (현행) |
|------|-----------------|-----------|
| 경로 | `widget/views.py` | `widget/v2/` |
| 패턴 | 직접 쿼리셋 조합 | Feature/Manager/Strategy 계층 분리 |
| 상태 | "TO BE DEPRECATED" | 모든 신규 개발 대상 |

### 2.2 핵심 도메인 계층

```
WidgetSet (위젯 세트 — 최상위 컨테이너)
  ├── Widget (개별 위젯, 타입별 Strategy 매핑)
  │     ├── WidgetOption (키-값 옵션, 델타 저장)
  │     ├── WidgetLabelRelation (노출/제외 라벨)
  │     ├── WidgetProductRelation (노출/제외 상품)
  │     ├── WidgetCategoryRelation (노출/제외 카테고리)
  │     └── WidgetTarget (비정규화 product_id 목록)
  └── location_type: 전체(1) / 상세페이지(2) / 메인페이지(3) / 리뷰게시판(4)
```

### 2.3 Strategy 패턴

V2에서는 위젯 타입마다 전용 Strategy 클래스가 매핑되어, 데이터 조회/필터링/캐싱/시리얼라이징을 각 위젯 특성에 맞게 처리한다. `WIDGET_TYPE_TO_STRATEGY_MAP`에서 매핑을 관리한다.

### 2.4 위젯 옵션 델타 저장 방식

- **저장 시:** 기본값(`WIDGET_TYPE_TO_DEFAULT_OPTION_MAPPER`)과 다른 항목만 DB에 저장
- **조회 시:** DB 옵션 + 기본값을 병합하여 완성된 옵션 반환
- **재구성 시:** 전체 삭제(`bulk_delete`) 후 재생성(`bulk_create`)

## 3. 공통 디자인 설정 (Global Configuration)

알파앱스의 위젯은 개별 위젯 세팅도 가능하지만, 전체 위젯 화면을 관통하는 통일된 공통 디자인을 제어할 수 있습니다. 설정 변경 후 반영까지 최대 15분 소요될 수 있으며, 미반영 시 브라우저 캐시 삭제가 필요합니다.

### 3.1. 색상 및 폰트
- 어드민 내비게이션 `[디자인 설정 > 공통 디자인]` 에서 브랜드의 키 컬러와 폰트를 지정하여 모든 위젯에 동일한 톤앤매너를 적용할 수 있습니다.

### 3.2. 작성자 정보 표시 및 마스킹
- **작성자명 표시:** 일반 고객의 경우 아이디(ID) 또는 이름 중 노출 형식을 선택할 수 있습니다. (비회원은 무조건 '비회원' 텍스트로 노출됨)
- **간편가입 회원:** 카카오/네이버 등 간편 로그인을 통해 작성된 리뷰의 경우는 아이디 대신 해당 텍스트(`카카오 로그인` 등)로 표기됩니다.
- **마스킹(별표 처리):** 이름이나 아이디의 노출 수위를 조절할 수 있습니다 (기본값 설정됨). 예: 뒤에서부터 2자리를 가려 "홍**" 등으로 표시 가능하며, 중간 마스킹(홍*동) 방식은 지원하지 않습니다.

### 3.3. 구매 인증 뱃지 (실제 구매 여부)
- 리뷰 데이터에 호스팅사의 '실제 주문 데이터'가 연동 결합되어 있는 경우 "구매 인증" 아이콘이 생성됩니다. 관리자가 임의로 엑셀로 기입한 서포터즈 리뷰나 스태프 리뷰에는 해당 마크가 달리지 않아, 고객에게 신뢰도를 부여하는 장치로 활용됩니다.

### 3.4. ShopDesign 플레이스홀더
- `{최대포인트}`: 위젯 내 텍스트에서 SNS 리뷰 제외 총 리워드 금액으로 자동 치환
- `{고객명}`: 재방문 팝업에서 고객명으로 치환
- 적용 대상 필드: `revisit_text`, `empty_review_text`, `empty_review_custom_pc`, `empty_review_custom_mo`

### 4.3 관리자 전용 위젯

`write_button`(작성유도버튼)과 `keyword_widget`(키워드위젯)은 `is_admin_only=True`로, 대시보드 위젯 세트 목록에 표시되지 않는다. 별도 자동설치 로직(`WidgetSet__Feature.get_manual_code()`)으로 관리된다:
- **키워드 위젯:** `ShopKeywordReview__Manager.widget_auto_install` 플래그로 활성화
- **작성유도버튼:** `ShopDesign.review_write_btn_show` 플래그로 활성화

### 4.4 위젯 타입별 전용 라벨 매핑

## 5. 특정 리뷰 지정 노출 (북마크/라벨링 기능)

상품 상세페이지가 아닌 메인페이지, 기획전(프로모션) 페이지에서 특정 상품이나 선별된 리뷰만 모아서 강조하고 싶을 때 사용합니다.

### 5.1. 리뷰 노출 범위 (`review_posting` 옵션)

| 값 | 라벨 | 설명 | 권장 위치 |
|----|------|------|-----------|
| `all` | 모든 상품의 리뷰 | 전체 리뷰 + 제외 상품/라벨 | 메인페이지, 게시판 |
| `product` | 위젯 설치 상품의 리뷰 | product_no 기준 자동 필터 | 상세페이지 (기본값) |
| `label` | 특수 라벨 리뷰 | 전용 라벨 연결 (§4.4 참조) | 메인페이지, 기획전 |
| `custom` | 직접 선택 | 포함/제외 상품·라벨 수동 지정 | 프로모션 페이지 |

### 5.2. 북마크(특수 위젯 구성) 활용
- 베스트 리뷰 위젯, 스마트픽, 스토리 리뷰 등의 "특수 위젯"은 어드민 `[리뷰 목록]`에서 해당 위젯 이름과 동일한 이름으로 **북마크 라벨**을 부착한 리뷰만 수집하여 게시합니다.
- 복수의 리뷰에 라벨을 부착할 경우, 일괄 처리가 불가능하므로 각각 개별적으로 세팅해야 합니다.

## 6. 리뷰 노출 제어 (위젯 OFF 및 고정/추천)

### 6.1. 특정 페이지에만 위젯 OFF (블랙리스트)
비밀 결제 링크, B2B 도매 전용 상품, 정기 구독 전용 상품, 부록 상품 등 일반 리뷰나 알파업셀(추천 기능)이 노출되면 영업 밸런스를 해치는 상품을 필터링합니다.
- `[추천 > 특정상품에 위젯 OFF]` 메뉴에서 제외 상자를 구성하면, 해당 상품의 상세페이지 영역에는 어떠한 자동 위젯도 노출되지 않습니다.
- 제외 목록은 상품 이미지 컷에 마우스를 오버하여 즉각 삭제(복구)가 가능합니다. 반영까지 약 5분이 소요됩니다.

### 6.2. 우선 순위 제어 (상하단 고정)
리뷰 목록을 볼 때 작성일이나 랜덤순의 한계를 극복하기 위해, 포토 퀄리티가 우수하거나 고객의 신체 스펙이 매력적으로 드러난 리뷰를 의도적으로 게시 순위 최상단으로 끌어올리거나(상단 고정), 악의가 없으나 사진이 보기 불편한 리뷰를 최하단으로 숨길(내릴) 수 있습니다.

### 6.3. 리뷰 없을 때 처리 (`review_none` 옵션)

| 값 | 동작 |
|----|------|
| `message` | 리뷰 작성 유도 메시지 노출 (리뷰 0개일 때) |
| `hide` | 위젯 자체를 미노출 (리뷰 < `review_minimum`일 때) |

### 6.4. 리뷰 정렬 옵션

| 값 | 라벨 |
|----|------|
| `-created_at` | 최신순 |
| `-like_count` | 추천순 |
| `-ratings` | 별점순 |

추가로 `sort_photo`, `sort_video` 옵션으로 미디어 포함 리뷰 우선 정렬 가능.

## 7. 캐싱 전략

### 7.1 위젯 데이터 캐시
- **캐시 단위:** 100개 리뷰/상품을 하나의 캐시 페이지로 관리
- **캐시 키:** `widget_{type}_{code}_{product_no}_{page}_{sort}_{device}` 조합
- **무효화:** `CACHE_OFF` 설정 또는 `is_dummy=True` 더미 모드 시 비활성화

### 7.2 CDN 캐시 제어 (TTL)

| 용도 | Cache-Control TTL |
|------|-------------------|
| 민감 데이터 (좋아요 등) | 30초 |
| 위젯 데이터 | 180초 (3분) |
| 상세 데이터 | 300초 (5분) |
| 설정 데이터 | 300초 (5분) |
| 5xx 에러 응답 | 86,400초 (1일) |

### 7.3 캐시 적용 위젯

게시판, CAFE24게시판, 상품별리뷰목록/게시판/비교, 스토리, 평점, 브리핑, 작성유도버튼, 키워드위젯

## 8. 프리셋 / 프리뷰 시스템

### 8.1 프리셋 (Preset)
슈퍼관리자가 쇼핑몰에 초기 위젯 세트를 일괄 생성할 때 사용. 9종:

| 프리셋 | 포함 위젯 |
|--------|-----------|
| `review_tab` | 포토리뷰, 평점, 공지사항, 게시판 |
| `simple_photo` | 심플포토 |
| `detail_top` | 포토리뷰 |
| `chat` | 채팅리뷰 |
| `floating` | 플로팅리뷰 |
| `main_page` | 포토리뷰 |
| `board` | 상품별리뷰목록, 공지사항, 게시판 |
| `rolling` | 롤링위젯 |
| `briefing` | 브리핑위젯 |

### 8.2 프리뷰 (Preview)
온보딩 과정에서 사용. `SYSTEM_PREVIEW_SHOP_ID = -1`에 저장된 프리뷰 위젯 세트를 실제 쇼핑몰에 복제(설치)하는 구조. 위 프리셋 9종 + 개별 위젯 타입 11종 = 20종의 프리뷰 타입 지원.

# [[리뷰 종류 정리]]

## 📌 한 줄 통찰 (The Karpathy Summary)
> 알파리뷰의 리뷰는 "누가(creater_type) x 어떤 방식으로(review_type) x 어디서(platform)" 작성했느냐의 3축 조합으로 분류되며, 각 조합에 따라 주문 데이터 존재 여부, 포인트 지급 자격, SNS 리뷰 작성 가능 여부가 개별적으로 제어되는 복합적인 매트릭스 구조를 가진다.

## 📐 도메인 모델 개요 (Domain Model Overview)

| # | 도메인 | 핵심 개념 |
|---|--------|-----------|
| 1 | 엔드 유저 직접 작성 (자발적) | 일반/빠른/SNS 리뷰. 회원/비회원/NPAY 비회원 구분. 주문 데이터 필수 |
| 2 | 엔드 유저 요청 기반 작성 | 미검증 리뷰(비구매 회원), 리뷰 요청(연락처 기반 알림톡). 주문 데이터 강제 생성 |
| 3 | 관리자 작성 | 스태프 리뷰(일반/SNS). 주문 데이터 없음. cafe24 싱크 미지원 |
| 4 | 외부 채널 수집 | 스마트스토어, 인스타그램(소셜), 네이버 블로그(소셜). 시스템 수집(creater_type=6) |
| 5 | DB 분류 체계 | review_type(1~4), creater_type(1~100), platform, content_type 필터 |
| 6 | 리뷰 수정/복사 버전 관리 | 원본(creater_type=100), 복사(creater_type=29), review_origin 연결 |

## 📖 핵심 도메인별 상세 (Domain Deep-Dive)

### 📖-1. 엔드 유저 직접 작성 (자발적)

| 리뷰 종류 | review_type | creater_type | platform | 주문 데이터 | createdAt |
|---|---|---|---|---|---|
| 텍스트 리뷰 | 1 (일반리뷰) | 1(회원), 2(비회원), 3(NPAY 비회원) | 알파리뷰 | 있음 | 최초 작성 시점 (별점 클릭) |
| 빠른 리뷰 | 2 (빠른리뷰) | 1(회원), 2(비회원), 3(NPAY 비회원) | 알파리뷰 | 있음 | 최초 작성 시점 (별점 클릭) |
| SNS 리뷰 | 3 (SNS리뷰) | 1(회원), 2(비회원), 3(NPAY 비회원) | 알파리뷰 | 있음 | 해시태그 인식 → admin 유입 시점 |

**회원 유형 구분:**
- **회원(1):** 자사몰 회원
- **비회원(2):** 자사몰 회원이 아니면서 자사몰 내부 시스템으로 결제한 경우
- **NPAY 비회원(3):** 자사몰 회원이 아니면서 npay 등 간편결제로 결제한 경우 (npay만이 아님)

**텍스트 리뷰 vs 빠른 리뷰 전환 규칙:**
- 리뷰 작성 유도 알림톡 버튼 클릭 시 → 자동으로 "빠른 리뷰"로 저장
- 빠른 리뷰에서 **텍스트를 추가 입력/수정** 시 → review_type이 "일반 리뷰(1)"로 변경
- 텍스트 수정 없이 이미지만 추가할 경우 → "빠른 리뷰(2)" 유지

**빠른 리뷰 타입 및 템플릿:**

| DB 테이블 | 설명 | 컬럼명 |
|---|---|---|
| review_review | 빠른 리뷰 목록 | review_type: 2 |
| shopreviewpolicy_shoppolicy | shop별 빠른 리뷰 타입 | quick_review_type (1: 커스텀, 2: 프리셋) |
| shopreviewpolicy_shopquickreview | shop별 템플릿 목록 | |
| shopreviewpolicy_shoppermit | shop별 사용 여부 | permit_quick (true/false) |

- **커스텀(1):** 관리자가 별점/만족도별 문구 직접 생성/수정/삭제. 만족(4,5점), 보통(3점), 별로(1,2점)
- **프리셋(2):** 기본 생성됨, 수정/삭제 불가. 만족(4,5점), 보통(3점), 아쉬움(1,2점)
- 별점/만족도별 템플릿이 여러 개인 경우 **랜덤 선택**
- ⚠️ 커스텀 항목이 다 삭제된 상태에서 빠른 리뷰를 켜면 → **하드코딩된 기본값**으로 자동 생성

**SNS 리뷰 상세:**

| DB 테이블 | 설명 | 컬럼명 |
|---|---|---|
| review_review | SNS 리뷰 목록 | review_type: 3 |
| shopreviewpolicy_shoppermit | 사용 여부 | permit_sns (true/false) |
| shopreviewpolicy_shoppolicy | SNS 유지 기한 | maintenance_sns (최대 7일) |
| review_reviewhashtag | 해시태그 | hashtag_type (1: 카테고리, 2: 상품) |
| review_reviewhashtagcategory | 해시태그 & 카테고리 연결 | |
| review_reviewhashtagproduct | 해시태그 & 상품 연결 | |

- **필수 조건:** 인스타그램 계정 연동 (페이스북 비즈니스 페이지 → 인스타 프로페셔널 계정)
- **해시태그 설정:** 각 기준별(모든 상품/특정 상품/특정 카테고리) 최대 3개. 작성 페이지에서는 최대 5개까지 표시 (앞에서부터 잘림)
- **인스타 게시글 연동 조건:**
  - 게시물 **최초 업로드 시** 해시태그 + 특정코드가 포함되어야만 연동
  - 이미 생성된 게시물에서 수정으로 추가해도 연동 안 됨
  - 특정코드는 order_item의 shortcode에 저장
- **작성 가능 조건 (5단계 체크):**
  1. 해당 주문상품의 해시태그 존재 확인
  2. 주문상품의 리뷰작성 가능 여부 확인
  3. SNS 리뷰 작성 가능 여부 확인
  4. 해당 주문상품에 SNS 리뷰 작성 이력 확인
  5. 대시보드 SNS 리뷰 토글 활성화 확인

### 📖-2. 엔드 유저 요청 기반 작성 (미검증/리뷰 요청)

| 리뷰 종류 | review_type | creater_type | platform | 주문 데이터 |
|---|---|---|---|---|
| 미검증 리뷰 | 1 (일반리뷰) | 11 (미검증 리뷰) | 알파리뷰 | 강제 생성 |
| 리뷰 요청 > 텍스트 | 1 (일반리뷰) | 기존 고객 일치 시 해당 customer_type, 불일치 시 4(주문생성 비회원) | 알파리뷰 | 강제 생성 |
| 리뷰 요청 > 빠른 리뷰 | 2 (빠른리뷰) | 동일 | 알파리뷰 | 강제 생성 |

**미검증 리뷰 vs 리뷰 요청 비교:**

| | 미검증 리뷰 | 리뷰 요청 |
|---|---|---|
| 회원 유형 | 비구매 | 요청 |
| 작성 가능한 사람 | 로그인한 회원 | 알림톡 수신자 |
| 작성 경로 | 쇼핑몰 상세페이지 > 리뷰 작성 버튼 | 개인에게 전송된 리뷰 작성 페이지 |
| 고객 수정 | O | O |
| SNS 리뷰 작성 | **X** | **O** |
| 게시/지급 | 설정에 따름 | 수동 게시/지급 가능 |

**미검증 리뷰 DB 스키마:**

| DB 테이블 | 설명 | 컬럼명 |
|---|---|---|
| shopreviewpolicy_shoppermit | ON/OFF | permit_unverified (true/false) |
| shopreviewpolicy_shoppolicy | 1일 작성 한도 | write_unverified_day |
| shopreviewpolicy_shoppolicy | 포인트 지급 여부 | use_reward_unverified (true/false) |
| shopreviewpolicy_shopreward | 리워드 정책 | title(100:별점 / 110:텍스트 / 120:포토 / 130:동영상 / 140:설문 / 150:SNS), value: unverified |

- 미검증 리뷰는 **빠른 리뷰 사용 불가**
- 작성 조건: 로그인 필수 + 기존 주문 상품 리뷰를 모두 작성했거나 주문 내역이 아예 없어야 함
- 1일 최대 작성 개수: 5/10/15/20개 (회원 ID 기준)

**⚠️ 리뷰 요청 특이사항:**
- shop에서 "비회원 작성 불가"로 설정해도 리뷰 요청을 통한 비회원 작성은 **가능**
- 비회원 포인트 기본값(default)이 적용되나, 비회원 포인트 설정 UI가 사용자에게 노출되지 않는 비정합 상태 존재
- 작성 제외 상품/카테고리의 상품에 대해 리뷰 요청 시 → 알림톡은 발송되나 리뷰 작성은 불가
- 주문 업로드와 동시에 배송 완료 상태로 변경, 가장 가까운 19시에 알림톡 발송
- 주문자 이름 + 전화번호가 동일하면 같은 사용자로 취급

### 📖-3. 관리자 작성 (스태프/체험단 리뷰)

| 리뷰 종류 | review_type | creater_type | platform | 주문 데이터 | createdAt |
|---|---|---|---|---|---|
| 스태프 리뷰 > 텍스트 | 1 (일반리뷰) | 5 (관리자) | 알파리뷰 | 없음 | 엑셀/단일 생성에서 지정한 작성일의 00시 |
| 스태프 리뷰 > SNS | 3 (SNS리뷰) | 5 (관리자) | 알파리뷰 | 없음 | 동일 |

- 스태프 리뷰는 **cafe24로 싱크하지 않음** (해지 시에만 일괄 이관)
- 체험단 리뷰는 100% 외부 업체 관리. 알파리뷰에서는 캠페인 생성(seed_seedcampaign)까지만 관리하며 체험단 리뷰 여부는 DB에 저장하지 않음

### 📖-4. 외부 채널 수집 (소셜 리뷰)

| 리뷰 종류 | review_type | creater_type | platform | 주문 데이터 | createdAt |
|---|---|---|---|---|---|
| 스마트스토어 | 1 (일반리뷰) | 6 (시스템 수집) | 스마트스토어 | 없음 | 스마트스토어 원본 작성 시점 |
| 소셜 리뷰 > 인스타 | 4 (소셜리뷰) | 6 (시스템 수집) | 인스타그램 | 없음 | 상품 연동 등록 시점 |
| 소셜 리뷰 > 네이버블로그 | 4 (소셜리뷰) | 6 (시스템 수집) | 네이버블로그 | 없음 | 리뷰 등록 시점 |

**인스타그램 소셜 리뷰 상세:**
- 메타 API를 사용하여 게시물 조회
- 게시글 원본 삭제되어도 연동된 리뷰는 삭제되지 않음
- 이미지 연동 개수 제한 없음 (인스타 게시물당 최대 20개 전부 연동 가능)
- **⚠️ 이미지 URL 만료일자 이슈:**
  - URL에 만료일자가 포함되어 있으며 랜덤하게 지정됨
  - 소셜리뷰 새로고침 + 인스타 피드 크론(새벽 4시)에서 매번 URL 업데이트
  - 게시물 연결 이미지를 **전체 삭제 후 재생성**하는 방식
  - 전체 이미지 중 `가장 빠른 만료일자`를 저장
- **게시물 조회 로직:**
  - 새로고침 시 업로드 일자 현재 기준 **1년 이내** 게시물만 조회, **최대 2000개**
  - 기존 게시글은 내용/좋아요/댓글 수 업데이트 (사용자가 좋아요 수 숨김 시 0개로 조회)
  - **1년 초과 게시물은 새로고침 시 일괄 제거**. 단, 상품 연동된 게시글은 제거되지 않음

**네이버 블로그 소셜 리뷰:**
- URL 입력 → 크롤링 방식으로 데이터 수집
- 원본 삭제되어도 연동된 리뷰는 삭제되지 않음
- 게시글 내 **첫 10장**의 이미지까지만 리뷰로 게시
- 작성자 ID와 cafe24 ID가 **정확히 일치**해야 적립금 지원
- 각 게시글별로 연동할 상품을 하나씩 선택해야 함

### 📖-5. DB 분류 체계 (필터 기준)

**review_type (리뷰 유형):**
| 값 | 명칭 |
|---|---|
| 1 | 일반 리뷰 |
| 2 | 빠른 리뷰 |
| 3 | SNS 리뷰 |
| 4 | 소셜 리뷰 (인스타 & 네이버 연동) |

**creater_type (작성자):**
| 값 | 명칭 |
|---|---|
| 1 | 회원 |
| 2 | 비회원 |
| 3 | NPAY 비회원 |
| 4 | 주문생성 비회원 (리뷰 요청) |
| 5 | 관리자 |
| 6 | 시스템 수집 |
| 11 | 미검증 리뷰 |
| 29 | 복사된 리뷰 |
| 100 | 원본 리뷰 |

**platform (작성 경로):** 알파리뷰, cafe24, 스마트스토어, 지그재그, 인스타그램, 네이버블로그

**content_type (리뷰 콘텐츠):** 사진 포함, 동영상 포함, 설문 리뷰, 키워드 리뷰 (리뷰 종류가 아닌 공통 속성)

### 📖-6. 리뷰 수정/복사 버전 관리

- **리뷰 수정 시:**
  - 기존 리뷰 → creater_type이 **원본(100)**으로 변경
  - 수정된 리뷰 → 원본의 creater_type을 따라감
  - OneToOneField(review_origin)로 연결
- **리뷰 복사 시:**
  - 원본 리뷰 → creater_type 그대로 유지
  - 복사된 리뷰 → creater_type이 **복사(29)**로 설정
- **리워드 적립 제외 대상:**
  - customer 데이터 없는 경우 (비회원은 회원가입 후 대상이 됨)
  - 제외된 상품/카테고리
  - 스태프 리뷰
  - 네이버블로그: 입력한 회원 ID와 일치해야만
  - 인스타 소셜 리뷰: 적립 불가

**전체 리뷰 종류 매트릭스 (교차 정리):**

| 종류 | review_type | creater_type | platform | 주문 데이터 |
|---|---|---|---|---|
| 미검증 리뷰 | 1 | 11 | 알파리뷰 | 강제 생성 |
| 스마트스토어 | 1 | 6 | 스마트스토어 | 없음 |
| 일반 리뷰 (회원) | 1 | 1 / 2 / 3 | 알파리뷰 | 있음 |
| 빠른 리뷰 | 2 | 1 / 2 / 3 | 알파리뷰 | 있음 |
| SNS 리뷰 | 3 | 6(시스템) / 5(관리자) | 인스타그램 | 있음 (관리자 작성 시 없음) |
| 소셜 리뷰 (인스타) | 4 | 6 / 100(원본) / 29(복사) | 인스타그램 | 없음 |
| 소셜 리뷰 (블로그) | 4 | 6 / 100 / 29 | 네이버블로그 | 없음 |
| 리뷰 요청 | 1 / 2 | 4(주문생성 비회원) | 알파리뷰 | 강제 생성 |
| 스태프 리뷰 | 1 / 3 | 5 | 알파리뷰 | 없음 |
